Hello,
We are just rolling out in Germany-Netherlands-Switzerland-Austria and probably Italy a standard BaaN-Module "GPA" (generic
process automation) which is espacialy devellopped for applications as y asking for.
It has a so-called process-configurator where y enter all the parameters to define the process. The status of the process is organized in a so-called Status-Matrix. If in the process somebody has to fill in data a is done in the "user data screen".
There is no need anymore to open all the sessions by yourself. The module if configured right will do this in the background.
The scanning is in principal done via the keyboard, but can be expanded via RF-scanners. Under UNIX this is absolute no problem. If you use Baan-Windows (BaaN IV under Windows-NT and BaaN-ERP) the communication can done via an ascii-file with polling, or via CITRIX. We are looking for a possibility with Windows CE and the BaaN-windows environment.
If y want to have more information pls write an e-mail.
