Dear Donnie,

Not knowing the exact item, I think the best way to model this is simply creating one normal warehouse (that's set to be ignored by EP) for each such customer. Transfer the stock there when sending the item to customer for trial. When he/she wants to buy it, make an SO for it specifying the customer warehouse as source for shipment. If he/she doesn't want it, simply transfer it back

Using consignment will also work, but it's just more complicated. You have to create consignment replenishment SO when sending the item for trial by customer. Then when the customer decides to buy it, you'll have to make consignment invoicing SO. IF the customer doesn't want it however, you'll have to make a return consignment replenishment SO. Using consignment features, however, will give you the following benefits:
You can't send the item to wrong warehouse, since each consignment warehouse has business partner attached to it
The documents are produced properly, i.e. on sending the item for trial only packing slip is produced (no invoice), while on customer actually buying only invoice is produced (no packing slip)

Rgds,
Paul
