Well,

I checked it at my end. You can add any activity as blocking step. You can block before the activity is executed. Like, I have kept tdsls4246m000 as blocking activity and I can still block the orders manually. The difference is that, the system will pop up a message that the activity is not yet carried out.

Check if you have maintained proper change reasons in Hold Reasons(tdsls0190m000). You can try creating a new reason e.g. General type and then try. Also, check if User Profile is defined for your username. That won't affect but you never know. You can check KB 1059434 for more info.
