Hi gabymore

This is transaction report. A transaction report unless taken from day one may show you negative balances. This is because if you take a report from some date after the transactions commenced, it does not take into account the transactions before that date.

Can you give a screenshot of tdilc1511m000 or tdilc1410m000.

However I have found that sometimes lot balances do go negative due to some bug particularly if you are using containerisation and inventory unit is different from storage unit. I find that users do amke mistakes in entering the correct unit at all times.

Run session tdinv0252m000 WITH THE FIELD 'ADJUST- YES/NO' SET TO NO'. If the field is set to yes you will get incosistency between Finanace inventory valuation and and Distribution inventory valuation
