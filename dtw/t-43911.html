You can through tools or DEM security limit the users ability into the various BP roles.

If the BP header (tccom100) is also a problem with the business owners then you could use field level security to limit the access to specific records. This however is extra burden on the database so performance suffers. So I'd push back on the manager requesting this. However if you were going to do field level security you'd need to define something to tell the system what is a Supplier, Customer or Both. You could use the numbering series such as what David suggests or if using LN FP2 or higher there is a hidden field tccom100.bprl which classfies a BP with 1 of 4 options (Customer, Supplier, Customer and Supplier or None) This field is updated based on the roles that have been assigned or not assigned.
