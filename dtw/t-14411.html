Hello Jochen,

Well logically speaking, this may not be the correct way of handling this kind of a change.

EDI is triggered by the buyer and any changes that need to be made should be made by the buyer (not the supplier --- in normal conditions).

My suggestion would be to communicate this change to the buyer (via email or telephone), so that he makes changes to the PO. This change can be automatically transferred to the supplier's SO using standard Order change message. Some advantages of this would be-

1. The operation would work in a normal way without the need to make any customizations.

2. Buyer can assign reason codes for the changes.

3. There will be single window that will handle the changes.

"The buyer should exercise control on the changes NOT the supplier --- this will otherwise overrule the EDI business logic!"

Hope it helps. ;)
