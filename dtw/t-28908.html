20th December 2006, 12:13
Maintain User Authorizations for Inventory Transactions {tdinv1110m000} has nothing to do with limiting warehouse access. This session is used to control which all types of transaction a user can use in the session Enter Inventory Transactions by Item {tdinv1101m000}. It wont limit access to Table Data which in your case is a warehouse.

Please clarify the statement where you say that the "user cannot do transactions like Sales order, Purchase order etc..". Do you mean the transactions in session tdinv1101m000 or the transaction in Sales Module, Purchase Module etc.?? Do you get any error message when you type this warehouse in any transaction where there is a field to enter the Warehouse?

Either way, the reason why i asked you to check the session Maintain Table Data Authorizations {ttaad4134m000} was because if i were to limit a person from using a particular warehouse in tables belonging to Purchase(tdpur041, tdpur045), Sales(tdsls041, tdsls045), Production(tisfc001,ticst001, ticst010) or even Inventory(tdinv100) then this is where it would be set up. After this, the user wont be able to see any transaction with this warehouse or even make one.

Coming back to your Commercial Warehouse. For academic purpose, when you say Commercial Warehouse, do you mean it as a term that is used in your business logic or is it a Baan term because i couldnt find this term in help section of Baan or in Maintain Warehouse session. Are you using a localised version of Baan4c4 which has this concept?

In any case, are all these users sharing the same menu in Maintain User Data {ttaad2100m000}?
1a) If the answer is Yes, then run session Copy Authorizations Between Users {ttaad2240m000} where in you copy the authorizations from a user where the warehouse is visible. Follow this with session Convert User Data to Runtime Data Dictionary {ttaad4200m000} and check if it helps.
1b) If the answer is no, then change his menu structure to the same as the others who can access this warehouse and then try step (1a).
2) Do check, when the user opens the session Enter Inventory Transactions By Item {tdinv1101m000}, if the session number is the same as the session number of the others who opens this session. Could it be that this session is customized somehow?
3) Try deleting the user and creating it again and try step (1a).
