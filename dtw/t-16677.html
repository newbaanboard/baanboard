Hi,

You could use field "Block Locations for Cycle Counting Orders" (tdilc000. take), but in this case wou will also block outbound movements.

When you set the "Block Locations for Cycle Counting Orders" (tdilc000. take) to "Yes" in "ILC Parameters", Inventory for which cycle counting orders are present must be blocked. To prevent mistakes, it is advised to block the inventory between the time that you create cycle counting orders and the time that you process your counting results.

This parameter determines whether or not stockpoints must be blocked for transactions that modify the inventory on the stockpoint during cycle counting. Advices are not blocked, but actualization of the advices are. Blocking only occurs when both tdilc000. take and tdilc101. take are yes.

Where this parameter is used: When ILC is implemented (otherwise the parameter is Block During Warehouse Cycle Counting) this parameter is used in combination with Cycle Counting Order Generated to determine the blocking status after session "Generate Cycle Counting Orders (tdilc5210m000)" is run.
