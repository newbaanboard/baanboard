defaults.edi is the conversion setup data for supported edi messages by baan. When you import this data the conversion setup tables will be populted. Get latest copy of defaults.edi from baan and load it.

First fill edi parameters, then create networks using which you communicate with your relations.

After that you have to setup your relations(supplier or Customer or both), populate messages supported by relation.
ie say i crate a relation 'A' as supplier to me then i will send purchase order which is 850S out. so A is supported for 850S out.
Every message can you multiple conversion setups. you can create your own conversion setups using defaults setups from baan.
When you print the the required document ex PO a entry is created in edi saying this transaction need to go throught edi.
When you run generate edi messages this is will generated a message.
This is very basic guideline... refer to baan documentation on EDI which will explain in details.

There is no Baan User Guide (to my knowledge) on Conversion set ups. Baan had the multi file structure EDI messages in triton version. The same conversion set ups are modified to make single file in Baan IV and above. There is no specific logic which can be defined for conversion set ups. Only experience in working with these conversion set ups will help in understanding them.

The combination of Relation and Order type maintained in the EDI setup enables the PO to be prepared electronically.
