Well, this is infor's answer :

" the session whinr1540m000 may never have negative entries, because it show the effective stock point.Only values >= 0 are allowed.
The figure in session whwmd4300m000 refers to the table whwmd215 (session whwmd2515m000). The value of the related field whwmd215.qhnd may have a negative value, but in this case the functionality of negative inventory plays a role.
Negative invenory is registered in table whwmd255 (session whwmd2550m000). From the screenshots I can see that negative inventory applies for the specific item and warehouse. This should only appear temporarily.
Background:
1. Goods arrived in the warehouse, but were ot yet confirmed received.
2. These goods shall be delivered and are immediately picked to fulfill a shipment.
3. When picking and confirming the shipment the negative inventory gets generated.
4. In order to balance the negative inventory a proper warehouse receipt related to the purchase order (or production order) needs to be accomplished. Otherwise the negative inventory figures will persist.
In such a situation negative inventory might still show up, even if a stock point in table whinr140 (session whinr1540m000) is available.
Please check whether there still are receipts to be carried out for the item.
You may check on data corruption, as well, by running session whwmd6290m000, first in test mode.
Please give a feedback.
Thank you and kind regards "
