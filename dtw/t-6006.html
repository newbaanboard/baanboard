Hi Nazeem,

well I think it's possible to set up a structure you like to have. There a several possibilities to proceed. In the first step you can set the % for the dummies to zero and only the final level to 20%. Or you simply delete the calculated commission/rebates manually for those levels you don't like to pay anything.

In the master data each relation can be assigned to a parent relation. That's the way you build up the structure. A similiar thing can be done with the agreements: an agreement can be declared as a child agreement. Both methods can be combined with each other. This may end in a jungle of links to calculate the commission/rebate. My advice: keep it simple! By the way, the Baan online help is not to bad in this module - really ;).

If you need further informations, please let me know how the parameters are set for CMS. We are on Baan IVc4.
