Normally in BaaN each relation is linked to an organisation. This means setup the "connection" between the two using session tcedi0110m000.
This is not very convenient as we wanted to allow all our customers to order through our web-shop. The orders placed have to be transfered to BaaN through the EDI module. Instead of creating a record relation/customer for each customer i've create a record with a dummy customer. In the EDI-message, in the part "global overhead" I'm refering to this dummy. In the orderheader however I'm using the real customer. This works great without having to worry about adding records in the table for each new customer.
