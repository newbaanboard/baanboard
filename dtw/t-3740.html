I guess you could describe function servers as a "transactional" approach to functionality. There are DDC function servers, which use Baan's DDC technology, and AFS function servers.

DDC stuff can be found under the "Baan IV Connectivity" menu. There is a session called "Function Server Generator" that will create a function server that can, with some setup, be run under DDC. A lot of people use DDC with handhelds and create function servers to automate processes such as cycle counting, receiving, shipping, etc. You probably need source code to run the generator session. We have it, so it has never been an issue for us when we create function servers for our clients.

AFS allows you to make certain session calls (i.e., mimic session functionality) without having the standard Baan source code. Thus, you could create a session to handle some input and process it as if it were entered in the regular session. There are API calls that you would embed into a program script you create. I am simplifying things and not doing AFS justice, but I hope you get the idea.

I hope this helps a little. If you need more information or assistance, please let me know.
