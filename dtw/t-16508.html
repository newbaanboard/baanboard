Hi,

Storage conditions are used to prevent some items were located in certain locations.

Steps:
1) Define storage conditions (tdilc0150m000)
2) Assign locations to storage conditions (tdilc0151m000)
3) Assign item groups to storage conditions (tdilc0156m000)

The paramater "condition type (kcon)" must be "Common" if the location is assigned to more than one storage condition.
Use "Present" if the location is assigned to a single storage condition.

Storage conditions are checked in: tdilc4203m000 / tdilc4104s000 / tdilc1121s000 / tdilc1122s000

Example:
Item Groups: IG1 / IG2 / IG3
Storage Conditions: SC1 / SC2 / SC3
Locations: L1 / L2 / L3 / L4 / L5

Relation between family groups and storage conditions:
IG1 - SC1 / IG2 - SC2 / IG3 - SC3

Retion between storage conditions and locations:

We want storage item group IG1 exclusively in some locations. So we define in tdilc003:
SC1 - L1 - "Present (for item/location)"
SC1 - L2 - "Present (for item/location)"
SC1 - L3 - "Present (for item/location)"


"Not present (for item/location)" is similar than "Present (for item/location)" but is it used for not let storage an item or an item group in a location.
