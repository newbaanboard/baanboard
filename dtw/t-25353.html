Thanks for all the input. The best solution to my issue was provided by asharnouby and was to utilize the planned inventory transaction module, specifically "inventory transactions by order". By only searching sales orders and controlling the sales order numbers input to avoid recent orders ( last three days ), you can easily see any orders that got "lost" ie..they did not become back orders and they did not have picks print out. It work because any open sales order requirement will generate an inventory transaction requirement. Thanks again for all the input
