Since tdilc301 is only appended to, it is possible to re-instate that data from tdinv700 by writing a new session. Difficult but possible. Only problem is if you have multiple locations, then you would need a 'compromise' decision as to which specific location to 'move into' or 'move from' to serve as a default. The other aspect is with Purchase or Manufacturing Receipts, where there are at least three records in ilc301 for each inv700 (Receipt into REC Location, Transfer FROM REC Location, Transfer into FINALLOCATION). If there is inspection required, it will have a different set of records. Each transaction type will have its own different way of logging into tdilc301, but most are one-to-one. The new session, if written, can become a good utility. (Using the Order Type and Order Number, one can refer back to the original Order information, if necessary).

There will be many people to convince about the decisions made, especially Finance.
