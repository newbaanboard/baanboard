Dear Bob,

By default, BaanERP groups SO line details by their planned delivery date and assign one WH order set per group. This means that if all the sales order line details have exactly the same planned delivery date, then BaanERP is going to assign them all to the same set number. This must have been what have happened in your live system. If, however, you set every sales order line details to have different planned delivery date, then BaanERP will assign different set number per SO line detail. This might have been what happened in your test system.

Rgds,
Paul
