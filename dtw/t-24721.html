The receipt transaction is registered in the following way:

1) production (receipt): against FTP
2) production manual (receipt): against actual cost
3) transfer manual (receipt): against issue price
4) cycle count/adjustment order: against actual cost
5a) linked sales return: against original issue price
5b) not linked sales return: against receipt price

Will MAUC be adjusted:
1) yes
2) no
3) depends; most likely it will
4) no
5a) depends; if between the issue and the receipt the MAUC is unchanged: no
5b) yes

I am sorry. You are partly right:

1) No, against FTP. Since BaanERP 5.2 it is possible to receive end items against actual costs. On the Costing tab of the Production Order (tisfc0501m000) the option "Use Actual Cost Price for Receipt Posting" can be selected. In BaanERP 5.0 end items are received against FTP.

2) You are right. This is of course against FTP.

3a) If no invoice relation is defined: issue price
3b) If an invoice relation is defined: transfer price. If no transfer price can be found, I suppose a message comes up. If that is not true, then I think you are right, FTP is used.

4) No, against MAUC. See SOL 107924.
