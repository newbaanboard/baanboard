I am not contradicting using BOI's / API's.

But the easy way of meeting the requiremnt with minimum efforts from both applications side is using Baan Exchange or Baan EDI.

If the quotation data also need to present in BaaN then as suggested by Darren Phillips earlier, use Baan Exchage to load quotation tables in baan and generate sales orders from there. For this you need to map the fields from the quotation application to the fields in Baan. Then you need create an ascii file in quotation application as defined in Baan exchange. This ascii file can be sent to the Baan server using ftp or any other means.

If no quotation data is required in BaaN, then use Baan EDI to insert Sales orders. The EDI setup has to be created in baan for 850 inbound orders. For this you need not develop any new DLL's. Baan EDI calls for sales order data in a particular EDI format which can be created from quotation application. It is just an ascii file in the predefined format.
