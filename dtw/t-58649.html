Hello,

tdpur040 is the actual Purchase Order Header and
tdpur041 are the actual Purchase Lines

Every time, data is changed (e.g Quantity of a PO-Line or goods receive data ....) a new line is added in history table.

Youre able to analyse, what data has changed historicaly.

E. g. PO lines (tdpur051)

It has to be read as:
The PO line for position is added (Origin)
A line with negative quantity is added against the origin.
A new line with different data to the previous line is added.
........
It's done, every time the PO-Line is changed by user or goods receipt a.s.o.

Hope this explanation will help.

The same logic is in Sales (tdsls040, tdsls041 and tdsls050, tdsls051)
