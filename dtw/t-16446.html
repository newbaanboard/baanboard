Dear Hendra,

On return POs, you can specify the original PO/invoice from which the goods were originally received. BaanERP will then make sure that only the lots received from the original PO/invoice specified can be returned in this return PO. This is a useful feature. However, it won't change the back order of the original PO/invoice

If you'd rather track it on the original PO, then don't raise a new return PO. Instead, simply add a negative order line on the original PO to return the goods. Still, this won't change the back order of the original PO lines. You still have to add yet more lines to receive replacements later on. It's probably not what you're looking for, but it's also another alternative

Rgds,
Paul
