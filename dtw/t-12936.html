Hi there,

recently I was wondering how the salesorder header discount is calculated. The following situation:

* Order Discount 2,08% (tdsls040.odis)
* Salesorder with a single item
* Quantity = 1
* Price 44,99
* Orderline Discount 30%
* tdsls041.amta = 31,49

No problem so far. According to the online documentation regarding the field tdsls040.odis, the discount is calculated for each orderline. So I thought it should be:

tdsls041.amta * tdsls040.odis / 100 = 0,65 (rounded to 2 digits)

But in reality the discount seems to be calculated like this:

tdsls041.oqua * tdsls041.pric * (100 - tdsls041.disc) / 100 * tdsls040.odis = 0,66 (rounded to 2 digits)

without rounding an interim result such as tdsls041.oqua * tdsls041.pric.

My question is: has this been standard functionality ever since Baan IV is existing? Did anybody took notice that they changed this calculation in the past?

Thanks in advance.
