The reason codes are built to enable you to drive the credit account to one other than what is set up in your integration.

Regardless of whether you use a reason code or not, the system will look to your integration setup for the debit account.

As far as I know, there is no "vanilla BaaN" report that will give you what you're looking for. The only report I know of in the Distribution Module that references the reason codes is Print Inventory on Hold.

Jack,

Linda is right: there is no "vanilla" report (at least in Baan IV) that lists or summarizes inventory adjustments by adj reason code.

The Inv Adj Reason Code is a great example of how Baan responded to user requests for functionality without providing the means to audit it. Most companies that use adj reason codes heavily develop a customized report (often via SQL or third party report program) to provide the required detail.

The integration scripts pull the ledger accounts from the set up in tdinv850 then override the credit account with the one specified in the adjustment reason code, if applicable.

You can also override the integration setup by specifying a ledger account on form 2 of the inventory adjustment transaction. On all the versions Ive worked on, an account entered on form 2 overrides both the standard integration credit account and one specified by Adj Reason Code.

Scott Johnson

We have not developed a report here - we print the integration transactions. The credit accounts that we affect via a reason code do not normally have inventory adjustment transactions hit them. Also, we have a one-to-one relationship between reason codes and credit accounts, so in effect we get the same data by printing the integrations for the specific account(s) that a customized report would give us.
