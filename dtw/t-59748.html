SITUATION IDENTIFIED IN:
"Freeze/Confirm Shipments/Loads?(whinh4275m000)

SITUATION DESCRIPTION:
Cannot confirm a shipment for a main item with components. Attempting to confirm the shipment results in the following error message:
Fatal error : [MR_INDEXOUT] whint.dllwhina.get.cogs.for.sales.o.cost.components index # from Dim [1] #

This occurs when the total number of cost components belonging to the item structure exceeds the value of the maximum number of effective cost components as registered in the ticpr099 (Cost Component Structure Parameters) table.

SOLUTION DESCRIPTION:
Handling of cost components in case of main items is improved. Shipments can be confirmed properly, independent of the number of cost components.

None.
