If you have the buyer change the Delivery Address (Warehouse) on the PO Header to the proper warehouse then EDI will automatically send this to the other company in the 850.3 record. Also remember that unless you turn it off EDI is going to pickup the delivery address for the line based on the PO Line warehouse in the 850.7 record.

There is a provision in EDI to maintain conversions for certain codes. Check the "Code and Conversion Tables1 ---> Conversion In" you will find a session to "Maintain Conv. of Delivery Address Codes by Customer (In) --> tcedi3110m000

Maintain the conversion between the incoming code in the message and the required outgoing delivery address code for the concerned customer and this should set things right.

for e.g. if the code of warehouse in coming message is 001. and the delivery address for your sales order is 100 for Customer E1000 then....

Customer code = E1000
Organization = "EDI"
Address Code ID = "ZZZ"
Code in message = 001
Code in application = 100

Hope it helps...
