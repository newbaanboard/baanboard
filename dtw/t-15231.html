Yes, Baan does have a default conversion format for EDIFACT.
You should be able to request for the "defaults" files from support. There is a session in BAAN EDI that will load these defaults. There is also ANSI X12, XML and a few others but I have only used EDIFACT, XML, and ANSI X12 so far.
The degree of change that you will be able to do without touching the message generation and interpretation programs does have a limit. You may also find that Baan saved some positions by eliminating low use fields from the published EDIFACT standard.

In cases where there are multiple partners and so many message mappings to manage, it may be better to use a PC based software like what Neal has mentioned.

My last project is for SMS ordering interface into a TRITON3.1b system using EDI and X12. It was simple and worked well.

Good luck !
