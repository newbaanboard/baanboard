Hi all,

we set up an EDI scenario with one company making Purchase Orders. They get transferred with EDI to another company, becoming a Sales Order.
For changes in PO's we use the change codes, so all changes are also visible in genrerated SO. This works.

EDI messages are generated with "Print PO".

Now the problem: When we DELETE a PO or just a line, this PO can not be printed again, because it is deleted.
So the "delete-change message" is not send via EDI.

How can we send these Delete Messages with standard BaaN EDI ?

Has anybody a solution for this ?!?

Any help apreciated,

Hi Jochen,

never worked with PO changes within EDI, but i have read in the Baan EDI User Guide (U7041C) that there is no way to directly transmit the chancellation of a purchase oder line. To do this, you must change the order line quantity to zero and process the line as a change.

For more information please read page 6-9 in the EDI User Guide
"Changing the purchase order (860/ORDCHG) IVb GLO and IVc only"

Hope this helps a little,

Juergen

Juergen is correct. The procedure is to first set the quantity to zero and process the change. Once the EDI transaction has been generated you can then go back and delete the line.
