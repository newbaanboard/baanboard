Seems to me you have a serious problem. If you get this message when trying to do a inventory transfer, you must have inventory on the item, without costprice? Lately some bugs have been introduced (and solved) in calculating cost price; in some situations it was possible that the old cost price was deleted (on recalculating) without the item data actually being updated. Also there were situations that a cost price in item data was set to zero.

I would advice to install the latest solutions for cost price calculation and evaluate what data in cost price part might be wrong (also think about incorrect revaluations in integration and finance).

The system gives error Cost price not present for this date in 2 condition

1. Cost price of the item not present

For this run ticpr2210m000 (Calculate cost price).

2. Cost price date is greater than the transaction date

For Example, if you have a item X of cost price date 07.03.2005. If this item is used in a sales order dated 03.03.2005 then the system gives error Cost price not present for this date
