11th January 2005, 18:05
Location control requires more steps to process - all of the inbounding and outbounding - and the requirement to not allow negative inventory. In general I recommend that a company that has not had the discipline of running location control or managing inventory effectively that they start out without location control, get to know Baan and then implement later if required. If they do in fact have a big inventory issue with stuff all over then perhaps location control is the answer. Consider also the personnel issue. It takes more to run location control. If this is a small company then location control may not be the best choice. I find that every company is different. Remember that you can always turn it on but once on you cannot turn it off.

Two main pro's in using ILC (in my humble opinion)

1: Possibility to store goods on locations instead of in warehouse (nothing new I guess)
2: All in-and outbounds will flow via the same "visual door", namely in and outbound sessions.

In case of larger number of records (items/transfers/locations) this makes it possible to process movements in a more controlled way.

Hi,
EVen if you turn off the location control by warehouse, the outbound steps are mandatory. Though system doesn't one go back, but it can be done if done with knoweldge. These are the steps (for non Project system).

1. Remove the outbound steps if present in any sales order or replenishment order. PLease adjust the open order and and the pmask of these orders.

2. Make sure that the inv001 inventory tallies with tdilc101 inventory

3. At this stage, the location control flag at company level can be turned off via Genral Table Maintaineance.

One will loose the location history from this point onwards. The old data will remain in the table.

The key here is to adjust the procedure mask of open orders.
