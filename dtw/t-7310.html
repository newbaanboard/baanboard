Having moved the stock to the receipt location by turning on ILC you cannot simply inbound as the stock did not arrive in the location be conventional methods.

Recently I also converted a warehouse in this manner. In order to move the stock quickly I used cycle counting orders and exchange. Firstly I loaded a file which reduced the stock in the receipt location to 0, then I loaded a file increasing the stock in the target locations to the appropriate quantity. This is fairly simple and quick to do.

Also, although converting a warehouse to use location control is not too complex be aware there are several things which should also be done. The warehouse in question will now be subject to both inbounding and outbounding which will have an impact on sales and purchase. ILC parameters should be checked. Picking lists need to be checked since location information may be currently retrieved from INV ond not ILC, and these can now show different values. These are just some of the things which spring to mind :)

If you need more information feel free to mail me.
