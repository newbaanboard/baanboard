Hi,

AFAIK, creating a new unit to the related items is the best and safest way to handle the situation. Changing/Editing the conversion factor might lead to data corruption due to existing transactions for the item.

But for LN FP7 onwards a new functionality has been introduced, which allows to modify conversion factors for items or item groups that have already been approved. Via the specific menu the session "Update Approved Conversion Factor" (tcibd0103s100) can be started and a new value can be entered. This is possible as long as the section "Usage in Inventory Data (blocking)" does not indicate the usage of the item on inventory level.

Refer KB#1043947
