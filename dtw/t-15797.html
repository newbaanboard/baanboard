Hi Ale,

Of course, you are right: whwmd217.

If I understand you correctly: the date in whwmd215 has no relationship with the date in whinr112/113.
The date in whwmd215 (whwmd215.ltdt) is the date on which the last inventory transaction was registered. So, it can be a date when the last issue was done.

The transaction date in whinr112/113 is the date on which the receipt was registered.

As MAUC is only calculated on receipt, the MAUC value in whwmd215/217 should be the same as the last one in whinr112/113. Note that in whinr, the MAUC price is stored (so per inventory unit), whereas in whwmd, the MAUC amount is stored (overall for all units).
The MAUC in whwmd215/217 can be more accurate as it is not rounded.

whwmd217.mauc/whwmd215.stoc = whinr113.mauc
