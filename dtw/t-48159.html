This is on Baan V (on later versions the parameter is on Item Purchase data).
When the parameter is no the approved source list will not be checked. If yes it will be checked and if the item supplier combination then exist then it is used. If the combination does not exist then it is also allowed.
The parameter is on company level and therefor it applies to all items. Most probably only for a subset of the items you want to use the ASL. For these items all possible suppliers must be part of the Item Supplier Information session, tdipu0510m000. You should use the blocked and approve status to make your ASL for that item.
