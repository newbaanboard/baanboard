The multiple units is because the inventory is being used by some process as that unit and baan shows stock as per the unit-wise inventory it has.

For example - EA and KLP. Let's say KLP = 1000 * EA

So, when Baan generates 2 lines when a Forced Count is did for this item, the user can choose to negate KLP and add it to EA, if that is a requirement. Depends on what you want. If KLP is made to zero and EA beefed up, next time, when cycle count is run, you wont see KLP there.

Inventory Structure

If one item is printed with different units, it is registered for this units. You can verify this data in session "Inventory Structure" (whinr1550m000).

When an item is printed with different inventory dates, then indeed it is because the outbound method is defined as FIFO or LIFO. Then again the inventory is registered against these different inventory dates.
