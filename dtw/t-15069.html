Rebuild Planned Inventory transactions for that item(whinp1200m000). You can run this session without checking the "update" check box for verification. This should remove the allocated Quantity of 10 pcs as the order was already delivered.

Second step in this is as you mentioned, you need to to adjustment for 10 quantity.

Rebuild...recalcultes on Order and allocated Quantities...

There can be cases like :

1. A production order is closed , But still you find entry in whinp100. In such case it deletes that record.
2. A purchase order is already received..But still Present in whinp100...It deletes that record... Some times,these kind of cases happens in Baan :-)

I have been using this session for the past few years at different clinet sites...There is no problem with this session..It never does anything wrong..

As per your second Questions...You can do any step first...You don't need to follow the Order..
