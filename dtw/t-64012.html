It should be possible to do this by creating an inventory commitment for the order you want to reserve the stock for. You can only commit what is in stock at a point in time, but once committed any generate outbound run that does not include the order with the committment will ignore that portion of the stock and only allocate down to the committed level, then produce shortages.

Hello,

Also there is an issue with the Inventory Commitments.

Standard ERP LN restrict the user not to generate the outbound advice for an order if the inventory is committed to some other order. But if you try to create the advice (Manually create a record) it will just give you the warning message regarding the commitment but you can save the record & release the advice.

So, make a note of this if you gonna map with standard Inventory Commitment functionality.
