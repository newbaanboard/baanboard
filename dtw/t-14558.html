We are a manufacturer using IVc4 with ILC and Lot Control.
In Baan standard a lot information is sent to the warehouse by generating an outbound advice in tdilc401.
We do not know the delivered lot at that time; this will be later picked by the warehouse and then sent by release outbound advice.
Has someone a solution to create an outbound advice without lot information?
There will be a problem with allocated inventory if other lots are released

The best way to solve your problem is to create an order type with the following steps:

1. Print Order Acknowledgement
2. Print Pick List
3. Generate Outbound
4. Maintain Deliveries
5. Print Packing Slip
6. Print Bill of Lading
7. Print Sales Invoice
8. Process Delivered Sales Orders


You can now print a pick list for the warehouse personnel to use to pick and record serial numbers (Lots) but now you will need to do a Maintain Outbound (tdilc4101m000 & tdilc4102s000) rather than Generate Outbound (tdilc4101m000) to tell Baan which serial numbers (Lots) were picked. Depending on the number of items to be reported, you may want to try and find a way to automate this process either by creating an AFS session or some type of data collection process.
