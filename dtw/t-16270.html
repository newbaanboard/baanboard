Hy Alejandro,

This is the help text for the first field that i am sure you already read:
"When generating inbound advice, BaanERP first checks whether any fixed locations are available where the items can be stored. If so, BaanERP then checks the inbound priority as it has been recorded in the Dados de localização por artigo (whwmd3102s000) session. With equal priorities, the location code is the decisive factor.

When items cannot all be stored in the fixed locations, BaanERP will consider locations that are not fixed if the Aviso de entrada por localização livre check box in the Parâmetros de dados mestres (whwmd0100s000) session is selected.

If more locations are available which are not fixed, these are also handled in sequence of inbound priority. This priority can be recorded with a number by location as well as location by item. The lowest number has the highest priority.

When there are many non-fixed locations for the same item, BaanERP will advise by sequence of location code (when priorities are equal).
The lower the code, the higher the priority."

So the inbound order first checks fixed locations, then the item data and afterwards the free location parameters (where you can define a priority for this in this session) to determine to where it should store.

The second field for the inbound is a way to order the storing list (arranging the locations in a logical and less costly way).

For the outbound is the same thing.. the first field defines the outbound priority for the itens, the second is the picking order sequence inside the warehouse.
