It is not possible to have landed costs codes out of multiple landed costs sets linked to one order. As soon as the first landed costs set has been found to the system stops searching for further sets (and will only look inside this set for multiple scenarios).

The system searches for landed costs sets on alphanumeric basis, so a cost set with code AA1 will be found before AA2.

Within the (one) landed cost set (for example AA1) multiple cost codes can exists an can be proportioned to the order (based on priority if you so desire).
