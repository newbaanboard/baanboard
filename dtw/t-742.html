We have this setup in baan4c4 supply chain I have not seen baan 5 so I don't know how different it is.

on 4c4 create a sales contract then a sales schedule for the contract and if required a shipping schedule, then use automatically create planned deliveries.

The generated orders can be changed in maintain planned deliveries, then released to in preperation, confirm planned deliveries and print advice.

The sales orders are generated at the confirm planned deliveries stage.

We now create the planned deliveries using a flat file and exchange scheme to populate the tables as the generated orders didn't always match what we were shipping to save having to type the changes in.
