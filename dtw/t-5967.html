Here are a few suggestions:

1. Negative inventory adjustment with a specific reason code. Not a clean way but very easy to do.

2. Create a return purchase order type. You need to create a new purchase order and return against the order. Steps specified in order type to be executed. Problem is that it is not linked to the original order in IV (Can be in V) and is difficult thus for Finance. Can use Ref A or B to specify original order number.

3. Add a negative order line on the original order. This way the return order is linked to the original order and for financial purposes very clean. Steps specified in order type to be executed.

Note: you can not generate inbound advice for negative qty. Move stock to receipt location and deliver via "Maintain Inbound Advice" if location control is switched on.

For handling indian localization aspects in purchase returns you have follow different procedures.

1. For rejection at incoming stage, steps to be followed are as under
a. Print return note. (tdpur4411m000)
b. Process excise invoice (purchase) with order type "Rejected after inspection"
c. Print excise invoice.

2. For line rejection steps will be as
a. Maintain purchase order with return order type. (Steps will include process excise invoice,print purchase invoice,PDPO), also return order field should be YES.

b. Process excise invoice with order type "Purchase return"
c. Print excise invoice.
