Hi,

Ok, if mrp isn't the solution, you can try this: in tdinv3201m000 set generate advice per warehouse on yes. In tdinv0101m000 set field tdinv001.uidt (use item data) to no, in that situation the economic stock is calculated using tdinv001 in stead of tiitm001 data.

If that doesn't give the solution there is always the possibility to correct the advice given, by printing it and in the report recalculate the quantity and use afs to change it in the planned orders.
