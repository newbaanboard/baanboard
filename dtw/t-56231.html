The Print Prices check check box was added by Solution# 224578, but that may not be your problem. The Price should have printed before the solution.

Solution 224578 ...

VERSION(S):
6.1_a3

SITUATION IDENTIFIED IN:
"Print Purchase Orders" (tdpur4401m000)

SITUATION DESCRIPTION:
It is not possible to print a purchase order without price information with session "Print Purchase Orders" (tdpur4401m000). On previous versions this used to be possible. This functionality is needed when a copy of the purchase order is sent with the goods. People handling this order should not always see the price.

SOLUTION DESCRIPTION:
A new field "Print Prices" has been introduced in the session "Print Purchase Orders" (tdpur4401m000). With this field the user can decide if the prices need to be printed when the order is printed.

Any customizations to the report?
