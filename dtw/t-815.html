18th January 2002, 09:23
It's a good idea to run this session in a job at night on a regular basis. Typically it solves a lot of problems.

In BaanIV the session you mentioned is tdinv0250m000, but there is also check inventory data tdinv0252m000 which you can run in the job as well with 'adjust' set to yes.
ilc users should also include rebuild inventory data (tdilc0250m000).

I've found it extremely useful to run these in jobs regularly (when no-one logged in) - especially on older versions of Baan. In theory they shouldn't be necessary or a lot of the problems they solve could be solved by installing patches / SPs.

Not sure why Baan should recommend against this. Perhaps someone else could shed some light on the subject?
sturla
18th January 2002, 10:57
I agree with Jason, run the rebuild session on a regular basis.
There is some problems, and thats maybe the reason that Baan don't recomend to run it without serious problems in the system.

I know companies using the A&D extention have had problems using the rebuild sessions, because they have to generata a new PRP (TP) after rebuilding, and that can cause problems. There have also been bugs in the A&D extention, that have made new problems instead of solving them.
Companies using PCS/PRP should do the same (run PRP after rebuilding), but they very often avoid to run PRP on regular basis, and skip the rebuilding.
On older versions of Baan IV, I have seen that tdinv0250m000 have made serious damage if special choises have been used at sales order lines (phantom item at the line and issue components). This have caused a lot of negative allocations on items, that sholdn't have been there. I don't know why. This bug is solved in c4.
Another reason to not run the rebuild is as Jason say, you have to do it when everyone is logged out. That is a problem for companies working 24 h a day or multi site environments having users all over the world (also working 24 h a day).

I know BAAN created a detail report for session "Rebuild Planned Inventory Transactions" (whinp1200m000) in the BaanERP versions: B50b and B50c.
