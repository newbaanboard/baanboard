Hi Vishal,

Order Types allow you to set up different scenarios regarding various sales process cycles that may exist in your corporation's standard operating procedures. Each type must have a definition of how this cycle is to be completed. There are 3 main sessions that must be run: Maintain Deliveries, Print Invoices and Process Delivered Sales Orders.

The mandatory sessions are to allow the system to properly track inventory, bill the customer to maintain finance functions and to complete the order cycle.

Frequently, companies will require the "Print Picking List" session as well to indicate to "order picking staff" the contents of the order.

I hope this helps.

Hi Vishal,

Purchase Orders are similar to Sales Orders in that there are certain required steps to complete the purchasing cycle (depending on the order type). A typical PO type may include:

*Print Purchase Order
*Maintain Receipts
*Process Delivery Purchase Orders

When each step in a PO is completed according to the "Order Type", certain flags are set in the PO header. These flags indicate which steps are "allowed" to be executed for the order type. What this means is that your system has been handed a "script" on how to complete the order and will follow that script to the end. As an example, if a purchase order type does not include the "Print Claims" step, you will not be able to print claims against a purchase order.

The reasoning for this (I imagine) is that there are some things companies purchase that do not require certain steps to be completed. For example, if you are ordering hand soap for your maintenance department, you probably do not need to print a "goods received note" since it is a straight-forward order of a standard supply (either it arrives or it doesn't). If on the other hand, you are receiving components of a controlled item (such as chemicals, firearms etc...) the supplier would most likely require a "goods received note" so they are "off the hook" for any liability claims.

In summary, each order type indicates a set of procedures that need to be followed for certain circumstances. Each of these scenarios should be mapped out for the organization to create an associated PO type that will allow that scenario to be properly handled. If a step is missing from the PO type, it will not (and in most cases cannot) be performed.

There is one step in the sales order procedure and one in the purchase order procedure that you may do even if it is not in the required steps. These are print order acknowledgement and print purchase order. If these are in the order type then the MUST be completed. If they are not in the order type then they MAY be printed. In order to print these you must be sure that the field "Order Line Types to Print" is set to "All Lines".
