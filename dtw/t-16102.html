26th July 2004, 14:13
Hi Gaurav,
try to give a short description what to do.
Before adding sales orders/lines do a call of function tdsls.dll1234.read.initial.values(), so you don't have to deal with parameter tables in your own script.
After that do a call of tdsls.dll1234.add.sales.orders for every SO-Line you want to add (you can get parameter list with bic_info).
After every SO you have to call the function dsls.dll1234.consider.additional.costs , which ends generation.
You have to commit by yourself.
In case of an error the add.sales.order function gives you a DALHOOKERROR . The error message can be checked with dal.get.error.message().
Hope this helps,
Andreas
