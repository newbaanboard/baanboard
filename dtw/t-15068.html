tapzhou

If your cardex system carries different prices per stock line in a manual fifo record, the first thing to do is to calculate an average cost price for the stock you still have on each item.

Then you will have to create the item records where they don't exist. If you have groups of items that are similar the copy item function is useful as is the default item data session where you can enter defaults for many of the item fields based on their item group and type.

When setting up the item data you will come to a field called purchase price. If you enter your average cost here, the system should ask if you want that cost copied to the average purchase price.

Where the records already exist, go back to them and enter the price in the same way. When you have finished with item setup, you must run the calculate and update standard cost sessions; this will enter your purchase price as the standard cost. Without this you can't issue the item to orders

You don't say whether you are using ILC or not - if you are, you will have to set up warehouse and location data.

The simplest way to take on the balances from the manual records would be to generate cycle counting orders. This should produce counting sheets with zero quantities that can be used to capture the balances from the cardexes. Then enter the balances to Baan as cycle counting results and process the cycle counting orders. This will populate your inventory file with the stock at the correct cost and give the first record in the inventory transaction history.
