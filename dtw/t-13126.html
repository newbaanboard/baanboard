Hi Harley

The input for cycle counting orders is table whinr140 otherwise whinh540 ( conditional)

Since in the new company there would be no records in either tables, the cycle counting orders are not generated.

For you a better and a simpler way would be to create records manually in the session whinh5540m000 and checkmark force cycle count.

Now try generating the cycle counting orders from session whinh5200m000 ( force cycle count again) Since there are no records in whinr140 , relevant records from whinh540 will be picked up.

Dear Harley,

We faced the same problem with one of our previous installation before. Turned out that the "Force Cycle Count on New Stock Point" and "Force Cycle Count on Inventory Zero" has to be set to "Yes" from day one. That way, BaanERP will remember to include practically everything if want to do a "forced" cycle counting (by settting "Force Cycle Count" when generating cycle counting orders).

A little note: If you don't do it on day one, then, on "forced" cycle counting, BaanERP will only remember to include the new items created after moment you set the parameter so

By the way, even after setting those parameters so from day one, I personally prefer porting the initial inventory data from Excel to BaanERP using BaanERP Exchange. It's faster than using cycle counting order procedure

Rgds,
Paul
