Hi,

Financial Transactions
Financial transactions are created when goods are shipped (Confirm Shipments) for issues. For receipt, financial transactions are created at receipt (Confirm Receipt). This is irrespective of other order steps. If the received amount is zero, financial transactions are only created if in ACP invoices must be matched against it. If you receive zero pcs for a transfer order; the amount will be zero. As it is not relevant for ACP, no financial transactions are created. If you receive a purchase order for a zero amount the Purchase/Result transaction is created for a value of zero.

Stock Transactions
1) generate O/A: allocation on pick/bulk level + on hand on staging location (whinr1542m000)
- pick advice: decrease allocation + increase on staging (whinr1542m000)
- ship: decrease on staging location (whinr1542m000) + decrease on item/warehouse level (whwmd2515m000)
Note: the quantity in transit in the ship-from warehouse will increase with the shipped quantity. If will decrease after receipt.
2) No receipt, no changes in stock level
3) Receive partially: increase of stock level for the received quantity. In case of final receipt no further receipts are possible for a transfer (manual) order. Hence, inventory in transit is reduced with total quantity.
4) Same as 3)
5) Same as 3)
