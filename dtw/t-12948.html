Your interface is most probably is using AFS programming internaly. When this is the case then I assume you are not using a recent Baan Service Pack. As of solution 115583 dated 2001-06-26 (Service Pack 9) the sales position session "tdsls4102s000" no longer gives the stock not available message you mentioned when the session is running in AFS mode (variable api.mode). The message and the subsequent menu were put in as a warning to the human operator and can not be suppressed bei any parameter setting.

Do you use MPS?

If so you can check to see if the 'Check available to promise for sales orders' parameter in Maintain MPS parameters is set to 'yes'. If this is the case, try setting the parameter to 'No'.

Also, in Maintain Sales Order Parameters you can try setting the 'Time phased inventory check' to 'No'.

Hope this helps
