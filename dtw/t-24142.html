Franseco,
The whole idea of VMI is that supplier manages the inventory. In past I have done following
1. Designate Purchase Order per component.
2. VMI vendor will supply in file, what component he has put along with qty and PO number
3. System would create a PO line and do a maintain receipt automatically
4. If the PO is exhausted (after consuming 999 lines) another Blanked PO would be issued

Ofcourse there was some customization involved. In this way there is no MRP exception messages. I don't think you can use standard Baan supplier rating as it will be always delivered on the date of PO. But apart from that it handles pretty nicely. My client is using this for last 6 years.
