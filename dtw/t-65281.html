Hi,

Check and repair checks for the inconsistent data in a few inventory tables and gives you are report if there are any inconsistent entries. If you run it in simulate mode, it will not correct the inventory in those tables but only give you the report about it. If you run in update mode, the data in the tables will be changed.

I will advice you to run it in simulate mode first.

Also, check these things once again:-

1. Check if there are any Outbound advice's against that item in session Outbound Advices (whinh4525m000)
2. If yes, delete them and recheck the stock point inventory for the item if its showing allocated or not.
3. If no, try creating a new advice as I mentioned with proper lot code and inventory date appearing in the stock point inventory for the item and lot.
If its still showing allocated inventory in stock point inventory for the item and lot, run the session check and repair inventory for the item and the warehouse in simulate and see if prints any report.

A way to run it is as shown in the snapshot.
