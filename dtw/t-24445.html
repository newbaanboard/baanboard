1. For the Revaluation, when the cost price updation is done, the system
prints a report almost exactly in the same format as you want. This shows
prev std cost, curr std cost, qty, change in invrntory value.
It is the normal practise by users to just display this report only instead
of printing. Then when accounts ask for details, they try to give the
Print integration details for Reval, which is not quite the format which
accounts need. So next time a reval is done. pls ask the user to print the
report and submit it to you. As a trial u can ask any one item to be
revalued and then check the report output also.

2. For the ppv, the standard integration report only shows the difference
between the std cost and the PO price in homecurrency. But most of the
operation people would like to see the PO price and std cost also in the
report. This is quite complex and needs to be customised. Care has to be
taken to differentiate between ppv-1 (diff between std cost and PO) and
ppv-2 (diff between invoice price & PO price).
We have customised this but were able to show the details for only ppv-1 and the staff are quite happy with that.
