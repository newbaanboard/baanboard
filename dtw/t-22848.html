When performing a slow moving analysis (tdinv6220m000) we experience the following problem:

For new items that have stock but hardly any requirement yet the analysis results in a "slow-moving" qualification.

Example:

New item A was purchased on stock 100 pcs
Sales orders for 5 pcs have been entered
Slow moving analysis calculates: turnover = cumm. requirement / stock

In this case turnover = 5/100 results in a slow-moving qualification???
This although the item is brand new !!!

This results in a slow-moving overview that is polluted by new items.

Has anyone any idea how to avoid this ?
Kees de Jong
21st January 2005, 23:01
No extrapolation

I would expect that item usage of new items is extrapolated. If e.g. in the last 4 months the item usage is 5, then the expected annual usage is 5 / 4 * 12 = 15.

For the slow moving analysis at least three periods of issue must be present. If not, the Expected Annual Issue of the Item master or Item/Warehouse master is used.

If this does not work it seems to be wrong to me.
