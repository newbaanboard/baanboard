We had a similar problem a few days back, and basically what we did was to set this value to 0 through the GTM.

If you too want to do the same make sure you set the backorder qty to 0 in tdpur041 and tdpur045.

Also you have to set Planned quantity field in tdinv150 to 0.

Note: We have a nightly job which Rebuilds Planned inventory Transaction.
