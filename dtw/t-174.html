Greetings,

Ask your administrator for the EDI Manual in the Documentation CD. Our documentation CD for Baan IV c4 is Documentation CD 5. You will need to download the default file from the Baan ftp website (ftp://ftp.support.baan.com) or take the link from the Baan knowledgebase.

It is not as difficult as it was in Baan IV b2.

Pay attention to relations. The network name should be equal to the relation code in the session "Maintain Relations by Network".

You must define Conversion Codes for Order Types.

Also, note the different "semaphores" used by Baan to check if messages are being generated or are being read.

Enjoy,

Randz

PJ,

Congratulations on you success. I just want to know if you found the documentation helpful or you noticed some more deficiencies that need to be noted and reported to Baan.

The interchange controller works fine, but you have to note that it runs a Baan session. This means that if you're not in the server, the probability of the session being terminated depends on your network and client reliability. I know some people who run the session in job mode off the server. If it is done this way, it may indeed be better to run the session from a background Unix job script. Unfortunately, my Unix script writing is a little rusty so I can't help you much on this at this time.
