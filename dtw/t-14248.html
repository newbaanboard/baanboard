Hi riazsayeed,

Please make sure that confirm backorder automatically is set to "yes" in sales order parameter.

I have tried with both scenario and it works.

For automatic backorder:
System will automatically create one new line in the session release to invoice as soon as I release to invoice the line which has qty delivered < qty order.

For manual backorder:
System will not create any record in the session release to invoice after I release to invoice.
Run session maintain and confirm backorder, select the order line and click "confirm" from specific menu. Then, system will automatically create 1 new record for the remaining qty as holdback order.

There is also another factor coming into play in this automatic back order creation thingy. If in the sales order, the delivery constraint was set to single delivery only, then even if you do just one partial delivery, BaanERP will not let you make any more delivery for the particular SO line. Worth checking that out
