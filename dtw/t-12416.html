In Baan IV it's possible to put the inventory valution method of an item on either 'lot price' or on 'FTP' (tiitm0101m000). If you have it set to 'FTP' you can only change it to 'lot price' in case your 'stock on hand' (tiitm001.stoc) is 0 and the cumulative issue (tiitm001.uscu) is 0.

I can imagine that you can do an inventory adjustment in order to get the inventory on hand to 0 (and then later you book that quantity back to inventory) but how to get the cumulative issue to be 0?

Hi,

First of all the item must be a lot controlled item to change from FTP to LOT PRICE. Try to go to the session tdltc0130m000 where a non-lot item can be converted to a lot item. In this session u have an option of choosing inventory valuation type. Define it as LOT PRICE and continue. I hope it solves your issue.
