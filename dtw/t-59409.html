Hi,

These activities are created at the time of releasing the PO to warehouse module and will be applicable only to those items in which you have defined the item as location controlled in the warehousing data. Also the warehouse must have locations.

In your case may be the item is not defined as location controlled so the activities did not get created. Since the receipt is already confirmed you cannot update the activities as "applicable" now. Try running the session whwmd6290m000 with the item code and location code in which you want to update the inventory, first in the simulate mode and update if the quantities are matching.

regards,
Soujanya

Hi,

It will update only the inventory related tables. No problem in updating.

Regarding the second query, sometimes the data does not get updated properly. After releasing the PO to warehouse, if warehouse is changed from a location controlled to non- location controlled or vice-versa, the activities are over written in warehouse order. In your case may be this has not happened.
