No you won't need a translator for this as the data needs to stay in the Baan EDI (BEMIS) format. You will only need a translator when you are converting into or from BEMIS.

I've never done anything with internal EDI but I think the only thing you need to do is to move the EDI file generated by the Print Packing Slips session from the appl_from directory of the outbound EDI network (sending company) to appl_to directory on the inbound EDI network (receiving company).

We have a piece of bespoke software on our system that handles the moving of the files and sends an E mail to whoever is repsonsible to signify a New EDI Message.

Hope this helps and good luck.

Hello,

Have you set up tcedi0115m000 - Maintain Outgoing Messages by Session.

Relations also have to be set up against each network and then the EDI message is linked to a relation. See sessions tcedi0128m000,0110m000,0111m000.

The ASCII file will be generated when you run session tcedi7201m000 - Generate EDI Messages.

Concentrate on producing the ASCII file at this stage and then worry about the format of the file. I believe that you will NOT need a translator.

Hi Abuibra,

do you get a message like "New EDI-Message generated" when you print the packing slip? If yes a new record is inserted in table tcedi700 (Messages to be Generated). You can check it in session tcedi7100m000 (Maintain Messages to be Generated) or tcedi7500m000 (Display Messages to be Generated).
If you don`t get such a message then there is something missing or wrong in your EDI Setup.

Abuibra,

I am using Internal and External EDI with BaaNIVc4, and BaaN only needs some kind of translation software on EXTERNAL EDI, changing the format of BEMIS to the format of Costumer.

In your case, I agree with Juergen, I think you have some problem with the EDI-message generation: first, you must prepare the EDI-Message with the session described in tcedi0115m000 (it is not a process to do with the incoming messages), then the prepared message must be generated. With that process, an EDI-message called how it has been defined in tcedi5112m000 has to be created on appl_from.

I have sent our EDI setup for our ASN to you to have a look through. Not sure if you've amended some of the following since you first posted the doc.

1) Maintain EDI Messages you do not have an Outgoing Library defined.

2) Maintain EDI Messages Supported by Relation you do not have a conversion setup (as downloaded in defaults.edi) defined.

3) Maybe worth checking form 2 on Maintain Networks where Supress Std Communication should be set to "Yes". (No idea why this is something I was told by a consultant a long time ago)

4) On the doc your Conversion Seup Relationship was blank this should have some fields in it see our doc.

A few other tips. Check out the trace files generated in the directory of your EDI network. Also check the permissions in the network directory.

From looking at your setup I suspect that files are not yet being generated.
