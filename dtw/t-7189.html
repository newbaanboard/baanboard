bisschops
16th October 2002, 16:06
Allowing negative inventory enables you to deliver goods for an order if the goods are physically available but have not yet been registered. This happens if the physical flow of goods is faster than the electronical flow.

Negative inventory

In such a situation, iBaan ERP can register negative inventory in the Item - Warehouse - Negative Inventory (whwmd2555m000) session. Negative inventory is intended to be only temporary.

iBaan ERP only creates negative inventory if the following apply:
The Negative Inventory Allowed check box in the Inventory Handling Parameters (whinh0100m000) session is selected.
For a serialized item the Serials in Inventory check box in the Item - Warehousing (whwmd4100s000) session is cleared.
The warehouse is not of the Consignment (not owned) type or the Consignment (owned) type.
The item's allocated inventory in the warehouse or location exceeds the item's inventory on hand.
Inventory on order exists for the item, warehouse, location and/or lot combination. Negative inventory cannot be more than the inventory on order.
You manually create an outbound advice in the Outbound Advice (whinh4525m000) session. If you use the Generate Outbound Advice (whinh4201m000) session, iBaan ERP does not generate outbound advice that result in negative inventory.


If the inventory on order is (partly) received in the warehouse (when the receipt is confirmed in the Receipts (whinh3520m000) session), iBaan ERP subtracts the received quantity from the negative inventory quantity. If the received quantity is equal to or more than the negative inventory quantity, iBaan ERP deletes the negative inventory record.
