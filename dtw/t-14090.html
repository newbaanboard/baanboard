I saw this once in Baan 5.0b, and the cause then was that the tax code for the sales order line was defaulting from the tax code field on the item record. If there was no matching entry in the tax codes by country table, it used another country where that tax code was valid.

I don't know if this is the same as your problem, but its worth a check.
