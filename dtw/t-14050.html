Hi,
All of these sessions was made to fix some errors in Baan.
By example:
tdinv0250m000 are designed to rebuild planned movements of stock in Baan. You must execute this session when you or you invenory planner/buyer detects some differences between the future stock and the current due to purchases, sales etc.
tdilc0250m000 rebuilds the stock data in your inventory. It takes the movements from inbounds and outbounds and calculates the real stock in your inventory. You must run this session when the inventory area detects some "reserved" items and there nothing to be reserved
tccom2210m000 rebuilds the value that you see in tccom1101m000
form 6. You must run this session when there some wrong value in this field.
tccom2220m000 rebuilds the value that you see in tccom2101m00 form 5. You must run this session when some wrong value is there.

Usually the Baan works good and this sessions are run only when the techÂ´s of baan support tells you.
In a personnal manner in my company often runs the first two of them in order to avoid the errors. Until now only few errors has been reported by this sesions.
Hope this help.
