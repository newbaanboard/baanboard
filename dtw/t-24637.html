Hi Deepak,

The feature was originally set like that so that purchasing people can make emergency changes to price even though the PO has been printed, countersigned by purchasing manager, and sent to supplier. Any of these "emergency" changes will be recorded anyway in PO line history, so any abuse of this feature can be monitored (but still can't be prevented). But if you are absolutely sure that people will not need emergency changes like this ever, then you can go ahead and customise the script (make use of the procedure mask to handle this)

Rgds,
