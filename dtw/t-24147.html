I think it is a bug.
Order header discount is only disabled (not defaulted from supplier data and also not editable) for cost orders.
For subcontracting orders however, the order header discount is defaulted from the supplier data. As a consequence, the user must be able to change the defaulted discount as long as no delivered line is present.
