Hi dhnish,

You may not delete any itemcode as long it refers to any other table in the company. In order to delete itemcodes but keep the historical data alive, you have to archive transactions, salesorders, productions orders etc. This is usually done by the standard archiving sessions available for ever package. During this process the data are physically moved to another company.

Hope this helps.

Item deletion program des not delete an item if it's used in any transaction tables like sales , purchase , production , histories , inventory etc . The deletion program ( suppose tiitm0203m000) deletes BOM / routing on its own if it's inventory is 0 .

The item deletion program however does not check reference in other logical companies (important if ur item master is shared across companies ) and it may give u fatal error .

U can use ttaad4229m000 program to check item references .
