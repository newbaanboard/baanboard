Hi Dinesh,

In this case, if only once you wanted to copy the master from one company to another company in that case you can take the sequential dump of the master table from source company thru session ttaad4226m000 and import the same thru session ttaad4227m000 into the destination company.

In case if you required to share the table between two companies in that case you have to follow the following steps:

CASE 1: If no data present in the table which needs to be shared.

1. Go to the session ttaad4120m000 insert the record like company, table and physical company details.
2. Save the record.
3. Run CRDD

CASE 2: If transaction is available in other company. In this case first you have to verify that there should not be any duplicate records in both the company. If duplicate record is there in that case both the record should have same parameteres (All the fields and its values should be same). After verification you can follow the following steps:

1. Take sequencial dump of table thru session ttaad4226m000 from child company.
2. Import dump thru session ttaad4227m000.
3. Go to the session ttaad4120m000 and insert the record like company, table and physical company details.
4. Save the record.
5. Run CRDD

this way data will be merged to master company and table of master company will be shared between the company.

Hope this will solve your problem.

regards,
