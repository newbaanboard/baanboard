Hi Moon Lok,

I agreed with you that item B should not in the project in normal situation.

Item B is actually a dummy item in the project. Some of our projects is used to manufacture space parts. In the project parts, we have a long list of the space parts item code. We make a dummy BOM with the parent as item B and all the space parts item code maintained as the child of item B. Item A is just one of this.

With this dummy BOM, all the items are linked and we can print out all the item easily with session Print Customized BOM(multilevel). This in turn help us to minotor the project easier.
