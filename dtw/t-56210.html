I have simulated this situation and it happens when the field "Issue status" in the form-2 of the Sales order line has the value "Issue Components".

As you know, for Manufactured items (incl Phantom items), it is possible to issue either the Main item or the components. The default is "Issue Main item" for Purchased items and "Issue components" for Phantom items.

I don't know if this is a bug or if there is a valid reason why a component with Inventory valuation as "Lot Price" cannot be used in the BOM when the the default of the sales order line is "Issue components".

But when i changed the value of the field to "Issue Main Item" the Sales order line got saved.

So in your case what do you plan to issue ? I suggest you try to simulate the order cycle in your test company and check all the documents generated (like Packing slip, DO, Invoice, etc..) and only then change the field value.
