Inventory adjustments up (increase inventory) will always be done at standard cost. There is no other place for Baan to get a value for the item, unlike when you buy it on a PO, or produce it on a production order.
An inventory adjustment down (decrease inventory) will be done at the value of that inventory.

Hi,
As one of the member mentioned, negative inventory adjustment happens based on the MAUC. And the correct answer for your positive inventory adjustments question is:-

when you do a positive adjustment choose the existing inventory date by zooming in "Valuation Date". Each valuation date stores different cost (i.e. based on the receipt / inv adj transaction. By default baan will put the current system date and time. Since there is no cost exist for the current date and time baan gets the cost from Standard cost.

Refer the session whina1512m100, it stores the values based on each receipt transaction date. (it's like lot pricing, but uses date to store the price). Hope it clarify your question.
