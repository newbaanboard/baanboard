Is there a way to avoid system searching at default sales price book for specific customers? In our environment system gets the sales price from the default price book and then applies discounts based on the discount matrix.

For a few specific customers we maintain a separate price book and search criteria in price matrices. If for any item the price is missing in that price book, system must show 0in the sales order. It must not bring the value from the default price book. Is it possible in ERP-LN?

HI,

Yes your requirement is very well possible in LN.

Follow the below mentioned Steps:

1. Define Matrix definition (tdpcg0110m000)
Here you have to select the define Type(Sale Price) attribute on the basis you want to define the criteria. As you said you want to define the separate Price List for different BP's so you have to select BP in Attribute. There are many more criteria available here.

2. Define Matrix Definition (tdpcg0120m000)
Define the Field's sequencing.

3. Define Price Matrix (tdpcg0130m000)
Select the Matrix Definition defined by you, and the PRICE BOOK to be used for the defined Attribute.
define the Exact Combination of Attributes on the Sales Attribute Tab.

In this Similer way you can define the Discounting Matrix Also.

But when an item was missed in the appropriate price book, default price book was referred by the system and that price was applied in sales orders. Those prices are way off the mark for these customers and users do not know until customer complains.

I defined another price matrix for these customers with the lowest priority and linked the default price book(intentionally, though it is not desired) and applied zero as price factor. This way system is showing zero price in the orders if the price is picked from default price book which is the last priority for these customers. This will stop users applying wrong prices I believe.

You are right, whenever system not founds any specific Price List for defined combination it will refer to the Default Price Book defined in Pricing Paramenter.

You can handle your scenario by Keeping the Default Price List on ZERO Prices, if some BP didn't falls in the specified criteria, it will go to the Default one but will find only ZERO Price.
