I can see why you would want to do this, we have suppliers that deliver every 'Tuesday' on their regular route, it would be nice if Baan's MRP created orders with the correct day of week.

I don't think there is a way in standard baan to do this.
If I was given this problem, I would probably create a processing session that would update the planned order date and planned delivery date in the mrp020 table (MRP planned Purchase Orders). Then when you confirm and transfer them into 'real' purchase orders, they would have the correct day of the week on them.

I would probably write the processing session so that it 'condensed' the requirements, so if you had planned MRP purchases every day for a particular item, it would sum them together, and leave you with only one planned purchase order, with the correct day of week information on it.

Typically I don't recommend manipulating the planned orders like this, but since they are going to be deleted with the next MRP run anyway, I don't think you would get into much trouble.

Another option would be write a report to gather the requirements into the correct buckets, and then have a user manually create the PO's based on this report.

It depends on volume and complexity, which approach you want to take.

Good luck.
