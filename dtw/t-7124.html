Dear karlovac,

PST is some simple yet useful data mart module for PUR activities. You can view a time phased summary of PUR activities (I like to call it PST's 4th dimension). The PUR activities can be categorised, for example by supplier group, supplier, item group, item etc (1st dimension). You can also choose whether the summary display actual receipt, order intake, or order cancellation (2rd dimension). The number displayed can be either amount, qty, weight, etc (3rd dimension).

Once set up, all you need to do is fill up the cubes with data from live company and you can arrange reports using any combination of PST's dimensions that you like (for example, the manager want to see the comparison between actual receipt and order intake amount per supplier group for every year during the last 5 years). And since the data are already in the cubes, producing reports that span large data in PST is much faster.

Once the cubes are filled you can archive the PUR if you need to and the data will still be in the cubes. Problem is if you have deleted the purchase order history for very old data, then those data can't be ported to the cubes anymore.

Rgds,
Paul
