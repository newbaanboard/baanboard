Hi

in releases like LN purchase requisitions have been introduced: The purchase requisition procedure is designed for the non-purchasing user (someone who may not know the standard procedures in purchasing). For example, engineers can order material and/or services, but they do not need to understand the entire purchasing process to do so.
On a requisition, the item code and buy-from business partner codes can be blank. This difference means that the requester can make a request for a new item or make a request from a new buy-from business partner.
Later the requisition can be turned into a purchase order or request for quotation.

Starting for B5.0x and above, you have "Order Input Controls" where you can decide which fields you want the user to input and other fields will be defaulted automatically. In that way, they don't have to tab through fields.

Check the session "Purchase Order Type - Input Controls" (tdpur0195m000)
