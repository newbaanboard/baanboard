Baan support respond to us. It is an standard functionnality. I call that an hidden functionnality because there no parameters and it is not tell in baan docs. Here is the full response of support :

We checked the sources and did some tests our internal system. Here the outcome of our analysis with the Product Owner of Baan ERP Warehousing:

What you observed is the standard functionality. When the shipping data, planned delivery date (& time) and delivery data are identical, the system consolidates the relevant order lines into one shipment ID.

The system is indeed not checking on sales order number. The address is what is important. There is no parameter to split this.

In case the following properties are the same, the same shipment ID will be used (as long as the shipment is not confirmed yet):

· Ship to code
· Ship to adres
· Ship to type
· Ship from type
· Ship from code
· Ship from adres
· Route
· Planned delivery date
· Terms of delivery
· Load
· Carrier

So basically if you ship something to the same address with the same delivery date and carrier along the same route, it will be put on one shipment Id. Which makes sense. It should not matter with which sales order the goods are delivered.


As said they have tested on them system to check that, so they was not aware of the functionnality....

So beware, if you using baan ERP 5.0c take care of that!
