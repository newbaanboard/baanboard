It was implemented with solution 142117.

Some details:
When generating the purchase consignment payment order, a search will be done for already generated purchase consignment payment orders. If found and appropriate for use, the order will be re-used. An order line is added to that order.
Fields that must match when searching a consignment payment order:
- Ship-from BP
- Consignment Payment Order Type
- Purchase Office
- Currency
- Self Billed Invoice flag
- Buyer
The Self Billed Invoice flag and the Buyer are defaulted from master data, before checked. Reason for this is that the values of these fields can differ per item.
An order line can only be added if the order is not yet closed. That means:
- None of the order lines is matched / approved in accounts payable in finance.
- None of the order lines is processed in session Process Delivered Purchase Orders (tdpur4223m000).
