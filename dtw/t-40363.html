This sounds like there is an existing standard item with the same item code as the configured one you are creating. On our system it will always choose the standard item over the configured one. Even if the standard item has a signal code which leaves it blocked, it gets chosen.

Our work-around, to ensure we can use the configured item, is to rename the standard item to have an "X" character at the end uisng GTM. Loses the link to inventory history etc. but it works for the moment.

Alternatively, you could make your configured item code smarter - that is more unique, so it's not the same as the standard item.
