Standard BaaN will not allow to run multiple EDI networks at a time.
The EDI network communication session has to be patched first to run multiple networks at a time.
Then dedicate each network for Outbound and Inbound respectively in each companies.
An external shell script/ program (based on the operating system) has to be developed that will pull and push the EDI files from the appl_to & appl_from directories to your third party EDI mail box. The semaphore EDI processing files can be used to start and stop this external program.
Run the EDI networks in batch as frequent as required to make it real time processing.
