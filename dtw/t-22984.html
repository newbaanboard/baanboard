17th February 2005, 10:49
AFS is Application Function Server. You can enter data through the front end of Baan, programmatically. This means that all default Baan messages/popups/errors are captured and written to a file and hence you upload data programmatically through the front end and hence without bypassing the application layer.

Prevents any upload error to a table if done through backend. There are many many posts on AFS in this site. Check them out if you are interested.

Session tppdm7260m000 looked promising initially but am afraid it would not be useful.

It requires that we maintain data in sessions like tppdm7550m000 and tppdm7156m000. These tables maintain mapping between the supplier's item-code and unit-code and that of the one found in tcibd001. If this is not maintained, we are not going to get update on supplier-specific price book or on Default price book.

Too many constraints as I go deeper. I gave up. I would write an AFS on session tdipu0101s000 to update Item Purchase Price so that it updates Default Price book automatically. It will be a different story if I have to update the Supplier-specific Price book.
