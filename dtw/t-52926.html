You can use Item description for creating the Requisition, however before transferring it to a PO you will need a Item code (that is available in Item General). Also you will need a one-time BP (Supplier), you can attach a address to this BP on the fly or use the dummy address.

There is nothing like on-time Item, however you can use cost items for all miscellenous purchases. Hope this clarifies...

Dear Mujtahid,

You would be entering item code (Cost item) in the requisition before converting it to Purchase Order. Enter the description of the item in the Requisition line text field. When the PO is created, then go to PO line session and remove/uncheck the 'standard description' field. This will print the PO Line text in the PO print. Pls note that the text from the requisition gets copied to the PO Line text.
