I'm not sure that you can do this with incoming EDI in Baan.

If you are using translation software, you should be able to do it there.

If it is intercompany (Baan to Baan), you should be able to do it on the outgoing transaction. In Conversion Setups, just replace the segment that maps to the tdsls040.ddat field on the incoming transaction with one with "date()" (without the quotes) in the "Field" column. Look at one of the Global Overhead conversion setups for an idea on what this should look like.

So in our setup, I would replace the line for tdpur040.odat on the outgoing transaction, with one that had all the same values, but with "date()" instead of tdpur040.odat. But test this, because in our setup this is a new record line and I'm not sure what will happen if it is changed.
