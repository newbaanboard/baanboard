Hopefully there is a wonderfully simple solution to this one:

We have implemented Lot Control for our finished goods. Each pack produced receives a unique Lot Code for traceability purposes. When we ship these goods, each pack is given a unique sequence/line in Outbound Advice and the subsequent Shipment. This is not a problem.

The problem lies in what happens to the Sales Order Line once the related Shipment Line has been confirmed. As part of the confirmation process, a new Sequence Line is created in Sales Order Lines (tdsls401) for each Pack/Lot shipped. This I could understand if the newly created SO Line specified the individual Lot that it was related to - but it does not - instead it specifies a Lot Selection of 'Any'.

This causes us to have a lot more (no pun intended) data on the system in the guise of the additional SO Lines and also it has a negative impact of our sales reporting abilities. We now have 5 page reports where there used to be 1 and we can no longer simply identify where an additional delivery has taken place or how many. Before we introduced Lots, a simple deduction was that 1 seqn = 1 delivery.

Is there any way to consolidate the shipped lot qtys against their original order line/seqn?
