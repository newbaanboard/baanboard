Onyour issue with Release to Invoicing - we have the same issue. We have some charge lines that we use a quantity, and some that we don't. WHat we did is to create a custom script that will go in and basically 'deliver' the cost item, once it sees a shipment for the sales order. Also, we developed a custom Release to invoicing session to check that ALL lines on the order were ready to be released, as the customer wants one invoice per order. If there are any lines not ready to be released, or if one of the charge lines with quantity were not yet delivered, it will not release the order and will send back an error report.
Also, could you add your freight without quantities and just use an amount?
