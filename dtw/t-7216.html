That is not how the cycle count works.

e.g

if you create a cycle count today then the on hand would be 1000, then you issue 500 to production order. tomorrow process the cycle count with a value of 1050.
The onhand stock would change to 550 and a cycle counting warehouse order would be created with the date of the cycle count adding the 50.

Dear all,

If I'm not mistaken, what bizen99 is actually asking is whether bizen99 has done the correct thing to just enter inventory transaction tdinv1101m000 with type inventory adjustment and past date as transaction date. If indeed that's what bizen99 is asking, then the answer is yes, that's the way to do it. Inventory adjustment can be set to happen in the past, just enter the past date in transaction date.
