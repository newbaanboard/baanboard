RE: Secondly how can you get the valuation of only accepted material, when Inv.on hand includes inv.on hold.

Sudhakar:

If I understand your terminology, the inventory on Hold that you are concerned about is inventory received (delivered) against a PO line where the receipt has not yet been approved. Accepted material would be inventory that has been received and approved.

If a PO line does not require inspection, the full delivered qty is approved upon receipt and there is no held inventory. So the only concern is goods that require inspection that have been received but not yet approved in Maintain Approvals.

I agree totally with Francesco that the received inventory has value that should be recognized in the ledger. That is how Baan functions. The receipt in Maintain Receipts generates integration transactions: a Purchase/Receipt and (if applicable) a Purchase/Result. The Purchase/Receipt creates ledger inventory.

If the line item requires inspection, then Maintain Approvals generates two additional sets of transactions. The first is a reversal of the original entries: negative amounts as Purchase/Receipt and Purchase/Result for the entire received qty. The second is a Purchase/Receipt and Purchase/Result for the approved qty. The value of the rejected qty simply disappears from the ledger after approval, regardless of what is done physically with the rejected goods.

Because the Transaction Origin (Purchase), Financial Transaction (Receipt), warehouse and cost price components are the same for all of these transactions, the integrations post to the same ledger accounts and you cant identify the Accepted value or the Hold (awaiting approval) value from the ledger itself.

If the Finance team wants to identify the value of Accepted inventory included in the On Hand inventory, you will need to calculate the value of inventory on Hold (not yet approved) and subtract it from inventory on hand. You will need to write a report or query as I not aware of any standard report that gives that information.

The simplest option is probably to value inventory on hold at a high level as Francesco suggests.

Another option would be to work from the receipts table (tdpur045). I think that if you select receipt lines that have Delivered qty > 0, Approved Qty = 0 and Rejected Qty = 0, then extend Delivered Qty by the Std Cost (also from tdpur045), you should have a reliable calculation of inventory on hold.

If ILC is implemented, you could instead produce a report of inventory on hand in your various inspection locations and extend it by std cost (or Lot price if LTC and Lot Pricing is applicable).

If finance says that it also needs to know what ledger accounts the held inventory posted into, the calculation will be considerably more cumbersome and beyond the scope of a simple query because it will be necessary to use the standard cost from ticpr300 and evaluate the integration mappings in tdinv850 for a Purchase/Receipt in combination with FSG, Warehouse, Item, Item Group and Cost Price Component. Frankly, I cannot think of a business requirement compelling enough to justify the effort.
