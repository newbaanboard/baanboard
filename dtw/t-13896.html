You should do some database tracing and save the uncertainty :)

DB tracing will record all db actions to a file for you to review when done. Just set up the trace when you log in, run the desired session, exit, check the logs.

Here's a link to setting up the trace.
Link with example (http://www.baanboard.com/baanboard/showthread.php?s=&threadid=1068&highlight=TTSQLTRACE)

The commands they are discussing go in the 'Command" field of your BW before you execute it (Right Click on your BW and say 'configure'.)

Here's a link to the a full doc on tracing:

Link to full tracing doc (http://www.baanboard.com/baanboard/showthread.php?s=&threadid=7665&highlight=TTSQLTRACE)

As suggested though, AFS would work if you want to duplicate the logic. DB Tracing and (if you are trying to ) backward engineering is not very reliable. You only get a result for running the data through 1 way. There could be many different updates depending on the data setup, especially in a complex process like that.
