There can be many considertions. Try to tell u as many as i can recollect.

1. Package combinations have to be same for all such companies. So separate
business logic for different logistic companies may require extensive
detailed designs
2.If the logistics companies deal in same set of items (ie same LOB) , the item master
, BOM / routing etc have to be maitained in all companies separately or
separate program / exchange schemes have to be written to keep these masters in sync.
3.All companies in the group must have the same home country / home currency. This may not
be the case in ur case.
4.All financial companies must have the same period definition in finance.
5.All companies must have the same master data for suppliers,customers,
and tables (stored centrally and applying to all companies within the
group). This may not be the reality if each logistics company has its
own set of supplier/customers.

So Baan IV multi-company structure can be useful for multi-location within the country
having same legal entity.

U can try the MCR version of Baan IV (if possible) with a leading currency as base currency
if it's spread across multiple countries. And have consolidation through
such base currency.
