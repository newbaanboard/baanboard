Hi,

System will not generate any receipt number for collect orders. Once you complete keying-in a purchase order line, subsession tdilc4113s000-Maintain receipts(ILC) will be invoked and you have to tab through that session which will default with the order line quantity. If you see the receipt no. in 'display receipts' session, it will show zero as receipt number. This is the standard functionality. In fact, while creating the order type, you might have noticed that system did not allow to add 'Maintain Receipts(tdpur4120m000) as an order step for collect orders.

If you want to change the received quantity, you have to go to the Purchase order line->special->zoom->Maintain Receipts(ILC) and do the changes. Also there is no back-order business for collect orders. Meaning, if you change the quantity in tdilc4113s000, PO-line will automatically change to that quantity.
