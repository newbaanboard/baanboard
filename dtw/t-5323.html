Don't use "create table from sequential dump" for importing data.
It's just a big DON'T!. You will have to do updates in tables (like (pcs) item master, and you can't do that by just importing data).

Use the exchange module, or a script to do it.

There aren't too many tables you will have to fill, so exchange will do fine and you can do this faster than using a BOI where you have to figure out all the possible user interactions (zoomed-to-sessions and so on).

Get in touch with your consultant to help you figure out the tables (should be done in a day or so, if you don't have too much specifics)

Yours,
Uli

Hello,

I have done the migration for the intial stock value several times now by using exchange.

I have done it in 3 steps:

1. create the locations with the baan standard session
2. import the relation item <-> location
3. import the cycle counting orders and follow the normal
procedure for cycle counting in Baan.
(Attention: If you have to import more than 999 records you
have to splitt your import file. It is not possible to have more
than 999 records per cycle counting order!)

If you need more information I can prepare some screen shots
from the import schemes.

Hi Michael -

We also are using IVc and had to bring in existing charges for projects. However, we went live with Distribution, Manufacturing, and Finance all at the same time.

It is my understanding that customized items go directly to a project when they are received - that is, they go directly into Work In Process and do not "sit" in inventory.

We utilized the "Maintain General Projects Costs" session (tipcs3140m000) in the Manufacturing module to bring in existing project inventory values from our legacy system. This, combined with affecting a separate specified account number when we imported our general ledger, allowed us to verify that the correct amount had been applied to each project in BaaN.
