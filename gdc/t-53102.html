A new blog entry has been added:

Working without BAAN source code

We never had source Code. Never wanted it.

Keeping Baan free of "Customizations" has allowed clean updates without issues.

What I did instead is to write a considerable amount of software in my own environment that accesses all my Baan Data in Oracle. I add the additional functionality I need there. I do not own any third party reporting package. Everything I need is provided by my own software including P&L's.

NOTE: I only Read the Baan Database. I never have and never will make changes to the Baan Database outside of the Baan Software.

The only problem with not having the source code is when a problem arises, you must contact Baan to make any changes necessary. Our problem is, we have the source code, and can look at it, but cannot compile or make any changes to it. In relying on Baan to make any changes, delays occur. Sometimes the users want a change in a hurry, and you cannot make it. Usually it is a change to a report, but, it still has to go through someone that can change the baan source code.
