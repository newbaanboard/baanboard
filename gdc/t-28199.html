The only customers on "Reger" (ERP 5.2a) are (to the best of my knowledge):
1) Barco
2) Sperry Marine
3) Manlow

Boeing is using the ERP 5.1a version, which is also known under the project name Verdi and at this moment they are also implementing LN (FP2) for the production of their new airplane.
Verdi and Reger differ a lot, but the same goes for Reger and LN.
If you want to go from Reger to LN, then you need quite some migration logic to perform that task. This migration logic is not present to complete that task. At this moment there is migration logic available from BaanIVc (standard, multi currency or german extension) to LN (only single site).
Recently also the migration logic has been completed from the Corelli version (SSA ERP 5.0c, no localization) to LN.

13th October 2006, 10:43
Hi,

The migration tool can migrate a multi-site environment from BaanIV to LN, but just as it is. What you see is that more and more customers would like to go for a "single logistic - multi finance" structure in LN. This scenario was not very well supported in BaanIV. So if you have a multi site structure in BaanIV and would like to change that in LN, then that is not possible. If you are happy with the multi-site structure in BaanIV, then with migration, you will get exactly the same multi-site structure in LN.
