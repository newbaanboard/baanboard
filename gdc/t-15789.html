Hi Mraguthu,

nice issues.

I still have a problem in understanding the job problems you have. A licence is taken by a bshell. So if you run several job daemon then you have multiple bshells and you consume multiple licences. The Jobdaemon is designed to get the jobs serialized. Only one job is running at the same time. If you want to start multiple jobs for one comapny at the same time then you can use the scheduler of the AS400 machine. Then the bshell is also only running while the job is executed and the licence is only used for that time. Multiple bshells also means parallel processing if you have more than one CPU.

About the MQ I have no experience. According to the second redbook of Baan on AS400 this should work.

OK, QSH is a product of IBM. So the most needed commands are implemented. Which ones are missing? Several of the functionalities of the missing commands can be created with the available commands in a script.

For all convert to runtime sessions there must be a flag about the items to handle. If this flag does not exist then the convert is not running. So our experience is that the convert does not run if you do something next to the sessions. What says the support about this?
