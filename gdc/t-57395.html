PMC is one of the good tools where you can easily gather the dependent solutions as well as generate/evaluate the dependencies for your in-house development if you use PMC Distributor module with it.
Refer to the PMC documentation for more info, you can also create co-requisites for your custom PMC solutions linked to the Baan stnd solutions.
You do not just install any solution into Production. With the proper Change Management and Code Migration processes, the solutions have to be deployed from Dev -> QA -> Production. If you have proper dependencies/co-requisites generated for your solution, it doesn't install if they are not present in the target system.

JoJo,

I am going to make an assumption that you only process the solutions that you need and not the monthly dumps.

You can create a second Update VRC from the standard VRC. In this second Update VRC process the monthly solution files. From this Update VRC you can run reports, or extract the dependency information and populate the web site.
