On the server there is this file in the bin dir: bic_info. Use the flag -a and you get all the information the DLL writers want to share with you in the DLLUsage and FunctionUsage documentation sections as well as technical information needed to call a DLL function. You can guess a bit what the functions would do by reading the "strings" section and other information provided. Use the -h flag to get some explanation on the usage of bic_info.

Hi,

even if you do not have a developer license for "ttstpbaandoc" you can still use following to help you:

Use "Main Menu\File\Run Program" and enter ttadv2439m000 in Proram/Session-Field. Tnis starts the Session "Print Objects by Program Script/Libraries". Chose your Library in the Selection Range area and select the Option "Show Exported DLL Function Prototypes" and the select the Option "Print". The exported DLL-Functions are printed with all the input Parameters (see example below). The Programmer Comments that the Session "ttstpbaandoc" generates are unfortunately missing here but nevertheless this is a very big help in most cases :)
