En -
I have been working with sych dialogs for a while, and specifically right now. Mine all work the way you see it usually - both on the same main table, but clearly the doc's show you can use it in your way. I don't find that the 'next form' option is needed to browse records - the 'next record' option is, is that what you mean?

From your description, it looks like during generate of the 'm' session, you may have had some options not filled in (that darn generator is a pain, and resets if you move around too much). The fact that your next form / next record buttons were not set right is a clue that generate did not go well. I think maybe if you regenerate your form it may help. Try generating again (a new number so you don't overwrite your current). Check that the following are set in the Generate:
Session Type: Display (does that work ok for you?)
Start Option: I use 44, that's what my doc says, can't see how that would effect your scrolling though.
Window Type: List Window
Main Session: Yes
Form Type: 2 or 3
View fields set appropriately
St. Command Set: Should default to M2.S or M3.S , or D2.0 D3.0. I think any of those is good, but make sure something appropriate comes in.
Occurances: Default should be 8 or 10.


Also, for double clicking, you can try this post and see if anything is new for your version, the default button should be executed on double click, but there is some doubt in V. I'll be testing some things out over the next few days if you do not get a better answer I'll post more.
