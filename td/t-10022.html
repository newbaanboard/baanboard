When using stpapi.put.field with enumerated fields, you might want to try using the integer constants for the enumerated values instead of the strings themselves. For instance, instead of stpapi.put.field(s.tdilc4203, "print.recom", "tcyesno.no"), try using stpapi.put.field(s.tdilc4203, "print.recom", str$(etol(tcyesno.no))). The same would also apply to the koor.f and koor.t fields. The etol function converts the enumerated constant into a long value which is then converted into a string through str$. In my rather limited experience, I've found that this usually works.

Just a suggestion: for AFS specific questions such as this, you might get a faster response on the AFS/OLE SIG forum instead of the Tools Development forum.
