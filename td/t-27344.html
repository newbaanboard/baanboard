The way I solved this was using arrays. So what I did was if the user wanted 5 columns(in my case weeks). Then I would pass the counter 5 to the report. In the headings and the details I would use the print expression to decide what to print. For example if they wanted 1 week the first column heading had counter>=1, second column had counter>=2 , etc. for the columns and details. I limited it to 10 columns(all that would fit on the report). In the program script I would then populate the array for each column. Example: if the user select input 3 for # of weeks to print

Headings: Week 1 Week 2 Week 3
details : array(1) array(2) array(3)

Where array contained the data the user wanted. Sounds like this would work for you - you can ask for the number of item groups. Pass that to the report and use it for you counter.
