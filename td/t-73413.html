In Billable Line session (cisli8110m000), there is a button called "Create Invoice".
The button, when clicked, will do its standard process and print out the standard invoice report without having a print session or any reports attached to the session itself.
The question is how can I add reports to that particular process of the button.
The desired result is that when user clicks the button it will show report selection rather than directly print out the standard invoice report.
JaapJD
7th September 2020, 10:19
You can create a report extension to select another report in the Get Alternative Report hook. In that hook you can use session (extension) fields to determine the report the user wants to print. So, no report selection from a list, but a field in the form that indicates the report to print.

JaapJD
7th September 2020, 10:28
You can create an enum domain (in tx) with the different reports to select from as enum values. Put a field on the form (with session extension) with that domain, and the user can select the report he wants. Use that field value in Get Alternative Report hook in a case statement to return the correct report code.
