If both of those table fields are defined as input fields on the report, then you can just create a new field in the appropriate layout of report associated with your SQL Query. When you define the field in the layout, just give the formula. You can just choose the domain used for those other fields.

This does require some knowledge of reports, but it is pretty straight forward. You shouldn't have to change your SQL Query.
morpheus
4th December 2002, 07:31
Evesely explained you about using the formula in the report layout.
Regarding using a variable, declare the variable (say...rqty) as extern in the script. Also, check the domain of other quantity fields and associate rqty with that domain. In your script use -
rqty = tisfc001.qrdr - tisfc001.qldv.
Define rqty and other fields in the report input fields. Use the variable in the report layout, and that's all!!
Hope this helps.
