Terminal I/O is quicker than disk I/O. I do not know ratios and I am sure these all depend on your drives, setup and configuration. Anyway when Baan writes to the display, it first goes to a tmp file and then is sent to the display. If you watch on large slow queries or sessions the display will start printing before the session is actually complete. So it sends it to a tmp file and then converts it for display. When sending to a file it first goes to the tmp file and then gets converted into the destination ASCII file. So more disk I/O and maybe even the conversion to ASCII is extra. Thats my take on it. Maybe a real expert will stop by and post a better answer.
