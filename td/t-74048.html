Basically (and there were more events) I just checked to make sure allocations were correct and would not let the leave the records until they were correct. Made a few of them mad - but that was the requirement by finance. :) What it did not do was prevent them from using end task. Probably something better and fancier for LN.


choice.last.view:
before.choice:
verify.allocations(false)

choice.def.find:
before.choice:
verify.allocations(false)

choice.find.data:
before.choice:
verify.allocations(false)

choice.next.view:
before.choice:
verify.allocations(false)

choice.prev.view:
before.choice:
verify.allocations(false)

choice.end.program:
before.choice:
verify.allocations(false)


choice.abort.program:
before.choice:
verify.allocations(false)

function verify.allocations(domain tcbool update.check)
{
if not check.allocations() then
message("Allocations not correct for this Project/CCN.")
| Do not do choice.again when hitting save button - save the record first.
if not update.check then
choice.again()
endif
endif
}
