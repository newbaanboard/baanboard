Check out the AFS/DDC/OLE Forum (http://www.baanboard.com/baanboard/forumdisplay.php?f=59). Do a search on OLE and you should see some examples of what you can do with VB. I personally do not trust OLE automation - this is a personal choice. Baan once told me it was no longer supported and I should use Openworld technology. The problem is that one portingset and client we used broke the OLE technology and I could not connect to baan.

Here is how I would solve it:
(1) create excel macro/vb macro to export data in format I want.
(2) Write Baan program to:
(a) execute macro created in (1)
(b) use client2server to move file up to the server
(c) use string.scan to read file
