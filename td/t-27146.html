What you can actually do is create an excel macro which would pass variables to a Baan library which would insert the record into the Baan table. Search this AFS/DDC/OLE: Function servers (http://www.baanboard.com/baanboard/forumdisplay.php?f=59) for excel. You should be able to find some examples of how this works. You might want to also search this forum - I think just recently someone posted the sample spreadsheets.

Personally I would rather write a Baan session which launches a macro and saves the file as a ".csv". Then you can process the file one record at a time in Baan. What I typically do is have the users save their files as a ".csv" then they can run my sessions to process the file. The only difference is you want it to be automatic - so I would then have the session launch a macro to automatically create the csv file and then load the baan table. Then you could schedule it in batch to run once a day or as often as necessary.

Way back in the old days I wrote some C code that would convert Dbase files to text files. But that was 15-20 years ago.
