Hi raikar_raviraj
Are you using a level II driver with SQL if you are not then you will have problems just loading the data directly to sql as baan will not display the data unless the index fields baan uses with a Level I driver are correct. there are are ways to create the index but it can cause problems if you get it wrong. But if this is data from outside of the baan system that is not used by any standard baan sessions then it is probably not a problem and there are post on the board that discuss this. If your using the level II driver then this will not be an issue.
Rather than use a macro to update sql take a look at the DTS module in sql it allows you to create jobs that will poll the exel file (any data source infact) and then upload the data to sql at set intervals.
