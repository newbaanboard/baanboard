•DBSLOG
See information on tracing with DBSLOG. This variable can also be specified in the db_resource file.
•DBSLOG_LOCK_PROF
The DBSLOG_LOCK_PROF parameter specifies the minimum duration of a lock that must be
logged. Any locks of shorter duration are not be logged. This variable specifies the minimum
number of milliseconds, which must elapse before a lock is logged. Lock time is calculated from
the time when the first record in a transaction is locked to the time of the commit or abort. This is
the longest time a record remains locked during a transaction. Note: The appropriate dbslog
categories must be set, for example:
DBSLOG_LOCK_PROF=2.0
This variable can also be specified in the db_resource file.
•DBSLOG_NAME
The DBSLOG_NAME parameter specifies the output of the DBSLOG file:
DBSLOG_NAME=/bigfiles/locking.log
This variable can also be specified in the db_resource file.
•DISABLE_DEBUGGER

•TEST_RETRY
The variable TEST_RETRY specifies the frequency with which the system must return to the
retry point during a commit. This variable must not be enabled in a live environment to avoid
performance and locking problems, such as:
TEST_RETRY=3
In this example, when a commit is executed, the driver returns to the retry point without committing the data. When the commit is executed again,
the process is repeated. The third time the commit is executed, the actual commit is done.
•USR_DBC_RES / USR_DBS_RES
The variable
USR_DBS_RES specifies the location of a db_resource file.
This contains a relative path starting from the $BSE environment. For example,
:
USR_DBS_RES=lib/defaults/batch_resource
This variable is used to specify a file other than the default db_resource file for the specific
sessions/jobs.
The USR_DBC_RES variable sets the location for the client, and USR_DBS_RES
sets the location for the server.

Call Graph Profiling
The Call Graph Profiler (CGP) can be used to identify functions and application queries that
consume a lot of time. The CGP includes a call graph functionality that displays the relations
between parent and child functions.

To enable the Call Graph Profiler, you can set the following command in your BW config Bshell Command -

For Unix based Server -

-- -set PROFILE_ALL=1 -set PROF_DIR=/tmp/graph your_session_codee.g.

-- -set PROFILE_ALL=1 -set PROF_DIR=/tmp/graphy tdsls4100m900For Windows based Server

-- -set PROFILE_ALL=1 -set PROF_DIR=C:\temp\graph your_session_codee.g.

-- -set PROFILE_ALL=1 -set PROF_DIR=C:\temp\graph tdsls4100m900For the latest porting sets, BDB_ALWAYS_FLUSH and PROF_RTIME are enabled by default.

In case not, you can use -

-set BDB_ALWAYS_FLUSH = 1 -set PROF_RTIME=1 in addition to the above commands
