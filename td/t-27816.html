Behaviour you describe is most probably caused by using check.input section, combined with set.input.error or by references in the table definition. If it is a reference it is not that easy to solve (or you have to remove this reference, but most probably you don't want to).
If it is caused by the check.input / set.input.error, you can change the script so check is done in i.e. after.input and use input.again in case of an error.
