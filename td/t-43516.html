In Baan, compilation has two passes (in fact, two different executables run one after another). First pass is to translate 4GL events to functions and it generates a file with endind "g", second is to compile the file from first pass. In your case, first pass is not successful, so your error must be somewhere in 4GL events/subevents. Must be something like a main event without mandatory subevent or vive versa. Post your code and let us check.
