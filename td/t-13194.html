Hi Muheeb -
I don't know if you have found a solution for this yet - we just ran into it. If not, we have not resolved yet either, but we have noticed the following:

It definately has to do with the app_start - if you comment that out it is fine.

It also has to do with the section and the start option of all things. If in the before program and the start is modify set, which we had, we get the error. If we change the start option to nothing, we do not get the error. If we leave it at modify set, and move the check to the choice.cont.process section, we do not get the error.

So, we have moved it around as a work around. I don't like it there as much, but we are living with it.

I would guess this is a bug of sorts - most likely a porting set issue or something, hopefully to be corrected. Post up if you have resolved.

The problem is that word ( or another apllication called with app.start ) send a message to the bshell that is unknown for it.

the solution is:
app.start(....
long dummy
dummy = next.event()

this extract the unknown event of the qeue and your trouble is resolved.
