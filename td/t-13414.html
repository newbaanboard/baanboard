does anyone have experience with these functions?

Situation:

I have created a sub-session which is used to enter some values and in turn they are exported to the caller on option end.program. These values are to be used in the calling session (main session) to restrict the records that can be accessed by its choice options. The functions store.occ.min and store.occ.max are used to set this range, but no records are displayed for the range.

example: table tchqm250 index1 = sern
index2 = user,ship,mato,date,time,sern

value ranges: user.f = "baanuser" = user.t
ship.f = 1 = ship.t
mato.f = "" and mato.t = set.max(mato.t)
date.f = date(2003,12,10) = date.t
time.f = 0 and time.t = 999999
sern.f = 0 and sern.t = set.max(sern.t)

result: no records are accessed in table tchqm250 meaning no records are displayed on the form.

Note: The second index is used to restrict access.

Does anyone know of some of alternate functions or some suggestion to resolve my requirement?

In case u are using export function to export values from subsession to main session then it will not work and consequently ur main session will not work.

U can try put.var functions or exit.val$ pre-defined variable in the before.choice of end.program using predefined process number variable 'parent' . And in the main session u can use skip.io in after.read/main.table.io section . This will give u desired result (restricting the display to the values entered by user)
