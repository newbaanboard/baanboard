Hi,
You can create a exchange scheme to export the exchange schemes & another one to import it in the new location.

These exchange schemes are used to export & import the exchange schemes (tuxch tables). For these exchange schemes, the text fields should have field type as Multi-line text in the ASCII file format.
These exchange schemes should contain the following tables:
· tuxch001
· tuxch002
· tuxch003
· tuxch004
· tuxch005
· tuxch013
· tuxch014
· tuxch021
· tuxch022
· tuxch031
· tuxch032

The exchange scheme to copy exchange schemes works flawless, as long as you define the text fields as type 'Multi-line Text' in your ASCII file Fields definition.

An easier method that I used to copy exchange schemes on the same server, is to create a bogus company with only the exchange module in it. Make the VRC equal to the company that holds your exchange schemes.
Then use the functionality in the exchange module to copy your schemes to the new company.
Change the VRC of the new company to match the VRC of your target company, and again using the copy functionality in tu, copy the schemes to your target company.

Finally, if your companies are on the same server, you can simply run the scheme in any company you want without any copying. Exchange will complain about the VRC's not matching, but it will run the script anyway.
Just ignore the error msg.
