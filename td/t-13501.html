I have created a report that will generate a Unix shell script to e-mail division managers a list of projects and their associated costs. The script gets generated as the report script runs. The report has four layouts:

Header
Before division
detail
After division

In the Before Division layout I first delete any pre-existing file (one unique file per division manager) recreate the file and insert the To's, From's and Subject as well as some text directed toward the user.

In the Detail layout I create an HTML table which has the output data embedded.

In the After Division layout I close the file, change the permissions of the file and then execute the file.

It sends out a handsome e-mail notifying division managers of any jobs where costs exceed sales (in red) or where costs are within 85% of sales (in blue).

My problem...

Any particular division whose list of jobs exceed one (hardcopy) page do not include the entire list in my file. For example if Division 1 has 40 jobs that fit the flagged criteria and the hardcopy of the report will print 20 per page, my e-mail will contain:

Header information from the before division layout.
The last 10 jobs on page 3 of the hardcopy report (detail layout).
Footer information form the After Division Layout.

Does anyone have any suggestions? Much thanks in advance!
