I am not sure that what you are trying to do is possible.

Firstly, obviously you are using a level 1 database driver which means that Baan is responsible for maintaining the indexes (the hash colums) rather than using the RDBMS index structures. This makes it impossible for 3rd party software to manipulate the data.

Secondly, I don't really understand your SELECT statement. Should the WHERE clause not reference any proper data table fields than hash columns?

e.g.:


select * from ttpur040120 where t$orno = "47";
