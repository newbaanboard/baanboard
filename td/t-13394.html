It's a customization.

For the 'T' option to be available, you generally need to have a text field on the main table of the session. That session's main table is tfacp102, which does not have a text field on it. You can create links to other table's texts as well, but somewhere you need a table text field to store the text into.

The straightforward solution for you, if you are willing to add the field to the table, is:

1. Add a field to the tfacp102 table, name it 'text', give it a domain of tctxtn, the rest should default properly.

2. Convert to runtime, with reconfigure checked (you should see your table come up), exit Baan, return. Check your results in Display Table Maintenance (ttaad4500), your new field should show there before you can continue.

3. Copy the Session component (Maintian Sessions) for that session from the Base (I velieve B50c is the base for this) into your VRC, and Under the 'Specific' pulldown, choose the optin 'Text Fields...' and you should be able to insert your table text field.

4. You can put a yes / no field on the form to indicate if there is text as well (tfacp200.txtn <> 0), but that requires either source code or wrapper tools to program.
Steps 1-3 do not require source code.

Don't know if you are into dev enough to do that, and some companies frown on table changes. But as that goes, this is a pretty simple / low risk one, with no index changes.
