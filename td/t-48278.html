There are a couple ways to pull it off. I have written a batch load that was run nightly to import the prices from another system's exported price list. After doing this for a bit, the users decided that the Baan price list provided a bit more detail than the other system, so we turned the interface around = maintain in Baan, export to outside system. Writing a custom session to import based on quantities isn't a bit deal since the price tables are pretty small.

The other direction would be to run (shell, run.prog) an executable that could get the data from the other system's price list structure by API or ODBC access.
