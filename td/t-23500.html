Hi Ash,

We have something similar to this and we have managed it in Baan IV c4. We have a session customized in Baan quality management system (QMS). This is for calibration history.

After calibration of an instrument on a given date the calibration sheet is scanned and uploaded to a seperate system other than the server or client as an acrobat file. This uploading is done by one person and the naming criteria for the file name is set as the instrument name (underscore) and the date of the document.

Later on any user having access to this session in Baan can go to the record and mark the record and click on the button available. The system then copies the acrobat file corresponding to the record selected to the local machine and then opens the acrobat file in the client system.

We have implemented this 6 months back and working fine and were able to reduce paper wastage and avoid manual filing and maintenance of paper in this area. The document is also now available to all users having a Baan access online. This was a good improvement and appreciated by the users of Baan.

I have given the sessions dump and the code that is used for the same. Hope this will be of help to many. Baan has got a lot of functionality, but it needs to be explored. The first session is for upload and the second is for viewing the data from any client machine.

Please let me know your views on the same. Any clarifications please let me know.
