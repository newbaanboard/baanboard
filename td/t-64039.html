The debugger option is NOT available from WEbUI.

The reasons are;

WebUI is a client interface with the least load over the computer, therefore the shell who allows the debugging is not present.

Apart from this the debugging process is meant to be done by the IT department. Therefore, the bw client is only to be installed in few computers.

Bhushan,

Your information not correct anymore. It IS possible to debug with WebUI, though you have to use it in conjunction with Worktop. This is documented on the InforXtreme support website under KB 809503 (August 2011).

1) In Worktop, run object 'Connector Daemon' (ottstpcondaemon). Click Listen and take note of the Listener Port.
2) For WebUI, change your URL from "http://<webaddress>/Login" to "http://<webaddress>/debugLogin" (correct capitalizaton 'debugLogin' is important).
3) Use the Lister Port and log in with your user and password.
4) When an object is in debug, it will come up in the debugger in Worktop.
