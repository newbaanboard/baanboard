Hi Babu,

yes you are right, the known limits are -


Tables
maximum record length is 3072 bytes
maximum number of fields is 1024
maximum field length is 3072 bytes
maximum number of indices per table is 31
Indexes
maximum index length is 120 bytes
maximum number of fields is 32 (no combined fields)
maximum number of index parts is 8

Yes there are limits. The maximum number of index parts of 8 can be a problem. You should check the Baan help texts in the maintain index session for how the index parts are created by Baan. Often it is possible to reduce the number of index parts by changing the order of some fields.
