i have faced same situation, but i think it´s a bug in a standart tools object, and it´s exaclty as they say "In some rare cases (not reproducable)", take a look to the solutions: 210855, 201312

SolutionID
210855

Solution Description

English
Tools error: unknown object type

TOOLS VERSION(S):
7.6_a1, 7.6_a2

SITUATION IDENTIFIED IN:
"Standard Program" (ttstpstandard)

SITUATION DESCRIPTION:
In some rare cases (not reproducable), the following non-fatal message pops up:

Tools error in session [sessioncode]; internal error in 4GL Engine ('unreachable case' failed)
File: ittstpeventloop0 (line 457), Description; fcsmgr.getObjectType()=8:
Unknown object type in evthnd.handlechangefocus.

*******************************

SolutionID
201312
Solution Description

English
Tools error about unknown object type prevented.

TOOLS VERSION(S):
7.6_a

SITUATION IDENTIFIED IN:
"Standard Program" (ttstpstandard)

SITUATION DESCRIPTION:
In some cases, while adding a record, the following message pops up:

Tools error in session [session code]: internal error in 4GL Engine
('unreachable case' failed).

File: ittstpeventlooop0 (line 1826). Description: fcsmgr.getObjectType()=8:
Unknown object type in evthnd.handlesetfocus.

This message should not pop up.
