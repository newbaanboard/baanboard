Filter is a new standard tools menu item in Reger and Gemini Tools versions. It is referred to as the query browser option (QBF-ttstpqbf) in the previous BaaN releases.

Martin,

Please add more info and screen shots in your posting, others cannot guess what's your requirement or in your mind. If you are explicit then you have a better chance that others can help you out well.

Refer to the session's help -

Filters (ttadv9593m000)
Use this session to define, or edit filters. You can use the filters to display subsets of records in an overview session, and to define a sort order.
To specify a sort order for the filtered records, click Define Sort Order on the Specific menu.
Note
The filters only apply to the records displayed in the session from which you started the Filters (ttadv9593m000) session.
Related topics
Overview of filters
To use filters
Field Description
Filter Name The name that identifies the filter.
Note
When you apply the filter to the overview session, the name appears in the session's title bar.
Field Name The field you use to filter the overview session's records.
For a record to display in the overview session when you apply the filter, the record's data in this field must satisfy the Value field by the operator you select in the Condition field.
If you define more than one field for a filter, the conditions combine to form an AND relation. That is, when you apply the filter, only records that satisfy all your filter's conditions will display in the overview session.
Note
You can only select fields that:
Are displayed in the overview session and the details session.
Are stored in the session's main table.
