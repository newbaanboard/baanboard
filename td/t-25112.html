The ttstpconf functionality needs to send the report somewhere, that is why if you do not specify a program, your local PC will handle the file extension according to what was setup for the type of file you specified (in this case, open .txt files with notepad).
Jobs are not supposed to send data to clients (that's why they are jobs, so that they run unattended in the server side), I don't know why you are sending job outputs to a client instead of saving them in the server. It would make more sense to save to a file in the server side and then pick up the reports thru ftp or other way.
In any case, save from server to client, etc. has been discussed already (you should have searched first), start here (http://www.baanboard.com/baanboard/showthread.php?t=12500&highlight=save+file)

On our system the ASCIF device first creates the Baan report in $BSE/tmp. I would expect the same on the windows environment. Next in converts the file to ASCII into the spool.fileout location. Of course the spool.fileout location is somewhere on the server. What is going into your spool.fileout? Of course I really do not know windows server setups. I would expect that you could specify another directory on the server in the output file.

Other devices we have like excel will take the file download it to the client and launch excel(or word). The download text file is in the client $BSE/tmp directory. If you want something to just download the file to the client with out lauching the file then I would write my own device driver. You can see this in the code and utilities session. All you would need to do is use ttstpconv to convert to ASCII, then server2client to download it.
