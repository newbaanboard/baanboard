Hi,

I guess you have to check few of the authorizations -

1. check the role templates they are linked to, if they have the authorizations on the tt-sql module.

2. also check package/table authorizations by company.

3. check the text group authorization and if that particular text group is set up in that company and also listed in the text group authorization session with a permission of update.

If all these settings are properly set and a runtime is made, logout and log back in for the effective changes and verify if that fixes the problem.
