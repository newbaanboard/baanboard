Your method is nice. Thank you for sharing.

So, I will share mine too.

Upon check-in process, there is a textbox requiring the developer to write commit messages similar to modern git. These messages can always be read in Extension History so that other developers can understand what has been going on on this specific extension.

Of course, this is only a way that I think is usable and worth sharing but I am quite sure that there must be other ways.

This might be not exactly what you are looking for but, I don't know, I feel like want to share something too. Maybe it is useful more or less.

And when you use this text option like tmannais states, it makes it easier to identify and revert if you like to an older version (prior to the fix). This you cannot read when using the comment method.

When changing code, comments are good, but a summary of the main change in the text is crucial.

It is even possible to "temporarily" go back to an older version, copy that version into a text buffer and revert to the current version.
