Okay - two ways to do this that I see off the top of my head. The first way is with the session script and should work since this is the before.report section. What you do is pass X as false until Y has 1 true value. At that point you pass a dummy record with X true. By a dummy record I mean one where whatever sorting you are doing this record comes out first.

Another method is to use the after.receive.data section - search for that and my user id. In this method basically what you would do is read through the tmp file that baan sends to the report. You would look for any Y is true. At that point you can reset the record pointer to the start of the file(maybe seq.rewind) and let the report continue to run. The example you will find is where I actually add a field to the tmp file and change the sort. You can probably adopt this to something you can use. Personally I like the above better than this one.
