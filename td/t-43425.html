Refer to the LN help.

Additional File
A generic component, such as an XML schema file, a GIF image, and so on. From SSA ERP 6.0a on, additional files are stored in a specific package, module, and package VRC.

Yes you can use Additional Files in a PMC solution, patch, dump etc. They are based on package-module-VRC combination.

Application help is now handled with the Webtop's webhelp.

"Additional Files" are software components, just like sessions or messages. You can add, modify, delete them with ttadv2570m000. You can use them to transfer files from one environment to another by PMC. The steps are then:
1. Create an additional file. Use the import functionality of the session mentioned above to register your file as additional file. You have to provide a package and module.
2. Link the additional file to a PMC solution.
3. Export the solution.
4. Scan and install the solution in the other environment.
Now you have the file in the other environment, but maybe not with the name and in the directory where you want to have it. It is in $BSE/application/<package><VRC>/b<package><module>/b<module><yourfilename>. You have to write some scripting yourself to get the file where you want to have it.
