Hi solepeder,

I had the similar problem some time back, I extended the enum definition for a domain and after that got the same problem what you are getting. I did all what you tried and it was not resolved. Finally I searched and removed the domain and table definition from the shared memory, created runtime DD for domain and then table and loaded object in the shared memory and it get resolved.
I observed that some time it happen that the definition of the loaded objects in the shared memory is not get updated and/or updated objects are not get refreshed in the shared memory. It is better that remove the objects from the shared memory, update the definition, created RDD and then load them again.

Best of luck
MRF
