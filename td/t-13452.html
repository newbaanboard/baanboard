The bic_info6.1 is an utility to get information on OBJECT files, not sources. The include files are in fact - only sources that are included in the main script at compile time.
You cannot get info thru bic_info about sources....

Try, by example, instead:

bic_info6.1 otiitm0101

to get info about the object of maintain items.

you are right wrong zardoz...

But if you want information on an include file or program script, you can open it for reading.

What you then need to know is that Baan allows you to work with different variants of scripts at the same time. The variants are rarely used, so most people only use variant 0 of the sources.

When you want to view your intended include file, you should use a command line like this:

view `explode6.1 itccom00070`
note: use backquotes to first perform the explode6.1 command and use its output as input for view;
note: add an extra 0 to take variant 0
