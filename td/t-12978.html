Another interpretation of your problem: you are trying to customize a script that, according to Tools, is in the standard B40_c4 VRC. If this is the case, and you do not have a source code agreement, you're out of luck. The source does not exist in the standard VRC! In Tools it is reported to be there, and that is correct (meaning, don't remove the entry/entries!), but the actual file $BSE/application/tcB40_c4/ptccom/pcom11020 does not exist. When copying, it copies the entry in the Tools table, but, finding nothing on UNIX, creates an empty file.

If, however, you know the file exist (for instance, you're copying from B40U_c4_stnd), check the file permissions of the pcom11020 file. If the permissions are wrong, the file will not be copied.

Hope this helps.
