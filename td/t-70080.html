There are actually several levels of this problem

1) The set up has to be good. The SMTP Server name and the port used for smtp has to be proper. This has to be verified by the system administrator.Generally the port name 25 or 587 is used for SMTP

2) Then you have to get it checked whether those ports are not blocked in your Application Server

3) Then you have to get it checked whether any Anti Viruses are not blocking the service requests

4) Then SSL Curl has to be enabled as Yes sometimes,

5) Even then there has to be dedicated SMTP Server installed in the server because with the default SMTP Server is not consistent as its protocol blocks many of your message sending abilities.
