I have a Problem with printing invoices and order responses.
The System is an IBM AIX 5.1 with BaaN IV c4 SP12.
I found a similar description at the BaaN support website:

SITUATION DESCRIPTION:
In a report the header is not printed on subsequent pages when it should be printed. This is caused by the printing of a detail-text.
When this text is so large, that it has to continue onto the next page, the header of this next page is not printed.

SOLUTION DESCRIPTION:
When a large text was printed, the program did not check correctly if a new page was started. Because of this, the header of this new page was not printed.

Haven't heard of a solution for Baan IV, but you may be able to work around this, by programatically forcing a page break or a need setting for your layout.

It shouldn't be to complicated. You will need to read the text table to find out the number of lines required for your text field, then set a 'need' amount for the layout, that the text is being printed on....

Other options, could be restricting the text to printing only the number of lines available till the end of the page, by setting lattr.textlines.max (based on spool.pg.length minus how many lines have printed minus footer margin).
