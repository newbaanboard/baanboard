Hi,

Helas, not possible. Since you're doing updates in a reportscript, you will not have dev kit, the program script sends data to the report, which you use to do updates. Even if you do not want to print anything, you need the data in the report to be able to do the updates.

Only work around is to send the output to a device of type ascii file in which output path is set to /dev/null.

Not clear whether u r talking of SQL query or a Baan standard program .

For Baan program (for which u do not have sources ) the only alternative left is using AFS stpapi.set.report function redirecting the output to ASCIF file .

For SQL queries , u can convert sql query to string , use sql.* functions, open the report using brp.* functions and redirect the output to ASCIF . This way u will achieve the objective.
