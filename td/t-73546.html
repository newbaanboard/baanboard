Create a report extension with calculated fields and import the values of the form fields in the Calculate Value hooks. Then Report Designer will show those fields as well.

Do you mean importing for example item.f in report extension for calculated field ext.item.f like this:

function extern void ext.bank.f.calculate()
{
| hook code
domain tcitem item.from

import ("item.f" , item.from)

ext.item.f = item.from
}

and ext.item.f will be available in report designer. Am I right?
JaapJD
10th November 2020, 16:52
Yes, that is almost correct. Field ext.bank.f will be available in report designer.

And you can do it with less code:

function extern void ext.bank.f.calculate()
{
import ("item.f" , ext.bank.f)
}


Or ext.item.f, but then the hook function name should reflect that as well.
