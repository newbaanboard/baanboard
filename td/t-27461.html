ganesh,

You can also have two different select queries, the first query will sum only the positive values and the second one will sum the negative values(put the necessary condition in where clause).
Then take the abs() value of negative total and add it to the positive value.

The reason i'm advocating the use of two selects is, rather than summing the amounts in selectdo, I would prefer the database do the sum itself which will be much faster.

However my solution hold true if you are selecting the records with a index fully mapped. If you cannot use the index then its better to total in selectdo as suggested by "gruizc" since we cant afford to sleect the same records again.
