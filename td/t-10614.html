I assume you are talking about the font that is displayed in the input form fields.

The problem is that the font is derived from the client. From memory, I believe the BW client uses regular Courier Font (size 11 or 12 pt?), and as you mentioned, unless you're a hawk you can not tell the difference between a zero (0) and a capital oh (O) with that font.

One method that I have employed for clients in the past who absolutely needed to tell the difference because they regular used item numbers with oh's and zeros was to edit the standard Courier font. I editted the font and placed a normal dot in the middle of the zero so you could tell the difference between the zero and capital oh. That font then needs to be installed on the client machine in place of the old courier font. It is not hard, but needs to be done on each client. You can also run Baan using a different font in the background, e.g. Times or Book Antiqua. You should use a proportional font. Neither of those fonts look that great to me.

One con of editting the courier font, is that if you use any other program on the PC, that 0 with size 11 (or 12) will have a dot in the middle of it - but I prefer that anyway (e.g. if you're using Word or something).

I don't have the font handy, but if you want it PM me (don't have it handy on this laptop) and I'll look for it.
