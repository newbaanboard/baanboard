We did a work around to do that.

Example:

extern domain tcpono number.of.copies
extern domain tcpono report.num

choice.print.data:
on.choice:
open the report.

collect data()

close the report.

function collect.data()
{
long counter

colect data()

for counter=1 to number.of.copies
report.num=counter
brp.ready()
endfor

}

report.num was added to the report .
Make report.num as the first sort field.
Add a layout with report.num after the header layout .
So each time the report.num changes it give's a new copy.

Hope that helps

I've been fiddling with the 'spool.pr.copies' variable and I've discovered that it also works with 'brp' functions - the trick here is that in the users device preference list the first device cannot be of type 'screen' - if it is the 'number of copies' defaults to 1 always.
