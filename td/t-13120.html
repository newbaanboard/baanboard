yes the create/convert runtime DD sessions use the bdbreconfig6.2 binary -

>bdbreconfig6.2 -U
-s : Suppress messages
-d : Database Driver type
O(racle) I(nformix)
S(ybase) D(B2) M(SQL-Server) N(onStop SQL)
<host> - remote system (name is <host>)
<driver>[<spec>] - driver name [ specification ]
-r <rows/transaction>: The number of inserted rows per transaction
-t : Specify directory for dump files
-o : File with succesfully reconfigured tables
-e : File with unsuccesfully reconfigured tables
-i : Ignore errors during reconfiguration (e.g. duplicates)
-N : table list (ex: ttadv200)
-I : File with table names
-O <file>: Redirects output to output-file <file>
-E <file>: Redirects errors to error-file <file>
-p <pack. comb.>: define package combination
-m : Disable domain constraints
-n : Ignore referential integrity constraints
-f : fast rebuild of table (first rows, then indexes)
-C : Company number for given table, format:
1. Specific company number (ex: 001 001 ...)
2. Range of company number (ex: 001-999)
-c : Test if reconfiguration is required
-M <number>: <number> is the maximum size per sequence
<number> can be specified as any number or a number
followed by 'K', 'M' or 'G', for Kilo, Mega, Giga bytes
Default size is 2Gbytes per sequence
-T : In case of domain change only, table is not reconfigured
-u : Short usage
-U : Usage
-Z : Reorganize
-V/-v : Version information

I would not prefer to use the command line option as the session does more like new/changed definitions for domains, where used tables, additional checks and sends the parameters etc to the binary.
