Hi there,

haven't tested this, but although the client runs on NT, the bshell will run on the unix OS.
So what about:
string arg(255)
arg="who am I > $BSE_TMP/tmpwho"
shell(arg,0)
Then seq.open and read '$BSE_TMP/tmpwho'.
If 'who' doesn't give you the info about the users IP, 'ps' might, if you know the process id of the bshell .... this should be contained in a runtime variable, can't tell you which.
No system ready to test on, but I hope this helps a little.
I personally wouldn't like to pass the IP by manually filling in a parameter in the BW client, if you are using DHCP this would be a completely wrong approach anyway.
