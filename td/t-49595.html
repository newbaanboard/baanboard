Same function I had used for couple of customizations. And I observed the improper behavior as mentioned by Niraj.

The single occ field on the form gets enabled when we try to edit any of the records, i.e. when command modify.set executes. Otherwise fields will be disabled, in fact, it doesn't appear on the form after saving records.

There are few baan standard sessions (eg. ticst0101m000 "Enter Material Issue for production order"), where this function has been used for showing editable single occurrence fields (which are not part of index), at the bottom of form having Form Type 3. And you can observe the same behavior on these sessions too.

I believe that there is no satisfactory solution for this, as the function exhibits a functionality that is against standard Baan behavior of Form Type 3

But, it will be a great help if anyone can suggest solution.
