Hi Gulya,

Modifying domains can be tricky. In this case the field tfacp200.refr is connected to the domain tfgld.refr. Be aware of the fact that the field is not only used in table tfacp200 but in at least a dozen of other financial tables including tfgld401 and tfgld106.

When you need to change the domain you must execute a create runtime DD ( ttadv5210m000 ) including tables, domains and re-configure tables. When you are working in a production environment you need to do this at a suitable moment. If users or processes locks one of the tables to be reconfigured this may otherwise result in an error 113. ( Table lock ). I also would make a back-up of the date before the tables are re-configured. If you did extend the domain you also need to check reports and forms involved because you want to see the largest possible values stored in the modified domain.

Good Luck.

JP
