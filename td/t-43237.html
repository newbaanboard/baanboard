You can use the client2server function (http://www.baanboard.com/programmers_manual_baanerp_help_functions_client_file_access_client2server) to copy any file to the server and read it using the seq.* functions (http://www.baanboard.com/programmers_manual_baanerp_help_functions_client_file_access_overview).

I hope it helps,

Hi,

As I understand your requirement is not that different. I mentioned in my previous post how to read a file from the client. If you do not know the names of the files you can follow JaapJD's suggestion and use app_start function. You can find the documentation here:
http://www.baanboard.com/programmers_manual_baanerp_help_functions_client_file_access_app_start

Using the app_start you can run a dos command that will create a file on the local that have all the file names. Example:

app_start("cmd /c dir /b "&strip$(local.dir)&" > filenames.out", strip$(local.dir), "", "", "")

The result of that code is that you will have all the file names in the file "filenames.out".

I hope it helps,

- George

P.S. The app_start is also in ottdllbw DLL. You have to define it in your script.
