What kind of form are you using? If this is a display session how come you use start.set and not def.find to find records? In a display session this is how I would expect start.set to work - especially if this is a multi-occurence with view fields. What exactly do you want start.set to do?

In one of my maintain sessions I use on.main.table to locate and find the view fields on the form. This way when the users hits the add.set option I have the correct records displayed which match the view fields. More info might help someone make a suggestion.


choice.start.set:
after.choice:
on.main.table(check_found)
if found then
execute(find.data)
endif

I see what you are doing now. I did something like this once before, but without start.set being active. I can see how you are finding the next record, previous record, etc. From looking at the document I wonder if the start.set event should be after.choice? I would expect the fields to be entered first, then in after.choice fetch the totals. On the other options you want to determine the next record (be it a change on account, dim, etc.) then fetch the totals - which should be in before.choice. BUT in start.set I think it would be in the after.choice even, because you want them to enter the data first. At least that is what I think so far.
