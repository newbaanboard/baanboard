I have done this both ways. In one case I have excel pull the data from Baan and in others I run macros on Baan reports. If I was you I would not use Excel to pull data from Baan - has more dependencies and is more complicated. The way I did Excel was to create a device called PCEXCEL. This device uses the fileout parameter along with the report name to know which macro to run.

For example a user runs a BOM report selects device PCEXCEL and puts LOU in the fileout parameter. I use this to look up on a table what the download filename is and which macro to run. The macro knows how to read the report and create what the user is doing. The using this macro you could pull other info from other resources.

For the parameters to app_start you can do a bic_info on ottdllbw to see its parameters.

Good Luck!
