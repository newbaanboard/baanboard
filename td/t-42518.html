Brendan Shine
28th March 2008, 07:36
Table dumps created with a newer version of bdbpre6.2 that came in with a newer version of the Porting Set which adds a new a line "#$!languages!$#" which is put in the dump files which causes the "Copy Company Data (tccom0214m000)" to error out with "FATAL ERROR: [MR_INDEXOUT] index -1 from Dim [1] 3072" as it is not expecting this line. The solution 222703 corrects this issue by accounting for the additional line.
Brendan Shine
28th March 2008, 07:39
I forgot to mention that you could alternatively remove the "#$!languages!$#" line from your dump files and then the tccom0214m000 would work as well, but the best long term solution is to install the PMC solution fix.
