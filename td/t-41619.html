You would need FP4 Tools & FP3_ERP Solutions:

SolutionID 222021
VERSION(S):
6.1_a2, 6.1_a3

SITUATION IDENTIFIED IN:
"Warehousing Orders" (whinh2100m000)

SITUATION DESCRIPTION:
A fatal error occurs in the "Warehousing Orders" (whinh2100m000) session. Fatal error : Dynamic link failure function 'dal.store.record' on whinh200. This occurs after installing a solution with components which are created with a wrong tools version.

SOLUTION DESCRIPTION:
By recompiling the script of whinh200 a new object is created. This component contains the correct tools version and will not give dynamic link failures.

SolutionID 222056
English Fatal error : Dynamic link failure function 'dal.store.record'
VERSION(S):
6.1_a1, 6.1_a2, 6.1_a3

SITUATION IDENTIFIED IN:
"Warehousing Orders" (whinh2100m000)

SITUATION DESCRIPTION:
A fatal error occurs in several sessions. Fatal error : Dynamic link failure function 'dal.store.record' on certain database-tables. This occurs after installing a solution with components which are created with a wrong tools version.

SOLUTION DESCRIPTION:
By recompiling the scripts of the wrong objects, new objects are created. These components contain the correct tools version and will not give dynamic link failures.

SolutionID 221812
English Fatal error: Dynamic Link failure function 'dal.restore.record'
TOOLS VERSION(S):
7.6_a4

SITUATION IDENTIFIED IN:
"4GL Engine" (ttstpstandard)

SITUATION DESCRIPTION:
The following fatal error can occur in some sessions:

Fatal error: Dynamic Link failure function 'dal.restore.record'

This can occur in sessions that use a Data Access Layer (DAL) that:
1. has been compiled using FP4 Tools and
2. that uses the with.old.object.values.do() or with.object.set.do() macros and
3. runs on FP3 (or less) Tools.

SOLUTION DESCRIPTION:
The bic_dal include file has been changed in such a way that when a FP3 (or less) DAL is compiled, it will not use the function 'dal.store.record'.
