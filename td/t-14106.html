If you define the exchange scheme as using a comma as separator (and quote as enclosing character), the internal code of the exchange scheme that parses the line read from the file into fields will always fall over on an incorrect number of fields.

So you will either have to change the source file (in Excel or otherwise) or change the exchange scheme. Perhaps you can define the input file in exchange as either fixed length or as 1 field long using some separator character that doesn't appear in the file at all. That way, the internal code will just read each line of the input file and parse it as being 1 field. You then have a job to do writing a condition to unravel it and stick it in variables that can be used to replace the original fields.

I haven't found an easy way to change excel's behaviour to only use enclosing characters in a csv file when required (i.e. when a field contains a separator character). If anyone knows how that can be done easily then please let me know.
