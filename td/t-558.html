Ankur -
There are a couple middleware solutions for use with DDC's and BOI's out there, Connectware is good from EpicData. My company RMCis puts out rmc3, which is targeted at smaller companies. The ddc can streamline processes from the shop floor, generally running on bar code readers such as Symbol or Intermec, instead of through the standard Baan sessions. You will need one of these solutions if you need to modify or streamline the company's procedures.

RMCis (and I'm sure EpicData) has standard Maintain Receipts software ready to run with the Baan DDC for any version. This code is based off the standard Baan Maintain Receipts script. You can make your own if you have source code by running the Function Server Generator session inside of Baan, but that produces code that almost works, and you need to understand function server coding to make it fully functional.

However, if you just want to print bar codes, you can use the bwprint options that come with Baan to do so. There are instructions on the Baan web sit on how to create a bar code font that you use for the fields you want to print in bar code format. As metioned above, you can also try 3rd part solutions for pretty reporting - like Optio or Streamserve.

For input, you can use a wand to input bar code information into the normal Baan Maintain Receipts session. Several existing wands plug into your computer to read bar codes, like a keyboard/mouse or serial port add in. When you scan the bar codes using the wand, the information comes into the Baan session screens as a number or whatever the bar code says. This requires no special processing in Baan, just the right wand setup.

So, it depends on the exact needs of the company how far you need to go.

Good luck!
