The debugger may not work for a particular session if someone else has the session active already when you compile it.

Also, I know that it is very difficult to debug DAL scripts in 5c. If any other session uses that script and someone has the session active then the debug will not come up, this also is true if GTM is active for the table whose DAL you are trying to debug. You are probably the only one working in your development VRC and that is why you have no problem with the debug there.
