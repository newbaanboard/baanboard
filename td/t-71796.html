Sync is for the BOD which is going out of LN, and Process if for BOD which is coming into LN, you need to define for both Sync and Process in the connection point if you are going to both send and receive the BOD, from LN Perspective.

Sync is used by the System Of Record (SOR) and it represents the complete object. Typically it is routed to multiple applications that need to be in sync with the SOR.
Process is used as a request to the SOR to add or change an object. The Process BODs are only routed to the SOR.
