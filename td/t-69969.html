I saw something like this in 4c4 - run the first time the fields were blank, but every run after that they were okay. So first make sure all the fields passed from the report script to the report are included in the input fields. Then check any report script fields and make sure they are not on the input fields to the report. I believe in my case I had included a field from my report script as an input field to the report. I know it had to do with report input fields - so make sure those are correct.
