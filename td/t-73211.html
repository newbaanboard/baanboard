I have a custom session with display fields on it. The session is scheduled in a Job which runs monthly. The two display fields Year and Period are getting set in the Job settings and every month admins have to change the job setting.

Is it possible to not save the display field settings while setting the session settings in Job or we need to set the variables pragmatically?
bhushanchanda
8th June 2020, 16:32
Yes, you can set the field values in your code in your function which has the processing logic.

You can use job.process to make sure you do this only if the session is being run in a job mode-


if job.process then
year.f = get.year() |#Logic to get year
perd.f = get.period() |#Logic to get period
endif

Well, In this case I would like to add a clearification:
* No, it's not possible not to save the displayed fields to the job settings. All forms fields get automatically stored, independent of display or input field.
* But yes, it's possible to change these settings when the program gets started by the job. This is what Bhusan wrote.
