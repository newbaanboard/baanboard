1. Create a new font (you should be able to find free ware 'font creation' softwares on the web) and assign your company logo to a charecter in this font.

2. Install the new font on the client

3. Edit the registry (using regedit / HKEY_LOCAL_MACHINE -> SOFTWARE -> BAAN -> BWPRINT) to include the new font.

4. On the report where you want to see the logo, Insert a new layout and set it as a 'User-Defined font' and place the logo-variable in this layout.

5. After compiling the report, send the report in a preview mode and check if the logo comes-up. If it is not, then right click on the layout and change the font there. It is just a one time setup. After the first time, you don't need to repeat this step.

It worked on Win-Nt/Win'98 for me. :)

In order to print to a postscript file via BW Print it would be necessary install such a printer device on your local windows workstation/PC. BW Print can print to whatever printer devices are installed on your local printer and the ASCII text is converted via the printer driver installed for that specific printer. And so if you want to print to PS or PDF that printer driver must be installed and used by BW Print.



On another note....
In BaanERP 5.X the printer fonts are stored in the local windows registery per report per printer device (NOT per use). This allows for a new printer device to be created within Baan to be used with user defined fonts. The local Windows registry settings can then be distributed to all of the necessary workstations with the appropriate settings for those reports.
