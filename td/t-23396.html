Hi there,

I do guess that you start the job via crontab.
The user that starts the cronjob has to be the 'system login' found in your Baan User Data (ttaad200).
That is to say if the cron ist started by root, then user 144 has to have root as system login. (session ttaad2100m000, don't forget to convert to runtime)

BUT ... note that the Baan-User is linked to a company. It is this company that the job will run in. (BTW that is the only linkage between job and company ... the value of $USER)
