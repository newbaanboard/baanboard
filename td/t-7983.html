You can add the "-- -set PACKAGE_COMB=<name of package combination>" to the command line. In that way you can be sure that you use that package combination.

I think when you do an "explode6.1 otdpurc041.b" you will see the similar error... in that case your package combination is not searching the C-layer. (if you have NT, use bic_explode instead, for ERP use explode6.2).

If you get a path returned, then check the permissions on the file.

Basically this problem is experienced in 2 cases

1. When the customized session is created, it may not be created using Generate Session and the developer copies one session to another and modifies the required information. When this session is ported to another VRC or another environment, while executing it gives problem.

2. The session object is not present.

What normal solution I follow for this solution is when I get this
problem, edit the session record by Maintain Session (ttaad2100m000 I think for Baan IV) and then modify the session description (may be by just putting the "." at the end, save the session record and recompile session. It works.

May be try this option.. it may help..!
