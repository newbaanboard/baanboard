Hi,

A mounted area is a work area that is located on another PC than your own. This could be a server or another client. In order to set it up, do the following:

- define your own PC as host
- define the server (or another PC) where you want to locate the work area as host
- create a directory on the server that will serve as your work area and make sure that you have the correct authorisations for that directory
- in Baan, define a work area on the server and make sure that the path that you put there is the path as used on the server (e.g. C:\......)
- define this work area as a mounted area in the session 'mounted areas' and put the path as used from your own PC (= the relative path, e.g. \\server\....)

Regardless of the correct set-up, I also ran into the same problem during an implementation as the one that you're experiencing (on Ln, FP2). This appeared to be a defect in the software that has now been solved by solution 212305 so also make sure you have that solution installed.

I hope this helps.
