Hi Sant,
Instead of making it a form 3 of table A, make it form 3 of table B, i.e., specify Table B as main table in "Maintain Sessions".
Now, assuming you ahve database relationship between table A and B, say,
tableB.field1 = tableA.pk
then, you can include table A fields as non-multioccurance (i.e. as a header) and table B fields as multi-occurance (i.e. detail lines).
You can use either of table A or table B in query.extension to restrict your query.

Hope it is clear.
