So, now I solved the Problem:

in Infor 10.2.1 the Display-Printer is a PDF-Format. So changing cpi from 17,1 to 12 has no effect. But when I reduced the column number from 215 to 178 the printing was automatically resized to the full paper. But I also have to do this for the not displayed section which have the "Output Expression = 0". If one section with Output Expression = 0 is still 215 col wide then it does not work.

How it will work on a real Printer is not clear, but I only develop for the pdf-printing. So I must'n care about nothing else.
