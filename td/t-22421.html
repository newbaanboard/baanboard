My recommendation would be to develop a API type function server which calls the Baan standard session in the back ground. You read the fields from the file, put them to the Baan standard session and then process them. We reduced the receiving process down to 1 session with only 6 input fields - it then runs all the Baan sessions in the back ground. We have done this for receiving, inbound, outbound, etc. Check out the AFS/DDC/OLE Forum. You could probably even search on receipt and get a fair number of hits.

Good Luck!
