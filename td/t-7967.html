To avoid the problem, the parameter ora_date should be set in db_resource. We forgot the same ... and now the database is big.

You can put 0 in a date by copying an other date which you know as zero, see PL/SQL blow.

regards

rem exemple de mise a jour de date zero dans BaaN
rem tcqms008 table instruments de module QMS non utilise
DECLARE
datzer date;
BEGIN
select t$lcdt into datzer from ttcqms008100 where t$inst = 'DATZER';
LOCK TABLE ttiitm001101 IN ROW SHARE MODE;
update ttiitm001101 set t$ltcp = datzer
where t$ltcp > '01-JAN-80';
COMMIT;
END ;
