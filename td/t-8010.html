Baan report can not print the image by itself (except as stated in ~vamsi 's post)

However u can use Baan reporting to write the output fields to a file , transfer the file to client and invoke your exe in after program section of the report. Your exe (may be in VB / VFP etc) should print the image easily.

U may use brp.open function and output the same to an ascii file.

We have done this successfully for our product designs directly coming from Baan along with other online information.

Stephan,

The other alternative would be to use ttstpconv option and view the report in a IE or any Internet Browser.


html The output of Baan is created as a html file on the server. The file can be loaded in a web browser to see it.Example:argument: htmlpath: /tmp/baan.outThe html file /tmp/baan.out is created and left on the server.Example:argument: html\/tmp/test %spath: /tmp/baan.outThe html file /tmp/baan.out is created on the server and processed by the script /tmp/test as the first argument. Together with the html conversion you can also define one of the following or all:-g<picturefile>-c<name>-f<fontsize of HTML>The picturefile will be included at the top of the HTML output. The name is a word that will be displayed additionaly on top of the report and fontsize is the size you define in the <font size=> tag of html. All these parameters are optional and can be used in combinations.


or you can generate the report in HTML format and write a small program to some tweaking of the HTML file to show the pictures.

Which method do you plan to use?

If you use the truetype font method. Very easy. You create a font called logos and assign different logos to different characters. You will then have an expression in the report that will output different characters based on your requirements.

Downside? You will need to install the font on every computer that needs to use the report. Also you will have to set the registry settings appropriately.
