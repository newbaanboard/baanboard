Hi Folks,
since my testing with the WebTop UI (Version 3.0 and 3.1, 4.0 is still under investigation) i came across some probs with my own sessions.
Espacially the functions in ttdllbw were prolematic.
What i found out is that the seq.read.local, seq.write.local and app_start didn't work when using WebTop.
As a workaround i changed my scripts in a way that the files are written on the server with the seq.* functions and then transferred with server2client (yep, this function still works) to the client.
The app_start function can be replaced by the function start.application.local (also in ttdllbw, don't ask me which version?) which works in bw and also in WebTop.

Maybe this info is usefull for some of the programming guys here on the board because there are many of referrals to the app_start function in different threads.

A list of some other WebTop problematic functions can be found in Solution 166790 on Baan Support Website.
