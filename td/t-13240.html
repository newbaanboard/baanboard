Yes, you can use it.

Here is a sample code-

function extern start.user.data()
{
string sdum
sdum = start.session(MODAL_OVERVIEW, "ttaad2500m000", prog.name$, sdum )
get.updated.info()
execute(first.set)
}

Please post your source code here will make it easier to find the issue.
1) The predefined variable 'background' stays on 0 in the called program, and the calling program does not 'wait' (like by 'MODAL' when the calling program is frozen till the child is exited).
You can use the sleep() (http://www.baanboard.com/programmers_manual_baanerp_help_functions_processes_sleep) command with the pid - current process id and then activate the other sessions, when you exit out of the other session then you can reactivate() (http://www.baanboard.com/programmers_manual_baanerp_help_functions_processes_reactivate) it.
2) the zoom sections are not being called in the called program (I need to perform some actions in case zoom from the specific session).
I believe the before.program section is executed in the called program, if the background variable is not set, then you can trap from which is the calling program and then do some additional processing/checks etc.
