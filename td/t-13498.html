Hello,

I have used cycle count orders to do things like this before,

1)
Create an exchange scheme based on tdilc501 (assuming location control).

2)
Export out some sample records, after you have :
- generated cycle count orders
- printed cycle count orders
- entered cycle counting data (setting counted inventory equal to zero).

3)
Based on your list of items, create (in Excel for instance) a flat file that has the same format as your sample records.

4)
Load using Baan exchange.

5)
Print Cycle count check report to validate impact.

6)
Process cycle count orders, which actually adjusts the inventory.

If you have used Baan exchange before, this is relatively easy,
1-3 hours work, depending on the data to be uploaded.
