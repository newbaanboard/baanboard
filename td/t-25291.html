Does decreasing the size of a based (dynamic) variable free memory or would I have to use free.mem and re-allocate my variable to get the memory back?

Example

long list.item(1) based

alloc.mem(list.item, 100)


Now, if I didn't need list.item to be as big any more, does the following free memory?
alloc.mem(list.item, 50)

Or, would I have to do this?

free.mem(list.item)
alloc.mem(list.item, 50)


I realize that I would lose the data doing a free.mem(), that is not an issue.

U can always re-allocate the variable . If the dimension is higher than earlier then earlier data is preserved . In other case (re-allocated dimensions is less than earlier one ), elminated elements will get freed and data therein will be lost.

There is no necessity for free.mem for re-allocating memory.

For string fields , u may not change the length dimension . Also for 2-3-4 dimension arrays u can change only 1st (2nd in case of string) dimension . Changing all dimension at random will result in data corruption.
