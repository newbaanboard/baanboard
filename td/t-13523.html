Hi Surinder -

Put the variable on the form, as in Omeluuk's description - 'text.yn'.

In the program script, you must declare it as an external variable:


declaration:

extern domain tcyesno text.yn


you can then have a field section associated if you want:


field.text.yn:
before.display:
do.something.to.fill()
when.field.changes:
so.something.when.it.changes()

6th November 2003, 17:55
Unfortunately, the calculation options is only available in the report fields, not the form fields.

You cannot add a non-table field to a form if it is not declared in the script (or a table field of a table that is not declared in the script).

The wrapper tools (like RMCgen, QKEY ) are perfect for this - adding fields and/or messages without having the source. You still create a script, but do not change (or even see) the standard.
