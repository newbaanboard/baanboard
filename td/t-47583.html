Yes, DAL is available BaanV owards(DAL) and enhanced in ERP LN with wider scope to server the needs (DAL2 or extended DAL)...but you want to understand the function flow, so that later if you want you can use it in later versions, here it is:

Function flow

When the DAL is used to manipulate the database, the main steps involved are as follows:

1.The user issues a command through the user interface to access a record in the database.
2.The 4GL engine loads the appropriate DAL (DAL DLL has the same name as the table name, and it gets invoked automatically ) and calls the hooks in the DAL to check the integrity rules for the object.
3.If the particular database action is permitted, the 4GL engine issues the appropriate database call.
4.Recorde set is identified and accordinglly action is taken.After the database has been updated, the DAL can perform further checks to determine whether or not data is passed back to the user interface.
5.The 4GL engine passes data back to the user interface (provided that the integrity rules permit this).


This is how it works...if you want to go into details, kindly refer the prgrammer's guide...
