The problem is not with sorting the data, but with emulating the point-and-click functionality.
Baan does not allow you to place buttons on top of columns (as a matter of fact, it does not allow you to place buttons anywhere specific since it handles the button placement for you).

You can retrieve and pass mousepointer coordinates, but you should ask yourself if you really want to go there.
In the end it is most likely a lot cheaper to train the users how to use Baan.

Another option is to put on a button to export to Excel. You can do this yourself, or check out Nazdaq's tools.

Psudo code would be:


choice.user.1:
|* Button on Form for Export to Excel
on.choice:
Select same data as on the form (same key info)
Write a .CSV file of this data
Open excel with .csv file as input, using the app_start command


I have done this before, it's not too slow. The excel sheet comes up on their PC, then they can sort, total, whatever they are used to. If they need to update, this would be more complex, but you could write an excel update using OLE.

En,

Press a different columnt, all records will be sorted in that column, etc.
If you have the columns as Index fields and if you enable Dynamic Index Switching you get the option to choose View -> Index.

The problem is not with sorting the data, but with emulating the point-and-click functionality.
Baan does not allow you to place buttons on top of columns (as a matter of fact, it does not allow you to place buttons anywhere specific since it handles the button placement for you).
This is possible in Webtop, but only for the Index fields.
Alternatively, you can add buttons on the form for different fields and add code to execute a rebuild.query().
Another option is to put on a button to export to Excel. You can do this yourself, or check out Nazdaq's tools.
You have to write more code to do that and Nazdaq's tools are options from the print session and not maintain. Alternatively, you can use the option : Edit -> Copy to Clipboard and paste in an Excel sheet.
