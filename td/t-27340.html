My first thought when I read this is for you to set up a user font in your print driver. The user font would turn on the character set for this character, print the character and turn it off the character set. So on your report for this field would select your user defined font. Never tried this, but sounds feasible.

Addition: You could also use the report script to print the escape codes for the printer to set the character set, etc. Using spool variables.
