Are your Baan Systems on the same network? If they are then you can setup the remote company, briefly use:
ttaad1100m000 - maintain companies
ttaad0150m000 - maintain systems
ttaad4110m000 - maintain database definitions
ttaad4111m000 - assign tables to databases

once this has been done you'll be able to open the remote tables using the following addition to a normal SQL statement:

and ttppp000._compnr = :remote.company

By the way the remote table definition of the table you want to open must be the same as the table definition on your local system. But this should be the case anyway because we don't change standard Baan tables, do we

If you mean different Baan servers you only need

ttaad4110m000 - maintain database definitions
ttaad4111m000 - assign tables to databases

Specify the INFORMIXDIR and INFORMIXSERVER (and their values) in the variable column.

If you just mean another server with e.g. an access database it really depends on what you are doing. Maybe your solution is already the fastest.
