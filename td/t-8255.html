Hi JH

First remove the physical tables, if you don't then Baan will report a DD/Physical table mismatch on the server whenever you start Baan. - not serious tho' (ttaad4231m000)

Secondly remove the references to this table - you may have to generate an IDEF1-X diagram (ttadv4131m000) to quickly show all references.

Thirdly remove the table definition (ttadv4120m000)

Fourthly convert to runtime all the tables from step two and three failure to do all the tables will result in a (temporarily) corrupted DD.

I would add few more steps -

1. Print References to Parent - ttaad4229m000

2. Print Where Used Tables - ttadv4430m000

3. Expire the Table Definition instead of deleting it. Someday, you might want to look at the table fields info etc or might realize the mistake of deleting it and want it back.

4. Remove entries in - Tables by database - ttaad4111m000

5. Check entries in - Logical Tables - ttaad4120m000
