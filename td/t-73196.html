I think the display device may be the origin of the problem. In LNUI you can configure the display device to generate a pdf output on screen. And somehow you can configure those pdfs to contain your logo (I never did this, so I don't know how to do this; but I know there are configuration files for pdf creation; may be in KB 1384719).

I don't think a session executed in AFS mode will run physically in the LNUI. I think it runs on the server without access to the LNUI. So, the display device will create no output, nor will it create a pdf (I'm assuming pdf creation is called via LNU).

My idea of solving your problem is to create and use another pdf device. I'm not quite sure if this will work, but I think you should give it a try.
I recommend checking some knowledge base articles: 1658894, 1658262, 1680655, 1384719
