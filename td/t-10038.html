Quite obvious... if you enter data with one session only able to fill half of the table, then you will find that all mandatory fields not filled are giving error. On the price field there is a restriction like <> 0 or so. If you want to do this, both sessions should also evaluate all tablefields, if there appear invalid data in either one field, you can take the option to zoom to the second session to get that field filled as well.

From the functional point of view this is happening with tdpur4120m000 wich can receive a purchase receipt, but when ILC is involved (AND if a storage unit, a storage date (LIFO/FIFO) or a lot code is involved) you will zoom to tdilc4113s000 to add on the other tablefields that may not be empty under that condition.
