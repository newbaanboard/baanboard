The ottstpstandard listing is the time consuming of the standard program. There is nothing you can do inhere. All actions (written by original and customized software) goes via the standard program to handle all treads to e.g. secure multi-user env., database actions, etc.
Focus on the listed procedures into the customized sessions. Remember that the profiling tool list the time needed to come back with the results, so if the database is slow by itself for whatsoever reason, this is reflected into the profiler too.
Good start is to optimize the database first, for e.g. Oracle first run the Oracle Optimizer before starting profiling the sessions.

Baancust,

As long as you use AND to specify more of a table's index, it will (or might) increase performance. Once you start using AND to fetch additional fields, or to build relationships with other tables, performance will suffer.

The best tip I can give you is to use BLAT to monitor your queries. It will give you all the data you need to analyze events before and after the DB driver.
