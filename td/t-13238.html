Help for ttadv5210m000 will be good material to start with and elaborate.

When using an NT Server slashes must be interpreted as backslashes.

SESSION OBJECTIVE
To create a runtime data dictionary containing software components by
package combination. This session is automatically called after you have
created a new package combination. It converts all specified components
to the runtime dictionary, including those that have not been
changed. The following software components will be converted to runtime
dictionary:
- table definitions
- domains
- enum descriptions
- sessions
The session will also dump the user data of users within the
specified package combination and the file fd6.1<package combination>
with redirections for software components.

The runtime definitions of tables, domains and enum descriptions are
stored in "d"-files in the directory as defined in Maintain Package
Combinations (ttaad1120m000). The runtime definitions of sessions are
stored in table ttadv999.

To create runtime forms and menus you must use the session
Create Runtime Data Dictionary (Forms and Menus) (ttadv5214m000).
Only the data definition of tables are created, and not the tables
themselves. To create the tables, run the session Create Tables
(ttaad4230m000).

This session must be run after changes in software made by utilities,
like:

- Import Data Dictionary (ttiex1285m000)
- Import Data Dictionary (5.0 dumps) (ttiex1286m000)
- Copy Software Components to New Package VRC (ttadv6265m000)
- Delete Software Components by Package VRC (ttadv6250m000)
- Copy/Move Language-Dependent Software Components (ttadv6200m000)
- Import Language-Dependent Software Components (ttadv6231m000)
- Maintain Enum Descriptions (ttadv4102m000)

This session must also be run after any changes in the VRC structure
made in session Maintain Package VRC's (ttadv1111m000)!


If Reconfigure Tables = yes, the system will reconfigure ALL tables
(the range of tables is not taken into account!) which are changed.
This will be done for the companies linked to the specified range of
package combinations.
Be sure that everybody has left the bshell, to avoid problems
with locked tables. If reconfiguring tables is not successful (e.g.
due to lock errors), it is possible to stop the process, by
answering "No" at the question Error during file configuration; try again
?. Later on, you can restart reconfiguring the tables, by running this
program again.
NT specific help
The output of the reconfiguring process will be logged in a log file
${BSE}\log\log.reconfig. You can look in this log-file, by using the
command "type" or "cat" on this file.

UNIX specific help
The output of the reconfiguring process will be logged in a log file
${BSE}/log/log.reconfig. You can look in this log-file, by using the
UNIX command "tail" or "cat" on this file
