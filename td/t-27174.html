Hi,

1) Command "set.synchronized.dialog" must be used only when the main-table is same for both the sessions.

2) If the main-tables are different for the sessions, you cannot use the above command.

3) If the main-tables are different, you can create a form command of type session or function. When clicking on this form command you can go to the desired session.

4) If you create the form command as session, make sure that you imported the desired field in the target session. If you create the form command of type "Function", you can use the command "start.synchronized.child" with matched index key fields. So that parent and child sessions will be synchornized.

Let me know if the above information is not enough to solve your issue.
