kbartelds
12th November 2003, 13:45
In maintain ascii file formats (tuxch0103s000), you can define the layout of the ascii file. There are 3 kinds of field: alphanumeric, numeric and multi line text. In case of multiline text the export scheme does not export the text number, but reads the actual textlines from tttxt010 (for all languages) to be put in the ascii file.

On subsequent import scheme (tuxch0122s000, form 3) the baan text field can be filled with the textnumber (field textnumber = yes) or a new textnumber can be created (field textnumber = no).

This, in combination with tuxch0112m000 (maintain ranges) gives you the possibility to use exchange to select exchange schemes to be exported (with scripts as text), so all relevant tu tables. On another system the same exchange scheme can be created to import the tu ascii files, including the conditionscripts, without overwriting existing textlines in tttxt tables.

I have done this many times: develloping exchange scheme and take it to the customers system, no problem!

Of course it is possible to have a seperate company in which exchange schemes are located. But if you want to transfer schemes from one system to another, the text might become a problem again, so use exchange scheme for it, that's why it is there for.

Best regards,
Klaas
