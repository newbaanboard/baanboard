JaapJD
1st June 2010, 09:44
In a Feature Pack upgrade there are two steps regarding tables. First is the reconfiguration. At that moment new fields etc. are added to the table. The second step is the Data Upgrade Engine, which updates the tables from a content perspective. So, if new fields need to be calculated from existing other fields, the Data Upgrade Engine does that part.
If you customize the table, reconfiguration will take care of changing the physical table to your needs. The Upgrade Programs that run within the Data Upgrade Engine don't handle custom fields, except when you have the source code of the Upgrade Programs. In that case, you can customize the Upgrade program to take care of you customized fields. But keep in mind that in this case you have to sync the installation of your customizations with the feature pack upgrade.
