It should work - if you are having problems on the form just try creating a new form variable called something like formfield_st1 - then set the form field in the script to the strip$(shiftl$(xx)). The field on the form should not have any preceding spaces - same if passed that variable to a report.

We always had a issue in 4c4 where the 4gl forms would sometimes add spaces on the form between fields(not in the field). So while it looked okay in the maintain forms - it was not perfect on the display form. But the field would be aligned the way we wanted it.
