There isn't. Yet.

I once started a project on sourceforge, to develop something like Baan tools in Java - exactly for the reasons mentioned. The project was sleeping a long time, but I am currently thinking about reviving that project and just did some major re-codings in the last months.

There is already a prototype of an inline SQL to java precompiler, I have a working data dictionary (like in Baan), I have a working application server and I am willing to donate a lot of code from a commercial application of mine (www.equilipresto.de) for building the form engine and probably also for doing a better relational-OO mapping.

So basically all the things that would be needed for building a Baan Tools equivalent in Java are already done - partly conceptually, most of them in some sort of code already; the thing is absolutely feasible.

But I stopped the project, because it's too big for just one guy to handle, there is too much actual coding to do. So if someone would like to join me developing the thing, send me a mail! I think with two or three people with some Java experience we could gain enough critical mass to find additional developers.

I think there is a need for an open-source development tool like Baan Tools, and there is nothing available. Meaning of course, that there is a potential market, and I strongly beleive a freeware ERP system could generate a lot of consultancy fees...

MooN can be found on

http://sourceforge.net/projects/moon/

There should be an installation software. You will need java 1.4, RMI and some XML extensions to have it working, but if you're interested and have difficulties, drop me a line - I can mail you a better working new version.

By the way, the project is called MooN, because sheep baa and cows moo...
