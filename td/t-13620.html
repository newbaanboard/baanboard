This may or may not help, but there are some configuration settings you can play with in the Crystal / Baan driver. A driver configuration file called "p2sbaan.cfg" is located under C:\WINNT\crystal\baan\config" (WINNT if you have NT or 2000). In this file are TEXT settings you can modify:

[STANDARD_SETTINGS]
.....
.....
.....
.....
.....
TRIM_TEXT_FIELD_LEFT=NO
TRIM_TEXT_FIELD_RIGHT=NO

You can try setting them to YES and see if that helps, or try what you preposed. The only thing with setting these is that it applies this to all reports.

Hello,

tttxt010 is a table that contains compressed text and includes control characters of different type (related to text positioning). In the baan report automatic conversion takes place and the user does not have to bother about the table tttxt010.

If you are using an external reporting tool, you would have to make the conversion yourself. The Baan programming language gives you the functions "text.read" or "text.to.buf" to help you with that.

I would not try to change tttxt010, you most probably will observe undesired side-effects in your Baan reports.
