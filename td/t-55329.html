this is a very common problem when using keyboard wedge scanners with Windows. The scanner is 'typing too fast', passing the data too quickly. This is not a problem when filling a single field, however as soon as you tab to different fields various things from the App and OS can interrupt/divert the flow of the keyboard buffer for brief moments.

There is no simple solution to this, you can try and slow down the data rate of the scanner but you are still likely to find the problem is only reduced.

If this is a one-off session requirement you could try and make a combined field to scan all the data then populate the rest of the fields.

The only proper solution is to move to an ASCII interface using a barcode data capture solution like the one found at http://www.arcgateway.com
