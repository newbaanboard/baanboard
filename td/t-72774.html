Depends if you need another or the same transaction:
* after.update.db: Same transaction, do not use db.retry.point and commit.transaction. If transaction fails all of updates are rolled back, also standard on main table.
* after.update.db.commit: Separate transaction, needs db.retry.point and commit.transaction in your code - after.commit.transaction hook
