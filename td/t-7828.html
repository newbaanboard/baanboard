Hmm..you are using the input only option on the form, and the input() function to force the input. I have not played with that - is there a reason?

If not, the standard is to declare these fields on the form as just input, not input only. On the form is the option for multi-occurance or not. If they are declared as input, not input only, you do not need the 'input()' function to get the user to the field, it's all controlled by the standard program. No coding required.

O..if they are not view fields, and you still want to make them single occurance, like in hours accounting...? You need the command in the before program:


before.program:
single.fields.on.mocc.form("tdobg010.frgt", "tdobg010.prft")


You also have to mark the current record you are on, at least that is how they do it in hours accounting. Then, when you are ready to go to that field, use the to.field(...) command.
