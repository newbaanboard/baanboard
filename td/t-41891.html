Hi,

Well, if you want to use a Job from your UNIX server then you will not have any connection with the client and you can not use the ottdllbw functions.

A workaround will be to copy a unique file from the UNIX server to the Windows server, and when the windows sees this file it will activate the DLL. In order to accomplish this you will need:

1) Write a FTP script that will copy this unique file form UNIX to Windows
2) Write a Baan tools script that will activate the UNIX script
3) Schedule a job in Windows to check this unique file
4) Write a batch file that will run when the unique file is found. the batch file should activate the DLL and delete the unique file.

I hope it helps,

- George
