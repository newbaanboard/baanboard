The Baan integration kit for Crystal is no longer supported, I think - this included a database driver for Crystal that accessed Baan tables in the same way as Baan application. However, you can use the standard Microsoft ODBC driver (SQL Client) with Crystal to access the underlying database. It is a bit more awkward to design reports as you do not get field descriptions or enumerated field values. Also, it is nothing like as easy to make the report run on a different Baan Company. However, the reports written this way run very fast.
