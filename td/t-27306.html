One solution is to write you own AFS session to call the program. What I do purchase orders is to provide a session with flags on it - file, accounting, etc. Then I print the PO for each type the user selected. I also have another session (cloned from print PO) that just points to a different report. The users also call this session through an AFS program.

Another solution is to create the AFS program, then export a variable for the footer you want to print. Ex. each footer would have a print expression sfoot=1, sfoot=2, etc. Then you export sfoot from the AFS program and import it in the report script. Not sure I tried this, but in theory it should work. You may have to actually control the footer being printed in the report script, but I think it should work as described.

I am sure there are other solutions, but this is how I typically control sessions where the users need to always generate mutliple reports without actually running the session twice and printing both reports.
