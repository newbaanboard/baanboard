I've had some strange effects some days ago when setting up a windows server printer. I changed one userfont to a large Arial font.
I followed KB 954187 by the word but it didn't work. The changed font was still printed without any change (default fount and default size).

So I searched inforxtrem.com again and found KB 1042513:

---------------------------------
Description: Custom fonts for BWPRINT.exe no longer appear when printing to a Windows Server Printer device type.

VERSION(S): ERP LN
SITUATION IDENTIFIED IN: BWPRINT.EXE
WINDOWS SERVER 2003
WINDOWS SERVER 2008

Resolution:
Bwprint uses registry key “HKEY_USERS\.DEFAULT\Software\Baan\BwPrint\Fonts” instead of “ HKEY_LOCAL_MACHINE\Software\Baan\BwPrint\Fonts”.
To add a custom font for BWPRINT.EXE the font needs to be added to:
HKEY_USERS\.DEFAULT\Software\Baan\BwPrint\Fonts> on the machine where your windows server printer device is installed.

Note:
The described workaround should not be necessary (compare KB 954187).
---------------------------------

I copied the font-settings manually with regedit from HKEY_LOCAL_MACHINE to HKEY_USERS\.DEFAULT which did the trick for me. Printing now used the changed font settings.

There's also KB 1130270 which may be involved, too. But I didn't look into that one.
