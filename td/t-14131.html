The predefined variable:

rcd.tppmmmxxxx

Holds the entire record buffer for the table. For example:

rcd.ttiitm001

Would hold the entire current tiitm001 record. If the table defs are the same and you want to back that up to, say, tiitm901, you can try:

rcd.ttiitm901 = rcd.ttiitm001

I think that would work. Update and commit. If that's what you want give it a test and let us know.

There is no real shortcut. If you copied the tccom010 table definition to tccus010 and added the new fields at the end, the copying of the record buffers will do, but don't use a normal assignment but copy.mem:


copy.mem(rcd.ttccus010, rcd.ttccom010)
tccus010._compnr = get.compnr()
| assignments of the new fields


If the new fields are not at the end, you can build two arrays with the fields of both tables (use rdi.table.column function) and get the values of the corresponding fields from the first buffer (with offset and size) and put them in the second buffer.
