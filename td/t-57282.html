Solution Description
English Convert to integrated session blocked checked out
TOOLS VERSION(S): 7.6_a5

SITUATION IDENTIFIED IN: "Integrate Dynamic Session and Form" (ttadv2910m000)

SITUATION DESCRIPTION:
During conversion of a static session with the "Integrate Dynamic Session and Form" (ttadv2910m000) session, the static session was updated correctly, but its form was not (i.e. the "Integrated" check box of the form remained unchecked). This resulted in an inconsistent session form relationship and made it impossible for the user to checkout the session (the message "Form not found" appeared).

This problem only occurred if the session was present in the current package VRC but the form was not.

SOLUTION DESCRIPTION:
During conversion an additional check is done to verify that the form is also present in the current package VRC. The conversion only proceeds if both session and form are present in the current package VRC. Otherwise, the conversion will skip the session and write an explanatory message in the session's log file.
