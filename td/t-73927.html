Hi,

- I have an enum field xxxyyy000.flnd in a table, domain xxdom.
- domain xxdom has values a and b
- I want the user to be able to change from a to b, but not from b to a

field.is.readonly is triggered before display and on checking the input.
If I code like this:
xxyyy000.flnd.is.readonly()
{
return(xxyyy000.flnd = xxdom.b)
}
the field is not readonly when it is displayed. But when I change it to b, the function is run again, this time it returns true, and blocks the change of the field, so I can never change the field to b.

How can I accomplish this? Should I check for old- and new value or is there an easier way?

I did somethink like this in the item basa data tcibd001 with extensions. (It's been a CDF.) But the use of the is.readonly hook prevented the save of the item data in other occasions/sessions.
