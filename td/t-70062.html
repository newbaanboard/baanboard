If you already maintained the Customization Project / Parts, then you can also use the Export Data Dictionary (ttiex1280m000) and the Import Data Dictionary (ttiex1285m000) to import into any VRC.

Bhushan's suggestion will also work, however you have to execute the PMC recipient setup in the Target environment. This is a better approach if you like to manage Software Components' delivery in long term so the dependencies are maintained.

However if you like to do one-off activities then the Export/Import DD sessions work faster.
