Yes, like Marc says, the fix.default.number function takes care of increasing the first free numbers.

This is how you call the function:

tcmcs.dll0050.fix.default.number(field, number.group, length, err.mess)

where:
- field = relevant table field
- number.group = the relevant number group
- length = the length of the number to be returned. [you can use tccom.dll0015.domain.length("fields_domain_name") to retrieve the length of the field]
- err.mess = message in case of error


Good luck!
