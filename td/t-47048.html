If your session is a Type 2 session (Update), then it will work in the init.form: section. You are then responsible to clearing the message when you do not need it anymore.

For Type 1 session (Maintain), if it is o.k. for the message to display for a small duration, you can use (for example) suspend(1000) after the message - in this case the message displays for 1 second and then goes away. This is because of the way the Type 1 session behaves when first presenting the form to the user. Again, this can be done in the init.form: section.
