Normally in the programs, this function is written to ge the user defaults. If the user has set defaults settings , then the imported values will get overwritten . If u need to delete user defaults, run the session ttstpdeldeflts for that purpose.

In the absense of set.fields.defaults() function is executed to set the fields to form field defaults. If the form.field has some defaults set at the form field level, ur imported value will also get overwritten. So in such case remove the form field default for orno.f .

If the value is changed after init.form in the prorgam , then display.all will definitely display the variable value as suggested by Mark.
