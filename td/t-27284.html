Hi,

The documentation of dir.entry suggests retrieval of files in a sequential way(?) which seems to be not happening in your case. Hence i would suggest a following workaround,

1. run command "ls | cut -f 1 > tempdirfile" using standard shell() function.
The above statement will create a listing of files in tempdirfile in alphabetical order.

2. Read the above "tempdirfile" file using seq.open , seq.gets functions.
This will get u the individual file name.

3. Use this file name to access the file.

I hope this solution works for you.
