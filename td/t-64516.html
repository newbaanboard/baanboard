We've had this strange effect too. We updated from FP6 to FP8+.

It seems you need to compile the labels first.

In FP6 the pop up windows asking for the report would display an empty line, if the label of the new report wasn't compiled yet.
In FP8+ an uncompiled label seems to surpress the report selection window and LN refuses to print anything (and it won't give any error).
