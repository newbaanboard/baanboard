The prinicple is easy: What you see as one session are in fact two sessions; the lower "detail" part is handled by the main session, and the list part is done by a sub-session (here: tisfc0506s000). You can observe this via the option dialog / start shell / ps.

The programming part is a bit more tricky. If you are interested, you should have a look at activate() and switch.to.process() for the synchonization between two processes, and sattr.combined for the layout behaviour on the screen.
