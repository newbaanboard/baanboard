This is due to shared memory and file caching on M$WinDos - even though on disk there is a newer object, you are presented the object from shared memory. First try log off logon again. If that does not do the trick, restart shared memory, if that won't help, reboot server.

Shared memory on Unix reloads newer objects automatically after they have been written to disk.
