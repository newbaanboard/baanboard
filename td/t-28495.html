1st November 2006, 15:46
My first thought is that the user id running the batch job does not have the same set of variables or mapped jobs. For example when you run the batch job you have a c drive. When the batch job runs in your environment there may not be a c drive. Just a guess.

We just ran into something like this yesterday - a user could manually activate a batch job and it would work. When the batch job was launched through the scheduler it would fail - turned out to be the environment variables for the user id which launches the batch. It needed another variable set.
