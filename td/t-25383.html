Hi,

If the definition file in the $BSE\dict is not corrupt then you can try this. Clear the tables ttadv501-503, change any one label in each table definition and each domain that you have created and run CRDD for those tables specifying full range for the Package Combination also ticking all check boxes.

If it still doesn't work then the issue is with the corrupted definition files.In that case, rename the dtable_name files for both the tables which is found under $BSE\dict\your_package_combination\ to something else and run Create Runitme DD for the tables your created and then follow the above steps again.
