We had similar issue and got the solution for LN Tools version. Check with Baan Support for a corresponding fix for Baan-4 versions.

SolutionID 226417
English Printing of reports to Outlook SP3 results in error message
TOOLS VERSION(S):
7.6_a1, 7.6_a2, 7.6_a3, 7.6_a4, 7.6_b4

SITUATION IDENTIFIED IN:
"Conversion to Word Processor" (ttstpconv)

SITUATION DESCRIPTION:
When trying to use ttstpconv to email reports to Microsoft Outlook 2003 Service Pack 3 from within Baan the following message is returned from Outlook to the user's screen: "The command line argument is not valid. Verify the switch you are using." This is only happening with Outlook 2003 SP3, problems do not occur on user's workstations having Outlook 2003 SP2 or 2002 SP3.

SOLUTION DESCRIPTION:
When Outlook was started with the command "outlook.exe file", an e-mail was created and the file was attached to the message. With Outlook SP3 the command is no longer sufficient: The switch /a is needed to attach a file to an E-mail message. This change is implemented in the object ottstpconv.
If your Current Tools VRC is:
7.6_a1:(FP1)
/ftpdownload/updates/7.6_a1_tt/224129tt.tar.gz

7.6_a2:(FP2)
/ftpdownload/updates/7.6_a2_tt/224129tt.tar.gz

7.6_a3:(ES 8.3)
/ftpdownload/updates/7.6_a3_tt/224129tt.tar.gz

7.6_a4:(ES 8.4)
/ftpdownload/updates/7.6_a4_tt/224129tt.tar.gz

7.6_b4:(ES 8.4.1)
/ftpdownload/updates/7.6_b4_tt/224129tt.tar.gz
