Our UNIX admin wrote some kind of push command file that would locate the new files in one application server and move them to the other server. Sorry but I do not remember what commands he used - we only have one application server now. This moved all objects including the data dictionary files. It was not automatic, but it was quicker than creating customizations and then exporting and importing them.

I guess you could create a common directory between the two servers. On the devolopment server you could export into this directory. Then have a batch job that would import what ever it found in this directory. Then delete it after the import. Just a thought on something you could try.
