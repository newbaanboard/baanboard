BaanBase is not ODBC compliant. Hence u can not have and ADODB connection to Baan.

However u can write a routine in Baan to to create a text file in Baan from table , transfer it to client, create a data source programmatically , and have an ADODB connection with Mictorsoft Text provider.

Remember with text driver there are some limitations like u can not delete a line in text file.

If u use Office 2000 or higher u can use Microsoft Query to query the text file and create Offline cubes also with the help of MS Excel & MS Query just from txt file . These offline cubes (.cub) files can be opened in MS Excel with the help of an olap query file (.oqy) . Microsoft is increasingly putting the multi-dimensional capabilities within the gambit of MS Office.
