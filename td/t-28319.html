Dear mr_suleyman,

normaly such software works like:

You catch the windows api message from the windows form (has nothing to do with baan) (e.g. Sessioncode, Sessionname, Company number), and you manipulate the message (change the colour) with e.g. Visual C++. ( Baan Form ----> Winodws API Message ----> Your Code (old: white // new: green) ----> Send Message to windows.

With colour changer (a very great tool by the way, and totaly pays back the invest in reduced support because of users working in wrong company) there is only one constraint, if the company number is not fully displayed on the form (e.g. a very long session name sample :: "tiitm9101m000 Very long Session Name [10"), then you have not all informations to colour the form.
