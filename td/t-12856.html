Hallo Santosh,

To my knowledge the only way to interact with the database backend is to use table triggers. Generally you have to do the following:

1- Create a simple baan table with say one field.

2- Using database tools create a before/after update trigger for this table. Place your call to the stored procedure in this trigger.

Now every time you want to call your stored procedure you just have to update the Baan table using a new field value.

You could also send parameters to your stored procedure by adding these as extra fields in your baan Table.

Hi,

An old thread, but one more option to execute stored procedure is to use sqlcmd utility if you have SQL Server 2005 or greater. I have used it in some occasions and it works fine.

Just create a Stored Procedure on your database, now create a bat file on server with the command to call the stored procedure. In Baan, use run.prog() function to execute the stored procedure.

One more way is to use the bat file as Scheduled Task, a call this schedule task from Baan using run.prog() command.

2 ways to call SP :-

1:-
SQLCMD -S ServerName -U USerName -P Passoword -d DBName -Q "Exec SP_Name"

2:-
sqlcmd -E -Q "EXEC usp_test @param1=N'%Param1%', @Param2=N'%Param2%'" (If you want to pass parameters to the Stored Procedure)
