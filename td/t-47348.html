Solution Description
English MR_INDEXOUT error during start of session
TOOLS VERSION(S): 7.6_a2, 7.6_a3

SITUATION IDENTIFIED IN: "4GL Engine" (ottstpstandard)

SITUATION DESCRIPTION:
When starting a session an error like the following is displayed:

1 : process 7 - Fatal error : [MR_INDEXOUT] index 251 from Dim [2] 210
2 : process 7 - Fatal error : Can not continue in whwmd2400m000 in DLL: ottstpstandard (gui.calcobjectorigin)[get.object]

This can occur if the form of the started session contains a group with only checkboxes, and one of the checkboxes is made invisible. Whether one of the checkboxes is made invisible or not, can depend on parameters, authorizations or other things.

SOLUTION DESCRIPTION:
The calculation for the position of fields went wrong. This is now solved in the 4GL Engine.

Solution Description
English Fatal error in / BW message about dal.do.reset.initial.fields
TOOLS VERSION(S): 7.6_a3, 7.6_a4

SITUATION IDENTIFIED IN: "4GL Engine" (ttstpstandard)

SITUATION DESCRIPTION:
1. The following BW message can occur when running sessions, after installing solution 223745:

Fatal error : [MR_INDEXOUT] index 38 from Dim [1] 6
Fatal error : Can not continue in <session> in DLL: ottstpamdll (dal.do.reset.initial.fields)[last function:set.mem]

2. Other fatal errors may also occur, like:

Error : SQLState QP999: Table does not exist (error 506) (tcmcs003.warehouse.is.wms.controlled() in object otcmcs003)
Fatal error : Error 506 (Table does not exist) on select

This is because DAL hooks are called in a wrong sequence, which can result in a wrong initialization of fields. In turn this may lead to reading tables in company 0, in which these tables do not exist.
