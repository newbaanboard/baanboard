When writing queries for group functions like min,max,sum,count(*)
we very well know that it returns only one row.
This is not always true.
Refer to the tools help manual -
The power of the GROUP BY clause is most apparent when it is used with the functions min, max, count, avg, and sum. In these cases, it produces summary values for each group.
You can execute the following code and see the results-
function main()
{
long counter
if spool.open("", "", 1) then
select ttadv200.cpac, ttadv200.cmod, ttadv200.cses, count(*):counter
from ttadv200
where ttadv200._compnr = 000
and ttadv200.cpac = "tc"
group by ttadv200.cpac, ttadv200.cmod, ttadv200.cses
selectdo
spool.pr.line = concat$("|", ttadv200.cpac & ttadv200.cmod &
ttadv200.cses, counter)
spool.line()
endselect
spool.close()
endif
}

The "as set with 1 rows" clause can be used to tell the bshell that no other rows need to be retrieved from the database. This can be used when there is no additional data or no additional data is needed.
