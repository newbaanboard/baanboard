Hi all,

I have a question about the functionality of function double.cmp().
What I know about this function is that you give 3 arguments as follows:
1) first double
2) second double
3) the tolerated difference in order to still be considered equal

I know that in many standard Baan scripts this function is used when checking values of doubles, even without comparing. For instance:
to check if a 'double'-amount is > 0, you will call double.cmp(amount, 0.0, 0.000), so then basically the amount is compared with the value 0.0, and the tolerance is 0.000, so if the amount is > 0, the return value of the function should be 1.

My questions:
1) When I applied this, I found that even when the amounts were 0, the result of the function was 1, i.e. that the amount is higher than 0..!! How come?? Am I doing something wrong??
2) Is it risky to check just simply like:
if amount > 0 and amount2 = 0 and amount3 < 0 then etc.?


Would appreciate if anyone could shed some light...

En.

I never use zero tolerance. I know that sometimes with double types a 0 is not really a "0". This has to with how the double types are stored. I remember from my HP days a double 0 could sometimes be stored as .99E-99 - well something like that. So when we converted to Baan and I used the double.cmp command I set the tolerance to 0.0001 or 0.00001. I never really checked to see how Baan and Oracle stored some of these values.
