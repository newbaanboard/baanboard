Value 'No action' and 'raised' of 'BW message window options' in
option dialog window keeps the BW message in the background. This helps
in some cases.

I have not been able to change the value of the options through program.

Some standard program too while evaluating expressions (e.g. invalid default
expression in tdsmi0150m000) or when starting non-existing client program
also raise BW message appearing on the screen. By writing proper validations
the BW messages can be avoided to a greater extent but can not be eliminated
totally.
