My first thought is to write your own device driver - instead of using ttstpconv, use your new device driver. Then you can manipulate the filename anyway you want - you can see an example of replacing ttstpconv in the code and utilities forum.

My second thought is that after the session script prompts for device you can always close it down, reset the filename and re-open the report with the new spool.filename.

As far as I now the ${variable} are pulling environment variables from the login shell - in my case a UNIX login. That makes things like ${HOME} and ${LOGNAME} work with is set at the OS level. I am not aware of anything on my system in the environment variables.

Of course you might want to complete your profile to include the OS of the server.
