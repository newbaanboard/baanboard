Hello,

We are managing English and French users. English Excel manages the . as decimal point and commas in the csv file.
To avoid a different environement, we use a different bshell and a startup programm (shell) in the desktop menu which sets the datecurr back to the right value.

Regards

bshell:

#bshellu utilise pour societes anglaises
. /administration/setenv
cd /baan/app/bse/lib

qui=$HOME
qui=${qui##*/}
grep "$qui" /administration/ut-anglais > /dev/null
if test $? -eq 0
then
cp datecurruk datecurr
fi
cd $HOME
$BSE/bin/bshell6.1 $*
