17th June 2004, 14:48
The discrepency between ttadv301 and actual form can occur in 2 cases.

1. programmer has used function input.field.invisible function to make the field invisible. Think less likely situation in ur case.

2. the form dump has been copied / delivered at the directory level without updating the same at tools level . In such case u can see the field variables at command prompt in unix by command 'strings <formdumpfile>' . We have quite few such cases of some programs from original installation.

So finding out such fields programatically is quite difficult.
