Sanjeev,

You can give authorizations to all modules in a package or selected modules. Refer to the session help -

Developer Authorization Template (ttams1151m000)
Use this session to define the authorization for developers in a template. The template can be linked to a SSA ERP user in the User Data (ttams1100s000) session.
You can specify in the template:
The package VRC for which the developers are authorized to maintain and develop software components in SSA ERP Tools.
The languages and modules of the specified package VRC for which the developers are authorized to maintain and develop software components.
The status up to which the technical writer is authorized to maintain the Help texts in SSA ERP Tools.
Click Convert Changes To Runtime DD on the Specific menu and restart SSA ERP Tools to make all changes to the template effective.
Related topics
To define authorizations
To define templates for AMS
To use an overview session
To maintain records in an overview session
Field Description
Template Code Specifies the code of the developer authorization template.
Package Specifies the package code of the package VRC for which the developers, that are linked to the developer authorization template, are authorized.
Package The description or name of the code.
Version Specifies the version code of the package VRC for which the developers, that are linked to the developer authorization template, are authorized.
Release Specifies the release code of the package VRC for which the developers, that are linked to the developer authorization template, are authorized.
Customer Specifies the customer extension code of the package VRC for which the developers, that are linked to the developer authorization template, are authorized.
Customer The description or name of the code.
All Modules If this check box is selected, the SSA ERP user, who is linked to the developer authorization template, is authorized to develop and maintian software components of all modules in the specified package VRC.
Note
If this check box is cleared, the modules for which the SSA ERP Tools user must be authorized have to explicitly specified in the Specific Modules fields.
All Languages If this check box is selected, the SSA ERP user, who is linked to the developer authorization template, is authorized to develop and maintian software components in all languages of the specified package VRC.
Note
If this check box is cleared, the languages for which the SSA ERP Tools user must be authorized must be explicitly specified in the Specific Languages fields.
Authorization for Components of other Developer If this check box is selected, the SSA ERP user has permission to modify software components developed by another developer in the specified package VRC.
Specific Modules Specifies the module for which the SSA ERP user, who is linked to the developer authorization template, is authorized to develop and maintain software components.
Note
You can only specify a module if the All Modules check box is cleared.
Specific Languages Specifies the language in which the SSA ERP user, who is linked to the developer authorization template, is authorized to develop and maintain software components.
Note
You can only specify a language if the All Languages check box is cleared.
Until Helptext Status Specifies the status of the Help text up to which the technical writer is authorized to maintain texts.
