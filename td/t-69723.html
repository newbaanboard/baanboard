.is.field.invisible only works when you manage the field using inputfield.invisible or inputfield.visible from the script. I never found a call that could be done to check field availability based on session personalizations. I have written my own function by checking ttadv900 by package/module/session (index #1), the field name (ttadv900.name), and the personalization fields for user, DEM role, or company. The 'visi' field can be used to determine if a field is hidden via personalization. ttadv900.valu will be "READONLY" for visible fields made read-only in personalization.
