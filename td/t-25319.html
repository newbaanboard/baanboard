Through Crystal Reports you connect to the Baan system through their interface using your Baan userid, password, hostname, BSE, and company number(s). Once in, it displays the Baan data dictionary tables and their fields for you to select from. You can select fields that consist of numbers, dates, strings, enums, and text and create custom reports in the report tool. Since you are actually logged into Baan, it will observe company, session, table, field, and range of data restrictions that you can set against any userid through Baan. The data that you report against has the same appearance as the data you would normally see from any Baan session. You do have to know how Baan data is structured when you construct a report because you are working directly with the relational structure. Keep in mind that not all data displayed in Baan sessions is directly off the table, but might be derived programatically. Don't know if this helps?
