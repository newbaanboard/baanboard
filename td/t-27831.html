HI nansat99,

Your problem is resolved or still open? As Mark suggested, put the report in the debugger and check the temp file that is generated for the report, open the temp file in the Notepad and check if all lines have same number of record elements(fields), the first record element and lat record element are same per line.

I suppoes that there should be some hidden "line feed" character with the data and you must find at least one line which has different number of record elements (record elements are separated with ","). Note this line, and also ckeck the line above and below as one is the part of the line which has problem. The last or first field in the line has some problem in the data, correct it in the related session.

May be you can get the some other information in the line like transaction type, batch number, document number etc. Print the report by providing these specific information and make it conform that report has the same problem.

I hope you have enough clue to fix the problem.

Best of luck
