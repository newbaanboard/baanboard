19th October 2006, 15:51
Are you talking about ttstpsplopen? If so I do not recommend using save defaults in it. This can cause problems. We have run into situations where reports stopped printing correctly because of this. I recommend using default printers for users. I have found that the defaults saved with ttstpsplopen become corrupt over time and in some cases it caused reports and scripts to stop working. The first thing I do when trying to figure out a reporting or printing problem is to "delete all user defaults".
