I think, the only way to use nvarchar will be using a Text field which would be rather a little complicated table design because you might also need to cast your nvarchar data to varchar datatype. Else, you can create a new string domain with say 500 characters and use it for your table field. Then map the nvarchar field to this new field. Or say create an array field of 500 char string and split the nvarchar value into these 5. Depends on how you want to implement this.
