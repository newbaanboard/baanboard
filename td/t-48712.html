Usually Baan company never gives tt package sources to the customers.

In some way, you can see something (not the code, but some useful informations) of the code (the dll calls, all the literals [SQL instructions are literals :)]) usin bic_infoX.Y on the object.

try:

bic_infoX.Y ottadv6265

Where X and Y are appropriate, by example for Baan IV:

bic_info6.1 ottadv6265
