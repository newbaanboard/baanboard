The question is do you really need source? There are a lot of people who have Baan and donÂ’t have source code. They using Q-Key from Fullscope (www.fullscope.com), RMCgen from RMCis (www.rmcis.com) or RUSH (Baanboard) to modify the code. (Do a search on RUSH and you can find additional information.) There was actually a presentation from RMCis at the 2003 BWU conference regarding this issue.

Additionally, remember that once you starting modifying source code you are creating a maintenance issue. Anytime there is a patch that involves a script that you modified you will have to get the update source and reapply you changes.

I think you will have to distinguish between a source license (which includes ALL the Baan sources) and the source of a particular session. You should be able to get the latter one by Baan support, but you must be prepared to pay a fee for it.

If that is not possible, I would strongly discourage you from using Baan - really. It is a MUST for running a business critical software to have the sources available if really needed, and Baan/SSA Global MUST be able to deliver those sources at least on demand if they want to stay in market. I do not know a single Baan installation in Germany without source code level customizations. QKEy etc. ist not an option, the TCO is far higher compared to creating (well documented) customizations in a Baan session script.
