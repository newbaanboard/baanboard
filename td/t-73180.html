Along with the link mentioned above, here are some things to check regarding
254 Error (Commit transaction has failed in the audit server):

◦The table ttaud110 Transaction Notifications needs to be created per company (either GTM allocate or Create Tables session) so audit can get the transaction number information needed when audited tables are processed.
Note: Check Tables session will not show this table as 506.
Note: Check if table is logically linked.

◦Check that all of the companies are in the $BSE/lib/auditdef6.2 file because if some are missing even though you aren't processing those you can still get a 254 error.

◦Run session ttaad4560s000 "Display Audit Sequences" put in the table that has the issue (should be in log.bshell log file) tab then put in the company. Then hit stop sequence button.

Note: Also could be related to file system being full.
