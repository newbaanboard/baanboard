Since the configuration.bwc file is essentially on the user's PC, one way to do this by controlling the write permission on the configuration file to prevent the <Save>.

To prevent the <Save As>, you would have to control the permission of the folder (for example) C:\Program Files\Baan\lib\user

Unless you have automatic ways to control these, it may not be a very cost effective solution - can become quite an administration head-ache to keep things in ship-shape condition.

Try to implement Wortop Central Deployment.

shortly:
You install BW and Worktop on some computer, activate central deployment (actually you just share couple folders on that computer).
Then you send link to worktop document to all users (It's not possible to save pasword in this case). + uninstall BW from clients PC.

There are some additional bonuses from WCD:
- Easier to upgrade versions of clients
- comunication via Worktop (sendind messages).
