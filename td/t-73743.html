I think DALNOMETHOD is raised, when the name of the function/method does not exist in the DAL. As far as I know a method used in dal.start.business.method() is an external function in the DAL.

Well, on our LN 10.7 there's no function named "update.customs.value" in DAL whin220 in the standard source. It does not exist in LN 10.2.1, too (so it is not a deleted function).

If you have an external function in the Functions hook of the table extension, you can call it from the session extension in two ways:
1. Add a #pragma used dll otxestwhinh220 and call the function directly
2. Use exec_dll_function()
