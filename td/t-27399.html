Yes mark is right(as always :) ) , just check your fourth field(as 3 fields are already scanned) declared in "report input fields" . As suggested run the report in debug mode , whenever the program execution halts, check the data values from fourth field onwards. You can probably find the culprit. If the fourth field is a string field then look out for any newline character or if the fourth field is a number then look out whether accidentally the value obtained is by dividing a number by zero.
