There is a limit to number of fields in a report (If I remember, 255).
However, there is a limit of 255 characters in a line, so it's highly improbable that you can do this, if the average length of the fields are more than 6.375 chars (without separation characters).
If you use the report to produce an ascii file, maybe it's better to use the seq.X instructions instead...

Yes - I have done it two ways. One report has one input field of like 200 characters. The report field is buff. In the session script I just did a select and then build the buff field. Using like buff = buff & sprintf(" |%s", some.field). Something like that.

The second way is in the report script. Basically the same thing except I was printing like the newest records(like last 5 PO's). So I knew the string would only be 30/40 characters long. Other than that it was a normal report.
