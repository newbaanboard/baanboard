I don't think this is possible. The XQuery in the business context model is executed for each visible entityType in the context message. Only the id fields of the 'current' entityType are available.
But you can extend the context message for InforContractSchedule and add the item code there.

Creating a session extension with Extension Modeler and changing/adding context there is the best way for 10.7. Ensure you have KB 2177731 applied as well. But changing the context model in LN Studio and regenerating the context library is also possible.
See following guides:
- Infor LN Extensions Development Guide
- Infor ES InContext Modeler Development Guide
