Like Mark said - this is a known problem that users tend to deal with after a while.

Some background may help. The read is done by index, and if you do not fill in the full index, there is sometimes a residual value in there, So... if you want to find
Sales Order=1
Position=10
Sequence 1

and you only put in
Sales Order = 1
Position = 10

it will most likely have the current highlighted record's sequence number in hand, probably higher than 1. When it does not find an exact match, it will go to the next higher record.
Another problem exists when all the fields of the index being searched on are not on the form. In this case, those fields will not appear on the find either, so there is no way the user can insure the full key is entered, so results are as you found, not predictable.

Moral of the story, if they can put in the full key, it will help to find the right record, but not always.
