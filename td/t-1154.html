They don't all have to by Dynamic, but that's why you are getting your start-up errors.

Dynamic forms get created empty, and you need to go to the Dynamic Form Editor to place even the basic table fields on the form before it will run. There is a button called add all table fields or some such that will put all the main table fields on the form. It won't run directly after generation.

Why is your Dynamic option always checked? I don't know. Mine is checked by default, but I can change it. Perhaps you need an updated tools patch...? What service pack are you running on?

As for moving your own customizations from IV to V - there is no tool but "Export Data Dictionary" from Baan IV and "Import Data Dictionary" in Baan V. The 2 versions should be compatible. You can run the printouts "Print Components in Package VRC" to find out what is custom on IV. You then need to upgrade them by hand to the Baan V environment (ie, new table names, etc.).

Good luck
