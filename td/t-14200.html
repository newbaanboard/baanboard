I agree, load that master data first.

One thing I have done in the past is create 'staging' tables for importing. The staging table has the same fields as the destination table, but has no references and has everthing in string format.

You can run the import to the staging tables. Then fill your tables from the staging tables when ready. The staging tables will accept just about anything an ascii file can throw at you, and you can validate from there, removing records as they are succesfully put into the final table.

Alternately, you have seen the path to a more complex solution. You can turn the ref. integrity off during a seq. load, and really all the seq. load is is an ascii file. You can make your file the same format as the seq. load is expecting, and use that.
