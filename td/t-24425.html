Hitesh's idea is good to make a OS independent functionality in BaaN. If its gets too complicated to handled multi level subdirectories, then you can use the OS options -

I see you are on Windows. Use the following command from the app_start() and redirect the output to a file and then read the last 2 lines in that files and you will get the information.
C:\>dir directory-name /s

68 File(s) 66,473,972 bytes

Total Files Listed:
1157 File(s) 600,298,492 bytes
296 Dir(s) 37,939,855,360 bytes free
