I have ran across this at a client who is running 8.4.

I am aware of the problem with the locale for ISO8859-1. Within the TSS_Locale6.1 files it defines the printable codes for the character sets. On Western character sets this is a "z" on other character sets it is usually a "y" with two dots on top.

No matter what character set I use, I get the tilde ~. Before anybody asks, yes I converted the user to runtime.

There is an issue on the latest 8.4 notes that states that there is an issue with SQL, 8.4, and the set_fmax function. But no detail.

Please share your experiences with this issue.

there is a table tttss002 with a field fmax.

The session to use is tttss0504m000 - password is the same as for validation.
Double click on the char set and the maximum form value can be changed.

With tttss0501m000 or tttss0103m000 you can do a convert to run time.

You have to be certain that you are not using characters that have a larger number than the value that you set for maximum form value.

Hope this helps
