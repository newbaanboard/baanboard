Yes - it was discussed also, but not sure where. I really do not recommend this, but what you do in the report script is count how many detail items or lines to be printed. So you have to get the page length, count the detail lines, footers, before.fields, after.fields. You have to figure out how many each get printed. Then total the number of lines - then take this and figure out how many pages are going to be printed. If it is a simple report then it would not be that hard - but for something like purchase orders it is just about impossible.

We use this at our site for purchase orders, but because of all the text it is not 100% correct. I could never figure out all of the ways text could skip to the next page. I really do not recommend this solution.

Good Luck!
