One method of doing this without sources, but with development tools, is to have multiple copies of the session that you want different reports to be linked to.

e.g.
session tdpur4401mo50
report tdpur4401m050 <-- company 50 version

session tdpur4401mo60
report tdpur4401m060 <- company 60 version

The have an upfront session that will call the appropriate session when in a company. Only add this upfront session in your menus or business process diagrams.

e.g.
session tdpur4401mo00 <- upfront session

has the code:

before.program:
on case get.compnr()
case 50:
activate("tdpur4401mo50")
break
case 60:
activate("tdpur4401mo60")
break
default:
activate("tdpur4401m000")
break
endcase
exit()
