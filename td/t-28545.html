Nope, Sole. Actually most of the time, Line is a variable set in the program script of the report session. It is usually a flag used by the programmer(s) of the report session to differentiate different type of records returned by the query. Since Line is variable declared by programmer (as opposed to predefined variables), don't expect it to have same meaning across reports. To fully understand the meaning of Line variable in each of the reports, you have to have access to the program scripts of all the involved report sessions

Rgds,
Paul

Adding to what Paul has mentioned, output expressions can be at times treated as the WHERE clause in a SQL statement. Based on what i understood from the years of working with Baan Reports, the expression Line = 1 is a flag which makes the section visible on the report. I have seen line =1 for making the section visible and Line =0 for hiding it, but i have never seen Line =2. Sometimes you can find that there is a criteria mentioned.
For e.g. Output Expression = tdsls040.txta (which makes the section appear on the reports only when there is a header text present.)
In short, expression can be treated as a criteria or a condition to show a particular report section..
