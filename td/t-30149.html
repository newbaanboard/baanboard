I am not sure this will help, but here it goes:
(1) If it is a custom session make sure field order on form is correct. It seems to me back in 4c3 we had this cause a problem on a job. You also might want to put it in debug just see what it is being fed. If it is a baan session I would report it to baan.
(2) I would do make a job again to a new sequence and print the job with session details to see if it does it again.

With the problems you are having with reports and this, it might be time for a tools update. I know in the past we had jobs become corrupt and we never found out why. I can't recall the last time this happened - I know it has been a couple of years. Not much help, but it is all I recall.

Now on SP20 we had a job that ran the same session 6 times, with different parameters. It test the 6th run would not work most times. We reported it to SSA, but then it started working. Baan did recommend we get the lastest tools patch, I have not heard any problems with it in production. What fun!
