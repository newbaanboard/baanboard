The Baan Job Process will already disable messages, instead writing them to the Job History table (ttaad512 I think). Unfortunately I'm pretty sure this hangs on questions.

If you use API programming to program the code you are running (or use Baan BOI's), then the messages are stored instead of displayed and the default answer for questions is take, or you can program your own answer. So, for example, you could make a custom session that runs a standard Baan session using API methodology and call that session from a Job. I have not tried but I think that would accomplish this if questions are a problem in the session you are trying to automate. API's are a big topic, search the boards for 'stpapi' for more info on that coding methodology.
