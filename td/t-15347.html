You can use ttstpdellocks to remove locks on source code, but I would only do this if you know it should not be locked. As for restricting code we use UNIX level permission on our production VRC - no write permissions. So we have to copy it to dev, where we can change it and compile it. Then when it is approved it is moved to all of production VRC's. There are other ways to lock down source, but this is how we do it.

You have to do it in 2 steps -

You have to create a developers authorization template (ttams1511m000) and then assign to the user data (ttaad2500m000) Authorization form tab and Developers authorization, and execute a runtime for the user.

Also, as Mark suggested you have to lock down the file access permissions for the VRCs/directories for which the developers can access or not.
