Dear Shah_bs,

baan made a modification for german customers for the tcorno, tcpdno domain because the customers have to keep records for 10 years because of legal requirements. (i guess from 6 to 7 digits).

I had to make this modifications, maybe some usefull hints:

(1) you have to check all forms and reports manually because you need the space for the domain enlargement in the layout.

(2) you have to check all scripts for stuff like"999999" (in relation to the domain, e.g. in SQL commands) and now you should set it to the max value of the domain (bad solution just use 2 more 9, better is the solution of baan using a baan function to check the digits of the domain).

(3) You have to open the reports, forms manually and dump/compile them afterwards. A CRDD is not enough!
Be aware you have to copy all Repors which are using the tcpdno domain to a customer/own VRC.
This is a lot of work, otherwise your reports e.g. for production are not showing correct data.

(4) This change is also for all tables using tcpdno like tisfc001 and others. So you need to convert them to RDD.

Other problems could arise if you use the tcpdno field as part of another table field e.g. concate a serial number (pdno & running number).
Now you have to deal with old 6 digits and new 8 digits.

You can check baan support for documentation for the german solution (should work on every system). There are also some check programs included.

Regards,

Kozure
