Hi,

This information is available in case the language is installed on the system, i.e. if you can switch the system to the French language, this information should be available.

When using a user defined date format in sprintf$ I don't think you can specify the language, but when using a pre defined format in Baan tools you can specify the language. So if you have a date format with the full month name in it you could print mont names. For French you would get something like sprintf$("%D012,4", your_date).

As an alternative you could define a domain for the months (I don't think a standard domain is available for this). If you make this an enum domain you can define the enum descriptions in multiple languages and receive the description of the required month using enum.descr(domain, enum_expr [, language])

The text comes form the runtime dictionary.

For non-tt domains, you could use maintain domains and maintain enum descriptions to add french descriptions, create/convert to runtime dd, re-logon, done.

For tt-domains, baan does not allow to enter anything. But I'm not sure if it is a problem if you have a closer look at the file $BSE/tools/dd/dtt.pd/dtt.pd ;-). There you can see e.g. :NAME:ttaad.moab and lines like e.g. :003march~Mrt~Mar... which obviously is dutch~english... (sytem language 1, 2, ...).
