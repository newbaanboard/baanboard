If you own source code you can modify it to fix problems. Otherwise you can notify Baan about the problem. Of course if you modify the code you end up supporting the code.

As a DBA there are certain things you can do in the back ground to help - my DBA's used to create indexes to help the backend queries. In some cases it was our code that was ineffiecient, but the indexes fixed the problem. Some cases it was Baan. We fixed our code and reported some sessions to Baan. All have been fixed - so far.

It is true that there is proprietary baan code that contains less than perfect SQL. It's been a long time since I've seen unmodified code shoot through the roof though.

The hints in 3GL are mostly ignored, depending on your porting set. The database driver adds hints of its own, and in the past I've seen many questionable ones from the oracle8 driver.

Forcing hints on the Oracle side is not what I consider "best-practice", but it is an option, as is adding indexes.
