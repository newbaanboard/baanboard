You must first compile the report in debugger, as I think you have already done. 2 ways to set the break. In GUI, find the line you want to stop on and double click. This sets the break, and highlights the line number. You can do the same in GUI or ASCII by focusing on that line and pressing 'b' <return>, which will also set the break point and highlight the line.

One little problem in reports (and exchanges) though, as I think you also see. When you debug, you see the standard report program, which is more complex, all around your program. If you do search for your script, and find it, you cannot set your breakpoint on it. I think this is because it is not part of the current source, which is really the standard program. What you do instead is set the break point in the standard program, on the call to your section, which is in the form of a function name. For instance, if you are trying to break on code in the detail.10, before.layout section, search for your code, and you will eventually find it in the function:

r.s.detail.10.bef.lay()

If you search around, you will find the call to that function, and you can place a break point on that.

Edited: ok 3 ways to set a break - in GUI there is also the break button, which you probably already saw :)
