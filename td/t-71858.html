Then I would try using the name of the local printer, not the sharing name. It used to work, but I haven't used it for some years. It may be worth trying.

In the field where you've entered the printers sharing name \\localcomputername\sharedclientprinter enter the name of the printer on the client.

This will work on every pc if the local driver is installed with the same name.

Tested it: I used my local printer "PDF24 PDF", thus typed PDF24 PDF instead of printer sharing name (the name of the local printer). It did open the PDF24 tool, when printing to it from LN. It did work on two PCs (PDF24 is installed remotely on every PC with the same local printer name "PDF24 PDF"). I did test on LN 10.2.1, but I do remember it from BaanIV C4 times. I think it worked on Baan5, too.
