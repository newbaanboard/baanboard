It is a well established fact that MS Excel is an excellent user-friendly
presentation (as far as quantitative information is concerned) and
analytical tool. With recent enhancements in MS Excel 2007 (http://www.erpjewels.com/excel2007enhancements.htm) and
Duet- JV between Microsoft and SAP, (http://www.sap.com/community/pub/flash/kw11_07_story_2.epx) it has become more evident.

Already Baan customers know about the importance of MS Excel and try many
ways to provide good Baan data access in MS Excel .There are many ways
and work arounds for data access and transformation of Baan data in Excel .
But none are complete for many reasons

1. Complexity of output formatting either through a tool or manually due to
lack of native data dictionary and deeper integration with Baan objects.
2. Longer lead times for getting data in excel
3. Lack of data transformation functionality in current alternatives
4. Lot of complexity and difficulty in changing backend Baan objects repeatedly.

All these leads to longer lead times for MIS generations and probable loss of user
Interest in MIS which in turn leads to delayed and inaccurate decisions leading
to more losses.


Today’s world is dynamic and so are the requirements of IT. Practically no
solution can satisfy the complete requirements IT in all situations . Different
situations and scenarios demand different approaches and different solutions.
Multiplicity of solutions to fit diverse requirements of a company is the order
of the day . This is what all leading corporations do to keep it agile and vibrant
moving forward fast. So deploying an integrated tool like ERPJewels will definitely
add to the vibrancy and agility of Baan customers .

ERPJewels is one such excel reporting solution on Baan which can provide easy
Baan data access and transformation in MS Excel. It provides great synergy
between these 2 great applications by combining their complementing functionalities
and providing deeper integration between these applications. One can view
Specific Native Data dict advantages (http://www.erpjewels.com/nativedatadict.pdf) in Baan terminology for more information.

For more information on this tool, one can visit our website www.erpjewels.com.

We have pleasure in making our wonderful Excel based reporting tool open source . We have
created ERPJewels open source project (https://sourceforge.net/projects/baan2excel/) for the same on 24th Dec 2007 . Even download page our website has more information about the same.

Names of contributors and volunteers have not been added as per request from many of them.

This tool bring life to dead (static raw) data in Baan tables and Baan reports by adding the much-needed interactivity and formatting/multi-dimensional capabilities of MS Excel and adding further data transformation power to it. This tool lets developer specify analyses fields (including user defined ) & calculated items (data polishing ) , map it to pivot table / plain excel , describe the field , add selection criteria (including user defined based on Baan 4GL code ) for any Baan table / Baan report .

As Baan 4GL functions and reporting flow remains same across versions , the tool can be of use in other Baan versions also besides Baan IV .

Connector to Baan report has worked across all SPs. It's tested upto SP21. There had been some changes in repgen6.1 about 2-3 times during it's journey from SP8 to SP21. All the times it has worked by making some fixes to the code within couple of hours so that it works on older SP as well as the new SP .

Enjoy this new year gift from Jewelex.
