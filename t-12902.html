There have been a lot of changes in apihandler and standard program. Not all of them, between service packs, are positive solutions and might have been reversed (or changed) within the next version of the standard programs.

In order to have a solid solution for users, there should be some skind of test procedure, after installing new standard programs all afs functionality should be checked. This can be done using test and production server, but installed solutions should be the same. Normally this should not be that much of a problem but with the ongoing development of afs test and production should be equal.
