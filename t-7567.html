Wim Kerkhoff
14th November 2002, 18:04
CES 2.5 won't be out until the end of the year at least. Another important thing to know is that CES 2.5 won't be compatible with CE9.0. That means that you can't upgrade to CE9/CES2.5/CR9 in 3 months. Since the API/SDK for Crystal Enterprise releases isn't finalized until it's out the door, it takes a while for CES to play catch up.

I've purged CE8.5 from our server and I'm installing CE8.0. The Integrated Reporting actually works, but CE8.0 is extremly buggy. All sorts of javascript errors, pageserver/cacheserver errors, etc.

The problem I had with CES2.0/CE8.5, is that CES would not install correctly. It did not set up the BaanAdmin and BaanViewer accounts in CE and didn't set up the default baan passwords. This meant that when you print from Baan, an error will appear in the IBR logs and the baan device logs about a bad or corrupt username/password.

I've no idea why printing to CE would would work, but not directly through CE to a printer. Perhaps give Crystal or Baan support a call. About the error your getting... if you check ttaad3520m000 (I think) it should give you the full error message. Also, see if the XML file is making is getting uploaded to the Crystal server.

Hope that helps... if it sounds incoherent, that's because I haven't had coffee yet.
