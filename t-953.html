First, we had a third party put the signature on a hardware DIMM installed in our Laserjet printer. The signature is stored as a custom font that prints only when triggered by a certain sequence of characters in that font. So basically the sequence of characters embedded in some PCL commands. The third party should provide you with this information.

Second, we use a third party software called Flexform for our check printing. Flexform provides a utility to let you insert custom PCL code into its own PCL output.

However, I'm pretty sure you can accomplish a similar thing with Baan alone. You will need to create a custom printinf file, based on whatever normal printinf file you'd use for the printer. In the printinf file, create a line for a user defined font (e.g. pfont1 for User Defined font #1) with the corresponding PCL sequence to use the custom font. Create a Baan device to use the new printinf driver you created.

Then go into your Baan report for check printing. Either create a layout section setting the font to the User Defined font # that you picked or add a form field in an existing section and set the font for that field to User Defined #. This will be where you want the signature to print. Then edit the layout, or program the form field, to output the special sequence of characters. Note that I have never tried anything like this in Baan myself.

I think the biggest problem with using Baan only would be precise placement of the signature. With a software like Flexform, you can specify exactly where you want that signature to appear.

We use a printer cartiridge font for our check signature. I believe we ordered it directly from HP to fit our HP III printer.

There are two sides to this. Baan and client.

Baan - define label for signature, modify check report

I created a label called sig that has a value of S. Within the check report, insert the label field where the signature should be on the check. I then gave that label a font value of 'User Font 12'.

Client - install font on PC, run BaaN check session - PREVIEW - define font to Baan User Font.

Once we had the cartridge installed in the printer, on a client that will be generating the checks, I installed the signature font (provided by HP on a floppy) via the printer properties for the HP III.
Then, (make sure that printer is the default) run the session(s) to generate the check - clicking PREVIEW at the printer dialog box. When the report is displayed on your screen in preview mode, double click the "S" that printed where the signature should go.
This should bring you into the fonts window. Scroll down until you find the signature font and select it.

You have now defined this font to print whenever the User Font 12 is chosen. However, a signature will only print when the report is printed from this client to the printer with the font cartridge installed.

As far as printing your company logo, the best I have found so far is to take a bitmap of our logo and make it into a font using
High-Logic's Font Creator Program (http://www.high-logic.com/fcp.html)
Then following the same principals as above, I created a label and called that label in my Baan sessions. The only downfall is that I need to load and define the new font on all of the PC's that will generate the reports.

Hope this makes some sort of sense.
