Configuring a network and its path, baan creates a folder "command" in the path. A file "comm.yes" (it may be empty) in there shows baan that the network is active. You can manually create this file or have it created by your EDI translator (eg. using ftp).

To initiate the communication, baan creates a "command.fil" file in the same folder. Then your translator can transfer messages. The translator has to delete the file afterwards. This is the signal for baan to look for incomming messages and to process them.
