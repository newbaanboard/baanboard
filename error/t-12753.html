I'm not sure what you are trying to do, but you cannot create a new item master record with just an item and a description. The session won't let you, exchange won't either. There is a whole range of fields on the item master record that need to be filled in with valid codes, and there are records in other tables related to the item that need to be created along with the item master record as well.
It is possible to load item data into Baan using exchange, but it is far from easy or straightforward.
Updating existing items is usually more straightforward and is a lot less likely to generate an error 606; if it does then it would be caused by data in your import file, for example an item group that does not exist in Baan.
You can find explanations for the error codes in the baan tools help section (I think under programming information, so go to the application development / customisation menu and look at help.

gfasbender
5th September 2003, 20:50
You are getting the 12 skipped records because you have options Overwrite and/or Update Records set in session "Maintain Table Relations (Import)". The skipped records contain Items that don't exist in table tiitm001. You can't update something that doesn't exist!

Turn on the option Add Record in the same session, then you'll receive an error record for the 12 skipped ones (since you don't provide all of the required referenced data need in the table)
