I also had this error but I don't remember the exact circumstances. You will have to trace the communication between Baan, the Crystal Baan Server (the part of Crystal Enterprise for Baan that does the communtication with Baan, interprets the XML File and upload it on the server), and Crystal Enterprise itself. Baan Support and Crystal support did help me,
probably you can find something in the knowledge bases of either Baan or Seagate. (By the way, the communication between the two support organizations seemed to work amazingly well).

I know that you can set some parameter within the Windows registry of the crystal Baan server that will make the crystal Baan server produce some (really helpful) log files.

In my case the problem ended up as related to our german version of Crystal, because the adminstrators group in crystal enterprise default installation is "Administratoren" in german, while the Baan server had "Administrators" hardcoded. (Or something like that...)

There should also be a document to be found at www.support.baan.com (in the user documentation section) on the Baan-Crystal-Connection. You should be able to download it from there.

If that doesn't help you, don't hesitate to log a case at baan support. The Baan-Seagate connection is still new, and not yet and on an install-and-it-will-work-always-without-any-trouble
level. (Nevertheless, I believe it to be the
future of Baan reporting)
