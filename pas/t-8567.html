Hi all,

Currently we are evaluating whether we should continue the Page Mode feature in Baan. It sometimes lead to confusion and could even cause errors.
An end-user can switch it on by using the session menu: Tools --> Options --> Message mode.

For a (technical) explanation what page mode is:
in page-mode, there is no checking done on the server when changing or tabbing through the fields, so the client accepts any value. Only when a command (like "save record") is started, synchronization takes place and entered values are checked.
Another aspect of page mode is, that all fields that are normally disabled by some logic will now by default be enabled.

As a consequence, ERP sessions can behave unexpectedly or differently than when run in interactive mode. This pagemode behaviour often confuses end-users.

I would therefor like to have your opinion on this topic. Do you within your organisation use this page mode feature?

Thanks in advance,
Reint Jan
