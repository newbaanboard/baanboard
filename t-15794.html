Salve Spartacus,

api.mode variable is inherrently a Sesssion thing and should be used exclusively in UI-Script and hence is it also predefined by Baan for such scripts.

A library - and for that matter a function - should have no knowledge of the UI-stuff. It should generally be concidered a bad programming practice when any of the following is used in a function or a library: display(), choice.again(), choice, update.status or as in this case api.mode.
