We are currently working on using Baan Finance. A few API sessions have already been created. I am sure more will be created eventually.

Not sure if this helps, but for our Inventory Stores session we previously had 2-3 people doing receipts and several doing the stores. We had a 3 day store window for parts and was not meeting that time frame. By automating these sessions we have reduced the number doing receipts to 1 fulltime and 1 part time help. Not sure about the stores piece, but from looking at the charts they presented last week we doubled the number of receipts will reducing the staff. We are also now meeting if not exceeding the 3 day store window. There has not been a reduction in force because of this, the staff is being used in other inventory functions.

We have processed thousands of inventory transfers and adjustments. This prevented the user from having to enter the transaction,locations and other info thousands of times. This is probably the oldest API session.

In all cases I can justify a savings on just keeping the user from having to manually enter data on 2/3 baan sessions when one session with only mandatory input is used(or using file input). It may only be minutes per transaction, but after hundreds of transactions this produces some noteworthy savings.
