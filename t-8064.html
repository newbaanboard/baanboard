A lot of the parameter tables are of no importance in the archive company as no transactions are ever done in it. It is purely a storage company but there are some you will need to view data. For example The authority table for inventory adjustments is not copied but if you want to be able to look up a transcation in the inventory adjustment session you need to be authorised. I think Bom units and base units needed to be entered. Financial periods for the financial archive sessions. languages, currencies and you have to have logical link on you companies table to the archive company. The are several parameter sessions in the live company that need to be set. The group company sessions needs the archive company entered and number of years to store archive data(you can only archive finalised transactions before that period). Purchase and sales parameter needs to be set for how to archive sales/purchase orders either by order or order line also some settings for statistics.
