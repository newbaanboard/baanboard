Hallo zusammen,ich möchte aus dem Produktkonfigurator auf den Kundenstamm (Tabelle tccom010) auf ein bestimmtes Feld zugreifen und die Information dieses Feldes im Produkt-konfigurator weiterverarbeiten.
Das Problem hierbei ist, das der zu suchende Kundensatz nicht in einer Merkmals-option mit seiner Kundennr. abgelegt werden soll.
Statt dessen soll über die jeweilige Variantennr. auf die Tabelle tipcf500 zugegriffen werden. Dort ist die Kundennr. hinterlegt. Danach kann auf die Tabelle tccom010 zugegriffen werden, wo der jeweilige Kundensatz gefunden werden kann (zur jeweiligen Variante).
Nun zu meiner Frage: Wie kann ich die jeweilige aktuelle Variantennr. in eine Variable einbinden, um so den oben beschriebenen Weg zu gehen?

Geht dieses überhaupt?

Hi Matthias,das geht nur, wenn in den pcf-Parametern "compiler" aktiviert ist.

Dann geht das etwa so (im Beschränkungstext): ! table ttipcf500
! import("tipcf500.cpva",tipcf500.cpva)
! select tipcf500.cuno from tipcf500
! where tipcf500._index1 = {:tipcf500.cpva}
! selectdo
! endselect

! select ... from tccom010 where ...
(das !-zeichen muss vor jede Zeile gesetzt werden, dahinter
kann beliebiger Baan-Code stehen, der normale pcf-Code ist natürlich ohne "!" zu schreiben)
der import-befehl funktioniert auf die Tabellen, die in den Sessions, über

die der Konfigurator als Subsession gestartet wurde, schon gelesen oder angelegt wurden, z.B. Variante, evtl. VK-Angebot oder VK-Auftrag usw.
