Hallo, in SSA LN müssen die Benutzer in Tools->User Management->General User Data->User Data angelegt werden. Dann in Tools->Database Management->RDBMS Administration->Links between SSA ERP Users and Database users eine Verknüpfung von dem SSA-User zum Oracle-User (meist verwendet man auf den Datanbank nur einen einzigen User) herstellen. Anschließend in beiden Programmen über "Specific" (Zusatzoptionen) das Convert to Runtime aufrufen. Das sollte es im großen und ganzen sein.

Hoffe das hilft etwas,

Hallo,

hast Du die User auch im Licence Manager angelegt? Wenn der Licence Manager auf dem Server läuft, geschieht das über das Programm /usr/baan/shared/bin/BclmCmd.

Ich habe das Script das wir verwenden angehängt. Wichtig ist/sind die Product IDs.

./BclmCmd -addusr $1 -p 10056 -heavy
./BclmCmd -addusr $1 -p 10146 -heavy
./BclmCmd -addusr $1 -p 10896 -heavy
./BclmCmd -addusr $1 -p 10996 -heavy

Die Groß/Kleinschreibung ist wichtig. Das Script liegt zur Zeit in dem Verzeichnis (daher ./...)

Wenn der Licence Manager auf einem Windows Rechner läuft, können die User über das Snap In oder über die Kommandozeile (dann auch mit BclmCmd) angelegt werden.

Rollen etc. werden zum ersten Test nicht benötigt.
