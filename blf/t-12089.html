Das ist ein Bug.
Solution 108685 sollte das Problem beheben:
Solution description
VERSION(S):

3.1_b, B40_b, B40_c
SITUATION IDENTIFIED IN:

"Generate Periodic Maintenance Planning" (tssma1224m000)SITUATION DESCRIPTION:
When the period for an already generated maintenance planning(with status "Not
advised") is changed and then again maintenance planning is generated for that
year, then the system cannot deal with the updated maintenance planning: extra -

non wanted- plannings will be generated.
SOLUTION DESCRIPTION:
While generating the maintenance planning in fixed periods for an installation
for a specified year, a check will be done whether already a planning exists
for that installation in that particular year.
If the planning is already present, then no planning will be generated again
for that specified year.
In order to regenerate a planning for a year, the old planning should be
deleted first by the user.
This is also explained in the helptext of the session under title "Periodic
Maintenance".
