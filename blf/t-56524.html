kurz zu meiner Person. Ich muss in unserem Unternehmen nach dem Weggang unseres Baan Administrators das System betreuen. Wir haben Baan5c mit einem SQL 2005 Standard SP3 auf einem Windows Server 2003 Enterprise 32Bit. Der Server hat 4 CPU und 16GB Ram. Die Datenbank ist mittlerweile 120GB groß und unserer Baan System extrem langsam. Ich habe schon einiges versucht, um mehr Performance zu erreichen. Ich gehe davon aus, das es schlicht am SQl Server liegt. Die Datenbank liegt auf einer FC Lun mit richtig IO's (20000 auf NetApp Filer) und transferraten von 200MB/sec. Transactionfiles und Database liegen auch auf unterschiedlichen Volumes. Schalter /PAE /3GB sind in der Boot.ini gesetzt und die AWE Option im SQl auf 1 gesetzt. Trotzdem ist das System extrem schwerfällig (Abrechung usw.) und wird nach empfinden der User immer langsamer.
Hat vielleicht jemand einige Tipps, die man probieren könnte. Ich muss dazu sagen, das ich mit Baan nur wenige Erfahrungen und Kenntnisse habe und mein Einsatzbereich eigentlich Citrix und Storagesystem sind.
