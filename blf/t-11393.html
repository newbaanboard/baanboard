In der Session Berichtsstruktur verwalten ctcat0112m000 hat man die Möglichkeit,
eine individuelle Bezeichnung anstelle der Bezeichnung laut Kontenplan einzutragen.
Diese Bezeichnung wird aber im Bericht nicht verwendet, sondern immer nur die Bezeichnung aus dem Kontenplan tfgld008.

Es handelt sich um einen Standardfehler in der Version c3 (ob auch in anderen Versionen, ist mir leider nicht bekannt), Abhilfe:Status changed to Waiting
bitte installieren und testen Sie folgende solution
VERSION(S):

B40_c SITUATION IDENTIFIED IN:

"Print Hierarchical Results Based On Report Structure" (ctcat3411m000) SITUATION DESCRIPTION:
When the ledger accounts are printed, the description of the ledger account was
taken only from the chart of accounts.If the description was changed in the

Report Structure, the new description was not printed. SOLUTION DESCRIPTION:
The description of the ledger account is taken from the Report Structure. If
the description is left blank in the report structure, then the corresponding

description from the Chart of Account is printed.
This solves defect number 74342. PATH NAME IN DIAL-UP DIRECTORY/WEB SITE:
Below are supported VRCs based on the Baan standard derivation structures. It
is necessary to download and patch all links contained under the VRC in the
order listed. The solutions may contain standard and localized components which

require multiple ftp links. If your Current VRC is:
B40L_c_cnt0 or B40L_c1_cnt0 or B40L_c2_cnt0 or B40L_c3_cnt0:

ftp://www.support.baan.com/updates/B40c5cnt/17592cnt.dmp COMMENTS:
These dumps must be imported into BAAN with session "Import Patch
(ttiex1284m000)". Please read solution 10389 for instructions on how to use

this session. Note for B40c users: because of the implementation of Euro triangulation
functionality, this solution may be incompatible with B40c, B40c1, B40c2 or
B40c3 software. If you did not yet download and install solution 15254, do it

now to avoid compatibility problems.
