Das ist eine Windowseinstellung in der Registry, die dieses Verhalten verursacht. Das Problem existiert AFAIK schon seit Win2000 oder XP. Infor beschreibt in der FAQ zum Worktop, mit welchem Registryschlüssel das geändert wird.

Worktop FAQ: Solution 22940671

"2. When launching Baan sessions, they appear in back of the Worktop application.
This is a common problem on Windows 2000. It is not restricted to Worktop but many applications will show this behavior. The reason is that Windows 2000 by default does not allow applications to "steal the focus". To solve this, there are three options:

- Download this file: /download/updates/port/Worktop/sol120792/stealingFocus.reg

and run it on the machine where the problem appears. It will automatically set the HKEY_CURRENT_USER\ControlPanel\Desktop\ForegroundLockTimeout attribute to 0 in the systems registry.

- Manually edit the systems registry: Open regedit, locate the "ForegroundLockTimeout" key in HKEY_CURRENT_USER\Control Panel\Desktop and set its value to 0.

- Download the TweakUI application from here: /download/updates/port/Worktop/sol120792/tweakui.zip

Extract the zip file. After extraction, go to that folder and find the "tweakui.inf" file. Right click on this file and select the "Install" option from the context menu. After installation, double click the "Tweak UI" icon located in the Control Panel and go to the tab called "General". There is a checkbox with the label "Prevent applications from stealing focus". Uncheck this box."

Die aktuellen BaanIV-Clients findet man in Solution 22881483. Ab Version 97 ist der Client offiziell für Win7 freigegeben.

"4) jan 25 2010 Note regarding Windows7 support
As of 8.6a version 4 and 6.1c.11 version 97 BW is certified for windows 7."

Ich setze hier die Version 101 unter Worktop 2.5.11 mit der beschriebenen Registry-Änderung ein.
