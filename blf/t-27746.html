Hallo,

ich hab da mal ein Paar Fragen zur Installation von Solutions.
Derzeit wird es bei uns in der Firma so gehandhabt, daß nach der Installation eines Patches folgende Schritte ausgeführt werden:

ttadv6200m000 Kopieren der Sprach-abhängigen Software-komponenten(kroatisch)
ttadv5214m000 Create RDD (Forms & Menus) über alle Pakete
ttadv5210m000 Create RDD zuerst Domains danach Tables
(Bei Problemen beim Cr. RDD ein "Convert to RDD" (ttadv5215m000) über die gesamte Range.)

Das ganze gefolgt von einem Neustart des Servers.

Meine Fragen nun:

Ist es notwendig für jeden Patch den Server neu zu starten, langt evtl. ein Restart des Shared Memory Dienstes oder ist evtl. nichts davon notwendig?

Muss das Create RDD über alle Pakete gehen oder langt es nur die Pakete anzugeben die durch den Patch betroffen sind?

Gibt es vielleich irgendwo ne gute Beschreibung zum Einspielen von Solutions?

Hallo,
die Solutions können ohne Neustart eingespielt werden.
Allerdings kommt es halt immer drauf an was die Solution für Komponenten mitbringt.
Wenn Tabellendefinitionen/Domains geändert werden muss man die Sachen zuvor aus den Shared Memory rausnehmen damit die Änderungen auch auf die Festplatte richtig geschrieben werden können (Sonst blockiert der Shared Memory die Dateien).
Ein Neustart von Worktop sollte dann genügen um dann die veränderten Data Dictionary-Daten zu bekommen.
