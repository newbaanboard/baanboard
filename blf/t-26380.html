This is Baan-message tgwmss0028, and is called on from the 3GL script tgwmsclient, which is a workflow script of client interface. In this script the program checks on a variable dont.read - I unfortunately am not sure of the means of this variable; I guess to do with data model configuration. However, when this variable is set to false, then the system looks for user/employee data in table tgbrg840. When not found, this message appears.

So I guess you have some setting (as I said I guess something to do with the enterprise model configurations etc, things I know extremely little about...) which require data to be present in tgbrg840. Ich vermute auch, dass sich diese Fehlermeldung auf Mitarbeiter im Projektmodell (tgbrg8140s000) bezieht. Beim Kopieren eines Mitarbeiters, der seine Berechtigungen / Menüs aus dem DEM bezogen hat wird der Teil der im DEM hinterlegt ist wohl nicht mit kopiert.

Lösung:
Über Projektmodelle verwalten (tgbrg4100m000) dann auf 'Mitarbeiter' klicken und den User zum Projektmodell hinzufügen.
