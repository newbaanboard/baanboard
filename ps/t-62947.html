Elements:
Build the technical Project Structure - (Like a Project BOM) in Projects like
Heavy Machinery, Turbines, Power Plants, Ships etc.)

Activity:
Entity to build the Scheduling Structure (Like Activities in a Project Scheduling Tool. Supports the Planning in a Project

Link in two ways:
1. Direct - Via Element Relations (Same Plan for all Budget Lines)
2. Indirect - Via Bottom up Budget Lines (Connected Data, with different Activities, if required)
Depends how detailed you need to plan. I prefere to use 1. detailed.

You dont need Elements in simlpe projects (pave streets, or so), but if you have a complex project, use them both (Elements) for the design integrity of your technical structure! Activities for the scheduling - in this case dont play with activity types - take Work Packages only - this is good enough.

Note: Currently (FP8) its only possible to link Service Orders with Activity Driven Projects! This need to be changed by Development!

Hi,

The invoicing can happen on the basis of TP project as well. In that case, you need to copy the activites of the project as the cluster breakdown structure. Then on the service order use the project on the header and select the activity of the project in the item field while creating the reference actitvity lines for the service order. Then the cost of the material, labour and others can be fallen on the project through the activity while invoicing the whole service order or invoicing just the single reference activity.
