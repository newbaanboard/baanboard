Ajay

I'm not sure why your default engineer is not being picked up from the configuration details; I seem to remember it is Preferred Engineer 1 that gets the work...

If you have not installed Automation Server you will not be able to run DSRP, but you can allocate engineers directly to Service Orders using tssoc2105m000. If this is done, it will be possible to release the orders with the button on the Service Order session. Obviously this would not be good if you are processing lots or orders daily. Another approach would be to use the Service Scheduler available as an extra product. You then have visual drag and drop allocation of work to engineers, a scheduling assistant to make sure all the constraints are balanced, and can transfer the orders for execution directly from there, including issuing them to engineers with E-Service Remote if you use it.
