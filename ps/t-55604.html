You can incorporate into the maintenance plan the concept of production hours as an additional indicator for reaching a maintenance threshold. It is not completely automatically generating the service orders, for that Infor needs to finish this item or a customization needs to be built. But without that still in combination with some business procedures you can come a long way.

Here is what out-of-the-box available is and how you can set it up:


Define measurement type: e.g. Operating Hours as dependent variable / Time as independent variable/ Increasing
Associate this measurement type with Service Item/Serialized Item.
Define Reference Activity (Inspection) with measurement type.
Define Maintenance Concept and set the time and (norm) usage interval.
Define Inspection Template for Reference Activity and Service Item (create one Operating Hours inspection and some condition measurements with cyclic increasing/decreasing trend with define follow up activity)
Generate Predicted Activities and check the increase in usage norm for each activity.
Generate Planned Activities and check the increase in usage norm for each individual planned activity
Transfer to Service Order (NOTE: this is an area for Roadmap improvement to further automate the comparison of actual value on serialized item with nom value on (non executed) planned activities)
Release Service Order and check Inspections
Fill usage inspection and check the value modification on serialized item (this also works in PDA)
Fill condition inspections and check the outside limits report (should highlight outside norm inspections)
Create Service Order for defined follow up activity from Specific menu in Inspections History. (NOTE: this is also an area for Roadmap Improvement to make a batch session for this step)



Let me know how you proceed and/or if you need more help getting it to work.
