Hi Carlos

Yes, a checklist can be used as a mini-script for the engineer to follow for detail within an activity/assignment. The checklist code and details are set up in service master data. You can choose whether the printed list has a tick box for execution, or one or a few lines of space for comments. There is even some limited functionality for determining the sequence of checks.

To invoke a checklist, you specify it at reference activity level. Then any service order actitity derived from that reference activity will result in the checklist being printed as part of the job sheet pack.

Once produced, it doesn't do anything, but if the required record keeping includes engineer feedback it can be useful.
