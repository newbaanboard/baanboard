Kiran Gholap
29th March 2008, 06:02
1. First you need to have a cluster defined in session tsbsc1100m000. Please ensure a proper usage class is attached to the cluster.
2. Attach the appropriate serial numbers to cluster as cluster lines
3. Define maintenance concept in session tsspc1120m000 for the items in cluster lines and usage class used
4. Generate maintenance plan from session tsspc2200m000
5. Planned activities would be generated in session tsspc2100m000
6. Alternatively one can manually maintain planned activities without following step 3 and 4
7. Then transfer maintenance plan to service orders from session tsspc2220m000. Please note that status of the planned activities as mentioned in step 5 and 6 should be released for this step to work.

You can also check if the status start date at the cluster line is set correctly, the planning will start off from the date the serialized item has been 'installed' under the cluster. Note that if the delivery time/installation time has been specified at the serialized item then it would have the higher preference.
