Hi,

Session tppss6550m000 (Display Material Transactions) is looking into table tppss650 and when you delete a project as part of your archiving procedure there should be no problems unless you still have transactions open in other applications like Distribution and/or Manufacturing.
So make sure all transactions in those applications linked to Project TP have been fully processed.

I agree it is strange that only projects without records in tppss650 get deleted without problems and archiving works in any case.

Make sure that you have all commitments fully transacted (if applicable).

I advise you to log a case with BAAN Support regarding this issue because they can dial in and see what is going on.
