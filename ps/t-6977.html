Hi Riazsayeed,

You can register hours in BAAN Project and have them posted to tssma3110m000.
For this you do NOT have to set the parameter "Baan Project Implemented" to 'Yes' in tppdm0100m000.

I will give you a high level overview of how to set things up in Project and suggest you try yourself after having read this message. In case you get stuck please reply to my post and I will help you further.

Sessions to execute in tp:

1. Create an Hourly Wage Type in tpppc2120m000 with the "Hour Type" set to 'Normal Hours'
2. Generate cost control periods and hours control periods in tpppc0210m000 for the years desired. To be safe for possible future use of BAAN Project it is better to also generate the Cost Control Periods with a Period Type of "Month". The Hours Control Periods are per week. You can view the periods after generating them in tpppc2121m000 (Hours) and tpppc0110m000 (Cost Control)
3. Create an employer in tphrs0120m000
4. Define a Wage Component by Employer in tphrs0110m000. Make sure the Type is "Hours" and the Default Allocation Code is "SMA (Services). You also have the possibility to check the field Hours Check and this will allow the system to let you know an employee worked more than 40 hours per week when you are booking his hours

Once this is set up you can start registering hours in tphrs1100m000. As soon as you open the session and click the new group button you will be able to zoom to the Hours Control Periods you generated earlier. By selecting a week the system will default in the dates and you will exactly know in which week and for what day you are booking the hours.

One important note in this scenario: the unit for booking hours must have a Physical Quantity of "Time". This unit is defined in tcmcs0101m000 (Units). The base unit for "Time" must be set to a second in tcmcs0100m000 so that when you define the conversion between a second and an hour the value will be 1:3600. This is done in tiitm0120m000.

Regards,

Edwin
