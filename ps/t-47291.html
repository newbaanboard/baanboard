Hi Sergio,

Once the status of a service order in LN is changed to "Released" there is no way to move it back to "Free". This is how the standard functionality works.
Perhaps it is better to use multiple service order activities so when the status of a service order is "Released" you can create a new activity with the status "Free" and then change this to "Released" when needed.
This will give you more flexibility with a service order.

Hi Sergio,

I assume you mean the user unintentionally changed the status of the service order from "Free" to "Released". In that case he/she should create a second service order activity. The status of this activity is "Free" and the status of the service order is "Released".
As soon as the user is ready the status of this second activity can be changed to "Released" and all costs and sales amounts can be booked against this one.
At the moment the service order has been completed the user can change the status of the service order, the first activity and the second activity to "Costed".

In LN you cannot change the status of the service order back from "Released" to "Free". Unless you have made a customization and I would not even think about going that route.
