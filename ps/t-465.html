Hi Sridhar,

It looks like you are entering the provisional amount against an element as Sundry Costs and not via an extension.
Then when booking actual costs you need some sort of a settlement with the provisional amount.
I do not believe this will work in standard BAAN Project.

In order to be using Provisional Amounts properly you should be linking Extensions to the project. Also entering the Provisional Amount at that time in session tpptc0110m000 (Extensions).

Another thing you need to have is a predefined code in the field "Sundry Cost Code for Provisional Amounts" in session tppdm6100m000 (Maintain Projects).

Then when you book your actual costs the difference between these costs and the provisional amount entered in tpptc0110m000 (Extensions) will be captured against the predefined code just mentioned as Sundry Costs.
Billing will be based on actual costs.
Hope this helps. If not please send me an e- mail with more information.
