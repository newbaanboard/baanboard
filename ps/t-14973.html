Hi Gokove,

In case you want to issue stock for a service order and out of a warehouse it is important the service order, service order activity and the cost line have the right status.
As long as these are either "Free" or "Planned" nothing will happen. The minimum status has to be "Released". If this requirement has been met you can use the "Issue Stock" button.

If you want to issue stock on a service order automatically then the right way to do this is via Generate Outbound in Warehousing. In case you defined a warehouse order type with activities to just generate outbound then this done in Warehousing and not in Service.
You can issue stock on a service order and out of a warehouse without going through Warehousing. Then you should not use Delivery Type "From Warehouse". This has to be "From Warehouse in Car" for example.

My question to you is: What do you want to do? Go through Warehousing and run Generate Outbound (automated process)? or use the "Issue Stock" feature in Service (manual process)?


Regards,

Edwin

Dear Gokove,

There are several Delivery Types available with each different consequences,
In case you use "From Warehouse in Car" the system will take inventory out of a warehouse without going through the process in Warehousing by using Generate Outbound.
If you use "From Service Inventory" then you will issue stock without using a warehouse at all. This is more meant for srews, nuts and bolts.

I do not have BAAN V Service in front of me so I am not able to explain the rest of the options here. If you want to know more about the other Delivery Types I would like to refer you to the help text or the BAAN V Service training manual.

Regards,
