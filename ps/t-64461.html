Elements are used to set up a - technical Structure (WBS) of Projects
Activities are used to create an - executional Structure (Plan) of Project

Elements and Activities can be linked in two ways:
- via the Element Relations - for a global planning of Budgets Lines of Elements
- via Element Budget Lines - for individual planning of Lines

IMPORTANT: In more complex Projects (Ships, Plants, Machinery etc.) the project should be structured with Elements and supported with Activities for scheduling (coming from MS- Project or other Tools like Open Plan)

The Element Structure can be changed if required if there is Design Changes or add ones i.e. if Scope of Project changes. If there is scheduling changes this can be done via changes in the link of activities to Budget Lines or Element. However, a change in planning does not lead to change in the Element (Build) Structure!)

This is a very important item that should be considered very carefully. If there is the plan, to set up complex projects with an Activity Structure only, this will lead for sure to conflicts between integrity of the Build (Design) structure and the plan or schedule of the project.

However, the original item was - Difference of Element and Activity. The Element is the "Lego - Block" that allows the set-up of the Scope (independent from planning aspects) The Activity is the Equivalent to the Planning Activity with Early / Late Start - Early / Late Finish - POC and the link with Resources.
