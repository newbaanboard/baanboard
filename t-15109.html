(1) No extra DLL's have to be written - just the session which does all of the controlling of the back ground Baan session. Please note that not all sessions are API friendly and sometimes owning source code helps. Some work arounds can be difficult.

(2) Yes it does use the edit checks from the session. Please note that not always do all events get called. This can result in some problems, but again usually you can work around this.

(3) No not that I am aware of. I run the session manually and just keep track of what it does.

(4) Not sure about Baan getting unstable, but you can run into problems with patches. Also if you put bad data then it can cause problems. Example you want to transfer 4 in a bin to bin and you send 6. It works but you caused the problem.

These sound bad but I use API function servers all the time.
