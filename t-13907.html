I am not sure what you are trying to accomplish by calling this function from VB. In this post (http://www.baanboard.com/baanboard/showthread.php?s=&threadid=13847) it looks like they are trying to do the same thing. If this is the RMC version of extending source then why do you need to call this function from VB? Typically what you do is create a Baan session with your modifications using RMC's product, QKey or the ~Vamsi solution. Then you would call this program using the STPAPI commands. Then when you did a insert the session itself would run the function that is mentioned. If there was error then it would be returned to the calling program - in this case VB. This function looks like it was intended to be imbedded in the program itself and not called independently. It looks kind of like Qkey, where it would call the parent object of the session.

One way to find the DLL is to look at the script itself. I believe Qkey includes a pragma used DLL pointing to its DLL. Maybe RMC does the same type thing. If it does not do that see if it attaches another library to the script.

Good Luck!
