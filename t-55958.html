I would say you need to contact infor and update to the latest STPAPI patches. I know at one point in time I had several API sessions which aborted with errors like this - usually after a certain number of transactions, but it would vary. Currently at this point we no longer receive the error - so I assume thru tools patches, portingset updates and API patches the problem was resolved. I recommend checking with infor.
