We had the same problem at first. Through Windows Event Viewer we found out that our broker and studio where not properly registered. We figured something had gone wrong when we typed in the registration code send by Baan using command line UI. With the help on licence manager documentation we found out that some xml -files with product codes where missing. Files where named <product code>.xml if I remember correctly.

Second installation we did on our customer server went ok, however.

Hi,
We had a similar problem with the broker service not starting. There is a file bclm.dll under C:\Program Files\Baan\Baan OpenWorld Broker or rather whichever is the directory for Broker. U just need to rename this file bclm.old or something and hopefully it should resolve the problem.

U can also check out the Baan Case # 940536 for more details.

Hope this helps.
