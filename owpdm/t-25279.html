If you start tmboaserver in a GUI mode you can set the logfilename as a parameter to log the errors. This file is created in the $BSE/log directory. You cannot specify a path.

tmboaserver [ -usermode <security mode> ] [ -l <logfilename> ]

If you are starting the buscomponents through a command line, you can again define the logfile to log the errors.


otmboaserver [-f <componentfile> | -busc <component_id> [-d]] [-l <logfile> [-s]]

Not sure which version of OpenWorld you are using but information on these options can be found in the document Installation and Configuration Guide for Adapter for Baan IV and Baan ERP

Hope it helps

Here is a sample commmand

bshell.sh -server otmboaserver -f /home/myuser/owstart/buscomp.200integ -d -l 200ow.log -s &

and the buscomp.200integ file can have a list of one or multiple buscomponents, such as


cn=bm_hal_200_20010,o=mycompany.com
cn=bm_hal_200_20013,o=mycompany.com

We are using OpenWorld 2.4 and I have tested this command and it works fine on our system.
