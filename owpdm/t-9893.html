Answer to Q # 1:

Integration between ESales and Baan (BaanIV or BaanERP) has following aspects.

1. If you want to use all the pricing/discount logic 'Live' from Baan backend, then you will need openworld to achieve that. This, by the way, is available in the standard integration between Esales and Baan.

2. If you do not want to use the pricing/discount logic 'Live' from baan and would rather use pricing and discount mechanism in ESales, then you will not need openworld solution.

3. The order data from ESales to Baan and back to Esales is done through EDI messages, and hence no OpenWorld components are involved.

4. The static data from Baan (Customer, Address, Products, Units, Currencies, etc.) are transfered from Baan using combination of Exchange Schemes and SQL Server DTS Packages.

5. If you want to use Dashboards/Lookups to see 'Live' read-only data from Baan (e.g. inventory, order status, etc.), then again you will need OpenWorld components to achieve that.

Answer to Q # 2:
Yes, you do need OW for Webtop.
