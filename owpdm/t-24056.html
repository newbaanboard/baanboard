shaboo
19th May 2005, 15:26
Is this bus.properties file is from your Baan Server or from the client machine which will connect to Baan using Excel?

With use of file based directory system, you have to create the same buscomponent on client side as well as on the Baan server (or copy the file over to other location, ldap.xml in your case). That way your Baan server will know which buscomponent are out there and then you will be able to start one using tmboaserver session.

Looking at your posts, it appears that when you started tmobaserver, you didn't find any buscomponent to start. So I would suggest to copy your ldap.xml file over to your Baan server. Make sure that your bus.properties file on Baan server is also using the same ldap.xml and then try to run tmboaserver session and see if you can find the buscomponent that you wanted to start.

Hope it helped.
