Hi,

Finally I could find the way to build the repository file using BOI Studio. The steps are as follows:

1. Go to project properties in property explorer. Specify repository file name in Domain Name. Set "Generate Domain" to yes.

2. Include all the QCD and DCD files of the BOIs to be generated. While including the QCD/DCD files, the only care to be taken is that all the QCD files should have unique ManagerDef i.e. the value for "Type" should be unique accross QCDs. For DCDs, "ID" of all ElementDef should be unique. Since, BOI studio generates java class files for each ElementDef, the duplicate name will have the overwritten class files i.e. any subsequent ElementDef with same name will overwrite the prior generated CLASS file and at the end we will have the class file for the last ElementDef, hence all the ElementDef in DCDs should have unique ID.

3. When the project is build, it will generate the class, source and jar files. The repository file will be available in the CLASS folder and will also be available with in the JAR file. This repository can be used by BOI Test Tool or XML Connector Utility.
