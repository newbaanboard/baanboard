Try adding the machine name of your client machine (XP Desktop) under the Adapter Client configuration and then setting the default BackEnd there.

Bus-components are single-thread in nature, meaning if 5 calls are made at a time, it will process them one by one. Then the other thing is to how many tmboaserver sessions you should run in a single bshell as it will mean running multiple session at a time which can also result in slower performance. So you have to think about number of bshells you can run plus how you will split your buscomponents among them. If the load is not too heavy you can survive with one bshell + one tmboaserver and one buscomponent for all users.

I have not installted the integration myself but here is what I think you need to do.


Go to Central Configuration Console and under Adapter Client add your machine name. This is the name of the machine from where you will be launching MS Project to connect to LN.


Once you have added the machine name as described in step above, you will find an option of selecting default bus component (see snapshot below). From the drop down select the bus-component you want your integration to use.


It should be that simple.
