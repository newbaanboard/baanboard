Here's how it was done in Webtop 2.0:

Create a HTTP Post request to the following URL:

http://<webserver>/<virtualdir>/servlet/com.baan.bfe.webtop.servlet.login.LoginServlet

The following arguments are compulsory for this URL:

Argument name Argument value
webtop.method silent.login
webtop.user user name
webtop.password password for user name

When the Webtop is configured to use BaanAuthentication then the following argument must also be passed:

baan.environment baan environment

The following are the other parameters. If they have no value, then they should not be included in the POST.

startupbackend Erp
startupform sessioncode (example: tccom0101m000)
startupkey1 search field (must be indexed) (example: tccom001.emno)
startupvalue1 search value for search field 1
startupkeyn Idem
startupvaluen Idem
startupwhere where clause (example: tccom001.nama like &quot;A.*&quot;)

Hope this helps.
