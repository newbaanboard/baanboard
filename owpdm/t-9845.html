Dashboard in E-Sales are basically meant for looking up data from the backend (e.g. BaanIV, BaanERP). Hence the dashboards are used only in a 'read-only' mode to see data such as sales orders, item inventory, Invoices etc. The standard dashboards are delivered with the specific iPacks (Integration Packs). For example, standard Dashboards for ESales - BaanIV will be available in one iPack and th dashboards for ESales - BaanERP in another. For dashboards, you will need to deploy OpenWorld components as well.

This was a very general overview. Feel free to ask any specific questions that you might have.

Dashboards/Lookups can be for internal employee as well as external customers. For example there are two different lookups for Sales Orders, One for a customer and one for an employee. Hence dashboards can be used by internal users as well as externals.

There is a standard lookup of Sales Invoices that provide overview of all invoices for each customer. It might not have all the info you want but atleast it is there.

We have two/three totally customized lookups that were specifically developed for us. And are extensively used by our customers.

hello Eva,
we have Baan E-sales with baan IVc4.
I think I understand what you are asking for.
You have a Customer Dashboard which includes several SSA's attached to it, such as:
Sales orders, Sales order lines, and sales order details.
You only want your customer to see the first one, Sales Orders.
Using this one SSA, you want them to be able to drill down to see sales order lines, etc. but not have the separate Sales order line SSA included on the main Dashboard itself.
If this is the case, you can go to to e-Manager, Dashboards,
Dashboard contents, and click on the SSAs you don't want, to delete them from the list...also make sure the linking is set up between Sales order and sales order lines SSA's.
