- a BW is active on the client

Now im facing similar problem. I did the following

1) Configured a bus component calld ERP_LN and into it , a connection point in this way

bidisocket:<ERP_LN server ip:8000>

2) Configured adapter client with only defaults value as:

Client Bus component : ERP_LN
Default Backend: ERP_LN

3)Copied OfficeIntegration.exe from client PC to exe (ERP Server running under Redhat Linux Enterprise server)

4)Started tmboaserver session from ERP Server and between it , started ERP_LN bus component. It started succesfully.

5) From Microsoft Word 2003 tried to create new ERP template an then received the same error

"Invalid port in URI bidisocket <ERP Server IP:8000>"

Infor Integration 6.2

pcolombo
26th November 2009, 18:03
Finally i could figured out what went wrong .The problem was the Adapter Client's configuration. I put the same value for both "Client BusComponent" and "Default Backend" fields and used the Bus Component that i configured to connect to LN . When i did this i received the "Invalid port in bidi socket" message. Leaving "Client BusComponent" field empty the problem was solved and now i can see LN from Word and Excel.

The "Client BusComponent" is for the client, usualy the client uses the first free port but can be defined to use a specified port. the host name or IP address must be that of the client.

I have only used the "Client BusComponent" when the client machine is using VPN and has 2 IP addresses and the open world client picks the wrong one.
