Yes, sequential dumps are platform independent. There are version differences in the dump formats, but the same porting set should accept the same dump format.

However, table ttadv335 isn't dependent on porting set; it contains information from the standard program, solution installs, customizations, etc... Unless the two systems you are dealing with are at exactly the same service pack level with all of the same patches installed and the same report customizations, there is a good chance that the tables are identical. That being said, if you have a downed system, it may be your best bet (unless you have a backup?).

ttadv335 has contains report layout related values in ascii characters.If only a particular report is corrupt and can not be opened , then export through ttiex4100m000 and import through ttiex1284m000 is better option . If a particular VRC has problem, u can export reports in vrc through ttiex1280m000 and import it through ttiex1285m000 .

Already compiled reports will not have any problems with corruption . If compile any report , take a backup of the same .
