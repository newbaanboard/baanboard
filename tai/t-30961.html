Unless you are using Unicode version, normally most international customers in asia are using two approaches.

1. Force the users in Asian countries to use English interface and input English data. This works if they don't have much domestic sales in Asian countries.

2. Installing a separate server for each Asian country. For example, one server in Chinese language for China office, one server in Japanese language for Japan office. For such case, single logistics and multiple finance does not work as it is not possible to share data among different application server with different locales.

Our suggestion is to go Unicode installation if you forecast a good business growth in Asian countries.
