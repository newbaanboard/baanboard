Pat,

Are you implementing these changes for SOX ? We have implemented the password aging (ttstppwdaging) and it doesnt ask me for any RDBMS change password, it checks only the OS/Unix Password aging. We have atleast 20-25 database per environment and the passwords are all set with no expiration date and the end users do not know it. Our DBAs logins do not have the password expiration too, but the Prod-Control/Deployment team have passwords expiration set to 60-90 days and the DBAs have provided them a Perl/SQL script to globally changed their passwords for all databases. Unless you have any users/developers directly working in the SQL plus you can get away from password expiring in the database.

You can look up the options for the binary - $BSE/bin/badmin6.2. Make sure you have the latest version, I reported a bug and it was fixed in the latest porting sets. If you are on Unix, also refer to the Standard include <pwd.h> to see the password structures and methods.
