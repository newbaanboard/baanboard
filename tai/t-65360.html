Archive does that mean you can't use the application?
Within the application you can use the so called exchange module which can be used to export tables to ascii files.

Without the application it's probably a bit more complex. You are telling us you have .dat files containing the data. Does that mean you did use c-isam to store the data? Because the baan own isam version did store the data in .bdt files if I remember correctly.

If it's indeed c-isam than you can use the c-isam libraries to develop some own code to read te tables.
If it's bisam and you can't use the application than you can filter the lines from the file. If I remember correctly deleted lines where ended with a null character and the others with \n.
