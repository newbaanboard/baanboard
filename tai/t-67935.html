Are you wanting to move only parameter tables or company 000 tables? That is not the same thing to me.

For parameter tables like tdilc000 - I would export and import those myself. These tables actually reside in the schema with the company. Like tdilc000123 - where 123 represents an actual company number. When we refresh a test/dev/patch company we only refresh from the production company tables - not company 000. So all of the parameter tables come along at the same time. I have never run tcmcs0295m000 so I am thinking it updates tables like tdilc000 - ilc parameters, and not really company 000 tables.

For company 000 I would just export and import (well the oracle dba's would) company 000 tables. Of course keep in mind if you are importing into an existing company 000 you would lose any development work that might be in progress. Once company 000 was built then you could make additional changes in the tools tables to point everything to the correct spot.
