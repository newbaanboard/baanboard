When I am using the Webtop on different customer's systems, I am experiencing veeeery slow response in the following use-cases:

* Maximizing a webtop window
* De-Maximizing it
* Using the Sort-By functionality, when the session has dynamic index switching enabled.
* This also seems to happen (but I'm not quite sure here) when closing modal windows (those that are "blocking" the one from which they were started)

In those cases, the Webtop seems to freeze for one or minutes, also the java console is frozen then. After that time, I get the response. I convinced the java console to give me some thread traces during that period, and I am finding some strange looking locks (Java, not database, just to make that clear) in Java, Baan and Jidesoft (a third party library infor is using for the webtop) classes. For me it looks like a deadlock situation.

(To replicate, just open the item master in Webtop and click on the maximize button)

This happens with Webtop 8.5 and Java 1.6.21 as well as Java 1.6.19 (on the client!!). I've heard from other consultants that they have been experiencing similar issues

Ok, so this post has three reasons: First, of course, does someone has a remedy?

Next, I want to get some feedback if others are experiencing the same problem (so please reply here if you have the same issue), because then I will open an incident and probably link it to this thread. Multithreaded deadlock issues are a nightmare for software vendors to debug and resolve, but if enough customers are concerned, we might convince Infor to look into it.

Finally, I've heard rumors that this might be an issue only with newer Java versions ON THE CLIENT. So, please, if anyone is running Webtop and another Java version (client side) then the ones I've mentioned above: Please can you post your version (Client java and Webtop), and if you encounter that problem or not?

Thanks a lot
