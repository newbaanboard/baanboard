Hi there,

I think you misunderstood a bit the concept of windows printing in baan. In principle windows printing means, that the GUI (not the server) prints the report on a windows shared printer.

So lets assume you have a windows print server called X with a shared printer called y, then you siomply enter as device \\x\Y.

If the printer in question is your default printer on the desktop you are working, then you can also simply leave the device empty and bwprint will use this.

rlp and other server related protocolls are not in use here (in case of printer type "windows" in baan). But often it is better/more stable to define the printer on your unix box as a remote or network printer and then define a printer in Baan of type "direct". But this is an alltogether different story.
