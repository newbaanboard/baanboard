On the form for the session "Delete DDC Message Guarantee Keys [tuddc2240m000]", there is a field "Delete Records with Retries greater than or equal to" at the bottom with a default value of 999999.

This has to be changed to 0 (zero) before running the session [in addition to of course, filling in the exact function server for which deletion is to be affected!].

The default of 999999 is probably to prevent 'click-happiness'!

Hope this helps.
