You would think that is the way it would work -- and it would be tremendously helpful if it did work that way. Unfortunately, filling those fields won't help you export the components you really want to export. It only specifies which customization parts (not the other objects associated with those parts) to export.

There was another post not too long ago about this as well. I think there is widespread agreement that this would be useful. Unfortunately, for now, you have to pick and choose the things to export using the ranges for each object type.
