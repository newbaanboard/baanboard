Error 512 is corrupt data dictionary. This can usually be solved by running Create to runtime / reconfigure tables.... Was this not part of the installation instructions of the patch? If you ran this already, you should check the error logs to see if anything did not work, especially related to those tables. This will usually give you a clue.

If you are on NT, you may also be having a shared memory issue. After installing the dump and running create runtime, you should shut down the system and reboot - or find another method to restart that shared memory (I find it hard to do on most NT systems without rebooting, but I know there are ways).

If there are still errors, another good session to try is Convert/Recover Tables (ttaad4228m000). If this still does not work, you can get more control and use bdb.pre and bdb.post to manually reconfigure the table...search the forum for help on those commands.

If possible, backup first.

Sometimes Baan has a mind of its own and would not run the reconfig even if you put on a new table definition for these tables. What I've done when I get a 512 error is to create runtime dd for Domains, table, reconfig . Since there is a paramater table involved you will need to make sure users are not logged since there may be a problem dropping the table. To ensure that the reconfig invokes the bdbreconfig you need to make a small insignificant change or if your really want to ensure that it invokes the bdbreconfig, add in index in which is unique -> convert to runtime dd with teh above options -> remove that index -> convert to runtime with the above options.

But before doing any of these. MAKE A BACKUP!!!! as well as do a sequential dump of the two tables!!!! . I would hate to see your next post coming from the job seekers section.

If you're not familiar with the Memory manager, a reboot of the server always works but here goes:

1. under start go into the Baan NT Manager.
2. right click on shared memory service
3. this brings up shared memory mgr
4. do a show loaded
5. click on the table object you want to remove and click on remove object
6. close it and Load again

Like I mentioned, rebooting also works.
