Experience: I just finished a fp3 -> fp5 and a fp3 -> fp6 update.

Basicly this is possible by putting the fp5 dict runtime files in your fp6 installation and run a create rdd. However don't forget that you have to run the upgrade engine as well to reflect table changes! Before running the upgrade engine you should validate all your data.

Hello Ian,

I would:

1) Clone the old machine to the new machine
2) Update Portingset and Enterprise Server to 8.6 on the new machine
3) Install FP6 in a new package combination with new update VCs on the new machine
4) Change Package Combnation by company from FP5 to FP6
5) Run the Data Upgrade Engine

It depends on the amount of data if you can do it in one run or if you have to split the migration to the new hardware and the Feature Pack Update in two steps.
