Hi Peter,

the way you describe it, it should be pretty clear, that this is not a problem caused by the baan application, but caused on the Tcp/IP level.

Do all Bann users face this problem at the same time? If so, then the Baan server might have an issue.

If only a single user at the time has the problem, then most likely a network device (switch) on the way is causing the problem.


If you connect a PC and the baan server directly with a hib (not switch), do you still have the problem?
