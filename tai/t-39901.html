We want to upgrade to a new hardware system and are in the process of installing the OS and Baan software on a new and more powerful server (the DB is installed on a separate server). During the installation of the Windows OS, we assigned a different server name than the one on the production server but we would like to reuse the old one once the data migration has been completed and we are ready to do the switch. That way our users won't have to do any changes to their client configuration. When we go live over a weekend, the hardware upgrade should be totally transparent to the users. Will we run into any problems by changing the server name of the new server? Are there any other steps required on the Baan installation?
sukesh75
31st July 2007, 08:43
It would be best to change the server name before doing the validation process or for that matter, before installing baan. Since you said your database is on another server, make sure that there are no connectivity issues between the servers after the name change.

How are you going to move the data? Sequential dumps? Are you going to restore the 000 company?
The problematic files surrounding a name change are License6.1 & tabledef6.1. All in all its highly advisable to keep baan installation to the end after you copied all the files to the new server and changed the server name and do an offline installation..

sukesh75
2nd August 2007, 08:40
As a long term solution, its better if you start using DNS. Lets say tomorrow you want to keep a standby server for your live production server and since they would be replicating data and talking to each other they would be having two different names. IF DNS was implemented, then When the Live Server goes down, all you need to do is to change the DNS Alias and the users would be re-directed.

So its not bad an idea to use DNS at all!!
