I came to realize that the Help Menu Tree shown reflected the first time I connected to Webtop from the client on each of the machines. Although I would have thought that the Webtop Cache would have refreshed, it never has.

I physically went into the users home directory on the client and deleted the Webtop Cache. The next time I reconnected, the help tree had all branches that I had installed and is now working.

I would appear that if you were to upload new help files that this might be a necessary step. I found nothing in the knowlege base that reported the problem.

I reported that bug and they fixed it in the latest Webtop release:

INCIDENT DETAILS
Details: We noticed that, in worktop you change the startup company and menu it changes immediately when you log in again. But when you sign into webtop, we can see the new startup company but the menu did not seem to have changed.

We cleared the cookies and objects/cache in the IE and yet the same old menu appears. We tried to create a new profile too and that didnt fix it.

Thanks
Attachments:

When a webtop user logs off, menu info is saved under C:\Documents and Settings\<USER Name>\WebtopMenuCache directory. Please remove those files if a webtop user does not want to keep the last menu tree when the users logged out previoulsy.

INCIDENT RESOLUTION
MR Reference: MR Priority:
Resolution: This issue was fixed on the latest WebTop version, Infor_Webtop/8.5_buildPL0108
