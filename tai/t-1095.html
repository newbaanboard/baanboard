I have setup the BI several times and it runs securely over the Internet with 5.0c. I assume that is what you are trying to accomplish.

All of your servers should have static IP's to start with. Get a public Internet address and use NAT (Network Address Translation) to protect the BIserver. The best way is to make a dmz off your firewall and place the server in the dmz. This will securely protect your server and your LAN.

The method that you access whether it is DSL or whatever will just affect the speed. BI Server is not lightning fast because it uses Java, but not that bad. You can use SSL to encrypt the connection made and it can easily run securely over the Internet for more than one customer, but port 443 has to open.

It has been a while since I did the setup, but would be glad to check it out if you would like any additional info.

I have to ask why you think you need to use DHCP and a static IP presents a problem?
