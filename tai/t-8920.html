Try to be more specific on your question, that one it too general.
I suggest also to get some tools training to understand how thinks work in Baan.

Baan has an application layer data dictionary (APDD) and a runtime layer data dictionary (RTDD).
Several parts of the APDD must be translated into corresponding RTDD ones (forms are dumped, reports are compiled, program scripts/libraries are compiled, etc.) because it's faster for Baan to load a file than to look into the database (also reduces the DB overhead).

As an example of what 'objects' are in use by a session, run ttstpsessinfo for a session to see the session object (compiled script), libraries (compiled libs), forms (dumps) and reports (compiled reports).
