The fact that a so called R file exists probably means a previous reconfiguration failed.
Are you sure that you have all your data in the table?
Perhaps something is wrong with the indexes?

With bdbreconfig -N ttdpu041170+ you can redo the reconfiguration however that will load the data again. If it has changed in the mean time you can't do that.
Check with your database software if all indexes exist. If not that you can create them with gcommand.
