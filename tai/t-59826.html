11th January 2011, 11:11
With 11.2, the Oracle (client/database) versions of 64bits stopped including the 32bits libraries (lib32). This means that any 32bits application that uses it does not work. If you installed Oracle64 bits and a portingset 32bits, you need to install an Oracle client 32bits to be used by the former to connect to the second.
If you are on version 11.1, you should have a lib32 under your Oracle home, you need to ensure that this lib32 comes first on your path for Baan, so that it picks the 32bits libraries first before the 64bits (on UNIXes this is easy, on Windows I think you need to define a different path for the Oracle account, that needs 64bits first, and another path with lib32 first for the Baan account).
