I would not assume a change in runtime to effect the data itself, so the values are still there, just not seen properly maybe. If there's no other solution you need to roll back your changes (restore) or change the DD of the affected tables back.
Also check this within the finance threads, I don't expect a change in DD to change the data itself...

And a convert does not run for all tables, it uses flags of changed tables to know what needs to be converted, that would mean that you a several tables with convert flags on. In the future, make sure that you select specific tables only.
