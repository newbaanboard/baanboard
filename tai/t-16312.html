You're welcome.

What I forgot to mention is that, although you don't need a developers license, I believe the user who creates the queries will have to be set up as a developer in Baan; at the very least they should be linked to an appropriate package combination since the reports generated are created within that structure. So if the current package combination for the user is set to for example to td ilc you'll find the report object in the directory structure for the location control objects and not, as you might expect, somewhere in tt. Not that this matters much under normal circumstances.
