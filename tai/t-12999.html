As for the possibility to send messages in B4c4, it does work!

I'm not sure about the technical concept behind Worktop, but I assume it uses the bshcmd6.1 (or exe) to actually send the message. If this is true, it might explain the problem. Sending messages with bshcmd is at best a pain. Most of the times, it only successfully sends the message if the sender is root or bsp on UNIX, and administrator or baan on the Windows platform. On UNIX this can be fixed by setting a sticky bit and a suid on the bshcmd6.1 executable. On Windows, I'm not sure whether it is possible to always start a program as a superuser...
