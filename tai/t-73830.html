16th July 2021, 16:29
Just so everyone knows the solution for this was a mix of things. So since new portingset solved one problem, but created this whole new monster. What we did was this to solve all the issues:

(1) Created a new BSE directory
(2) Upgraded the portingset here (users use the normal bse for our production so this does not impact them)
(3) Put a new bshell6.2 into the new BSE (essentially the bshell6.2 is from a soon to be released portingset, but not for 4c4). I was told 9.3j is last for 4c4 - so be really sure you want to update to it.
(4) Modify the Billable Cost Report job to run under the new BSE. Essentially all users and all other jobs run under 9.3e portingset and only this BCR job runs under the newly created BSE and 9.3j portingset.

Not an elegant or recommended solution, but it gets to where we can finish closing out rates on years. At that point we can truly move baan into an archive of some sort.
