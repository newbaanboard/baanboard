Hi Mick,

Checkout this thread for discussion on AIX:

http://www.baanboard.com/baanboard/showthread.php?threadid=10392&highlight=aix+sco

Basically SCO have revoked the authority of IBM to use AIX. So the future for AIX seems slightly uncertain.

Computer Weekly has lots of articles on this:

http://www.computerweekly.co.uk/articles/article.asp?liArticleID=122719&liArticleTypeID=20&liCategoryID=2&liChannelID=28&liFlavourID=1&sSearch=&nPage=1

64bit Oracle favours very large databases. Ones which have say a shared memory size greater than 2Gb. Baan doesn't have the necessity for this power, or a large shared memory. 64bit Oracle will double the memory footprint, the binaries are are larger, and the subsequent throughtput through memory is thus slower. Yes more powerful, but not faster.

Also note that Baan have not released and are not going to release a 64bit Porting Set - so the Baan Application will always be processing in 32bit anyway.

Though the : 64bit OS - 64bit Oracle - 32bit Porting Set, is a very popular combination amongst our members with no reported problems.
