Sorry it took so long to get the thread validated. Moderators have to approve the first post. As to the issue itself - I moved it here because I think it will get more responses hopefully.

We have 4c4 on UNIX platforms and never had an issue like this. But I have seen some pretty weird things between the clients and the servers. All of our jobs run on the app server and then once the app kicks off for the sessions in the job they connect to the database server. I have seen network issues between clients (on windows) to the app server, and have seen some network issues between app server and database server. I do not know if that could be your case or not. In our case the DBA's and Network teams could track down the issues between app server and database server. It was a lot harder with the clients and the app server. This one could not be duplicated on demand. The first thing I would do is log a case with infor.
