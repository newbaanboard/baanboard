Many BO solution do have 2 separate installation parts.
In KB 22945150 this is described as :
Solutions to be installed often contain 2 parts: one part is the dump for the LN application packages, the other part is the dump for the bo version 2.1 package. In order to guarantee a correct installation, it is required to configure PMC so that both parts are always installed together.
Start the Update VRC's (ttpmc2140m000) session to link the Update VRC for the bo package (most often the Update VRC is 2.1U_b6_bo) with the Update VRC for your LN Feature Pack (e.g. B61U_a_stnd) through a VRC Combination. If no VRC Combination exists, then first start the VRC Combinations (ttpmc2150m000) session and create one VRC Combination (e.g. PROD, for your production environment). In the Update VRC's (ttpmc2140m000) session link the Update VRC for the bo package to a VRC Combination and link the Update VRC for your LN Feature Pack to the same VRC Combination. By using the VRC Combination, PMC will automatically install related solutions for the bo package and the LN application packages to avoid fatal errors because of missing solutions in one of the packages.

Please verify this setup. If not there then you need to verify whether the mentioned solution has been installed for both parts.
