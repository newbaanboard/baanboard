hello.

idea 1: Use DEM. In ERP LN there is session Application by Component (tgbrg5115m000) and it's possible to generate Sub-apps, i.e. all subsessions. All components are collected in 1 list, so steps to do depend on situation.

Idea 2: use "Print where used session" session (ttadv2410m000) whith following conditions:
package " " - "zz"
session " " - "zzzzzzzzz"
your PC
specify forms and script of session you need (remove check boxes for menus and functions).
leave only 1 check box in the bottom (print session where used).

1-st way is preferable, but I do not remember how it works in Baan4 (maybe someone else can help), 2-nd one has some limitation - it will not catch situations like <filed - menu - session> (You may use session "print where use menu" to find out list of menus called from your session and to bypass this problem) and it works only for 1 level whereas generation of Sub-apps looks for ALL subsessions (sub, sub-sub,.....).

If session is not very complex way offerd by lakoon could be more reasonable.
