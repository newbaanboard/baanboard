From what i know ttiex1285m000 would let you import your dumps into any existing update vrcs but i dont think it would let you rename it in the same session.
1) If the so called customized session tdcus9450m000 doesnt exist in the vrc you are trying to import into, then you could go ahead and import it and then rename it to tdacp9450m000 after the importing is over.
2) If tdcus9450m000 does exist and if its not used anymore, you could choose the "overwrite" option in ttiex1285m000 session and then once importing is done, do the renaming...
3) Last but not least...if tdcus9450m000 exists and you want to keep it and have this one as a new session then i recommend you create another vrc combination and import this dump into that. Once imported you could create a copy of this session from your new vrc to the one that you desire with the name tdacp9450m000..

Hope its clear..

>>can I cange the (9403m000 to 9450m000)?
After importing, you could. Make sure the module is present in your vrc on the new server(ref. kaustubhs post), then create a copy of the session (tdcus)9403m000 with new name (tdacp)9450m000.

Bottomline: You can rename an object to any name you want(by creating a copy of it) after you import and not before or while importing..
