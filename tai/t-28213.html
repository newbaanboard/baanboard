Refer to the tools solution 214729
English Trying to save grid customizations results in a 'Error saving customizations. Contact your system administrator' mshown in the statusbar.
TOOLS VERSION(S):
7.6_a2, 7.6_a3

SITUATION IDENTIFIED IN:
"Customize Grid" (ttadv9200)

SITUATION DESCRIPTION:
Trigger customize grid option in a session. An option 'Apply to zoom session as well' exists and has been checked. Press OK, an message 'Error saving customizations. Contact your system administrator' will be shown in the statusbar.

This happens when the session was not 'grid customized' and OK is pressed without having made changes. And can happen when the session was already 'grid customized' in a LN version before fp2 and OK is pressed

SOLUTION DESCRIPTION:
When the option 'Apply to zoom session as well' has been checked, the grid customizations will be saved for the zoom variant of the session as well. It could happen that it was tried to save a field customization for a field, that in this LN version actually didn't exist anymore.

Before the save for the zoom variant is done, the fields will be checked on existance in the LN version.
