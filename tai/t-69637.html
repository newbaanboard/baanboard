I have been tasked to plan and implement a upgraded of a BaanIV server to a new server and upgraded SQL server.

The current server is BaanIVC SP12, with portingset 6.1c.17 Level 1 drivers on SQL 2008 R2. The plan is a new Windows Server 2016 and SQL 2016 with level 2 drivers. Also Baan Licensing to SLM licensing.

If I am reading the technical notes correctly, the upgraded to 8.7b.01 is fairly straight forward, directly from 6.1c.17. Infor has suggested that we upgrade 1 portingset at time and install all patches and make sure all requirements are taken care of before moving to the next portingset.


The next issue is the level 1 to level 2 driver upgrade. Has anyone done this going to a new server?

Should the upgrade be done on the current server, full export/import of data, or should we install the level 2 drivers on the new server, move Baan and then export all the baan (company 000 and production companies) from the level 1 server and then just import on the new level 2 server?

Thank you for any suggestions or experiences regarding this,

Hi mike,
I recently upgraded my linux servers from rh4 to rh7.
My experiences have not been smooth as I wished.

First of all, I planed to install oracle 12, and it will be required rh7. That was not the problem. From the baan side, to run rh7 I need at least, ps 8.xx.
This is because with rh7 all is 64 bits native and also the oracle db is on 64 bits.

Since I don´t use SLM, also I needed to to that conversion (I was on ps 6.1c.10).

When I converted my test environment to SLM and then I can move to ps 6.1c.17), I experienced some drop user´s without explanation, and some cryptic messages.

I have invested around 2 months to get the test environment up and runing, but I´m afraid to said that it will not a succesful migration from 6.x to 8.x

I have finished to install rh6.5 and oracle 11 in order to install ps 6.1c.19 on the baan side to get all up and running. This is due to, in ps 6.x the baan binaries are 32 bits, and in oracle 11 you still have 32/64 client versions.

Hope this helps.

It's do-able.

Just make sure that your old baan environment is able to use SLM.
On the new server install the rdbms software, install latest portingset. Install SLM, get the license key.
Than export all tables, import them on the new server.
Rename the new BSE, copy the old one over and merge them (not overwriting the portingset!)
