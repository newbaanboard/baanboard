I am trying to upgrade my LN6.1 SP2 to FP3 and the installation fails when creating runtime dd. I have found out the reason for this failure is that when installing LN6 wants to install pmc solution 226039 only the problem is solution 226039 introduces a new field in a table which requires a new domain, but this domain is not in the solution. The new domain is delivered with solution 226545 which is NOT being installed. Hence the complete installation of FP3 fails. By the way 226545 is also superseded by 227699 which doesn't get installed either and 226039 is also superseded by 227832 and 227331 but leave that.
Is there any way I can force solutions to be installed along with the solutions for FP3.

You can also stage 226545 in the StagingArea in the same directory where the FP3 dump is (B61_a3). Note that you must not put the .tar.gz file there, but the unzipped solution dump. If you use the StagingWizard to process the 226545.tar.gz file, this will be handled automatically for you.
