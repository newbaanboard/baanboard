Did you submit the new Activation Key ?

Refer to - SLM 2.1 Installation and Configuration Guide

To submit a new activation key
The following command lines show the syntax for submitting a new activation
key to the license server.
BclmCmd actkey <activation key>
BclmCmd actfile <key file>
The key is directly entered as text (argument <activation key>) or is read
from an ASCII text file (argument <key file>).
Next example shows you how to activate a License file:
C:\>BclmCmd -actkey 47388f86fcd99221e6e064398103702f
Submitting activation-key to license-server.
Activation-key : 47388f86fcd99221e6e064398103702f
Key length : 32
OK
Note
When the key is submitted and the requested license file becomes active, the
License file has changed: the change and add elements become active
and the delete elements are removed. In addition, the version number is
incremented by one.

The easy way for make changes in validation is using slm client. Start slm.msc or bclm.msc.

Another possibility is BclmCmd. Try command BclmCmd -U, you will see options.
First you have to get licence file (BclmCmd -getlic ...), edit this file and after that save it (BclmCmd -newlic ...)
