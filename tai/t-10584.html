First of all, bshcmd is a standard Baan tool. Since you're on 5.0b, the "script" is probably called bshcmd6.2. I put script in quotes, since it actually is a compiled program! Nice utility as well. :)

On 4c4, the command bshcmd6.1 -p <bshell pid> gave me the following (partial):
PID PPID PGRP SESSION TICKS FLAGS S CMP MEM
1 0 1 ottstppollmes 185 00000060 R 100 215936
2 0 2 ottstpstdlib 111738 00020000 S 000 321976
3 0 3 ottdskmbrowse 882803 00000060 B 100 268280
4 0 4 ottdskbrowser 20824 00000060 B 000 414104
6 3 6 ttaad4100 97810 00000070 B 000 428032
I assume you're asking about the column S. Well, according to the help (from the Option Dialog, activate the Shell, and press ?) S is the process queue status, with the following possible values:
S - Sleeping
R - Running
B - Blocking
Z - Zombie

Hope this helps!

Blocking queue
When a process is waiting for input, the scheduler moves it from the running queue to the blocking queue. So, the running queue contains only processes that can be run. When a blocked process receives an event, it is moved back from the blocking queue to the running queue.
