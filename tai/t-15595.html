Hi there,

bad news for you: You are running Baan on windows and in this case shared memory will not always get invalidated, when it should.

Stop and start Shared memory and it should be fine!
If this does not help, reboot your system.

I know this sounds like a bad joke of a unix guru, but I once spent half a day on such a problem after installing a patch wich contained a new table.

Reboot solved the problem. Sad but true!

Regards

Markus

P.S.: I assume your user is in the right package combination :-)
