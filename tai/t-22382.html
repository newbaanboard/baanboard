Segmented domains are evil. Stay away from them...

The only place that I know of where they are used is in item data
to allow the project / item code to be joined by the same domain.

so, in theory, you could start your implementation with ITEM = [Project 1][Car] and later on decide that you want to have ITEM = [Car][Project 1]
