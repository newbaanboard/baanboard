To check if your text tables are logically linked, run this from Unix shell:

grep "txt" $BSE/lib/compnr6.1

If you see output similar to the following then tables are linked (in this case, all txt tables in companies 031, 032, 033 are linked to company 030).

tttxt:030:031
tttxt:030:032
tttxt:030:033

If that is the case, then you might consider renaming the "refint6.1" command so repairing reference counters fails during the reconfigure so that you can run it manually afterwards on just the necessary tables using the -s option (single parent table).

In Baan IV this option isn't used when called by bdbreconfig6.1 (fixed in Baan V) so it will look at current tables parent as well as child tables.

Review the log.reconfig6.1 which will show list of tables just reconfigured. Check their "dd" files for REFC entries with reference type "r".

For each of these run
$BSE/bin/refint6.1 -p $pkgcmb -N $tablename -r -s -C $compnbr

I've used this to repair reference counters only where necessary thereby bypassing unnecessary repair of text tables.

For more info or if questions, please send an e-mail.

Regards,
Brendan

These are the steps:

1. Rename $BSE/bin/refint6.1
2. Run Create Run Time DD
3. Track the tables to reconfigure
4. Run Reorganize Tables (ttaad4225m000) with the options Reference Integrity, Repair Reference Counter and Repair Reference Counter of Parent only checked.
Use your own discretion and confirm from Baan support
