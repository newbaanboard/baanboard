Hello,

you have to import the source code dump in the B61_a VRC of LN and then apply the feature packs/solutions. PMC "looks" if there is a source code in the base VRC and if there is, it applies the (changed) script in the update VRC.
The license itself has nothing to do with if the source code is loaded or not.
You need it to able to compile the standard sources.

It seems Infor changed the source-installation with FP8 indeed. I've checked the latest info in FP8 (KB 1155124 "Latest Information for Infor LN 10.2.0) and stumbled upon this:

"26 September 2012: From this version onwards, sources are only installed if SLM contains the correct Source License. So before installing or updating an environment, SLM has to be configured with such a license, otherwise no sources appear in the environment. This is in contrary to the behavior in previous releases, where this license was necessary to start the development activities only. The current Source License has SLM Product ID 7026."

We're planing an update from FP6, but haven't done it yet.
