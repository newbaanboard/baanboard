The "Package Combination" is used twice. Once as an attribute of the company number, where it defines the data model that company uses. Once in the user data, where it defines which software version a user uses. Of course, both has to match somehow (the user cannot use software that operates on an other data model than that of the company he's working in). That's why this check is done, when you just switch the company.

To remedy that issue, you will have to switch the package combination of your user ("Maintain user data" in Tools). In the same session you are also able to enter your start company, which would be the test company. Change it there, do a convert to runtime. Logoff, logon, and you should be able to test.

Last note: Don't confuse the package combination of your user (which you set in the tools user data) with the "current package version". The latter defines in what (package) version you are developing, but that can be different from the (package combination) version your *using*.
