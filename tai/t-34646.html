Restoring dict and application is a lousy form of rollback. If you use backupTrestore for rollback, than you also need to restore company 000, as it will contain some parts of the patch.

Actually what you should have done is to use simply PMC to uninstall the patch.

But a normal application patch should not create trhis problem at all. If all is done right, than a CRDD should work in 99% of all cases and if not the 1% of errors can be successfully debugged.

I assume all of this is happening on a test system and not on the life?

Regards

Markus

Maybe a stupid question, but...

Have you tried to restart your server after patch installation? Your profile says that you are on Windows. On Windows servers, table definitions in shared memory often do not refresh after creating RDD. In such case you will see the changes in Maintain Tables, but not in ttaad4100/4500. All you have to is a server reboot.
