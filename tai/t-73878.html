ttstpsplopen can write to files and the filename can be set in the "Output File" box.

There you can input something like:
UNIQUE_FILE_$$.prn

This will send your data to a unique file with the name UNIQUE_FILE_{somenumber}.prn like in UNIQUE_FILE_4562.prn

I assume {somenumber} is the PID of the report process or something alike.

Now I got a request to use the DATE instead of the somenumber.Alt solution is to run a cmd shell script right after the job with this content:
`@move UNIQUE_FILE_*.prn DATENAME_FILE_%DATE:/=%
This works for me because my %DATE% var contains / as field separator and it will be removed in the %date% variable.
