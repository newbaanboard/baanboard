Question: when printing a report to notepad, how to keep the file name as defined in the device path

For example, if I define a file name as report.txt in the device path, I expect the file opened by notepad will be report.txt, but each time the system will create a temporary file tmpxxxx.txt on the client PC. How to force it to use same file name in the device path?
mark_h
19th March 2013, 15:29
One way is you can do this is to write your own device driver. A sample can be found in the code and utilities forum (http://www.baanboard.com/baanboard/showthread.php?t=513). Basically your device driver will capture the report, convert it, then use the server2client to download the report, then launch notepad.

PS - the server2client is where you would use the spool.fileout command to keep the file name for the download.

Mark_h, thanks!

I finally notice the setting on session "Maintain Device" today. For the "argument" field, if we put "ASCII:notepad", the file name created on client PC will be tmpxxxxxx.txt; if we put "ASCII:notepad %s", the file name will be the one you put in the "path" field.
