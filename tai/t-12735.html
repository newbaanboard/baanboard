Hi gous

If you set up the scroll mouse for universal scrolling, or, in other words, to simulate the cursor keys, they will work in BaanIV on one condition: the cursor has to be positioned near or on the scrollbar. 29th August 2003, 11:30
It all depends on how the mouse driver "translates" the scroll function. In XP there seems to have been added a separate "scroll" function that is not implemented in the BW client. If you use a mouse driver that translates scrollwheel movements into the new scroll function, it will not work in Baan; if it translates the scroll wheel movement into "filling the keyboard buffer with 'arrow key' keystrokes". Using a mouse driver from floppy usually will do because they use the old way...

Hello all,

after I got a new USB mouse (Saitek Touch Force Optical) I've had some problems with the scrolling wheel function in Baan IV (W2K with BW068). This is what I found out:

Saitek advise to use the built-in windows driver. Sorry, no scrolling here.

Logitech Mouseware does support the scrolling wheel (positively checked with Version 9.24). But mouseware detects the mouse's manufacturer and refuses to work with 'foreign' mice. Sorry again.

Microsoft Intellipoint 5.0 works with non-microsoft mice but does not support scrolling in Baan IV. Aaaaargh!

Microsoft Intellipoint 4.1, finally, does not refuse foreign mice and supports scrolling in the menu browser as well as in multi-occurrence forms. Hooray! But why did they remove this feature in the newer version ???
