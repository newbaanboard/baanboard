On XP you can run a netstat -o that shows the PID associated with each connection, but this flag does not exist on previous versions.
If from the username you know from where it is connecting, then you can use the w2k resource kit pulist.exe (read/download from Microsoft here (http://www.microsoft.com/windows2000/techinfo/reskit/tools/existing/pulist-o.asp).

I think I saw some utilities (non-free like this here (http://www.astrontech.pl/html/body_wint_tcpwiev.html) ?) were you can see the IP address associated with a PID (other possible option is to see the IP's thru a firewall installed on the server...).
