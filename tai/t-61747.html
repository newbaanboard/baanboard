Hi Chris,

Check the package combination in which the users are working. Is the table part of this package combination?
It looks like the table is being used e.g. in some program but is not part of the package combination.
You could check in %BSE%\dict\dd<package combination\dtudll if you can find the dict file for that table.
If it's part of the package combination and you can't find it in the runtime than a create runtime could probably solve it.
