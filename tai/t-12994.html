Moreover, if I try to proceed top-to-bottom in order of installed dependencies or vice versa, some of the dependent solutions do NOT change to "To Uninstall" status....they continue to remain "Installed" even after running "Check to Uninstall" on them.

You need to think on the reverse of the installation. If A was dependent on B, if you uninstall A, B will be left. If you uninstall B, then A will be selected to be uninstalled.

Now, the process of installation may not be so straightforward as that, if a solution is not going to uninstall status, check the uninstall report for a cause. You need to find the solution that did not went to uninstall and run an uninstall on it first, and keep repeating the process untill all are uninstalled.
