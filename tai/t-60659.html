first make sure rexec Rsh rlogin ftp are on
add rsh, rlogin and rexec in the etc/securetty file

u can use following commands

chkconfig rexec on
chkconfig rsh on
chkconfig rlogin on
chkconfig vsftpd on
cat >> /etc/securetty < rexec
rlogin
rsh
EOF
/etc/init.d/xinetd restart
vi /etc/vsftpd/ftpusers
vi /etc/vsftpd/user_list
service vsftpd start
chkconfig vsftpd on


important point : don't forget to add ur windows ip address( from where u r installing) and hostname in the /etc/hosts file of the linux server

15th June 2011, 12:23
Just some more things to check:

- Please also check the logfiles in /ERPLN/bse/log.
- Is there a logfile called log.oracle8 or log.dbserver_dll and what does it say ?
- Make sure that "norman ordanel" is member of the administrator group or use the bsp account for installation.
- check if you can connect to the oracle database via sqlplus and use the same login/passwords that you enter at the installation wizard

- not sure if this still an issue with Oracle 11: There used to be a script called "changeperm.sh" in the $ORACLE_HOME/install directoty that relaxes permissions on certain files. With Oracle 10g you had to run this script otherwise you got an "error 510 - database not on" in the log files.
