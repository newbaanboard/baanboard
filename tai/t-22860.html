Hi Hagen,

I don't have any advice for quality information sources, but we have created many exchange schemes (including generating an entire test company from an exchange scheme). We are also on Baan IVc3 which will be different from Baan ERP. However, the general process should remain similar.

The main process we use is as follows:
Set up the basic information
---------------------------
*Maintain Exchange Scheme
--Gives a name to the batch, date format, seperator, location of files, etc....
*Maintain ASCII Files
--Indicate which tables will be used for your Import/Export Scheme
*Maintain ASCII File Formats
--Indicate which fields of each table will be imported/exported
*Maintain Batches
--Set up how many batches each exchange scheme will have and in which order they are run (this will be important with multi-table exchange schemes since some tables need to be imported before others due to "lookups")

Set up the Import/Export
------------------------
*Maintain Table Relations
--Set up the file names for each table (and for each batch) that the exchange scheme will be importing from or exporting to.
*Maintain Field Relations
--Set up the fields to be imported/exported with the option of modifying the data "on-the-fly" with fixed values, incrementing values etc....
*Maintain Ranges
--Allow you to set up any restrictions on what data should be imported/exported
*Create Import/Export Script
--Generates the script that will be run by Exchange
*Import/Export Data (Regular or Non-Regular)
--Performs the import/export at per your schedule (runs the batch)

This is a very generic description since I am not aware of your familiarity regarding the exchange process. I hope this helps to some degree.
