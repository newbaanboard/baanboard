I had a similar problem in a previous life, but i solved it in a different way.

I created a baan table with fields like Trigger Name, Trigger Table and Status
Then deleted it and created an oracle view of dba_triggers with the same fields (and same name..)

then I created another table with like trigger name, trigger table and maintained that via Baan.

Then I had a report that would run in batch every morning that would
check to see if there were any tables in the 2nd table that didn't exist in the
first table, which would indicate that trigger was dropped.
