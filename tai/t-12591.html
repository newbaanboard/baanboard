NPRao

There are different monitoring tools to do that but it depends what is your idea of "big" process is - CPU Intensive/more memory or more database actions.

If you like to see the CPU intensive, then you can look from the shell command option from BW/Worktop and using ps command to see the process tick size.

To see the Oracle memory size by the bshell process-

[DEV:bsp]/app/common/home/bsp
>ps -lp 18895
F S UID PID PPID C PRI NI ADDR SZ WCHAN TTY TIME COMD
1 S 18292 18895 18872 0 154 20 599a6a00 1437 897690 ? 0:00 ora8_srv6.2
[DEV:bsp]/app/common/home/bsp
>
block size 4kb = 1024
1437 * 4k = 5.6 M
30-80 mb -> Oracle-8 process
