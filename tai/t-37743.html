There are two options to print the pre-requisites of a solution:

1. Run the session you mentioned already: "Print Dependencies by Update VRC" (ttpmc2440m000). If you install solution 203819 first, this session will run much better and you can prevent long running times by using the new options to suppress certain output.

2. Run the "Check to Install" from session "Process Solutions". Solution 139594 introduced a new option "Retain Status". If you check this option, the "Check to Install" will just check, and will not change the status to "To Install". All solutions that would be installed, will be printed.

Note that in both ways you need to download the pre-requisites first, otherwise PMC does not have all data. To download them automatically you can use session "Download, Scan and Connect Multi-Level" (ttpmc2210s000).

In Infor ERPLN 6.1 there is an option to download the pre-requisites automatically during "Check to Install"!

JaapJD
25th June 2007, 11:00
"Check to Install" prints all dependent solutions as far as they are downloaded and scanned, but not yet installed.
"Dependencies" shows just one level of dependencies.
A dependency is a relation between two solutions. A pre-requisite is a type of dependency. It means the the pre-requisite solution should be installed before the current solution can be installed.
A co-requisite dependency between two solutions means that both solutions have to be installed, but the order of installation is not relevant.
A post-requisite dependency means that the solution must be installed after the current solution is installed. Note that PMC only knows the post-requisite dependency after the newer solution has been scanned.
