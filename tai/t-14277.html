Hi,

i have had the same problem, but in ERP50c.
This was solved with solution 139224.
Here is the description of the solution :

SITUATION IDENTIFIED IN:
"Check Solutions to Install" (ttpmc2203s000)

SITUATION DESCRIPTION:
If you checked a PMC solution to be installed, the solution kept the status
'Available', although the report didn't show any error messages.

This was the case if there was a large tree of pre-requisite solutions and also
co-requisite solutions for a localization.


SOLUTION DESCRIPTION:
The check algorithm has been improved; if there are no restrictions to install
the solution(s), the status will be changed to 'To Install'.

See also solution 132988:

SITUATION DESCRIPTION:
It was possible to connect more than one Update VRC for the same Base VRC to one VRC Combination. This is not allowed.

VRC Combinations are meant to handle co-requisite dependencies between different Base VRC's (e.g. standard solutions and localization solutions) and not for updating different Update VRC's with solutions for the same Base VRC's at the same time.

When PMC was setup in this way, unpredictable errors might occur. One of the symptoms of this setup is that during the checking of solutions, the solutions keep the status 'Available' and there is no error message in the report why the solutions could not be installed.


SOLUTION DESCRIPTION:
It is not possible anymore to connect an Update VRC to a VRC combination if another Update VRC for the same Base VRC is connected to that VRC combination already.
