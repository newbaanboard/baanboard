What is your client, a windows PC or a unix [client] system also?
If it's a windows one you need to install a Xserver first and have it running before continuing.

In both cases, telnet to the Baan server, set BSE and BSE_TMP, export the DISPLAY to your local system IP and then start a bx6.1 in the background.

If anything mentioned is not clear, then forget about it, there is no reason to use bx6.1 for normal operations. The ASCII (ba6.1) is much better, from admin. or speed point of view.

Now, I don't see any advantage of using bx6.1 sinec it's too heavy, both on the server and mainly thru the network and not so user friendly as the BW.
The only advantage of using bx6.1 is if you can run it locally (like BW) and your client system is a supported one for doing it so (so, a portingset exists for your UNIX OS).
