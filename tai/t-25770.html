Nandan
30th November 2005, 17:43
No need to validate the second environment. Just copy the tables ttiex300 and ttiex301 (tools company) from the working environment to the test environment. Run the configure applications. (works if both Baan environments are of same version)

I have never seen two license daemons of the same baan version working on the same machine. Mostly there will be a problem as the both licenses daemons will compete for the same port number. If the environments are of different version, there is no problem to run multiple license daemons.

For the license problem you are facing, try to stop the license daemon of the working environment. See if you can login to Baan in demo version. If you are able to login then, copying of ttiex300 and ttiex301 should work. If you are still not able to login in demo period. Try to login to Baan using BW configuration, with command set to "ottiexpatch".

Nandan
1st December 2005, 11:34
Follow this sequence.

1. Start License daemon of Validated Baan.
2. Login to validated baan system. Use ttaad4100, navigate to ttiex300 use export functionality to export the table to a directory. do the same for ttiex301
3. logoff, stop License daemon
4. Login to test environment. You are logged on to demo version now. In demo version you will have access to all the sessions without any restrictions
5. Using ttaad4100, import the tables earlier exported.
6. Without logging off from baan, start the license daemon of validated system
7. Run configure application
