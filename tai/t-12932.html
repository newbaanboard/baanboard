Indeed, if you have a software problem and you are behind with installing sp's, you give support a possiblity to state that you first have to install the latest solution first. The other side is that if you allways install all weekly dumps from baan, you're the victime of all the new bugs.

Realizing this, I made my own installation procedure, which enables me to install the latest solutions, while still controlling what might be the consequence of it.

Procedure:
- download and scan weekly dumps every week
- use a query on pmc tables to get a list of solutions that can be installed without installing other solutions (dependency), also stating type of solution (script or table change)
- check the components and the short description of the solution using pmc, check on customization
- for the more important sessions and "heavy" solutions, check knowledge base on latest solutions
- install solution

This takes up time every week (about 1 hour), but at the end there is no problem in installing sp's (installing sp13 took me about 15 secs). You might consider to only scan weekly dumps up to 4 weeks ago, and to write a query to get a list of solutions by component (sessions).
