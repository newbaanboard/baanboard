A VRC Combination is a set of two or more Update VRC's in PMC. With this concept PMC can handle co-requisite dependencies between solutions of different Base VRCs. Suppose you use BaanERP 5.0c with the bgl0 localization. For both Base VRCs (B50_c and B50L_c_bgl0) an Update VRC is present. If you link both Update VRCs to the same VRC Combination, PMC will install the standard and localization solutions together. If an Update VRC is not linked to a VRC Combination, the co-requisite dependencies to other Base VRCs are ignored.

Session ttpmc2240s000 will generate the Update VRCs and VRC Combination. This is a quick way of setting up the environment for PMC. It creates the Package VRCs and can also change the Package Combinations. Assume you have the following VRCs installed: B50_c - B50_c_sp02 - B50L_c_bgl0. Session ttpmc2240s000 can change this structure to: B50_c - B50_c_sp02 - B50U_c_stnd - B50L_c_bgl0 - B50U_c_blg0. The Package Combination on B50_c_sp02 will be changed to B50U_c_stnd level, the Package Combination on B50L_c_bgl0 level will be changed to B50U_c_bgl0 level. Also B50L_c_bgl0 will be derived from B50U_c_stnd instead of B50_c_sp02. Note that the last step cannot be performed manually, so you need session ttpmc2240s000 to do that for you.
