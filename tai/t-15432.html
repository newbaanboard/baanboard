Yup.... it works fine.

Before you run this session, you have to take a sequential file dump of your company data in a folder. All the seq table data files created there are with a ".S" extension.
This session changes the old company number to the new company number with the data present in each of those files.
There will be more than 2000 such files.
That is the reason why it asks you for the folder name only and not each file name.

Hope this helps

You can take dumps by using the tools session.. ttaad4226m000 Create sequential dump of table.

Choose the below options:
specify field seperator.. "|" (pipe sign)
directory for table files : (only the folder path... no file name)
name of error file : c:\err.txt (to whichever path u prefer.

The output is a file for every individual table for whichever company you are trying to dump.

After this you can run the tccom0214m000 session.
