In Baan IV on UNIX tbase is part of the Baan product and therefore installed by default. But tbase was and is not supported for Baan in Windows platforms.

So if you install Baan on Unix, you have the option to directly install into tbase, oracle, informix etc. In the case of tbase, Baan will install the actually database binaries and the database data files by itself. In case of the other databases you have to install the product yourself and create the database files before installing Baan.

For Baan on Windows it is a bit different. Tbase is not an option. You have to install the data base product first yourself and then Baan. For some databases Baan is able to create then the data files automatically.

In my experience, the windows case depends a lot on the date of your installtion CD. I tend to install into SQL Server (because that always works well) and then copy the data to my actual target database.

I know, the situation is a bit of a mess, but that's the way it is.
