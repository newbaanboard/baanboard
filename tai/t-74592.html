Nevermind. Infor has a solution.

This KB is for info on the release of new functionality to extend dates beyond 2038.

With the release of porting set 9.1b, and the corresponding tools and application solutions Infor LN can support dates beyond 2038.
To enable this and other feature a new session is introduced in tools: "Features" (ttaad1130m000).
This session can be used to enable available features in the porting set, See KB1871111 for more info on this session.

........................

To enable the new feature for extended dates there are some pre-requisites:
• Tools 10.5.1 and KB1871111 and KB1874434.
• Application 10.5. Install LN application solutions through Nov 2016
• Porting set 9.1b, see KB22923520
• Enabling this feature is irreversible. Once timestamps beyond 2038 are in the database (through the conversion sessions or transactions) this feature cannot be disabled anymore.
