It is surely very strange - those authorizations. There are only two possibilities:
- either an authorization for a module (tdinv) exists and you exclude the authorization for some sessions of this module (Exclusive) or
- you give an authorization only for selected sessions (Inclusive).

I suggest:
First try to convert to runtime DD for this User.
If nothing happen then you should delete strange authorizations for this user and give them back again. Then it has to be everything ok.
