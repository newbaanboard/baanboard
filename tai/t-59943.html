For applications not to obtain help and you know the baanerp/B61a#stnd folder has content in your WebUI server, definitely the problem is Fallback scenarios.
A trick you can use to determine how to enter the correct fallback scenario is.

a) run session ttadv1511m000
b) jot down the string for the package combination to which all your users are associated, e.g., B61Caprod
c) Your fallback scenario (in WebUI admin console) for the applications help should be
baanerp.<string from step b> -----> B61Ua#stnd
For instance say the package comb for my users is B61Caprod and I have FP7 help installed (which installs in B61Ua7stnd) my fallback scenario would look like
baanerp.B61Caprod -----> B61Ua7stnd.

Molina.
