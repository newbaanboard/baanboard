It appears like your source environment data dump does not use the same locale as the newer environment which is Unicode.

Refer to the KB and the PDF - Infor Enterprise Server Unicode Migration Guide

Thank you, NPRao!

You are right!

The problem is that user locale (defined in [User Data Templates] and saved to %BSE%\lib\user\u<user> file) was different than defined in %BSE%\lib\tss_mbstore6.2.

Once we make the locale the same in both places the error is gone.
