Using Terminal Server has a lot of advantages but also disadvantages. We use TS ourselves for some of our applications.

Advantages:
- TS itself is a zero client
- Almost any application can run within a Terminal Server environment. So it makes any thick client a zero client.

Disadvantages:
- you now have two desktops, which is more difficult to work with using also apps at the same time not running via Terminal Server
- runs well on a LAN but can run into performance problems when running over the internet since all keyboard and mouse input must be send immediately to the TS and UI updates must be send back
- Needs configuration of accessing local resources like printers, copy to clipboard, etc.

We also get sometimes the requirement to run Webtop within a Terminal Server. I really would like to know why you would like to do so...

Regards, Ivo
