If this is Baan IV

From the OS do
explode6.1 d<Your table name>

such as

explode6.1 dtiitm0101

(if it's Baan 5 and up, use explode6.2)

does a file come up? if not - then your convert/create runtime DD was not successful.

If it does - then most likely you are using a new domain in that table that has NOT been converted to runtime DD.

this usually happens when you're using a domain from a different package (like a TC domain in a TD Table) if you just do create runtime DD for TD then the TC domain never gets converted.
