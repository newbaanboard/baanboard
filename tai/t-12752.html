Refer to - Performance, Tracing, and Tuning guide

dbsinit
Specifies database server optimization. The value of this variable must be read octal. Currently the only valid value is 01:
01 specifies that an optimistic approach must be used when checking for references in parent tables. The referenced row in the parent table is not locked, improving the overall concurrency. If this flag is not set, optimistic reference checking is not used.
Sometimes in Baan IV, the value 010 and 004 can be found when using multibyte. This is not needed anymore in iBaan ERP.
The 020 value has become default for a long time and cannot be set anymore.
