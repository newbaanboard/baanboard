From where it is reading DataDictionary in test server ? is it from Shared memory or Disk ?

write this command in Command option of BW config
-- -dbgsrdduse -logfile trace -keeplog <session code>

the above command will create a file called trace in $BSE\tmp folder. Open that file u will get the info about the place of DD.

If it is reading the DD from Disk the spead and the first connection will be slow. Load the DD into shared memory to make the speak faster.

We had problems similar to what youre seeing and it was licence related. If you try running licmon6.1 (assuming youre using a network licence) on the server that has teh slow responce to connect to the bshell you may also notice a slow response in getting licmon6.1 to respond. If it is you may need to get the latest licd binary from Baan. Also check your OS's message queue to see if it is maxing out. Somtimes your would have already connected to the bshell but the user interface does not come up.
