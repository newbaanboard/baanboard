Somewhat late response but it may give some more insight in what is happening... as you will know the bw client is talking to the bshell process where the actual actions (running sessions) is done. The bshell "talks" to the BW client every once in a while and requires response. When this response is inadequate or not present at all, the connection with the client is terminated with that error.

There are settings you can use when starting the bshell process, that influence this call-response mechanism.
-set DS_MAX_WAIT=value the timeout duration [seconds] in which BW has to respond to the sync message, and override the default of 30 seconds
-set DS_TIMEOUT_DETECT=value the timeout duration [minutes] before the bshell starts sending a sync message to BW

And - but you did not mention your settings - there is also a setting that terminates the idle BW instances:
-set SESSION_TIMEOUT=value the number of minutes the bshell can be inactive before it terminates itself, when the timeout is triggered a message is sent to the UI and the bshells stops

In case you are curious of this Client-Server communication, there is this option in the client:
Right Click the BW icon in the systray, opt for Debugging/Tracing. Choose Output to Console and set the other options
- Show Events
- Show Messages
- Show Socket Notifications
