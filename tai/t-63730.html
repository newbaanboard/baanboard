I have done this last year with unix systems. The master application server and the application server in that setup are running aix.
On both servers you basicly need to use the same portingset. I don't have experience with using mixed operating systems.

Basicly you need to use the BSE_REM variable on the application server which points to your remote master application server. You need to configure a r<user file> as well so your application server knows how to login to the remote master application server. Some files from $BSE/lib you need to have setup on the application server as well.

Long way back we needed these things as cpu's where not that fast as they are now. Nowadays it's not needed for performance anymore. The setup above was due to security reasons.

Wouldn't it be more wisely to migrate the unix baaniv environment to your windows machine? Or are they using BISAM, in that case they need to migrate their data to a RDBMS.
