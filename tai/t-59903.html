Dear Noor,

just some information about the topic enlarging order number.

http://www.baanboard.com/baanboard/showthread.php?t=44151&highlight=enlarging

I have done this twice ... depending on your customizations this project can take weeks.

The solution from baan offers some check reports which help you to identify forms, reports, scripts which have to be checked e.g. if there is enough space for 7 digits or if the form contains a hardcoded max value like "999999", this reports are realy helpful!

Another point is, that you have some time both, 6 digits and 7 digits in your system (current orders), i. e. the users have to be trained to work with both e.g. in reports or in business logic.

As far as I remember all standard forms, reports, scripts prepared for enlarged order numbers are shipped with sp 15 from baan ... but baan did this job not 100 % so you have to check your system with the check reports installed by the solution mentioned in the link above.

------------------------------

LN: I have done one migration from baan iv to infor ln fp 3. My point of view, that the business logic is realy different so users have to be trainend with totaly new processes ... the gui is more complex (dash boards), so you have a lot of information but you need users who can benefit from this.

There are some features to transfer e.g. reports from baan to infor ln, but i would recommend to check what is realy needed and develop it new.
