Hi,

this security file inconsistency can not be resolved by reinstalling and re-validating SLM.
It's an inconsistency at the ERP LN server itself.

You can check if the security file mechanism is consistent by running: bic_info(6.2) -S all

It's Infor support who should be able to get that resolved. They should be able to identify what the cause of the inconsistency is.
