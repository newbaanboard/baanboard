Hello @ all,

I found the solution. The problem was that the user root000 did not have the permissions for Audit.

To be able to process audit sessions the users have to receive extra permissions by root. Now it was surprising for me to see that root000 did not have the permissions. Anyhow the problem was solved after root gave the audit permissions to root000.
