Hi vishbaan,

If you made any changes in the user authorization, this is stored in table ttaad400000. Convert RDD checks this table to get the changes.

If you changes the tables directly, Convert RDD don´t recognize this. You must also update the table ttaad400000.

Unfortunately I don´t know exact which entries must be there, but probably you can check this, if you make some changes manually in order to see what entries where made.

In the p<username> files, are the DB-permissions stored.

Hope this helps.
