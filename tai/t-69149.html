Hi,

It can be done through Role Based Authorization.

Just create a role and add this session to the session authorization by session with "No Authorization" flag. Now, attach the role to the user and he won't have the access anymore.

Similar can be done using session authorization by company. Just give "No Authorization" to the company you would want to restrict the user. So, the switching wont work.

Hi,

I am not talking about the AAD roles. By role, I mean, AMS roles(ttams2100m000)

For creating a role for super user, you can give full table access, full library access and full company access for the companies they are permissible.

For this, you may use -

Session Authorization by Company
Table Autorization by Company
Library Authorization

Now make the super user a normal user and assign this new role to the user.
