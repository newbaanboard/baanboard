I found on Infor Support, what is wrong. Your environment is probably same as below:

Clean installation of the Infor Enterprise Server 10.3 fails with "ttbsi.iex.erpa: Error in import".
Following conditions cause the installation to fail:
- Single-byte installation
- used Porting set 9.0a or later

There is one workaround described for this situation:

Disable the row versioning when installing Infor ES older than 10.4. The work-around is

1. Install the porting set only.
2. Modify $BSE/lib/default/db_resource file: Disable/delete bdb_use_row_version resource by adding a hash in front of the line.
3. Continue the installation of the Tools & Apps
