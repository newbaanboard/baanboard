Hi there,

if you moved to a window system, then isamdef should not be important.

The last sentence "adjust path in the applicaton" means that you have to use session ttadv1115m000 (maintain directories of software components).
If paths have changed and you simply adjusted them in the fd6.1PACKCOMB file on OS level, the next "convert to runtime" will overwrite this file with old information. The same applies to paths of audit files, you have to use session ttaad4116m000 for the same reason.
Hope I didn't forget something important .... but usually it is very easy to move a running system.

good luck

Hi there,

in case of Baan V it should be fd6.2xxxxxxxxx where xxxxxxx is the name of the package combination. These files are to be found in $BSE/lib.
Use ttaad1120m000 (maintain package combination) and check, whether the paths there do exist on your machine.
Check whether there are entries in $BSE/log directory.
