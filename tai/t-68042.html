Are you wanting to know what something like ps stands for? If that is the case look for maintain packages - on 4c4 it is ttadv1100m000. That will show show the name for each package on your system. You can also see on 4c4 in ttadv1101m000(maintain modules) what each module means.

Or are you talking about the sub-directories under one of the package vrc. Where there are files like ftdinv, mtdinv, otdinv, etc. Then f is forms, m is menus, i is include files, o for objects, p session and library scripts and r is for reports.

There's another type of directories: b - additional files. That's where you'll find the icon-gif-files etc.

You'll find the possible software components when looking in session ttadv1115m000.

f - Forms
m - Menus
o - Objects
p - Program Scripts
i - Functions (includes)
r - Report Scripts
b - Additional Files ("XML-Models")
