I have received a similar error in the past when the BSE\tmp directory was not cleared before installing a service pack. It seems that some times (and this is conjecture), that a temporary file with the same name was used. Clearing out the temporary directory and rescanning worked for me. You may have to go into GTM and modify one of the PMC tables - I remember having to do that to change the status. Sorry I can't be more specific, but check your temp directory as a first try.

The temp files that get created may be corrupted. You need to re-scan the SP again with overwrite option, then redo the installation. However, prior to do this you need to change the solution ID in the PMC table that has the status 'installing' to 'available' first using GTM.
