L.S.,

Recently we had the same problem in a session. It turned out to be a parameter-problem. In this case, the parameter for TP-project in table tipcs020 had the value "No". When turned to "Yes", the problem was solved.

So, the session is trying to update a table that is not "selected for update".

By the way: this is not a Baan-problem, but a malfunction in your adjusted program.
