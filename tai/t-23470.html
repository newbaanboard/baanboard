A long time ago, you would get this error in Baan IV with Oracle, when the character set configuration of your database did not fit to the locale setting of a user and the key field did start with special characters like german "Umlaute".

What was happening is the following: Baan would query the database in the sense of "get next row". Then the database would return a row according to the sorting of the characterset in the database, but the key of the returned row was in the ordering of the Baan locale before the allready given row. Then the error would be thrown by Baan.

To solve this make sure locale of Baan and DB are the same or use ISO_BIN as the locale in Baan.

Maybe you have a similar problem now in DB2,
