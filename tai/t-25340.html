It may be helpful to you to think of Package Combination and VRCs similar to the relationship between Microsoft Office and MS-Word, MS-Excel, etc.

For example
a Package Combination (B61Ua) = Microsoft Office (MSO v.2.0)
is made up of
Packages at a given release (i.e., VRC) level (Package + VRC = PVRC)
tc VRC 1.1a = MS Excel (v1.1a)
td VRC 2.2b = MS Word (v2.2b)
tp VRC 3.3c = MS Project (v3.3c)

You could have another Package Combination using the same Baan packages but different versions. Say you want to keep using the "old" version of the tc package / MS-Excel, but use the newest versions of everything else.
Package Combination (B61Ub) = Microsoft Office (MSO v.2.1)
is made up of
tc VRC 1.1a = MS Excel (v1.1a)
td VRC 2.5 = MS Word (v2.5)
tp VRC 3.5 = MS Project (v3.5)

A package combination is tied to a Company and a user as a means of informing Baan what release level (i.e., what code base) that company or that user is to be using.

Hope this helps.
