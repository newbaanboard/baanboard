In Report Servers (ttrpi1500m000) you can specify the webservice URL to the SSRS you want to reach. But I assume you did that correctly already. Second thing is the Report Folder. Probably there is a $hostname variable in it. That is substituted at runtime with the server where the request comes from. And that is different. If you want to share report definitions between servers, you have to remove that variable from the Report Folder (in both servers) and move them manually in Report Manager. Or alternatively, deploy the reports also from server2, but in that case you have two copies, you need to keep in sync.
