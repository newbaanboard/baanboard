Hi there,

in the data dictionary you find described all domains, enums and their usage in tables.
Accordingly the information shown in the programs ttadv4100m000, ttadv4102m000 and ttadv4120m000 are what is written to the DD.
In session ttaad1120m000 you find the path of the DD for each package combination.
The structure of the DD itself is not complicated. For each package you have a subdirectory dxx.pd which contains the file dxx.pd ... the description of all domains and enums in package xx. For each module yyy in package xx you have a subdir dxxyyy, where the tables are described.
