Hi there,

in Unix the logs are written in $BSE/log, but in several files. The bshell, ba61, filter6.1, database drivers all have there own files.

To see the latest written logfiles just type:

ls -ltr

To see the end of a logfile, do:

tail -n 100 $BSE/log/log.bshell6.1

Slightly off topic, but you can get this information through running a db trace on your bshell when you run the session. This will create a log with the info on what table caused the fail.

Here's a link with some good documentation on tracing.

http://www.baanboard.com/baanboard/showthread.php?s=&threadid=1068&highlight=TTSQLTRACE

Which setting to use can be confusing. I'm bad at this, but you can try:
-- -set DBSLOG=01366 -dbgobj -keeplog -logfile mylog

You may need to play with the number, this will give you WAY more info than you need, but should be able to see it somewhere in there. Or check that post to understand better. It will create the 'mylog' file in your home directory.
