That doesn't look very good ...

First of all, you should decide what is the best series the continue (let's say 2).
* Then check/set the field "blocked for all input" (session tcmcs0147m000).
* Check all your jobs!
* Inform your users which series number to use now (lets say for purchase orders).

Just to make it clear: If you set the field "first free number" to 0, the system will have to check 200001, 200002 and so on until it finds the first gap. That might slow down your system!

So you *must* begin to "reorganize" your data i.e. archive your projects (session tipcs2260m00). This will take some time; we calulate 2-5 minutes for each project. This will increase the number of gaps so that you can reset the first free number to 0 without slowing down the system too much.

If you have done that for lots of projects, it might become necessary to use database tools ("update statistics") to increase the performance.
