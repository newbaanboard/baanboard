One hint for all those who have tried to install the SLM 2.3x up to 7.x on a virtual machine: Although it is not supported by Infor/SSA, it is still possible to install and run the latest SLM in a virtual environment. The setup only seems to recognize VMWare environments (atm) - running the setup (for example) on Sun's Virtual Box works fine.

Important: It seems that SLM 7.x recommends a new host ID. Thus the old license.xml doesn't work anymore... Also, as installing the SLM in a virtual environment is not supported by Infor/SSA, it is not recommended to use it for in a live/productive environment.

The SLM server checks the Mac Address of the virtual network card. Part of the MAC address identifies the manufactur of the device. On a VMware virtural machine, VMware is the manufacture. The SLM server shuts down as soon as it identifies the network card.

If you are using a notebook, install SLM server on the notebook and ERP on the Virtual Machine. You can also point the server.xml file to another SLM server on the network. If the Server has named users, you must be a named user. SLM also identifies the BCLMID for the development server. So you will probably not be able to do any development, but you could run the applicaton.
