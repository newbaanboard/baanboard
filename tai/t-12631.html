Minhaaj,
Is there a possibility to "copy" packages and all its components and change the package code ? (A "tt" session perhaps ??)
Yes. There is but inside the BaaN - dc/du tools. I developed that tool when I was in BaaN and its not released to customers.
How would this effect the data dictionary ?What all would I need to do to accomplish this ? (IF I were to write a 3GL to update the tt tables.... would it mess up the dictionary) ?

Yes this affects the datadictionary. You can only do it if you know all the tools table structures the way the data is stored etc. I prefer not to write this program now as you would need those functions/methods which are located in the tools includes (we need tools sources to make our scripts compileable) and are always subjected to changes.
BEWARE If you make any changes directly into the tools tables at the customer end, due to BaaN Copyrights rules, they are not responsible for any mess that you created and the support becomes void.
I suggest you log a case with BaaN support and try to get that session. But I built the tool for BaaN-5 series. I am not sure if it was ever ported for BaaN-4 series.
