Your installation is UNIX, right? I don't think that you need to re-install anything!

I would do the following:
1. mv /baan2 /baan (i.e. rename it)
2. ln -s /baan /baan2 (create a symbolic link, that might help to keep the system alive, and that might be removed later if you are sure that all other things work fine).

Normally, the only key to baan is the environment variable $BSE.
(For EURO-triangulation we had trhee different baan environments on one machine!)

But (shell-) scripts from outside baan, cron entries and all that stuff might have hard-coded paths, so that has to be checked as well.

If you follow Slawek's advice, please check at least the following files (and related tables) for hard coded directory names:
- $BSE/lib/fd*
- $BSE/lib/a*
- $BSE/etc/*
- %BSE/lib/sr*

If you only want to remove the Baan envrionment it is sufficient to remove the /baan2 tree. But check if everything was installed on this disk. It is possible to install parts on other disks. And of course tables should be revomed before or after manually.
