The domain is connected to the table field and describes the type/layout of the table field.

F.I.

Table ttccom010

field tccom010.nama domain tcnama
field tccom010.pstc domain tcpstc
field tccom010.sndr domain tcyesno

domain tcnama = string field, 35 characters
domain tcpstc = string field, 10 characters
domain tcyesno = enumerated field, can either be "yes" or "no"

So it is not possible to store text in a enum or nummeric domains. The domain describes what value can be stored in the table field and how it should be stored ( number decimals, lenght of the the field, lowercase, uppercase, predefined choices using enum values etc. )
