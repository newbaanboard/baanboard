The problem you have is that your table structure in the company in which you try to import is different than the one from which you exported.

The trick is to copy the table definities from your export environment to the import environment, than import the files and run a create runtime dd afterwards.

Of course this can only be done if the import environment isn't used!

Another way to solve it is to check what's different regarding the structure and try to match this in the sequential files.
