One of the mayor dependencies can be the use of the ability to print to XML. This caused a change in the $BSE/include6.1/bic_* files used to create objects. So objects created with these include files also need the support of this in the tools objects (in casu the ttstprepdll). This change also required a portingset update (because bic and repgen use the same $BSE/include6.1). This dependency is also documented in the SP requirement.
