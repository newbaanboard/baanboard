Hi Guys,
This is second time we are migrating our Baan from one server to other server. In 2002 we did migrated our Baan from 730 IBM iseries to 840 IBM iseries, this year we are planning to migrate our Baan from 840 to 570 IBM iseries in next couple of months.
In AS/400 Baan companies are generally go by libraries. In our general practice we do these following(this procedure only apply if you want to copy all your Baan instance)

1) Copy the Baan Data library (Baan 000) and rest of Libraries (other Logistic & finance companies)
2) Restore the all users and user authorities in OS level
3) Copy the all application directories (Baanerp/bse/application)
4) Make sure that you copy the db4 users files to other server
5) Change the host name in tabledef6.2 file. (/baanerp/bse/lib)
6) Change the host name in license6.2 file. (/baanerp/bse/lib)
7) Start the Baan on new server
8) You wont see the licmon6.2 job in new server.
9) Run the patch objects after error solving. That put your system in demo mode.
10) This point I prefer you better to run all create runtime tools & application process, just for safe.
11) Change the host name in Database definition session and convert run time.
12) Validate the new system with Baan validation department to get permanent licenses.
13) Like that you have to re-validate all you Baan software after moving the Baan to new sever (like Open world, Bridgelogix software etc)

Good luck,
