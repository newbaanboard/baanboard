There are a couple other places to check. Verify that the user is using the same Package Combination. If the data dictionary is different (domains and tables) you can not access a company with a different Package Combination. But if the data dictionary is identical, you will not get an error message and the session/dll path can be different.

Check the security of the DLL at the OS level. Make sure it has the correct values. Verify the user belongs to a group that can access the DLL. Sometimes when applying customiztions or patches, the files pick up the permissions from the user that are not correct.

Verify the $BSE/lib/user/u(username) file is being updated. If the permissioons are not correct, when changing the users values and converting to runtime, the OS file is not updated. View the contents and make sure that it is being changed.
