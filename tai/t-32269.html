I'm running B40c.75 at this time. The scripts seem to run properly (as well as inquiries, reports, etc), but when I try to maintain a script, it can't find the '.cln' file (using wordpad or notepad). 'temp' and 'tmp' were assigned in the environental variables (local) by the Baan setup.

**** resolution ****

The fix was to check the 'tmp' folder for rights. I was not able to copy a file to that folder without having vista asking for administrative override (permission) to place the file there. Since this Baan account is not a local (full) administrator, I changed the security (properties) on the 'tmp' folder to allow my Baan user (joined to a corporate domain) to have full access (after having Vista ask me a few times if I accept this action).
