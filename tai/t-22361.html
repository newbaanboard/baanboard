Hi Suresh,

You are lucky, because you are running Oracle, so the solution is pretty simple:

If you are on Unix and your users connect via the ASCII Interface, then you can just use the built-in password aging restrictions of your Unix box.

If your users connect via the GUI, then you can still get it working, by using a wrapper script around the bshell binary.


If you are using Informix, then Baan is using the OS passowrd for authentification, so changes of the password by the end user would give you a headache.

I am not sure, wether there exist an actual product for this issue, but a bit of scripting will get you there.

Hi Suresh,

the good thing about Baan on Oracle (compared to informix) is, that the user does not need to know his oracle password. Actually you do not even need a dedicated oracle user for each Baan user. This way, the IT department can maintain and change regularly the Oracle password and adopt the changes in the ora_users file.

Iin regards to the bshell wrapper, please search the forum first. There are lots of references to the general concept of a bshell wrapper and what it can be used for.
