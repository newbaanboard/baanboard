Hi,

as Andreas stated this is quite a task if you are a new Infor admin.
Generally it is:
- Install the new 10.7 server,
- install two VRC structures on it (one with 10.7, one with FP7),
- export the company data from FP7 (old) and import in to FP7 (new box),
- then change package combination by company from FP7 to 10.7 on the new machine
- run the data upgrade engine and fix all errors it will report (consult the installation guide bhushan provided)

There are some issues you have to figure out when you face them (e.g. the tools tables have partially a different structure..).

Good luck & have nice day

We also a have the a8 VRC (and our customer variant) on the LN 10.7 Server. So I think you will need a copy of your a7 (FP7) and of your customer VRC an the LN 10.7 Server.
So basically you need to have two package combinations on your LN 10.7 Server. The first one is a copy of your live package combination of your F7 Server. The second one is the package combination of LN 10.7.
The idea is to switch your LN 10.7 Company to the copy package combination of your FP8 server. Export the data from the FP8 server and Import it onto the LN 10.7 server. (That’s why you need to have a copy of your "old" package combination on the new server.)
Then you switch back to the LN 10.7 (new) package combination. After reconfig of the data the Migration Tools are executed, to convert your old data into the data needed in LN 10.7.

That’s the basic thing as far as I can tell. I can't give you any details (like what session you gonna need to start the data conversion; never did that. I guess it should be somewhere in the Infor documentation for the new release.)
