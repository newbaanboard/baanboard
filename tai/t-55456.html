I guess you just want to disable the user for a temp period. In Baan IVc, the easiest way that you can temperary disable the users from the HP-UX side.

Solutions 1: change the file /etc/passwd if you are root;
Solutions 2: change the user $BSE/ora/ora_users.

In either way you need copy and backup the current file, and create a new file contains only the lines with the users you still want to enable the access.

After you finish the maintenance, you could just move the file backed up to the original one.

I understand that, you would like to baan in SINGLE USER MODE.
If so, please try the below steps.

1. Take backup of $BSE\lib\ipc_info file.
2. Ensure all users logout.
3. Modify the below line $BSE\lib\ipc_info file.
"bshell s 0 0 p ${BSE}/bin/ntbshell6.2" by adding a prefix or postfile for 'bshell'
Example : bshell2 s 0 0 p ${BSE}/bin/ntbshell6.2

4. Change BW configuration - field of 'Bshell name' as 'bshell2' instead of bshell.
5. Any other user who is using 'bshell' in BW configuration, will not be able to log on.
6. After finish of your work in SINGLE USER MODE, you can revert back the changes in ipc_info - 'bshell2' as 'bshell'

Another easy way is to change the permission on ${BSE}/bin/bshell6.x or ntbshell.exe

Lock all users except one:
Give execute rights to only the user that is allowed to log on.

Unlock all users:
Give execute rights to group bsp (unix) or designated group for Baan users in windows.
