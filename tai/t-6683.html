If you can, start the ttstpshell process (via Option Dialog - Start Shell) and press ps for the process list. Starting ps again will offer you the possibility to see changes in the process list.

If you succeed, then the bshell process does respond and you can see what processes do get processor ticks. If it is ttstpshell only that adds up ticks, then the other processes either died or wait for input.

If you cannot go to the option dialog (because it is a background job process) you can try to do the same with the bshcmd binary, you need to know the pid of that particular bshell though.

If you do not succeed, then with bshell command you cannot reach the bshell either...
