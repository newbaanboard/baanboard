Hi,

The database groups relates to the users and the rights provided to users. If you refer to the definition of database groups in Prog Guide, you will get to know its a group of database users with the same authorizations and rights. When you add a database user to a database group, the user automatically receives the group's authorizations and rights.

So, I guess you will need to keep separate databases for different sets of companies and then link the database groups to those databases. And, link the users to the database groups.

The database group becomes a separate instance name of database. You will need to study complete DB & Application Architecture to implement this.
