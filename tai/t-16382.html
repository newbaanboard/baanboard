I believe you can Import the data from your existing servers to the new severs from the tables ttaad200(User Data), ttaad220(Company Authorisations), ttaad225(Company per User Autorisations), ttaad230(Module Authorisation), ttaad231(Session Authorisations). Convert to runtime DD of the user data.

Yes, you can copy the contents of the tables, but the Convert will not do much, because the Convert-Indicators are missing. Once way to fix this, is to create these indicators yourself.

Another way to do it, might be the export/import session found in tools under Triton migration.

All of this will only help you, if you have not changed certain things like the names of Package Combinatios and VRCs!

Markus
