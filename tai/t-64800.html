Hi,

From one of the documents:-

Device type "Windows Server Printer" is specially introduced to be able to print from Webtop. When logging into Baan using Webtop, you do not have locally Bwprint installed. In that case printing is done from the Baan server.

It comes with special "silent bw print".

To know more about devices, try KB 22838233

About, brp.* commands, they act same like rprt.* functions, where you can specify which will be the default devices, which report will be printed at run time.

We use Windows printing. Makes it quite easy. You just specify the Windows printer in Device quite (like \\<Print-server>\<Printer>.
If this printer is installed on the client, Baan will direct the print job there, if not it gives you the option to install the printer and then print to it.

The windows server printer is especially used for printing from jobs. Check the script to start jobs in %BSE%\bin. It uses ntbshell.exe --server to run the job, there is no bw used.
Reason is that you don't have a desktop available if things are started from the taskscheduler.

I am not sure but before webui downloaded bwprint to the desktop and this was used to print.
