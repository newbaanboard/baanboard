To your 2nd question: I can only tell you how we avoided the problem: we are using the employee number in Baan with a b as first character as login (except for some special users). i.e. b<your number>. We need the b only because unix doesn't accept numbers only. But I think it would be very much work for you to change all logins, homedirectorys ... :o . The advantage of this login is, it is easy to find the employee from the login code i.e. if you need adress data, email, whatever.
