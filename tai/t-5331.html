In my opinion, there is not a general tool that can answer your question.

I think we have to make a big difference between going from c2 to c4, than going from c4 to ERP, for example.

ERP is a new application. You cannot go from c4 to ERP simply applying patches.

Let us call migration when moving to cx to cy.
Let us call install a new application and try to implement it and transfer data from an old application when moving from cx to ERP.

Some modules does not exist in ERP, logic changes, etc... Maybe is easier going from BaaN c4 to BaaN ERP, than going form SAP to ERP.

We call all Migration, but those different migrations are too DIFFERENT.
