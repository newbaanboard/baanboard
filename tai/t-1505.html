Looks like BS and DS were validated with the same validation of keys, otherwise they would not be sharing the same license server; two diferent validation set of keys mean no sharing of the license server.

If that is the case, you need to apply for a DS own license validation keys and re-license it (and updating the DS license6.2 to point to its own):

DS + BS same validation keys => 1 single license server on the main server (the "properly" licensed DS)
DS own validation + BS own validation => each needs/has its own license daemon
