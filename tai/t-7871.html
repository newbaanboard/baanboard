I assume the error message will be something like "Cannot start session" wich is correct to inform the user about what is happening, but in such cases it is not enough to inform the system adminstrator about what the cause is. We are lucky to have the complete error messages in the log files in such cases.

Here also the logfile itself (a tail or a copy of the complete logging of the event in any of the logfiles) will explain more about the reason. This may be permission error, non-patched binary, lacking table definition etc...
