Error 512 points to a corrupted DD, according to the help. There could be another reason, namely that the logged in user has a different package combination than the company. If this is the case, GTM (and all other sessions accessing the table) will report the error. Meaning that the table's definition in the package combination that you are working with, differs from the definition in the company where the table belongs. Check this out, and if the problem persists, let us know.
