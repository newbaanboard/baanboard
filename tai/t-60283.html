I have observed that while taking seq. dumps in LN FP5, the character "U" is prefixed to string data. In case any string field is empty then only "U" is stored in the sequential dump.

It becomes a problem when someone has to work on sequential data dump in excel, then he has to get rid of prefixed character "U".

Can somebody let me know how can I create seq. dump of table without "U" being prefixed for string type of data.

set variable PREVERSION and run bdbpre again:

-set PREVERSION=3
