Yes, it is possible to migrate. Only depends if the current system is already been used for some time (data, users, patches - SP's - etc.) or not. If not, maybe it's faster to start with a new environment, if yes, then migrate.

You need in advance to add the Oracle8 function and create your database, users, etc, maybe even upgrade the portingset...
Then you use bdbpost to dump your tables (all, including company 000), stop Baan, adapt tabledef6.1 to point to the new database, start Baan and then use bdbpost to load the data.

Have backups at hand...
