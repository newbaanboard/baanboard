Error 205 will come up in insert or update, when one field of the tableset will not follow the domain-definition of the table-definition. Or one field is empty but marked as mandantory in the tables definition.

205 will also occur, when you try to write a string with specialchars (like umlaut) into a non multibyte string on a multybyte installation (had that one myself two days ago, when importing some data from BaanIV into LN).

The logfiles of Baan/LN will log the fieldname, that caused the error (if you're using a Baan/LN windows-server, you need to check the windows event log).
