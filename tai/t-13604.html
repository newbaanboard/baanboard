I think it's the correct procedure.
You can have more than one BSE environment on the same machine, so you can install c4 on a different directory, migrate the customizations on the new environment, test the all and after you can simply switch the users to use the new one, deleting the old one when you are sure all it's ok.
I think also you have to require a new license code for the new environment, but it's better to check it with BaaN.

you have two options

a) you install baanIVc4 above baanIVc3
and create in this new branch a new prod VRC. If you have customiation you must redo it in this branch. you can then migrate the company from VRC. ==> this reconbfigure your data automatically for the tables that has different structure in BaanIVc3 and BaanIVc4.

b) create a new $BSE and install therein baanIVc4. Create the new prod VRC and also a cdummy VRC wgere you copy the datadictionnaries from you porid BaanIVc3
create you companies in this vr'c and import the data from baanIVc3
the migrate company from VRC.

remark : this is working in unix environment if you are on NT I think that you are limited to case a.

case a is more easy but you can only test with test companies not the prod company or you need to copy your production data to new company(ies).

case b is a little more complex but you can install in baanIVc4 a copy of your prod baanIVc3 (same companies) and do testing.

I would definitely advise you to go for plan b.

This should also be possible on Windows NT. If not, then get yourself temporaryly a test server to prepare everything there and copy everything at migration day.

If you go for plan a, then you will never get rid completely of your c3 installation. On top of that, as long as you have not finilized your migration, you will be working with c3 application objects on top of c4 tools objects. Thisis possible and some baan users do it, but it is kind of not supported, I guess.
