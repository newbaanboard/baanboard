Yes it is possible. There are 2 ways to accomplish this:
* first you can use bdbpre/post on tu tables, if scripts are present do not forget tttxt tables, and be sure the receiving company doesn't have any data in it which could be overwritten.
* second: create export scheme for the tuxch tables, using range active; do not forget to change the flag for text from numeric to multiline text; in the receiving company create an import scheme for tuxch tables.

The latter would be the wisest I guess. In the past I have done this numerous times, e.g. for hours accounting schemes, without any problem.
