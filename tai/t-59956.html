Hello everybody,
I have a question regarding decimal and thousand sign.

My customer has two companies. In both the companies the software language must be English: labels, forms and reports must appear in English for the company users.

My problem is: for one company, users must have "." as decimal sign and "," as thousand sign, while for the other company users must have "," as decimal sign and "." as thousand sign.

Since decimal and thousand signs are specified on software language, this is a problem.

Users of the two companies are different: they belong to the same system, but if an user works on a company does not work on the other: there is no overlapping.

Is there a possibility to implement this avoiding to create a new language and copying all the language-dependent software component to the new language?

Dear lvergnano,

yes it is possible but not with the baan standard.

We had this issue once with an system running in us and mexico.

It was not only confusing but also dangerous e.g. cheque documents.

The solution works as follows (unix tru64, baan iv):

There are 2 datecurr files on the system. One is original for your settings for language "2" named "datecurruk", the second is modified (e.g. datcurrus) where the thousand and decimal sign is changed.

Within a login script a text file containing the "special" users is checked. If the user is e.g. us company, then the current datecurr file is replaced on os level with one of the source datecurr files (either datecurruk or datecurrus).

This avoids having a special us englisch language.

One possible risk could be that the copy process is not 100 % stable, but we never had this issue with this system.

Link with similar topic: http://www.baanboard.com/baanboard/showthread.php?t=6925&highlight=datecurr
