Please make sure you understand these things! If not you can end up with bookings in your live data!!!
The online help of tccom0214m000 contains a detailed description of the procedure. Did you noticed that?

Within tccom0214m000 you should replace ALL 3 companies if they are related. So use 800>900;801>901;802>902;

You run this for each company, start the session in the original company!!!
ALWAYS check the sequential files if the modification has been done! E.g. in ttccom170*.S

IMHO: the error you get has nothing to do with the above process. Probably you have still users working in that company? Check the eventlog or logfiles, they should contain more information.

Please test this first in a separate BSE if you have never done this. Check if the application keeps working, especially if you don't have bookings in the wrong company.

Error 510 (first error on your log) This indicates that the database has not been started. Start the database to correct the problem.
Error 506 on your log means taht the table do not exist.

Try to delete all tables from company 900 then create all tables for company 900 toi be sure all tables had been created properly.

Once done it repeat the load process with the flag "Append if table exist" turned on. This should fix your issue.
