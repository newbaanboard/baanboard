As Thangtq739 pointed out, in session User Data Template ttams1110m000 you can define the timezone for each user. You have to make sure that the user does not have a different template assigned (in User Data session ttaad2500m000) as you could have users in different timezones and therefore LN allows you to create different templates where you can assign different timezones to different groups of users.
As importantly is to update the DST files that are delivered with the newest portingsets ($BSE/lib/zonerules and $BSE/lib/zoneinfo) (check sol 148218)
These files is how the application determines according to the timezone and date and time of your system what time should be in effect for each user.
