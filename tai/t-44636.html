For the Infor Webtop to start any session it needs to convert the form definitions into a format that it recognizes. This is the reason why it takes time when opening any session in Webtop. The delay can be reduced by doing this conversion process(Online Form Cache Inititalization) before-hand. It creates the form definitions in an xml format inside a pre-defined folder(set during the conversion process) for all the packages in the range provided(at the time of executing this process) of the same package combination the user(baan user login) belongs to.
