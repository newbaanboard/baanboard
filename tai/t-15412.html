Hi there,

what you see is confusing, but quite correct!!!

The form/table, which s used to enter the security key always contains exactly one row. This is the row, where the session "Print requested system conf" was executed last.

So if baan send you two security key, one for seq no 1 and one for seq no 2, simply fail over, enter the whole data (ie. the seq. the configuration keys and the security key) and validate.

You might find that the field to enter the security key is not enabled. In the ascii Interfacee you had to press the group key "s" to get there. In the GUI it should be that button, which looks with a bit of imagination like a butterfly.
