Barcode printers like Intermec, Zebra, TEC etc provide own codes to generate barcodes. You will check the printers manual to retrieve the correct codes for printing a barcode from intermec.

Basicly you will need to put a the printercommands on the report layout. Standaard Baan layout commands and barcode formatting wil normally not work when using Intermec, Zebra etc. as these printers a a different printer language instead of PCL 5 / PCL 6.

So, check the programming manual of your printer device.

Hi Altrev,

At our site we are printing from Baan to Intermec Label printers.
As we have seen it - it is not as straightforward as printing to laster printers.

The reason is that label printers mfrs have their properiety languages like IPL or DP for Intermec.

We use bartender (BT-E3 BarTender Enterprise ) to design the label and convert the format to a file in the printer language. Bartender needs to be licensed - :rolleyes:

The file is placed in a Unix directory, with placeholders for "variable data".

When printing a report - you need to write a custom dll to replace the variables with actual data and send the file to the label printer (like an lp -d)

No driver is required in Unix - in this case.

Hope this helps.
