Solution Description English SCM for Windows based platforms.
TOOLS VERSION(S): 7.6_b4
SITUATION IDENTIFIED IN:
"(De)activate SCM and Component Management " (ttscm0501m000)
SITUATION DESCRIPTION:
For Windows based platform SCM has not been enabled.
SOLUTION DESCRIPTION:
SCM has been enabled for Windows based platforms.
This solution contains a Functional Addition. Please read solution 134994 for the background of Functional Changes.

SPECIAL INSTRUCTIONS:
The following actions have to be executed to enable SCM for a Windows based platform:
1.
Install RCS binaries on Windows Server machine.
These binaries can be found at /ftpdownload/updates/7.6_b4_tt/230630tt_rcs.zip .
The original binaries can be found at http://www.cs.purdue.edu/homes/trinkle/RCS/ .
For normal SCM actions the 'win32' binaries are used, for merge action the 'dos' binaries are used.
The ZIP file also contain a small 'mergewin.bat' script for merging purposes.
The following directories are present in the ZIP file
- .\rcs binaries needed for SCM (including small bat script)
- .\rcs\dos original dos binaries.
- .\rcs\win32 original windows binaries.

2.
Via the environment variable 'RCSDIR' the location of the RCS binaries must be set.
This must be done at the server e.g. in the logic service.
E.g.. RCSDIR="D:\RCS"
This path should be added to the PATH variable (before $BSE/bin).
Also a global system variable 'TZ' must be set.

In case of problems a special logging activated. This is done via the environment variable 'WIN_SCM_LOG'.
The logged data with become present in $BSE/tmp/log.RCS

Users also should have write permissions on c:\windows\temp on the server.

3.
To make the SCM/CMT sessions visible in the menus the refresh button (or F5) can be used.
