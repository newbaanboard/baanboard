1. ERP is normally bound by I/O, since you run your db on 'real' hw it think this should not be a problem. VMware has some overhead although this nearly disappeared.
2. licd will NOT run in vmware, it's being detected. However since your db is running on physical hw you can easily install a so called fat client on that machine which enables you to run licd on that server.
slm can't run too in vmware. But's more easy to configure it, the slm server can run e.g. on a simple xp station. However in order to be able to run slm you need to be on tools SP25 (not sure about the SP but it's one of the latest) and you need to have one of the latest portingsets.

So basicly I don't see why you can't use Baan with the hw setup that you propose.
