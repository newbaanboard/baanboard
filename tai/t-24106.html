I am surprised nobody answered to this yet.

Unfortunately you are on completely the wrong track here. If you look at a typical 3-tier architecture, then you will find a pretty low traffic between the UI-tier and the application layer and a very high amount of traffic between the application- and database layer.

This holds true for Baan also. So if you install on your client a fat client, then you will create a very high bandwidth between your client and the database. Baan recommends at least 100Mbit for this. Compared to this, you should be able to run 2-4 GUIs (depending whom you ask) via a 64kbit ISDN line.

You can improve user experience even more, by using a terminal server and server becoming a bit less sensitive for network latency problems.
