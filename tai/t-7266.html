This is from Baan support:

Check the $BSE/log directory ( for NT check Application Event Viewer of the Server) for relevant log file information.

Host Mode:
- The file permission for $BSE/lib/tabledef{REL} may be wrong.
- In tabledef{REL} the database type( oracle7,oracle8,informix etc) must be present in $BSE/lib/ipc_info file
- Not enough resource available to start a driver process.
- $BSE/lib/ipc_info file has SPACES as field separator instead of TAB.
In Baan Client-Server(Baan Net):
- All the possibilities mentioned for Host Mode.
- $BSE/lib/user/r{$USER} file contains wrong password, permission.
- There is a difference in server name in $BSE/lib/tabledef{REL} and $BSE/lib/user/r{USER}.
- Format problem in $BSE/lib/user/r{USER}.
- Client and server network connection is down.
