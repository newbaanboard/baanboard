Are you asking at a Unix level or Baan Level? If Baan do a bdbpre/post of the users tables with the right setting this will only add new users in to your tables. But be carefull make sure you get all the tables required.

If at a Unix level so long as the O/S is the same on both systems copy the password and shadow type files across, but again be carefull as root and other system logins might have diff passwords on the 2 systems.

For copying users, you need to update your /etc/passd, home directories, <db>_users file, create them in the DB, upload the [corrected fort the test system] user tables (session authorizations, user data, etc.), also upload the entries in ttaad999 (runtime) or create entries for ttaad400, etc., etc...

More tricky than you think and if something is wrong, your user(s) will not work as expected. If possible, create the ones that do not exist yet fresh on your test system.
