10th July 2005, 19:13
This has been discussed already, a SUPER user can access all sessions and cannot be restricted. If you want to restrict session usage, do NOT create a super but a NORMAL user instead and then grant access to the sessions needed.
Create an export of the live company and then import it into the demo company - this is the standard way of copying company data. Make sure that the data definitions (DD) of the tables is the same bewteen the VRC of the live company and the demo company to avoid DD errors while importing the data.

manojsharma
27th July 2005, 13:33
Hi Ganesh,

As far as I understand, you require a super user with no authorisation for tools.

When you create a super user, he/she has no authorisation for APPLICATION DEVELOPMENT, APPLICAITON CUSTOMISATION, SQL QUERRIES, MENU MANAGEMENT etc.

And when you give permission in session ttadv0142m000 (Maintain General Developer Authorisation), then the super user able to access the above menus.

But if you don't want to give permission for entire TOOLS, then you have to create a normal user and give permission by module using ttaad2130m000 as said by RITA.

I think this will solve your problem.
