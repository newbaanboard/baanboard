Not sure about your environment. But, on LN when I faced this issue, I created a bat file and scheduled it right before the jobs. What the bat file does is, it restarts the License service and makes sure that no orphan processes are running and makes a way for my job to run without any issues.

I know from a customer who free'd the licenses to by resetting the license server. They got big problems after Infor audited that server. Don't think that is legally allowed.
With the old license server you can use an option to clear orphaned licenses, use that to free them. However that will not help if all your licenses are indeed are used.

A better way is to use the SESSION_TIMEOUT to kick out users after they are idle for a certain period.

However since abottoir is using unix he/she can easily write a wrapper script to prevent all licenses are used.
