I don't know if it is the same way or not anymore, but it used to be that if you used DEM module when you generated the users desktop it would create more session permissions that what was on the DEM flows. That was an issue - but I also found it hard to believe than an employee would just guess on run program to hit a session. Anyway, log story short, we created our own run program that checks only for sessions on the DEM flow directly - then runs it. So for example - if the DEM created access to tisfc0101m000, but it was not on one of the DEM flows you cannot run program.

As far as I know if you are not using DEM flows you still need permissions to the session to use run program.
