(from worktop technical notes)

A common Windows 2000/XP problem is that when sessions are launched from within Worktop, they appear in back of the Worktop application. This problem is not restricted to Worktop but many applications will show this behavior. The reason is that Windows 2000/XP by default does not allow applications to "steal the focus".

To solve this, you may do one of the following:

1. you can edit the systems registry manually: Open regedit (win2k\xp) or regedt32 (win98), locate the "ForegroundLockTimeout" key in HKEY_CURRENT_USER\Control Panel\Desktop and set its value to 0.

2. download the TweakUI application from the Microsoft site. Extract the executable. After extraction, go to that folder and find the "tweakui.inf" file. Right click on this file and select the "Install" option from the context menu. After installation, double click the "Tweak UI" icon located in the Control Panel and go to the tab called "General". There is a checkbox with the label "Prevent applications from stealing focus". Uncheck this box.
