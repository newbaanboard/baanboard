I am not sure if this is what you are looking for but the easiest way to possibly set view (read) only authorisations for Baan would be to use Database Authorizations functionality (Session ttaad4130m000). As long as the user is set to Normal you can then set authorizations to read only at the package level. Do this for all packages except 'tt' - Tools.

This should get you started in the right direction, if you are using DEM, this should also generate the session security for the required user, but I do not believe that it sets up Database Authorizations.

Create a new role, which has only read access for all companies/packages based on session. You can set authorization at the database level to make it read access for all companies/tables. But if you do that you prohibit updations to some of the tools table entries like session defaults, queries/filters/printer queue, application locks, address book info etc.

The above option is based on the Tools Authorization Management System, you have to explore the similar DEM Role setup linking sessions and processes.

I am on the BaaN-5.2 system, you are on BaaN-4 series, you can find the corresponding sessions/table codes to handle it.
