Hy Every one

Here it is a gold solution for this Issue:
I you have installed batch install SP1-22 and now you can not execute any baan session, getting this error, so you can implement this solution:

1. Navigate to $BSE/lib/install and check if the tab.reg file is present. If it does not exist, put the attached tab.reg file in this folder. If it exists, then make a backup/copy of the current file and put one tab.reg file .

2. Navigate to $BSE/tools/tt7.1_a. Check if the ittbic folder is present. If not, create it manually and take care it gets the same permissions as the other folders 3. Put a ibic___slm___0 file in the folder created in step 2.
4. Navigate to $BSE/lib. Open and edit the fd file for the package combination in which you are working. E.g. the fd6.2.b50c file in case your current package combination was b50c. Check if a line is present for itt. If not, then add the following line:

itt:${BSE}/tools/tt7.1_a

Also for this step please first make a backup/copy of the file before you start changing it.

Then finally go do step 5:

5. In the "Directories of Software Components" (ttadv1115m000) session, check if the marked line is present of the second screenshot of the secfile2.doc document. If not, add this line.

This is the best solution we could have.

Thanks and good luck to your work
