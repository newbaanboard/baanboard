You can not share tables if companies are not in the same package combination.

If table structure is same in both package combinations then you can create database script, which will periodically copy data for selected tables from first to second company. I am not sure how it will work and what is risk.
Anyway, you have to organize work in such way that people change selected tables (insert, modify, delete) in one company only, database script will "synchronize" data for second company.
