Spartacus,

You can also use - ttaad4228m000 - Convert/Recover Tables.

This session can be used for two purposes:
- to convert tables to another database type
- to recover tables

The first item means, for example, that you can convert BAAN
Base tables to ORACLE tables. In that way you can transfer data of
BAAN Base tables to an ORACLE system.

Recovering tables means that you can convert data of tables to the same
table. This means that the table will be rebuilt. See also
Reorganize Tables (ttaad4225m000).


HOW TO USE THE SESSION
This session is a combination of the sessions Create Sequential Dump of
Table (ttaad4226m000) and Create Table from Sequential Dump
(ttaad4227m000). First you select the tables you want to convert or
recover. After that you can use a number of options, as in these two
sessions.

IMPORTANT
If you create tables with "Ignore Referential Integrity Constraints" =
Yes, it is essential to run the session Reorganize Tables (ttaad4225m000)
afterwards, with "Repair Reference Counters" = Yes.
