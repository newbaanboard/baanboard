Hitesh Shah

U can create ur own index on Baan table after copying it to a new customized VRC .

When u do this , u have to port the changes to the table in Baan SPs or bug fixes manually in customized VRC. Further primary key should be created using table's original primary key to avoid any reconfig or runtime problems. Due to these additional efforts , modifying a standard Baan table is done only after careful consideration of all pros & cons.

Sure!

If you create the index in Baan, then it is customizing, with all the pros and cons.

If you create the index in your database directly, then you must be aware, that the index is lost, if Baan reconfigures the table (becuase of a patch for example). In this case Baan dumps, drops and imports the table.
