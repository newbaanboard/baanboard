Yes - you can do it from AFS. Of course it depends on what you are trying to do when you say "print a document". So for example for generating outbound - I believe we run it twice - run one prints the outbound advice to the selected printer and then run two prints the outbound shortages to the printer. What you can not do is run one report and then send the report to two different printers - but you can run the report twice and send it to two different printers.

I do not know what you mean by "windows server printer" - but it should work sending it to any printer type that works in baan. So define the printer in baan, make sure it works and you should then be able to use that printer with AFS.

I use a virtual printer called redmon http://pages.cs.wisc.edu/~ghost/redmon/ to do something similar where I want a print sent to printer and a copy sent to an email. You can setup redmon to run a batch file when it gets a print job. In my batch I send a copy to a printer and a copy to a command line email program. In your case just write a batch to send the output to different printers. For more complicated printing we use autoform lasernet from efstech.
