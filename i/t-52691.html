As far as i understand you need the following procedure:

(1) Delete the order position (with status 7)
(2) Delete the order header (after all positions are deleted)
(3) Run the session tdsls5201m000, now all History data which have no current data (in tdsls040/tdsls041) will be deleted.

If you have no archive company you will loose your historical data e.g. for turnover querries etc.

You can check the parameter session tdsls4100m000, mask 4, "Method of deleting order data".

Marking the session in the menue tree, pressing F1 will give you some usefull hints.

If you look up the on-line help of the session tdsls4223m000, it gives rules about when the data is deleted:

Sales order lines are only deleted if a number of criteria are met:
- the backordered quantity must be 0
- financial transactions must have been created and the turnover history
updated
- the project for project-related sales must have been archived
- if the order is an installment order, the customer must have been
invoiced for all installments, and the order must have the status
"Closed"



Maybe some steps remain to be done in your system that prevents the orders from getting deleted.
