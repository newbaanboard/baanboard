Yes & No option will based on transaction type update mode.

if it's other mode, then yes option is not working.

Hello Finamp,

if your transaction type mode is " Real ", then you can use yes option in trail balance. system will include all non finalized transaction for those transaction type.
if it's other mode, then yes option is not working.

If Transaction Type Mode is "Real Time" - then it is updated directly when the debit values entered for a document correspond with the associated credit values .

i.e. it does not wait for the batch to be finalized and hits directly tfgld106

Where as in case of "Finalization" mode it awaits in tfgld102 table for finalization to hit tfgld106 using session tfgld1110m000.

Hence option in TB “Non Finalized Trans" if Yes - then Picks from tfgld102.
