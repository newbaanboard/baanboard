As a policy we are not allowing users to exceed their department quota . So users can never get logins in excess of their departments. If that's what u need , all u need to do is follow below-mentioned steps .

1. Create a table (e.g. tijwx650 in 0 company )
2. Maintain all OS users (not Baan users ) in lower case in this table
with theri OS ID , dept and login allocation .
3. Create this session (in all package combination with no forms and a 3gl program script say tijwx6666 as per attachment)
4. Add this session to startup sessions of all users and convert to runtime.

Now users can never exceed their dept allocations .
