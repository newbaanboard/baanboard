Hello,

Another fast alternative is to use ttaad4100, after creating and converting a new table definition.

1. invoke ttaad4100
2. input your data
3. depress the 'y' key

If the message 'File does not exist, allocate?' answer yes and you are finished. Otherwise an error message is displayed.

Hi there,

there could be all sort of reasons for your problem:

a) The table is in an VRC, which is not visible in your Package combination. If you look into ttaad4100, do you get the right table description?

b) Could be a typical Baan on NT problem. On NT baan does not properly sync new things on disk and shared memory. Often the only solution is restart of baan.

c) Even more evil: I once had a similar problem on NT. After fiddling around for a day, I got frustrated and rebooted the server. After this, the problem was gone!

In your case looks like the new table alredy exists in the database.

Try this run session ttaad4231m000 - Delete table, key in your company no, package, table no. If it says 1 table deleted then it is OK. After that go to ttaad4100 and create this table again.

should work.

When you create a new table def with all the required fields and indexs you can do a "Convert to Run time DD" and "Create Table" from form 2 (Details) session itself.

The .NEW file is created only when the table already exists and you are trying to add / delete / changes a field or indexs.

Hope this helps.
