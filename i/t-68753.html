28th March 2016, 11:47
What happened is that the data dictionary file ($BSE/tools/dd/dttaad/dttaad260) and the table ttaad260000 in the database are out of sync because you didnt check the "dump data definition" switch.

What you could do: edit the data dictionary file so that it is in sync again with the database table, then log in and then do the create RTDD for enterprise server with "dump data definitions".

The data dictionary file is a plain text file and you can delete the entries that the error message indicates (desc, catg, etc.) but you should know what you do and make a backup of the file before you start.

Consulting Infor is also not a bad idea !

Have a nice day,

lebowski
