We are on Baan 5.0b and we eventually created our own external system to manage purchase requisition approvals for MRO items.

We found that Baan did not have enough controls in it to ensure that an end user did not request over their given approval limits or to financial coding that they are not approved to use.

We had a consultant company build an external requisition system that runs on our intranet so that any employee can create a requisition and if it is not within their limits, they would have to send it to a person that has sufficiant approval limits. It would then e-mail that person with a link to the external requisition for them to approve or reject. Only after this external requisition was properly approved would it write the requisition into Baan requisitions, and from that point, the buyers would then go forward with the purchase.

The system has it's pros and cons and Baan would be smart to look at expanding the requisition system to include similar controls.

Another draw back that we have for MRO purchases on Baan is that we use a single cost item for all these purchases (otherwise you'd be setting up new items like crazy). In Baan 5.0b, you can not convert the requisition to an RFQ without losing all the financial coding. Generally we want to be able to send some of these sort of requests out for quotation but we can not use Baan for this either. I am not sure if Baan 5.0c has corrected this issue or not. When we need to send a request out for multiple quotations, this generally has to be done manually, otherwise we convert the requisition into a PO and use a modified PO report as a Request for Quotation. Messy but effective for us as we are not high volume and it doesn't slow us down too much.

Another draw back of the single cost item is that all requirements are entered into the text, and it is very difficult to search the system later for items you have purchased in the past.
