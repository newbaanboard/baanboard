Finally installed ERPLN FP6 on windows 2008 R2 and MS SQL2008 SP1.

The initial DB connectvity problem was because of SQL DMO packages were not installed, after installing them seperately the problem got fixed, but got stuck in another problem where the installation was using sspi login instead of rexec, as Lebowski suggested in another post we can fix it with a work around by manually configuring it to rexec, but the latest porting set has done that job for us, so when you use this portingset it doesnt use sspi.

The problem was with sspi login which was fixed in the latest portingset (8.6a.02),

Hope this info can help others in installing latest OS, DB and ERPLN.
