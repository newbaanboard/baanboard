Actually I'm quite surprised that there are companies that allow ordinary users to do GTM ????
Table maintenance should be restricted to IT staff / administrators anyway, and any GTM I do is either; - because of testing in a seperate test company - to correct any structural system error / bug that has created inconsistency that cannot be reversed with the normal programs - to correct/adjust an occassional orderstatus (e.g. sometimes and order gets blocked due to margin control settings where actually it should not be blocked.)
The last two GTM adjustments should be accompanied with proper documentation about cause/resolution by IT, and if applicable by the request from end-users.
Accountants will also want to see this kind of tracing.
If you really need 7-10 GTM adjustments per day then something needs fixing :-)

p.s.: I think as per BV GTM also enforces consistency and business logic using DAL
