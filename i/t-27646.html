29th December 2006, 14:30
Using the error message from the qptool I searched Oracle Metalink and found a document that detailed the same sort of error code. The fix was -

The dynamic loader, dld.sl, is attempting to bind to (load) a 64-bit libclntsh, when the application is 32-bit.

dld.sl is pointing to $ORACLE_HOME/lib. Notice it has lib and not lib32.

The library directory structure has changed from Oracle9i compared to Oracle8i.

In Oracle8i lib contain 32-bit libraries and lib64 contain 64-bit libraries.
From Oracle9i lib contain 64-bit libraries and lib32 contain 32-bit libraries.
SolutionTo implement the solution, please execute the following steps:

1.set SHLIB_PATH to $ORACLE_HOME/lib32.
2 set LD_LIBRARY_PATH to $ORACLE_HOME/lib32.
