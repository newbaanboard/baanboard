Hi,

Use the following command in the BECS command line and run the slow sessions, this will create the tracing info in the desired location and from the reports this generates you can track down to the query level as it shows which query is causing the slowness.

-- -set PROFILE_ALL=1 -set PROF_RTIME=1 -set BDB_ALWAYS_FLUSH=1 -set PROF_DIR=<specify your own directory where you wanted to create these files, with out any angle brackets, just the path>

Also make sure all the parameters are set properly, like BSE, BSE_HOME etc.

All the best.
