I have found on the following solution on infor site,

Description:
How does SLM validate licenses?

Resolution:
General informatin on SLM license validation:
- When a session is started the related product ID is “taken” for a license.
- If you close the session the license is not released immediately. It depends on the timeout settings (in the license file and shown in the snapin). By default 15 minutes, after which the license is released again.
- Which session takes which product ID can be shown with the “bic_info.exe –S all” command. It will show for example:
<sessionentry session="cprmp4504 " sourceid="7125" objectid="7124" />. In this example objectid="7124" is the product ID it takes to run the object ocprmp4504.
- SLM validate licenses only after launching a session, the validation does not happen at log in.

Hi,

the "Confirme Time" can be checked in the SLM snap-in.

Select License Data (F0).
Right-click Options Data (L6) in the right area and, on the shortcut menu select Properties.

Properties:
Confirm Time
The time, in seconds, in which a license request must be confirmed. The default confirm time is set to 900 seconds, or 15 minutes.


See also KB 1023275 "What is the significance of "Confirm Time" in SLM? How does it influence the licensing?" in the Infor Knowledgebase.
