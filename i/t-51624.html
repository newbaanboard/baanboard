Refer to the Latest Design Principles document for the Package VRC naming standards:

Package VRCs
Definition

The package VRC defines a specific version of the package.
Coding standard

Code: pp vvv t rr ccc z
Characters Description
pp Package
vvv version (2.1, 2.9, 210, 211, 5.0, and so on)
t type (in uppercase)
rr release code
ccc customer code
z sequence number (alphanumeric: 0,1,2....9,a,b...z)

Possible values for type:
Value Description
B business-specific version
C customization by dealer
D dealer-specific version
J junior (compact) version
L localization
O own development by customer
S standard

For standard software the code position t remains blank.
Change your Package Combination/VRC to use 'O' instead of 'L'.
