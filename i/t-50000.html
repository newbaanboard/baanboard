If you want to completely disable this session, one way (brutal way, IMHO) is to remove (or rename) the object script from the directory tools:

$BSE/tools/ttXX/oMMMNNNN where MMMNNNN where MMMNNNN are the third 7 chars of the session (discard first 2, the module name).

Just rename it as the same name.DISABLED or something else

Nobody can call this session after.

Otherwise, if you want someone have already access to this session, you have to manage the session authorizations (this means that all users have to be normal users, a lot of work and a slightly increase of load of the system...)
