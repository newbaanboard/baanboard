Wrapper is using a product like RMCis, qkey or ~vamsi source extender. What you do is write code that runs with the original Baan Object. A baan object is made up of events that can be called from outside the program, the wrapper program allows you to write code that runs with the original baan program by calling these events. In essence this code wraps around the Baan object - thus the name "Wrapper" or "source code extender".
