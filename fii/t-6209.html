As Juli mentioned, you need to look in the tfgld4532m000 session. This will tell you why the integration has not posted. Using this information, you will then need to create integration setup in order to post the transactions into the general ledger and finalize the batches created by the posting before you can close the period.

When you try to use the session: tfgld0107m000 "Maintain Period Status" and if you change the period from Open to Closed, it will think for a bit and then tell you: "Not all integration transactions are posted for year... period..." At the bottom o the screen it will tell you the type of the source and transaction. If it is PCS/Act. Cost of Goods, one thing that has worked for me was to close the Project and then Post the resulting Integration transactions using tfgld4201m000 and selecting the resulting batch for finalization using tfgld1110m000. This cleared out most of my offenders but now I am stuck with some projects that are not ready to be closed even though they have un-posted integration trx...
