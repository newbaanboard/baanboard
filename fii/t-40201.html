tfacr2240m000 / tfacr2250m000 will not clear interim sales account. It will only adjust a/r control account amounts.

In Baan IV you'll just do a manual entry to clear this amount. But you should log a case with Infor support as the rounding you are getting should not occur.

The biggest difference in LN over Baan IV is that integration accounts like interim sales can be reconcilied using specific functionality in financial integrations. If you are familar with how the Goods Received Not Invoiced reconciliation functionality works in Baan 4 or 5 then it is a little bit like that but much better. Basically this tool pulls all the transactions and points you to things that might be wrong.

There should have been a documentation CD with a reconciliation reference guide with the LN install CD or you can search for it at the Infor Support website.

Projects in LN are different from Baan IV - much more robust functionality. You can post interim results if that is your question. I'm not too familar with the details.
