Anthony,

We have the same problem, and I haven't found a workable solution yet. Anyone: would this be possible in BaaN ERP ?

Anyway, IMO it is not a bug but quite logical: if you value items against lot price, you'll need a 'lot'. The direct delivery procedure skips all the steps of physical handling the goods in your warehouse. So no inbound movement -> no lot -> no lot price.
Hence why it is not possible to do this with lot valued items.
