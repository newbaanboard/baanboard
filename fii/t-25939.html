Hi,

first the customer is right. It is an EC Sales if the ship-from address is in one EC country and the ship-to address is in another EC country and if the invoice-from-company (so the company issuing the goods) and the invoice-to company (so the company receiving the goods) both indicate their VAT number,so that they can occur on the invoice.

What you need to do is to create and use a correct Invoice-to-address that the user has to choose correctly in the Sales Order. This must look like this. As I attach an example I refere to the countries in my docu n ot on your example but it is basically teh same. In my example goods go from germany (so assume u are a german company) to UK but the Inovice goes to Hong-Kong. So create an address wherer you enter on the first form the country Hong-Kong and the address in Hong-Kong. The country in this case is needed to select e.g. the correct address format. Usually as well it determins the VAT country but.................................

............................an enhancement shiped with a solution has been done to define on form two a country for VAT. And if you enter a country there the system uses this one - not the one on form one - to determin the VAT country. So u enter UK. And now the VAT calculation will find DE to UK which is an EC Sales. As well you have to enter in the address the UK VAT Id-Number, else system does not return EC Sales, it returns Germany / VAT code from item master file as the system believes the customer has not VTA number and is a private person.
