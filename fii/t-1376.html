Dear Ken,

the general way how to use Dimension is not very complicate, in fact it is as easy as how to use the chart of account and you can learn it from the Baan help or User Documentation.

But the details how to set them up is also as complicate as to set up the chart of account. You need a concept of cost accounting, booking rules, reporting necessities and Integration Logistic-Finance before you set them up. A good knowledge of the logistic processes helps too.
In my opinion this cannot be described in a general documentation but you have to work it out e.g. with a finance consultant.

hi,

dimensions are conceptually subledgers and are attached to GL either by making them mandatory or optional.
if it is made mandatory when GL posting is done it searches for the subledger(dimension) to be posted.
this ensures the subledger is also posted while posting to GL

by this way 5 dimensions (subledgers) can be attached to a GL thereby ensuring that the growthof GL is not drastic and altered during the course of the business which also ensures the legality of the object(COA).

this is a USP of baan and if put effectively eases the job of Users of Finance in baan and also provides an excellent report for the MIS.

the sessions wherein this can be done is in the Integration with other modules B-object in GL module and u can use the session available in this b-objects.
