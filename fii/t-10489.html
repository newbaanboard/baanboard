The rebuild session is tfgld3206m000 and should only be run when there is no one working on the system. This session rebuilds your history totals from the ledger transactions(tfgld106). The trial balance print values from the history totals. This will at least ensure that your trial balance report is correct.

Did you run the select financial statement values before running the financial statement? This could also be a problem.

30th June 2003, 09:55
If you rebuild your ledger history (tfgld3203 or 3206, they do the same job in a different way), all relevant records are removed from the history tables (tfgld20*) and rebuilt from your transaction history tfgld106 (and from tfgld102 I think).
If you have closed your previous financial year that's ok. But if you have only provisionally closed the previous year, this causes a problem since the provisional year end only writes the opening balance totals to the tfgld20* tables but does not create a transaction. As a result, the rebuild will wipe out your opening balance and does not rebuild it since there is nothing to rebuild it from. You will have to re-run the session to create a provisional opening balance yourself to correct this.
