Hi,
Maybe u have a way to recover tfgld418 missing records using tfgld410...as u may have noticed records in tfgld410 relate to tfgld418 in the following way:
tfgld418.ocom=tfgld410.ocom
tfgld418.tror=tfgld410.tror
tfgld418.fitr=tfgld410.fitr
tfgld418.trdt=tfgld410.trdt
tfgld418.trtm=tfgld410.trtm
tfgld418.sern=tfgld410.sern
tfgld418.line=tfgld410.line
tfgld418.ttyp=tfgld410.ttyp
tfgld418.docn=tfgld410.docn
tfgld418.fcom=tfgld410.fcom
tfgld418.lino=tfgld410.lino
tfgld418.cprj=tfgld410.cprj
tfgld418.prin as no equivalent in tfgld 410 but is normally = tcyesno.no
tfgld418.sint is set to tfgld.sint.posted if tfgld410.podt <> 0(empty) else is unposted or erroneous
tfgld418.tmpk=tfgld410.tmpk

The main index is the same, so u can rebuild tfgld418 from tfgld410 with this criteria...
Maybe u should try it first with existing records in both tables to check if your script would generate correct records for tfgld418 out of tfgld410...
Hope this help.
