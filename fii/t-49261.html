Problem could be in the following 2 tables:

tfgld418
tfgld417

Please check the value in the table field "fcom" (financial company).

The reason is there are 1 ore more datasets with value "0" because the correct value e.g. "100" is not set.

The select within the programm calls for the table in company from field value "fcom".
This is now "0" -> BaaN tries to select from tools company.

Solution:

Manually edit the correct values with GTM (ttaad4100) and rerun the program.

This happens only 1 time up to now in the system.
