Hi Laksmi Software,

That's not how our 5.0c SP7 behaves. It simply uses single MAUC value per item code per company and uses that for integration. Did they change the feature on SP8+ ? It does make more sense to have MAUC per warehouse in companies which is geographically dispersed in large country with each warehouse supplied by different supplier (for geographical reason), but that would make tracing back integration transactions a lot harder. Honestly, people can go so far as to calculate MAUC per supplier lot, but then at one point, the advantage of MAUC system disappears. MAUC assumes that when issuing goods from warehouse, people don't really know which exact shipment that particular unit came from (and what price). By using MAUC and simply averaging the cost of the good before being issued, the accounting of inventory can be somewhat simplified

Rgds,
Paul
