If I remember correctly, Baan does not integrate indirect labor transactions to finance. We use ADP payroll provider and have to do a file dump up and back during the payroll process. We have written our own scripts that takes the hours accounting data for indirect labor and assign accounts/dimensions to the transactions based on the task number and employee's home workcenter. These are then imported into the GL.

We use a similar process to read hours accounting records to create a payroll accrual for the end of the month (we only accrue indirect labor).

Yep, we have done the same. I work for a consulting firm, we developed a full Time and Attendance system to go on top of Hours Accounting & Interface to payroll. We wrote JV posting for indirects (among other things).
If that's not something you would be able to code, we could provide just the indirect posting portion. It is not freeware, as mentioned we do consulting, but that part alone would be pretty inexpensive, and supported by our firm through upgrades and service packs, etc.
If you have the coding ability, you can fill the JV Import table tfgld213, tfgld214, tfgld215 and then run the standard Baan session to Import as a JV Transaction.

I should also mention a decent non-conventional solution that we found at one of our client sites. To get the integrations to go through, they just set up a couple production orders that they never closed, and assigned them to post as needed. They entered production order transactions instead of indirects then.

It worked well for them for a long time. (Until they went with our system that posted indirects).
