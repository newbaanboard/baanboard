There is limitation in baan system. At a time only one user can be super user. Whatever the user ID that you update in CMG parameter that user will become a super and he can edit any batch, which you would be aware already.
There is no other authorization session as per my knowledge.

Now when user try to open the payment advice it will check two tables tfcmg109, tfcmg000.
I think you have to overwrite this with little bit customisation.
