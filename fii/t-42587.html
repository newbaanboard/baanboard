The error you are mentioning comes when the batch status is "Free". Normally batch status is free when you have inserted a transaction type in the maintain transaction session, but not yet entered a transaction (or it is unbalanced)

But it is unlikly for an integration batch status to be "Free" because there are bound to be transactions in the batch. Still I remember integration batches too create empty batches (due to some bug which we later resolved), especially sales batches.

Check the batch status, and check the transactions inside the batch. If the batch status is free, and batch is empty, you can delete the batches. If the transactions are present and balanced, you can correct the batch status to "Ready" by a GTM. If the transactions are not balanced, correct the transactions by GTM, then change the batch status through GTM.

Above solution is just based upon your description of the problem. I advise you get a professional help before doing the GTM.
