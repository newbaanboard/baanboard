this is data corruption.

if the status is registered, there cannot be a matched amount.

normally you should report this to baan support.

if you want to do this yourself:
check tfacp200 and table where the matching is in (tdpur0 something in baanIV). If no matching lines are in this table, you can reset the status in tfacp200 to registered and removed the matched amount.
