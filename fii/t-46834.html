Hi there,

No, do not switch a pure logistic company, all of a sudden to a both company, in which you probably also switch on "Finance implemented". You will have a lot of open orders, which only create financial transactions from the date on which you turned the company into both, which will lead to balances on integration accounts, which should not be there.

If you have finance implemented, then indeed you need an active mapping scheme, to create any logistical transaction. However, this is not too difficult. You can just create a mapping scheme, with just one document type and map it with a default account. Check it and make it active, that should not be too difficult. Of course remember, that in the end you need to make a good mapping scheme, because all the integration transactions which are being created from that moment onwards, need to be mapped to ledger account/dimensions and need to be posted. But, like I said earlier, any active mapping scheme will do, even with just one document type, mapped to a default ledger account.

Michiel

Is the setting under sales Parameter (tdsls0500m000) and purchase parameter (tdpur0100m000) -

Log Financial Economic Transactions set to NO

This setting will not stop the logistic transaction also will still have the financial transaction post to the integration table, but will not post to finance until the mapping scheme is setup.

Hi there,

It will only stop the "On Order" postings while registering the orders, if you go further in the process, like receiving/shipping in warehousing, you still need an active mapping scheme. So like I said the easiest way is still to create a simple mapping scheme with just one document type and map that to a default account. Check it and make it active.
