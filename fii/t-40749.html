Create a financial batch and Reject the cheque which will return the invoice(s) to an open status where either a new cheque can be issued or a credit memo applied. There should be a transaction type defined within the Bank Relation (tfcmg0110m000).

Rejecting a cheque requires the same steps as clearing a paid cheque (reconciliation). This assumes that your payment method was set with an anticipated step. In Payment Methods (tfcmg0140m000) you should see a field 'Document Accepted/Sent' under the heading Anticipated Payment/Receipt Steps. Hopefully this field is 'yes'. It would be highly unusual for cheques not to be set this way.

I complete agree with John's comments. To add to that, there is one point mentioned in the first post: Paid status of the Cheque. Cheque status will become paid only when it is reconciled. If the cheque is already reconciled and the reconciled batch is not yet finalized then only user can do something about it. User has to change the reconciliation status from Complete to Open first. Then the cheque will become anticipated. Now user can reject the cheque by changing the status from Open to Rejected.
