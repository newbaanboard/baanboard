Try to enter also a currency translation account on the complementary side in the company parameters.

I think the year-end program checks both before it starts processing. The reason is that if you want to know whether complementary accounts have been used in the system, it will have to check the full account table whether complemenatry accounts have been used. This costs time, so that is the reason I guess that both accounts (staturory and complementary) are checked eventhough maybe the dual accounting functionality has not been used at all.

I know that with SL158180 this check is done in tfgld6203 and the complementary currency translation account does not have to be filled in, in the company parameters. So either fill in this complemenatry currency translation account or install SL158180.
