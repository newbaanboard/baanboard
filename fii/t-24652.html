I am not sure if this answer would help you, but still thought of sharing with you. Since it is working fine for one company, it is possible that the data is corrupt in the 2nd company. When I say corrupt doesn't mean physically corrupt, but possible that some data is not the way the pogram exepects it to be.

For example, in the session Write-off currency differences (tfacr2250m000), we found that if one customer has a document in home currency(HC) and the same document with a different line number in foreign currency(FC) and if the FC document has a $0.01 balance to be written-off, then when we execute the write-off session, it simply hangs. When we talk to Baan support, they don't have an answer and say that they cannot reproduce the same. But the fact is that the session hangs.

Similarly, there could be something contrary to the program's expectations. You can try to run a few customer code ranges and see where exactly the problem lies and then analyse the data to get a better picture.
