Hi,
there are two other rebuild sessions but, you can only run them for the Year. The tfgld3206m000 session is the only one that gives you the period range, and wether you want to rebuild the Ledger, the Dimensions, or both. No one must be in Finance while any of these sessions are running.
You are always dealing with History ( tfgld106). What these rebuild sessions do is recalculate the Opening balances and ytd activity based on what is recorded in the tfgld106 Finalized History table. It depends on which session you run and what parameters you select but, these are the tables involved in the recalculations:

tfgld201 ledger account totals
tfgld202 dimension/ledger totals
tfgld203 opening balances ledger
tfgld204 open balance dimension/ledger
tfgld205 history dimension/ledger combination

The tfgld106 is more or less the BIBLE of the finance module. It is not properly indexed for these sessions to run at maximum efficiency. ( in our version c3 mcr0)
The tfgld106 table is almost always a large table, and this causes the sessions to run more slowly. Make sure you have enough space roll back segments. roll back segments being to small has caused this session to abort for me.
Hope this helps to clarify,
BB

Hi,

First of all, indeed make sure that no financial transactions are made/posted. These transactions update the history tables (tfgld201+) that your rebuild session also alters. The result may lead to wrong figures that needs to be corrected ... by rebuilding!

It's quite common for a number of our customers to rebuild just one or a few periods per night; thus requiring more than a week to rebuild an entire year.
