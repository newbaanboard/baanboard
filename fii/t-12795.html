Hi Abuibra,

The tables below are used:
table ttffbs000 |Budget Parameters
table ttffbs005 |Budget / Year Master Data
table ttffbs101 |Budget Amounts/Quantities
table ttfgld004 |Financial Company
table ttfgld005 |Period Master
table ttfgld007 |Period status
table ttfgld008 |Chart of Accounts
table ttfgld010 |Dimension Master
table ttfgld201 |History - Ledger/Curr
table ttfgld203 |Opening Balances - Ledger
table ttfgld001 |Companies

Tables tfgld201 and 203 are used in the report. Table tfgld201 is the sum of all finalized transactions for a specific year. It is possible that a rebuild of your ledger totals isn´t finished completely. The effect is that the total amounts in tfgld201 aren´t correct or completely empty. During a rebuild of your ledger totals the first step is a remove of the existing totals from table tfgld201 and other tables in the 200 serie. If after this remove the rebuild run is killed you do not have totals and therefore zero values in your report.

In case of budgets it is possible that you forgot to fill in a budget code in the session or that the budgets aren´t filled in in the budget module.

Some hints but it is still possible that the script of the session of report is corrupt due to a bad fix.
