Yes, I have come across this problem recently.

This happens because there is a finance integration batch somewhere in your system which is in use. In my case this happened in a multi site environment.

To solve:

Find the original company num, transaction origin, financial transaction, date and transaction type of the transaction.

Look on tfgld430 for the batch info for that original company

Find the batch and check it's status. If it is in use change to free and rerun integration. It should post now.
