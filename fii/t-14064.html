I think that even when you have the integration with finance switched off, Baan will still generate the integration transactions. Once you implement finance they will be posted.
I believe this has been done to make it possible during an implementation to go live with the distribution part of the system before finance is fully implemented without loosing the integration transactions.

You can of course delete all records from the tfgld41* tables, but that may stop you from matching purchase invoices against existing orders. It may also have some implications for things like GRNI matching, WIP and stock reconciliation etc. I'd get some advise from Baan support on that.
