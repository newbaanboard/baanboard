Hello Yolanda,
If you are getting this error when you try to close the GLD module, and indicates that is on the tfgld416, this mean that there integration transactions that have not been transfered to finance.

You should, look at Integration errors in session tfgld4432m000, there will appear all the transactions that have not been transfered to finance and the cause for this.

If there only one transaction with error (that you have described), the most sure is there no code defined for a transaction without asignation. So I recommend you that you create a temproary transaction to make the transaction have their accounts. After this, you should delete this code in order to avoid erroneous transactions.

Hope this help.
