We need some advice how to change the way we use currencies (mainly Canadian and US currencies, with some Euros thrown in).

Our accounting department wants to be able to use different currency rates for balance sheet and P&L accounts. The balance sheet accounts should use a fixed, end of month rate, while the P&L accounts should use more of an average of rates throughout the month.

Currently we only enter one rate at the end of each month.

Accounting would also like to roll standard costs (which is done every month) based on the average rate.

Some ideas we have had:
Does anyone have any experience with Baan's multicurrency extension? Would it help with this?

If not, we are thinking of creating a table similar to Maintain Currency Rates (tcmcs008) that would hold the P&L rates. We could then write reports that use the rates from this tables and use these to create journal entries for the differences.

For standard costs we are thinking of just changing the purchase rate to the P&L rate, rolling the costs, then changing the rate back to the balance sheet rate.

Has anyone else faced this problem? How have you handled it?

We use multi-currency extension and I don't think it would help with your requirement. You still only have 1 standard cost (albeit stored in 3 currencies) and exchange rates are stored 1 per currency pair per date.
It is quite common for P&L transactions to use average rates and balance sheet, end of month ones. However, you don't know what the average is until you get to the end of the month. So I think you have a choice of entering rates in more regularly (so each P&L transaction could use the "actual" rate) and then balance sheet accounts may need to be revalued at month end using the month end rate.
You can only run (i.e. store) standard cost once per day so maybe you could do that the day before end of month (e.g. 29th June) using the average rate at that point in time, that would be near enough.

Hope that helps a bit.
