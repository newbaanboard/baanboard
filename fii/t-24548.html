Hi,
This session is used to set more than one discount on the same order line,, any way if you are using this session or any field in the sales order or sales order line for discount it has the same accounting..

When you post the sales invoice, you get this entry :

-- assuming no tax

Dr.Customer
Cr.Turn over account

and the following data is entered in the integration tables..

centeral invoicing (sales)/sales revenue
centeral invoicing (sales)/Cost of goods sold
centeral invoicing (sales)/discount

this discount is the one came from the sales order -- and which can be changed before release to invoicing using change price and discount after delivery --

now the first entry has the turn over account less than what we should have because the discount is not there..

so we add it as follows

centeral invoicing (sales)/discount

Dr.discount
Cr.Revenues

Note that revenues is the same that is used in centeral invoicing (sales)/sales revenue not the turn over account.
