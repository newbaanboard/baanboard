can you get a look at the records in the tfcmg504 table? That is the table that Maria C. is referring to. It holds the record type data for EBS. The session script checks the first record in the ascii file and compares it against this table. If no match is found, then it gives the error you described. Compare the record in the tfcmg504 table with the first record in the EBS file.
hope this helps,
