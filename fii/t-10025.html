I agree with the previous reply that not sharing the finance tables is not recommended by Baan, but as long as you are sensible and test properly, it can be done.
I'd try not sharing tfgld426 and use separate integration elements for separate companies. That allows you to post to different dimension codes for the same workcentre in different logistical companies.
Your post mentions different ledger accounts. They are not defined in gld but instead in for example tdinv8150m000 /tfgld850 which isn't (or doesn't have to be)shared anyway, so different ledger accounts should not be a problem.
