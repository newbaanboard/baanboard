Sales return order generate credit notes in finance AR module. Is there a way to link the credit note automatically to another Sales invoice which was generated from a Sales order?

At present the Credit note has information about the Sales return order and then we will have to check the Sales return order and then find the Original order or ask the Sales department, which is a time consuming process.

Please let me know if there is a solution or work-around to this in Baan ERP LN 6.1 (FP5).

Hi,

I found that while creating the Sales return order, in the header details, there is a block with the heading as "Returns" in the session "tdsls4100m000". The fields in this block needs to defined as follows;
Original Document Type: Invoice
Original Document Number: <Company number>, <Original Sales Invoice number>
Return Reason: Select the appropriate Return Reason code.

Then include Sales return order lines and process the sales order and the credit note that is generated will be automaticall set-off with the original invoice amount. If the Sales return order credit note value is less than the original Sales Invoice value, then the credit note will be fully set-off with the Sales Invoices and the remaining balance will be shown in the Sales Invoice.

This could avoid the finance department from spending time on assigning Credit notes to Sales invoices where the Sales return order process can define the original Sales invoice number.

I could simulate the same and hence thought of sharing the same with all Baan consultants in Baan ERP LN 6.1 (FP5).

Hope this information helps.
