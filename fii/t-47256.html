Hi there,

You should ask support for the latest porting set. For ranges on strings, the system will use for the maximum on the to-value always something like set.fmax(maximum.value, domainof(maximum.value)). This maximum value depends on the domain of the variable and also on the LOCALE the user is using. This LOCALE determines the ASCII set which is used. In a UNICODE environment the user can set a restriction that no high ASCII values are allowed. However in the runtime tools the function set.fmax() was not changed accordingly, this means that the maximum value is for example the y with dots, but while inserting in the database this value is not allowed and the 205 error is thrown. The portingset has been adjusted for this, so this means that you are using an old porting set, which needs an upgrade.
