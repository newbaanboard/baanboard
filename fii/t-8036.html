Hello, we had a similar problem, not caused by tfcor0111, but with a "hanged reconfigure table". The only way was to recover from previous backup, one with tfgld106 still in good shape, and :

1 - Full Database Backup of Baan

2 - Restore Backup From Tape

3 - Dump tfgld106 content ( use | as field Separator )

4 - Restore Backup of step1

5 - Delete Table tfgld106

6 - Re-Create Table tfgld106

7 - Create Table from sequencial File ( one of step 3 )

Important.

A - Since tfgld106 is the table for all finalized financial transactions we must be shure that no transactions were finalized since the "good backup", or else you would have to do the same with table tfgld102.

B - If we do not have finalized trans. since your good backup, you should be fine.

C - Finaly must recover ledger/dimension from transactions ( tfgld3203m000)

Good Luck
