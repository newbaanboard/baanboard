Hello,
for me it seems to be a question "who has manipulated the database" and not which Baan-User did something.
Of course, a user ist able to delete data via using the application, even accounts can be deleted as long as they were not used in the past (no history, no references).
But the field "Financial Supplier Group" in supplier data is a mandatory field, the content can not be deleted by a user in the Baan-application (a user might be able to change it which causes enough problems :-), but not to delete the content). So, whoever deleted the content of the field was able to disregard checks of references in the Baan-application.
I would ask the colleagues in the IT-department, especially in the team managing the database. ttaad is possible as well, but for that the related fields/ records seem to be too selective.
