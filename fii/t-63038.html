Hi, I know this is kind of crazy but is it possible to take two invoices and combine them?
WE have one sales order but added a line after the first line of the sales order had been invoiced
(e.g. on inv# 12345) and now we realise that there should have been a second line on that order, so it was added but of course it leads to a new invoice#. (e.g. on inv# 12346) Can I change the tables to get both lines on one invoice after the two invoices have been printed?

if the environment is baanIV then i guess it may be possible.
If the invoice is only printed and PDSO is not done. May be you can Check tables tdsls41, tdsls045 & PDSO tables. Make sure the invoice date and times match for both order lines. Change the document status of 2nd invoice to deleted. Try this in a test environment first.

Most importantly, what is the status of these two invoices now?

If Printed : baanravi has already advised
If Posted : you ll need to make changes to a lot of tables like tfgld106, tfacr200 etc and then run Rebuild History in GL, Rebuild Opening Balance for Customer, etc

Better would be, to create a return sales order, generate a Credit Note to nullify the existing two invoices, and then later create correct sales order lines and invoice to get a single Invoice.. :)
