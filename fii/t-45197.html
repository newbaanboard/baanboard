Hi there,

I do not really understand what you mean with the phrase "The problem is when we checked our g/l all the transaction is as per dummy mapping".
So you had a "dummy mapping scheme", but you still used that one to map and post your integration transactions? I am sorry to say, but that is of course a bit stupid.
If you did not post them, then you can of course easily remap them with your new active "correct" mapping scheme.
If you did post them already then you can use the session "Remap Posted Integration transactions". With this session you can revert the already created integration transactions and can map and post again, see the help (it is session tfgld4282m100). I am not sure in which feature packs of LN this session is available by the way.
