I've received a at least a couple of private messages in the past seeking info on SEPA. I've chosen to respond as a post rather to maintain the spirit of the forum and also because any ideas exchanged here could you useful for any other users interested in SEPA.:):)

These are the steps I followed to implement SEPA
1. We are on Baan 4c4 SP27. I had to still install loads of patches to get things working. Some patches were already available. A few were bugfixes I obtained during implementation.
2. Generated a default XML layout. Modified the layout as per bank's requirement.
3. I've infact extended the layout to cover some of our non-EURO payments as well.
4. Got our finance maintain all our & supplier bank account details.
5. Setup a new bank/payment method and specify the path to export the XML file.

There is a SEPA implementation document available on Xtreme which I found quite useful.

I'll be more than happy to answer any further queries on this.

Hi, since we implemented SEPA there have been many bug fixes a complete list can found in the excel attached with the KB 1180901 on Inforxtreme. Below is a list of PMC solutions, I suggest to cross check with the list on inforxtreme. Once these are installed you then
1. Use session tfcmg0524m000 to setup XML file layout. You can a default XML layout by selecting from the menu
2. Maintain the default XML structure to suit your requirement. You can select/deselect, make tags mandatory/optional, and change "Fixed Text" or link with required baan fields. Your bank can provide you with a XML structure they use and you then try to match with that structure.
3. Setup a payment method (tfcmg0140m000) with XML file = 'Yes'. Provide XML file path in tfcmg0145m000.
4. Test by generating XML file before moving to processing a real payment.
