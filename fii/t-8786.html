Hello,

You can enable a new dimension, but you got to note couple of points here:

1. Your existing dimension history will still remain same.
2. However, I beleive there are some correction sessions which you can get from Baan which would even correct the existing history.

More importantly, this change will be affected for the group company to which the financial company belongs to.

If you have multiple group companies, then this change will NOT reflect in other group companies. Because, regarding multi-site implementation, the only rule of the game is with in the group company, all the financial companies should have same number of dimensions.

Hi,

It is not enough if you just maintain new dimensions. You have to specify the usage of the same, in chart of accounts also. So far you were not using the 4th & 5th dimensions. Hence, the usage of the same must have been specified as "not used" in all accounts, I suppose. Now, if you want to open the new dimensions for use, you have to set the flag for these dimensions to either "optional" or "mandatory" in the respective chart of accounts.

You can set it to "mandatory" for a ledger account only if no history presents for that account. Otherwise, the system does not allow you to do that, because the earlier transaction would have already been recorded without this new dimension. In that case, the only option available to you is to determine the 4th & 5th dimensions for each of the history transactions in the respective ledger accounts, and input the dimensions for those transactions through GTM (may be, some baan program must be available for this... check out with BaaN.)

However, if u want to set it "optional", you can go ahead and do it without any hassles.

This new setting does not affect the other groups. But within the same group, it depends whether you have shared the chart of accounts and dimension tables across the companies or not. If you have shared the same, this will affect all the companies within the same group. If you have not shared, it will not.

You can check the sharing status of these tables tfgld008 & tfgld010 in session "Display logical tables".

Hope this will help you
All the best
