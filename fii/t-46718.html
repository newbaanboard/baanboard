Hi there,

Must be a problem in the query which fills the session. The system works with doubles, so floating points and depending the database and operating system, it is possible that even if the value should be 0.0, you can still have very small values in the database. If the query is using then something like, tfacp200/tfacr200.balh <> 0.0, then also the invoices which have actually no balance, but such a very small value are shown in the grid. The query should use a tolerance. I guess you need to contact support for this.
