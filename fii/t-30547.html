Your best bet is to change check to see if its already attached to a finalization run number buy looking at tfgld100. If it has a finalization run number, you need to go back to the session "finalize transactions" and look for that run number, enter it and then finalize.

Chances are that the batch is there in a "Yes" Status which is causing it to be hung up.

Simple changing the status to deleted may cause a error in the month end close becuase the finalization run number will still be open.
