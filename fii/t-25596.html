manuf. integration is quite complex question...
backwards, your points:

3. financial value of transactions based and splitted in cost price components.
First of all, you should check CPR module, sessions of "Maintain Cost Price Componens", "Mt. Operation Rates" and "Mt. Cost Price Calc. Codes" (menu: Cost Accounting / CP Master Data)
You can learn basics via these master data.

Cost price component maybe the most important base: this object defines the type of the cost, e.g. operation or material cost component.

2. HRA module is always implemented with manufacturing modules so you should set up HRA master tables as well - although later you can run SFC module without using functionality of HRA. You can report hours from SFC module, with backflushing method.
Be careful how BAAN calculates actual operation costs sent to finance:
Rates for reported operation costs (machine and labor) are taken from CPR module or from HRA, depending on HRA param. "Actual Operation Rates".
So, BAAN will calculate with standard rates defined in CPR or with Employee/Machine rates, defined in employee master and machine master.

1. Do you mean transaction types?
Operation Cost transaction is booked when operation (hours) is reported.
Completion (Receipt) booked when you report prod. order completed (inventory receipt of finished goods). Each transaction can contain Operation cost component.
