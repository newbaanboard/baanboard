Table Timestamp Definitions (ttadv4136m000)
Use this session to define timestamps for LN tables.

A timestamp is an additional column in a table that stores the date and time of the last change for each record.

Timestamps are used, for example, by the extraction logic of the CPM Enterprise Analytics. The timestamps enable CPM to extract data incrementally. For example: based on the timestamps in a table CPM can perform a weekly extraction of records that were changed during the last week.

To create timestamps:

Enter the tables, for which you want to define the timestamps, in this session.
Run the Convert to Runtime Data Dictionary (ttadv5215m000) session to convert the table definitions to the runtime data dictionary, and to reconfigure the tables.
Log off and log on again.
