We only ever run any of the finance repair options on a needs to basis. Immediately after implementation we were running some on these sessions quite frequently due to various issues (normally user errors or bugs). Since we have managed to make our system more stable we hardly ever run any of these sessions.

I suppose another factor to be taken into consideration here is the size of your data and the number of companies you are running. I also notice you are running Baan V which may for whatever reason require these sessions to be run more freqeuntly.

Atul,

We run the session only when we have to, i.e. when we had to make changes to table tfgld106 or when a finalisation run failed and has caused problems with the trial balance figures. The only 'regular' run is before year end. Running it does take a couple of hours, we schedule it as a batch job to be run overnight, so it's not really an issue.
I found that in Baan V there is a lot less need to run it since there are less problems with finalisation runs failing.
