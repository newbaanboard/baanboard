Hi,
I have seen this when there is an automatic process involved where it afects the table tfgld102.
Normally this is due to a lack of a lock on the record on the table. So when another process comes and try to write on the table, a duplicate error is shown.

Since the problem is on the gld102, there is no big deal in order to solve it. I recommend you to put a case on Infor in order to review it.

Another approach is to review what is the last number used by series of documents and try to match with the wrong document number in gld102.

This error shows when the db driver fails to update the table tfgld017. This is most uncommon error due to, if there is a non finalized transaction, the db must do a roll back instead of a commit.

Hope this helps.
