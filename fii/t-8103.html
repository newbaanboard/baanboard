Hi,

Integration Options or elements have nothing to do with capturing the dimensions for AR. It is maintained in the session Maintain Ledger Accounts by Customer Group, which is available as a zoom from the session Maintain Financial Customer Group (tfacr0110m000).

Here you can map dimension for the particular AR Control A/c, provided the dimension usage is set to Mandatory or Optional in the session Maintain chart of account (tfgld0108m000). If the same is set to "Not used", you can not maintain dimension in for that account in tfacr0110m000.

Hence, please check the dimension status for your AR account in session tfgld0108m000 first, and set it "Mandatory". Then come to tfacr0110m000 and specify the dimension to the account.

Note: If the dimension status is not already set to "Mandatory" and some transactions are already recorded in that ledger account without dimension, you will not be able to change the status to "Mandatory". Then the only option is to make it "Mandatory" through a GTM correction in the table tfgld008. But this gives raise to problems at the year closure. So if you decide to resort to this, better take a professional help before proceeding further.

The other, safest way is, to create a new account itself, with dimension set to "mandatory" and map that new account in tfacr0110m000.

Hope this helps you.
