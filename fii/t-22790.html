Have you used print session tfgld1401m000 "print non-finalised transactions" this will tell you what is meant to be unposted. If there is a genuine unposted transaction it will tell what it is and you can deal with it through the normal interfaces.

If it returns the message "no data found" then you may need to look at other methods as described previously by other replies.
