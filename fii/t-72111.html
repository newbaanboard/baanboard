The problem is solved.

Solution description:

* credit line was missing in two tables tfgld495- reconciliation data and tfgld102.
* Generally when we post an invoice from invoicing module it generates revenue analysis entry in integration transactions(tfgld482), at the same time it generate a reconciliation reference line in tfgld495 which was not generated.

* To insert the reconciliation unique identifier, I have done a simulation in test system it's generated a unique identifier.
With the same unique identifier I have inserted a line in tfgld495 tables in production for which the invoice reference also linked manually.

* in tfgld102 also I have inserted one credit line and for which unique identifier link entered manually(the same unique identifier which is mentioned in tfgld495)

* the bath was able to finalize

* after batch is finalized the reconciliation performed for interim revenue 5 there was no difference found.

It's happened for only one record therefore the correction done manually. If there is bulk transactions it's always advisable to approach infor for correction program.
