if the dimension is only for use in one comany then if the companies are sharing the chart of accounts then you shouldn't make the dimension mandatory on any of the accounts.
If the companies are not sharing the chart of accounts then the only problems you might have is if finance want to make the new dimension mandatory on existing accounts but this is not possible because of history in tables like tfgld202 so unless you resort to GTM the new dimension can only be made mandatory on new accounts. There is a way round it using parent child accounts which as been posted on the board before.

Also when we wanted to add a third dimension after the initial setup I couldn't set the third dimension to yes in the maintain group comany parameter session I had to use gtm to do it
