Dear Frank,

I think you should use dimensions (tfgld0110m000) to solve your problem, and relate them to the accounts of investment. The advantage is that the amounts on the dimensions is booked with every invoice booked, and you can in parallel maintain the budget for the same dimensions in the fbs-Module. Then you have a simple comparison of both figures, for example in tfgld3501m000 and tfgld3520m000, and tffbs1502m000. Additionally you can use the dimension in the FST-Module.

If you use in IVc4 the controlling Package, you can even maintain a BAB with them. Unfortunately you didn't write in your profile which Baan Versio n you are on.

If you don't know how to set up the dimensions, don't hesitate to ask for more information.

Dear Frank,

this message want to explain that in the past/ history this account was already booked without dimension, so you cannot set it on mandatory, because that would mean that there are always (also in history) is a given dimension.
You have two possibilities now:
1. you accept to set it only on optional, and give strict advice to the booking personal, that they should always give a dimension (e.g. cost center); if the integration scheme is set up properly those bookings shouldn't be a problem. If sometimes they forget, you can book correct this afterwards with a journal voucher, but that means double work.
2. you block the old account and create a new one (e.g. only on adding a zero more) which you can set now on mandatory. With the sublevel you can sum up the old and the new accounts.

Hope that helps.
Bye, Juli

create a code using maintain business information codes example 001 give it a description "Machine serial no"

use maintain business information types create one for each fixed asset group that you want to use the serial no code with. You an set mandatory optional or not used like dimensions (set mandatory doesn't seem to work on our system IVc4 sch1 (sp9).

You can insert the records for old assets using use maintain business information details. for new assets should, I believe activate the session while inserting the new asset.
