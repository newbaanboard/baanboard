Hi,

If you accept, finalize and archive these orders, they can be eliminated. But the global acceptance session does not have any 'tolerance' and hence it will accept everything, if you select 'debit and credit balanced' =No. In that case, you can use the correction session to make the balance zero, where you can give tolerance. Remember, the correction session creates entries in GL.

Also, did you run the session print invoice specs-financial, with full specification = 'yes'? If so, at the end of the report you will get the difference between transactions and ledger history. It must be zero.

As far as GRINYA module is concerned, everything is done at the interim table tfacp135. No entries are created in the GL(not tfcor1240m000). Archiving just deletes the records from the interim table and puts a flag in table tcmcs095 with the year and period. Once you archive, you cannot select the records before the archived period. But at any point of time you can redo the GRINYA reconciliation by resetting the archive period in table tcmcs095 and start reconciliation from the beginning. So, experimenting is not a problem.
