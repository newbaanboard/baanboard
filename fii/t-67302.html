Hi,

You can try session 'Print Integration Transactions' (tfgld4482m000). Select the option 'Select by Integration Document Type' and input 10030111 as Integration Document Type. Input transaction date as 28/12/2014. Select report 'Print Financial Integration Transactions'. The report should give you details of integration transactions for the input range with their current status. Check which transactions are not of status 'posted'. Based on their current status, they need to be either mapped or posted. If they are in error then error needs to be fixed before they can be posted.
