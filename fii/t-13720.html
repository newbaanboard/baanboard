From 2.* to 4.* this session never kept a log anywhere in the system. All it does is rebuild ledger history tfgld201-206 tables from transaction tables tfgld102/106 as per the transaction type update mode for the period selected by user. Closed periods are not allowed .

So if the process is aborted for some reason , you can always restart the same.
