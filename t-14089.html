My afs session is running in a job.
Can I choose where the afs.log is created ?On Unix you have the rc.startjob script where you can pass bshell variables (to turn afs log on for a job). When the file cannot be written to your ${HOME} dir it will be placed in the ${BSE_TMP} dir. But use logging of an AFS session (in a job) only for debug purposes, so rather in a copy of the rc.startjob script than the operational one.
