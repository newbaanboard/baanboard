Debugging is easy, you really should have a look at it.

Code: I often make some code to get quickly to what I want to see, e.g "before.program: e = 0". That starts the debugger as soon as the session starts; otherwise the debugger will start for the first line of your code, and if you have a type 1/2/3 script with only little code, it might take some input before the debugger will be started.

Code: In your case, I would recommend that you always use commit.transaction after each db.insert or db.update. That makes debugging easier.

Debugger: In the debugger, you can search with "/db.insert" (and may be repeated by Return), an you can set a breakpoint on that line with "b". When you press the Continue button (or enter c) to run, the session will stop at every breakpoint, and then you can check values (enter their name in line 2; be careful for vars that are identical to debugger commands - just add a shlash to them).

Code: Another simple way is to add a variable e.g. "extern long foo" too your code. Add lines like "foo = foo + 1" to the interesting part. Then you can use the debugger command "t foo" to trace the variable. The debugger will stop each time the value changes.

Hope that gives you some appetite to give it a try!
