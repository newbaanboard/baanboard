Normally good redundancy is required for baan and db . We have Raid 1 configured on separately for each of following for best performance and data recoverability and online backups .

1. OS & cluster
2. Baan
3. SQL log file and application
4. SQL dbf files

Such configuration is not possible with 3 disks only . Configuring Raid 5 of 3 disks and keeping db log file , baan temps and sql dbf in same partition is not recommended for best performance and data recoverability purposes.

14th December 2010, 17:31
You should put baan and the OS on the first disk.

Then mirror (RAID 1) the 2nd and 3rd disks and use that for your database.

Database + RAID 5 = poor performance.

Your database is small enough that you can easily back it up nightly, and backup the tx log 2x per day. so you don't really need the fault tolerance of raid 5. The parity calculation for inserts kind of trumps any performance gain you'd have going across so few spindals in a raid 5.
