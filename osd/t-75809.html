1. You will need a portingset that supports the combination of RHEL 8/Oracle 19c. That will likely require you to use SLM for licensing (if you're not already). If you're not using SLM, there are usually a few solutions that need to be installed to support that (depending on if you've kept solutions up to date, which isn't likely based on my experience in Baan IV environments). You will also need to look at any integrated/3rd party products and make sure they're portable to the new plaftform.

2. Yes, this should be able to run on VM.

3. Need to make sure you have the expertise for the database upgrade. Any Unix scripts you have written should be validated, although since you're staying on RHEL, I wouldn't expect issues. Make sure you address migrating devices and jobs/cron.

4. Execute full-cycle testing - not just technical testing. Make sure all processes work and there are no unexpected results.

Hope that helps!
