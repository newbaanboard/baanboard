Could the db instance name be different?
If you installed SQL Server on the new server it could happen as the install process asks for an instance name which may not exist on the previous installation on the old server (e.g. if you had upgraded on the old server).

If it is different, your tabledef6.1 would need to be edited to look something like this:

tccom001:200:msql7(MSQL_LEVEL1=1,MSQL_SERVERHOST=192.168.10.2\BAAN1):Y

where "Baan1" is the instance name.
