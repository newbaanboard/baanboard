Probably a mismatch in the UserIDs of your database user(s).
Assuming that the DB user is "baan":

Open SQL Management studio
There are two logins for "baan": one is under "security" of SQL server and one is under "security" of your Baan database.
After the restore the user IDs differ because one is from the installation of c4 and the other from the backup.
You have to delete the user from the Baan database and then assign him the database again by checking the appropriate checkbox in the SQL servers security. Aftter you checked that box you see his login under the Baan database again and the error should be gone.
Make sure that the users have the same password (the installation user and the user from the backup).

Good luck.
