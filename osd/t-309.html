Here goes something I just got from SSA Global - BaaN support:

For Baan IVc ( and your current PS 6.1c.07.04)
======================================

Supported: SLES 8
Oracle database: Oracle 9.2, 10g
Hardware: minimal Pentium-III

Usage of this portingset requires a license key. Solution 142799 provides a correction program to add the related commercial function.

In case an existing Baan IV environment is moved from another OS to Linux: Be sure to install solution 142799 and run the correction program before moving the BSE environment as installation afterwards is not possible due to the required license-key.

For more information about Linux and Baan IVc, please read the following document:

ftp://ftp.support.baan.com/updates/port/6.1c.07.04/U8474A_US_SSA_BaanIV-Triton3-Technical_Notes_for_portingset_6_1c_07_04.pdf


If you want to go to the latest Portingset (6.1c.07.07), please read the release notes for the PS.

ftp://ftp.support.baan.com/updates/port/6.1c.07.07/Technical_notes_portingset_61c0707.PDF


For BaanERP Ln ( and the latest PS available now: 7.6a.02)
===============================================

Supported OS version: SLES 8, 9
Oracle Version: Oracle 9.2, 10g

Oracle
Suse SLES 8 with SP3 or SuSe SLES 9 is recommended.


For information about Linux on BaanERP Ln, please read the following document:
ftp://ftp.support.baan.com/updates/port/7.6a.02/Technical_notes_for_portingset_76a02.PDF

If you have any questions, contact SSA Global.
