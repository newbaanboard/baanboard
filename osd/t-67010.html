Yes, definitely some work involved in mapping the tables from Oracle to SQL using SSIS. From a resource perspective, it is actually quite a bit faster than bdbpre/post. I think we did a 750 GB database in 16 hours or so with SSIS whereas it took 30+ hours using bdbpre/post.
