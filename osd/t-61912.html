By the way, the issue might not be due to the virtualization but due to the fact that you moved the database to a seperate server. There is a major difference on running queries (or other that translate into a lot of traffic between client and DB) locally or thru network. You might need to pump upp the network?

On VSphere this is also very noticeable: 2 servers (one app. frontend and one DB) with vmxnet3 (that defines 10Gb interfaces), 2 same physical ESX hosts connected to 1Gb LAN. Running the servers on different ESX the traffic between then goes thru the 1Gb real network, but when on same ESX, there is no LAN traffic, all runs within the same ESX host, and here uses 10Gb: 10x faster!
