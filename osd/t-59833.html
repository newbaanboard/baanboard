Hi,

I think it's not that Infor wants to discourage you but it's more a matter of what the market requests. Personally I don't see a lot of informix sites anymore although it's a good RDBMS.

If you just use the plain rdbms you can use oracle or sql-server as well. However you probably have to sell new licences which can be costly.

The portingset 'hides' the OS for the application so the move to linux or another OS can be done pretty easily. Especially if you have used relative paths for you package combination, directory path for the components, pmc setup etc. It's always wise to use the ${BSE} variable!

Furthermore find out if you have used paths in your own software.
Next thing that can be an issue are custom interfaces.

If you are not familiar with linux than make sure you have someone at hand to assist, it's a complete different OS. You probably need to configure the kernel before being able to run a RDBMS.
