Hi orma01,

I´m not a guru, but we have done nearly the same installation last year.
We have migrated from Unixware7, Baan 4c4 and tbase to SuSE ES8 and oracle 9i. Hardware: HP Proliant ML 570, 8GB ram, 2 Xeon 2,8 GHz Processors. We have 175 concurrent Baan users.

If you follow the installation instruction from SuSE and Baan, you shuoldn´t run into trouble.

Here is a link for installations documents for oracle on SuSE -> http://www.novell.com/products/linuxenterpriseserver/oracle/documents.html

For Baan, please refer to Document U7028H US

We are very satisfied with this environment, we don´t have performance problems and the system is very stable. One big issue is of course is to set up oracle properly, but this is not a linux problem.

We had only one problem with rexec protocol, used by the BW clients. We got a message "Where Are You", becuase in SuSE ES8 they have changed the rexec package, I don´t know how it is in other versions. There are some possible solutions to fix this problem. Either install an older rexec package, or you follow the instructions from this link ->http://www.starnet.com/KB/error/where_are_you.htm

Hope this helps
