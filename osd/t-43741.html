There is no baan/unix driver for SQL Server. so that will not work.

If you trust your DB to windows then you might as well trust baan to it as well, since the DB does most of the work.

What most folks would do is Baan on Windows and DB on a Unix box.

Support confirmed to me what dave_23 already said:
For SQL-Server the problem is that there is no Unix-porting set that has a driver for this database.

A theoretical option would be to create a second Baan application server under Windows that communicates with the database, however, for performance reasons this is no option.
