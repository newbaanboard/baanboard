I faced something like that before wiz a customer... BaanIVc4 with d481 extension.

Look..
(( GLOBAL VARIABLES, INSTANCE NAME, SERVER NAME, OPERATING SYSTEM, BSE...etc SHOULD BE SAME ))
1- Start the Installation on the new server
2- Compress the BSE and DB from the live server
3- move the files to the new server
4- use SHUTDOWN IMMEDIATE to shut down the DB Instance for the new server
5- Stop the Logic, Shared memory, SLMServer and ASM services.
6- Now decompress the folders on the same locations
7- start the DB and the services
(( The Application should be the same typical as the Live one )) tested

hello,

I have done this a number of times.

1. change all the paths in tools to ${BSE}/application etc
and then convert to runtime.

2. Create the user baan on the new system.

3. copy ${BSE} from disc to new machine.

4. export import database including company 0

5. Install the binaries again even on windows.
This will install the SSA/Infor manager
Do NOT install the BW first as this will set the BSE wrong in the regestry.

6. Check the lib/ora files for user & group passwords and db parameters + tabledef
In the database you need not only dbuser, dbgroup but also roles r_dbuser, r_dbgroup
The dbuser does the data changes so needs insert, update delete etc. The dbgroup does the object changes so needs create table, delete table etc. You can look on the live machine to see what rights these users and roles have.

7. Create the other users on the new machine.

This also works for a copy from windows to linux etc
