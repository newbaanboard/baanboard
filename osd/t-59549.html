Believe u were on Ln . Am i right.

For ur query .
1. First install SQL 2k8 on win2k8 . Install baan on the same machine .
2. Get level 2 driver validated with new valiation key along with MSQL driver (not sure driver requirement for SQL2k8) . Ensure the database definitions refer to level 2 driver (check tabledef6.1 MSQL_LEVEL = 0).
3. Ensure nobody is working on baan .
4. Update porting set to ur target porting set in new server.
5. Export all tables including 000 company tools table in existing tables in old server.
6. transfer all tables to new server.
7. Import table for 000 company deleting existing company using bdbpost (not import from session).
8. Import non-0 ccompanies deleting existing tables and ignoring referential integrity & then rebuild ref integrity.
9. Copy application & dictionary components in package combinations / pvrc folders to new machine.
10. Copy fd / user / compnr6.1 etc in lib folder to target folders.
11. Except baan user , other users will have to be configured using ttdba0110/0115/0120 etc.


Please do some tests if u do it urself or else getting an expert hand to do this is best.
