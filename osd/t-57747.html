I am currently running BaaN V on Windows 2003 server (32 bit) with my Oracle database running on a separate 2003 server (64 bit).

I went to a split environment because of performance. The BaaN Application is tuned differently than the Oracle database. The Application server has 16 Gbytes of physical RAM while the Database server has 32 Gbytes of physical RAM. My User group is over 400 users with about 1000 concurrent sessions.

One of the best ways to handle memory (IMHO) is to make sure that your C:\ pagefile should be set to about 1 Gbyte and your main pagefile(s) should be on another platter group assigned to a different logical mount point. I currently have pagefiles of 64 Gbytes on 2 other logical drives with C:\ static setting of 1 Gbyte.

I use the logical E:\ for my BaaN and have 2 main pagefiles on e:\ and d:\ (this drive is basically for all non-BaaN Application scripts, jobs, server maintenance, etc.).

Hope this helps a bit.
