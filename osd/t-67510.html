I would opt for Oracle 12c, it will not long before infor will not support 11Gr2 anymore. See latest 9.0a.01 platform support matrix.
You have 2 options: export all you data with baan tools, build a new oracle database and import it again; or upgrade your oracle environment.

Latest is probably the fastest however you need to have oracle knowledge how to. Oracle can deliver you all the documentation you need.

For both options I would recommend test the end result. Especially the performance both interactive sessions and your jobs. After all issues are solved perform the final migration.

Hi ?,

You keep baan on the same server? Because without a support agreement you don't get a new license wich you need if you use a new server.
If you have enough space available on your server than I would keep the datafiles of the old database save in case something goes wrong.
If you keep using the same instance the only thing you need to do is to change the tabledef6.1 file. There the connection to your database is defined.

However.....
The portingset you currently use, does that one support 11Gr2? Not sure if that will work with your version. You need to check this.

Another tricky thing since 11Gr2: oracle doesn't deliver 32 bits client libraries with that. So using this with a 32 bit portingset can only being done via a separate 32 client version installed in another ORACLE_HOME.

IMHO I think the best you can do right now is to discuss this with someone that has done this before. It's not rocket science but you don't want to end up with a not working system assuming it's being still used in production.

Baan IV supported with Oracle 11i or 11g

Description:
Oracle is talking about ending support for 10g in a 2011 or 2012, so it will be necessary to upgrade to at least
Oracle 11g.
Is there a specific SP or portingset that is required for this?
Resolution:
According to the technical notes of the porting sets, it would necessary to upgrade the porting set to at least
6.1c07.21 in order to upgrade to Oracle 11g. In other words, 6.1c.07.21 is the first porting set that supports
Oracle 11g.

Check the release notes: ftp://ftp.support.baan.com/updates/port/6.1c.07.21/Technical_Notes_PortingSet_61c0721.pdf This only mentions 11.1, not 11.2. Perhaps it will work but I would recommend to use only the combinations that infor supports according to their technical notes.

From the technical notes of 6.1c.19:
Oracle 11.2 support on 32-bit porting set
Starting with Oracle 11.2, the 64-bit Oracle software does no longer contain a 32-bit client interface.
Customers must install separate 32-bit Oracle client software. Refer to the Oracle installation guides
for more details.
An Oracle Database Net service must be configured in such a way that the 32-bit Oracle client
installation can access the Oracle database.
The setup of the Infor Baan IVc4 Oracle driver must be based on these resources:
• ORACLE_HOME (of the 32-bit Oracle client installation)
• ORACLE_SERVICE_NAME (the Oracle Database Net service name of the database).
A connection between a 32-bit porting set and an Oracle 11.2 database based on ORACLE_HOME
and ORACLE_SID is not possible
