The things that I've run into in moving to the recent OS/DB versions for a Baan IV environment have to do with the porting set changes. The old licensing method is not supported past a certain porting set version (which doesn't support 2008 and above, I believe) so you will have to move to SLM if you are not already. Also, if you use DDC for anything, it is not working in 8.9a.02 and above, I believe. Other than that, not a difficult process. For SQL Server, you can't migrate directly from 2000 to 2012/2014. You have to use an interim 2005/2008 server to change the database version.

Thanks,
