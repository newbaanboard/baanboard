I'm not sure you can do an in place upgrade for Windows. But there are relatively easy ways to "move" a Windows installation to a new server. Basically, install a new portingset on the new Windows server, move the existing BSE to the new server (other than the portingset related directories), and make sure the tabledef6.2, etc., are pointed to the right database server.
