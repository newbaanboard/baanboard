What is the exact error message, is it something like "Where are you?".

Please check the ip adress range of the clients ... could be a dns problem, that some ip ranges can make a look up, some higher ip ranges are not able using other dns server or method, happend once for new installed pc's (replace of old ones or new employees).

We had this issue once, only solutions we found:

(1) enter the client ip address and computer name in the /etc/host file of the baan server (works, but not a realy good solution)
(2) use baan login instead of rexec, another advantage is that your passwords are now crypted, with rexec the passwords are uncrypted transferred over the network.

we also had this in the past when DNS had problems. Mainly with laptop users. On HP-UX you can disable the reverse name lookup with the -c option in /etc/services I believe. Maybe a similar option exists in your linux flavor.

Other solution is to have DNS records flushed more regularly. I believe the default is 7 days in standard microsoft DNS. We have now set this to 1 day so laptop users that travel are not so much affected.
If a client still has the issue you can also run the register DNS on the laptop so it updates on the fly.

Hope this helps
