17th November 2014, 17:14
The process we use to refresh our test companies could be adapted pretty easily to suit your requirements. Run the session count records by table (ttaad4222m000) to get a list of all tables that contain data. Then edit that list to include only the tables you want to keep. Save that list as a file and run bdpre from a command prompt specifying that file. You will get a dump of just the tables you want. Then delete all the tables in your test company. Then import the dump back into your test company and finally run create tables to create the remaining empty tables

When you said, "table you want to keep", does that mean ALL the non-empty tables or you select tables based on what you want?

When we refresh our test companies we take a dump of the prod companies. To make it take less time we only dump tables that contain data.
Actually what we did the first time was to split the list into multiple files based on the size of the tables and run the dumps in parallel.

For our situation we keep all the tables that have data.
For the OP's scenario I would delete all tables except the "static data" tables. Obviously you would need a pretty good idea of what references there are between tables to keep your referential integrity intact.
I have never done this so it was just a suggestion but I do not see any reason why it would not work.
