What I see as consultant in the Netherlands is that most customers choose SQL-server since you need that anyway to use ming.le which is a plugin in sharepoint.
So this can be a cost issue.

With oracle the optimizer sometimes decides to use another strategy to retrieve the data after the tables are analyzed.
Furthermore in Oracle 10/11 you need to install a infor 'code' page to be able to use unicode.

It's often also a political choice: if your company has a DBA with knowledge of one of these it' s probably more logic to use that RDBMS.

As far as I know Infor has no document to describe the differences. You can find, probably sponsered, information about the differences between Oracle and SQL-Server.
