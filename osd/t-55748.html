8th July 2009, 22:29
We've been working with Infor on a project that will upgrade us from 4c4 to LN using SQL Server 2008.

We discovered a major problem with SQL Server 2008. If you do a full install of SQL Server 2008, the system will become incompatible with the Infor Install programs for both 4c4 and LN. We could not even install a porting set under this situation.

We've were unable to repair the system and as a result, we had to rebuild the machine from scratch.

Therefore, until a fix is made, when you install SQL Server 2008 only install:

Database Engine services
Client Tools Backwards Compatibility
Management Tools

With only these parts installed, we were able to do a complete install of ERP LN.
