Rajendar,

There are always different opinions on this issue from BaaN support side. If you cannot avoid playing with live environment and need to do it, you can change domain of your live server.

Although, you should think and ready for worse case. If something fails, you will have to re-validate your software, so need to have last back up and if it fails you have 30 days to validate it. BaaN validation broadly divided into two categories:

1. Soft crash (Steps available here - http://www.baanboard.com/baanboard/showthread.php?t=1555&highlight=back-up )
2. Hard crash (Steps available here - http://www.baanboard.com/baanboard/showthread.php?t=23152&highlight=crash )

Since you are not changing any hardware, you have to perform soft crash in worse case.

Hope this helps.

Or to quote somebody who's name I forgot: "You can't plan for success, but you can plan for absence of faillure."

The only consequence that I can foresee is having to point all your clients to the new domain.
Make sure to check all your server scripts for hard-coded (bad!) references to the current domain, and finally make sure that any and all OLE functionality is pointing in the right direction.

There should be no consequences to the Baan system itself.
