I never come across any list of services because I think that Baan only uses what NT has availble for it, like, if you do not have "Distributed Link tracking Client" you will not be able to access NTFS filesystems on other systems, but this is transparent to Baan.

The only one I can think, that Baan relies on, is the "Event Log" service, since Baan relies on this one to log its messages.

All the others may be or not used (you can discard a lot of the network services because you can have Baan on a local workstation, not connected to a network; discard the Print Spooler if not using printers, etc.).
As long as the minimum OS services are running (plus the database ones), you should be able to install and work with Baan.
