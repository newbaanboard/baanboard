Usually the Baan installation CD's are getting "frozen in", when a realease is done. In this case the installation CD is not aware of the latest version supported by Baan.

To avoid this issue, I tend to install Baan on a definitely supported database(on unix I choose tbase, on windows I choose SQL server), then I upgrade to the latest portingset and then I dump and load the data into my target database.

I know it's not nice, but it sure works.

Most likely there are other ways of doing it, like copying the installation CD to disk and replacing some files etc. But this wil need some trial and error.
