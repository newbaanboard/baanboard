We have just migrated from Informix to SQL Server and the performance is really great, an hour's run previously take only a few minutes. It has to be because we have 4 cpus and 6 Gb of RAM. where previously it was only 1,1.
However we experience slowness in the printing of reports when we send it directly to our dot-matrix printers. This does not happen when we were using the Unix Server.
To get around this problem, we would output the report in a file and send it to the printer thru the 'copy' command in command prompt. This way we would get a faster print from the printer.
Hope you gurus out there can help us avoid this step.
