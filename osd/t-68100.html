Hi people:
We have migrated (yes, right now 2015) from an HP-UX environment where Baan IVc4 was running with Oracle 9i in the machine to a distributed environmente over Linux SLE9 (VMware virtual machines).
So, we are experiencing performance problems with any task that can take long over 1 minute. As we thought, the new environment should give us an improvement of performance but just the opposite is what we've got.
So tasks that were done in 10 minutes now delay for 30 or more minutes.
We have deploy the system into two VM's:
- One for Baan Application, 24GbRam, 18cpus, 4NICs
- One for Oracle DDBB: 72GbRam, 24Cpus, 4NICs

So, have you any tip for the kernel, baan, Oracle or any other configuration than can help to improve the performance? I think that the main problem is that the procesess in Baan machine have a lot of delay waiting for something I don't know.

Please, can anybody give some light on this?

Hello again:

After hours and hours of tunning worked out by a group of 5 technicians I think that the key was in the configuration of the Oracle VM.
A the very first moment we reduce the number of cores of the VM from 24 to 6 if started to give us a very acceptable process times.
For sure we changed all that can be changed to improve performance in Baan and Oracle that maybe have helped but the main milestone was the number of VM cores in Oracle.
