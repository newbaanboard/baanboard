I do have to say though, using the OS time command isn't very consistent in determining baan performance.

you'd be better off doing TT_SQL_TRACE=02200 qptool6.1 <bla bla>
and parsing the logfile.

that will get you actual rount-trip query time instead of random overhead of starting up a qptool process. (which can fluxuate quite a bit)
