This is stored in, potentially, one of several files. Check $BSE\lib\tabledef6.1 (may be tabledef6.2, depending on your portingset) or $BSE\lib\defaults\db_resource. It should be one of those. Change all of the references from the prod SQL server to the test SQL server, then login. I usually log into the test SQL server at that point and verify that there is a corresponding process launched to verify Baan is connecting to that server.

Once you've done the above, you will also want to update the database definition in Maintain Database Definitions to point to the correct server (in Baan IV, for both your non-audit and audit definitions).
