Just to let people know. Do not try to install the Baan IVc master CD on SQL Server 7 once SQL Server 7 Service pack 4 has been installed.

It will give a version error because the sql server version has increased from 3 characters to 4 and Baans version checking code can't handle it!

Hope this stops anyone from falling into the same hole as me!

The problem has been fixed, but the problem is that it takes a lot of time before new media become available.

For BaanERP5.0 new media will become available very soon (based on porting set 7.1d.01).
For BaanIV it will take some more time.

In the mean time, use SQL Server 7 versions with a build number below 1000.
SQL Server 2000 is not affected by this problem.

I had reported this Bug in October 2001 to BGS.

There is another bug with this version of master cd.
If you try to install on Oracle database, and if you set the size of tablespace all to 2GB, the installation crashed with an Oracle Error (database busy).
The only way to install from this master cd with large tablespaces is to create first the database and then install ERP on the existing database.
