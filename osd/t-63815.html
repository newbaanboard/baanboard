you can add index on sql server table, LN will just ignore it. make sure there is no activity before you do this so you might have to reboot before you run your script. Ln will delete your index if you perform a re-org (i doubt you would). test everything first on a test server, including the performance of your sql query. if it doesn't do much, or if the performance improved for a little bit only, then it's not worth the effort. if you are using SSRS (you should since you are in SQL server) you can run a snapshot of the report early in the morning and the users will use it throughout the day. the downside is it's not realtime but something to think about for long-running queries.
