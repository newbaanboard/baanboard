How many clients do you have.

Is it realy the server or the network ... if it is slow for the users and you run a bshell directly on the server do you have the same effect.

Have you measured some explain plans of sessions running normal, compared to the explain plan of the same sessions when they run slow. You could also measure this with the baan logging getting an idear which part gives slow response.

We had some issues with an oracle based system ... it runs fine for weeks and then some standard session using item master e.g. runs extremely slow.

Reports run before 1 minute, than runs for hours.

Reason was a problem with the optimizer and the statistics. One solution was to update the statistics, helps in 3 of 4 cases ... but sometimes nothing helps. We than went back from cost based to rule based and the sessions runs with acceptable answer times.

Other effects i remember was hardware issues connecting patch wires to some plugs causing parts of the network to slow down.
