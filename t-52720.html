You didn't say if you are using Oracle or not. However, Baan uses chr(10) within each sequence of 240 characters to specify a Line Feed (Carrage Return). This means that even if you only have 1 sequence of 240 characters in the note, it can have many, many physical lines when placed on a report.

To this end, I got carried away with my SQL and created a command that does a good job of formatting the text so that it can be used in a report.

Sorry If you do not use Oracle. I wouldn't want the job of converting it to SQL Server.

select trim(both ' ' from (replace(replace(trim(trailing ';' from (rtrim(trim(leading ';' from replace(replace(t$text,chr(10),';'),';;','')),' '))),'; ',';'),';',';'))) from baandb.ttttxt010XXX where t$ctxt=523141
