are you calling the configuration file directly ie.

baan4.application."the name of the configuration file"

if you call baan4.application it will use the default configuration file. if it is using the default then both users are using the same clsid in the hkey_classes_root portion of the registry which maybe causing the problem


could try this, create a configuration file for each user call it the same as the username and then have the vb program connect to baan4.application.%username%
