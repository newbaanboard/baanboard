Hi,

Yes, this we already faced earlier and functionality of LN such that once you reject the piece through QMS it will generate the adjustment order and throw the inventory out of the system, so you need to create another adjustment order with the same serial number and bring it back to one warehouse (say named as Scrap warehouse) and issue the same piece for rework production order.
if you want to specify the rejected serial as a separate item code (say 'scrap-item001') then you need to maintain another item code and against the same item code you need to raise the adjustment order with the same serial code.
