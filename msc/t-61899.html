Hi,
There is a below mentioned work around on which you can think over:

Item Setup:
RING001 (End Item) - Inventory unit ‘pcs’ and Sales unit ‘pcs’
GOLD001 (Raw Material) - Inventory unit ‘gms’
GOLD002 (Scrap Material) - Inventory unit ‘gms’

Activity:
In Production Order, BOM will be like this:
GOLD001 (100 gms)
GOLD002 (-5 gms Negative Quantity, Suppose Scrap is 5 %)

Execution:
There will be two type of transaction against that Production Order
Issue: GOLD001
Receipt: RING001 as well as GOLD002

Can you let me know if it’s work fine, or any other complication in process?
