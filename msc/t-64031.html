Hi FrenkR,

There is a standard solution (session) to link a BOM to a routing.
The session is called tibom0140m000 (Material - Routing Relationships). This session can be started from the BOM-session using Specific and then the first choice.

Be aware that this session is only active if you leave the operation field in the BOM to empty(=0).

In the relationship session you link the material with the routing code and eventually with the operation is needed.

Hello,

A simple custom table would be with the following fields.

1. Parent Item
2. Routing Code
3. Position
4. Sequence
5. Child Item
6. Quantity

The first 4 fields will be the index fields.

Initially at the production order level, Estimated materials will be defaulted from the Main BOM based on the default routing code. But when we change the routing code on the header, the estimated materials will be updated from the custom table through an AFS.

Hope the information is useful to you.
