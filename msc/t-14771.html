Well, I'm not sure if this is a bug..
I guess someone changed the order policy from MRP to SIC. If there are MRP-orders present at this time, they might not be deleted during the next MRP run (because they are no longer an MRP-items!).

When we change MRP-items to SIC, we ensure that existing MRP orders are deleted manually.

I agree with just_fro that a regenerative run should solve the problem, but sometimes the Baan logic is too perfect ;)
