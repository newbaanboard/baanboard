Baan checks the following conditions:

1. All subprojects have the status Closed, if there is a main project.
2. All production orders for the project are closed.
3. All purchase orders have the status Closed.
4. All purchase receipts have been invoiced, and the invoice matching procedure carried out in the financial accounting.
5. All sales orders have been closed with the Process Delivered Sales Orders (tdsls4223m000) session.
6. All service orders for the project have been processed in history.
7. All transaction hours for the project have been processed.
8. No project inventory is still present. The project inventory value cannot exceed the value entered by the users. Remaining inventory less than this value is automatically deleted by BaanERP.

Did you either scrap or return to the stockroom any excess inventory? Check your project inventory reports and clear up any leftovers there and try it again.

Run report tipcs2402m000 Print Inventory Values of Projects to Close.
You can either deal with this inventory by returning it to stock or by invoicing it from the project.
Alternatively (depending on what internal controls you operate!) you can enter a value in the Close Projects session in the field described as "Delete Project Inventory up to Value". This will write any left over inventory off to cost of sales.
