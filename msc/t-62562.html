Let me try if I can make things clear.

With domains you can restrict e.g. a string field to contain only capitals, numeric e.g. only to contain the numbers 1 to 10. Also the format can be defined.

With reference tables you make sure that you can only have order lines if the order exists. Since baan doesn't use specific database logic (we support different ones), this kind of information is stored in the baan dictionary, not in the database.

Reference mode can be e.g. mandatory (so field value must exists in the reference table) or mandatory unless empty.

Akt. is indeed activated, the other column in front of it is Mandatory yes/no.

tdilc006: blocking reasons
tdsls092: not present on the server which I have available right now
tdsls093; same
tcmcs001: Units
tiitm001: Item data
tttxt001: Containts texts numbers as stored in your table, must be used with tttxt010
tdsls094: not available right here.

But it's simple :

Package Module 9999 M/S 000

tc mcs 0102 m 000

Means :

Package = Common
MODULE = Master Common Data (System Tables)

Sessioncode :

At position 2 the type of session is coded :
1 = Maintain
2 = Update
4 = Print
5 = Display

In position 1,3,4 the main table for the session is coded.
In example its table tcmcs002 = Currencys

M = Mainsession
S = Subsession

If the last the numbers are 000 it's normaly a standard from BaaN .
