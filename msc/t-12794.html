Hi,

This is a common problem for many in the past that are using LOC and LOT. Much of the problem is down to W/house procedures. Against the SFO is default locations where the materials are to be Backflushed from. If the materials are located in another w/house for instance then the materials can not be issued. As the previous e-mail suggested, take a look at the planned transactions, because that way you will identify the location issues.
Also be careful, because you can distort the actual Stock-on-hand by not consuming the materials actually used.

run generate inbound advice for all the production orders.
in session tdilc4203m000
( you may get No data within selection.. perfect.)
if u get data then run delete inbound tdilc4213m000 to delete the inbound generated)

Actually this problem is with tdilc402. Doing the above changes the recieved qnty 0 and shifts it to advised.

Now you should be able to close the production orders

Usually this problem will occur because Baan allows us to issue inventory out of reciept location. so the order steps are not fully completed.
