Martin Jung
20th May 2003, 14:52
Hi Old Vens,

using RPT-items or not should have no impact on the MRP-run. In our case we could reproduce the problem with RPT-items as well as with a standard item.
The Time Horizon can be changed in the MRP Parameters (timrp0100m000). The online help for this field contains the following hint:
.. you can adapt this to your company's needs, but note that: a too long value results in error messages from the MRP program. A too high value results in unnecessary memory usage..

Hope this helps a little bit more..

Martin
