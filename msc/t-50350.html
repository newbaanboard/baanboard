19th April 2008, 09:27
In the session Rebuild Planned Inventory Transaction {tdcor0250m000}, there is a field (somewhere in middle of Form1) by the name "Update Mode". If this field is set to "No" then it will only give you a report showing the tables with incorrect values and what their new values will be once they are fixed. This is the reason why i recommended you to use this session with the "Update Mode" set to "No". As long as you run this session with [Update Mode] set to "No", you dont have to worry about affecting any tables.

Another thing, do not forget to run only for the ItemCode that you want to fix as it might take some time for this session to give you an output...
