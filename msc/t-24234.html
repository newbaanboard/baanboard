9th June 2005, 15:32
Unfortunately, in Baan V, Baan eliminated the EDM parameter that allowed you to pick the application to use for viewing drawings, documents, etc. You are now at the mercy of the image.display function in library tiedmdll0020.

There are different ways to get around this. If you have source code, you can modify that library. If you don't, you can use one of the tools for modifying program scripts without source code (e.g., RMCgen, Qkey, rush). And there are others.

We've modified the library for our clients as part of overall production order document printing integrations. With these changes, the drawings and/or documents (AutoCAD, PDF, Word, etc.) associated with the finished good's revision print collated with the regular Baan production order documents.

I hope that helps. Good luck! :)
