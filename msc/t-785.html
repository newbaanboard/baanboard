30th January 2002, 18:27
1) Ensure that there are no active records in Baan for the items you are planning to change: such as Sales Orders, Purchase orders, Plan orders, etc.

2) You should expire the BOM for these items.

3) Then there should be no major issues with changing them from Manufactured to Purchased. Remember that you will have to complete the normal fields for a Purchased item, probably change the Item group and Product type and run a cost calculation update.

4) Change plan code from MPS to MRP or SIC.

The previous message mentioned to expire the BOM. This is correct or you also can delete the BOM.
Also you should remove the task from routing along with the routing code for the item. This will ensure proper clean up for future rollups.

Have you determined if this will be a permanent change. If not then you might consider creating a subcontract work center and task. Maintain the task in the subcontract module and maintain the routing with this change. You can also retain the original routing as an alternate route. This gives you the ability to flip back and forth if needed.

M.A.D.
