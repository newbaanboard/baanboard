A simplistiv view of what we do - is generate and print all available outbound each night for the users. The print outbound loads a custom table. This allows us to sort it based off the index the user wanted - so you could create your own index. The user can then release the whole display screen or individual items - it runs stpapi commands in the background to release each individual item. Of course ours is sorted by run number(and a bunch of other fields), but you could use whatever index works for you all. All you need is a way to group the print outbound data into a custom table. Just a thought.
