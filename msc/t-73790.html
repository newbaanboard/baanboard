Well if you are using 4c4 you can use tisfc0102m000 to re-open operations. Our time keeping system allowed us to make corrections, but we still needed to open operations backup when they were completed incorrectly for one reason or the other. We actually wrote a little API session that took the production order and operation and would go re-open it for the user, but we would not reopen backflushed operations. I noticed that in the code (but only one site used backflushing).
