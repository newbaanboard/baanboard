Have you created a routing for the manufactured part or parts?

It is not the cost type in item data that determines whether the operation costs shown. When you calculate the standard cost price, the operation costs are generated based on the routing.
The routing details the operations performed using tasks/work centres/machines. The work centre details the operation rates to be used for the operation(S) and the setup/production rate/run time of each operation on the routing determines what the operation rate will be.

Hope this resolves your problem

Master Data must be set up. This involves defining the item data, the logistic tables (such as warehouses and seasonal patterns), the supplier data, the routing data, and the cost price calculation in the ITM, MCS, COM, ROU and CPR modules. The debit and credit accounts must be defined in the General Ledger (GLD) module. The (purchase) order types must be defined in the Purchase Control (PUR) module.

Hope this gets you up an running,
