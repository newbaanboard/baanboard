Hi Halvorn,

probably there is a delivery line where there is a not-yet-confirmed back order quantity (tdsls045.bqua > 0). In that case, this delivery line get's the order status "to delete" after processing sales orders, however it's still not really at it's final state (because you still can confirm the back order and create a new delivery line for that order position)

Hope this helps
Uli
