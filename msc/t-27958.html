Hi Z,

The message you're receiving is actually an old message and has been changed by solution 109711. The correct text is "Operations are not (all) planned, check the resource storage horizon and the calendar availability of the item's resource(s)".

Now first check whether there are records in cprrp200 for your operations. In case there's an operation missing it might be that that operation has been set up with run time 0. In that case no records are generated in cprrp200. The solution then will be to set up the operation with a run time > 0. If this is all alright, then it might be that there's no calendar available to plan the operation(s) with.

Good luck,
Frank

Hi,

Cprrp200 is a table not a session, so you have to access it via either ttaad4500 or ttaad4100 (both without m000). If you haven't got experience with tables it'll be better to look at it with your administrator. But what you can do yourself is check the run time in the routing operations. If both run time and setup time are 0 in one of the operations then no record will be created in that table and you will get the message that you mentioned.
