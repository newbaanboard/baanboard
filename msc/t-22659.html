Were running IVc2 and......

We use Quantity Dependant Routings and multiple routing codes per item. For each item we set up the routing, then we just copy the exiting routing to as many new routing codes as needed. Rather then change the run times we just use different Routing units for each routing code in tirou1102m000:Maintain Routings Codes by Item.

The result is that MRP will automaticly pick the correct routing code based on the 'Up to Order Qty' in tirou1102m000. It's work on the front end, but our routings don't change much so maintenance is low, otherwise it would be a lot of work.
