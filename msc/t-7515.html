Normally Assayed Items is one type of process item where its potency varies from lot to lot so Assayed item must also be a Lot controlled item.

A scalable material provides the ability to adjust the batch size based on the material quantity available during production.

In the situation where the components come from multiple lots with different potencies, they must be defined as assayed items.

When the assayed item is received the potency of the delivered goods is entered. Baan automatically determines the inventory delivered based upon the standard potency and the actual potency.
For instance:

Standard potency of 90% per kg

Purchase Receipt 130 kg @ 65% potency

Inventory = ((65/90)*130 => 93.8889 kg

This means that using the 130kg received would be equivalent to 93.8889 kg at the standard potency.

Using this method is not necessary to adjust the formula qty since for each lot delivered Baan records the...

Lot Number, Delivered Qty, Qty @ Std Potency

When generating outbound advice the correct qty (based on the formula and lot potency) will be issued.

For example,

Items: M1 (Manufactured Std Item) made from 1 kg of P1  (Purchased Process Assay Item Std Potency 90%)

Purchase Orders: No- 82 for 130 kg, No- 83 for 100 kg

Lots: 18, 21 respectively

Lot Potency: 65% => 93.88 kg, 95% => 105.55 kg respectively

Prod. Orders: 100015 for 100 off M1

Outbound Advice: P1 Lot 18 130 kg => 93.88 kg @ Std Potency

P1 Lot 22 5.7 kg => 6.02 kg @ Std. Potency
Normally this field will be set to YES or NO the wile creating the process item in the session tiitm0101m000,

The filed value in the session tiitm0130s000 is defaulted from the session tiitm0101m000 for all of its containerized items.

Once this field is set , you can't change for the item and containers. However You can change the filed value in the session tiitm101m000.
