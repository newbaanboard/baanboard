This used to happen to us every few months in FP3. It seems to have gone away since we upgraded to FP6. I always fixed the issue by using GTM to change the qty to be back flushed to zero in the Production Order Operations table - tisfc010 (step 4 below). Try the first three options before you attempt GTM.

Take a look at KB Article 1043271 ...

When closing the Production order , system is giving error ‘ There is still some quantity to back flush for a production order’, following could be the possible reasons, sometimes it could be due to data corruption.

1. In session ticst0101m100 (Material to Issue for Production order), check for quantity in Subsequent Delivery or To Issue. Solution: Make it zero or issue it.

2. In session ‘Report Order Completed’, check for ‘to back flush’. Solution: Run session back flush Materials and Hours or make it zero (All related material will be back flushed).

3. In session ‘Report Operation Completed’, check for ‘To Do Back flushing’ (If order status is not completed).Solution: Make it zero or Back flush it.

4. In table tisfc010, check for quantity in field tisfc010.qtbf ( When order status is completed).Solution : Back flush it or by back-end make the quantity zero.
