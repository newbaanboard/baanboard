What is the best process to use for the scenario described below?

A customer submits an order for a customized (ETO) product which we have never produced before. The request needs to be sent to engineering to create new part numbers, BOMs and routings. But also the raw materials have long lead times, therefore we wish to order the critical raw materials (anticipation) before the engineering process is complete to streamline the order-to-shipping lead time. We are currently using this project order entry screen called “logbook”, creating work orders for purchasing the raw materials off of the project, then deleting those work orders and creating new ones After the sales order has been created for actual production. We don’t create a sales order until after the item number is configured and entered into Baan, which can take several weeks.

This seems like an unwieldy process to go thru. Isn’t it possible to create the sales order upon receipt of the customer request then update the order later once the new part number is configured/created? How do we do this?

Hi,

It looks like you have b40_c4 and not b50_c or ERP LN.
In that case I would indeed suggest to use a sales order in combination with PCS Project and BAAN Configurator.
PCS Project can be used to capture all engineering related costs.

Using TP Project in b40_c4 for your ETO process would be cumbersome and time consuming.
However, if you would be using b50_c or ERP LN then it would be a different story.
