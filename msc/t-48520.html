You definitely have to execeute "Process inventory variances". In contrary to Baan IV, LN doesn't create integration postings for e.g. purchase price variances directly. Instead, so called "inventory variances" are created, which you can use to indicate if such a difference is something that should be reflected in the MAUC or not. Those variances have to be processed, otherwise some interim accounts won't be balanced (if you set up the logistics-finance integration the way it's recommended).
