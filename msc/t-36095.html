Hi,

1) the option should only be disabled in case the part is not a generic item AND no product variant is present in the item data (tcibd001)

2) check whether the field tcibd001.dfit (derived-from-item) of all the components is filled. It's quite common that this field is not populated in case the item data have been migrated from Baan IV. In case it concerns a customised item, then the value in that field will point at the standard item it has been copied from. In case it concerns a standard item, then the value should be equal to the item itself. Bottom-line is that the field should be filled for all items in the item table.
