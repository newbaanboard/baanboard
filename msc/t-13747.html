Hi,

I think you can do it in this way. Define multilevel BOM and routing for the main item for which sales order is made and further continue the same exercise for all manufactured items in BOM of the main item. In this way all the items are linked till the lowest level.
Then define the main item in maintain project parts Â– tipcs2111m000 for the required quantity for which sales order is made.
Now when you generate planned PRP orders for the project, PRP production orders will be generated for all the items till the lowest level and PRP purchase orders will get generated for the purchased items defined in the project.

I hope this will resolve your problem.

We've never had an issue with this.

Sun-assembly items need to be To Order. If they are Anonymous then PRP will not generate production orders for them.

When you look at the Customised BoM at any given level, the sub-assemblies within it should show as Customised Items, not Standard items.
