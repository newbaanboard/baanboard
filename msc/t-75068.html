Well just using ttadv3180m000 gives you a lot of power. You can query one table then go to the report and create all kinds of layouts to do all kinds of stuff that I won't go into. We use baan 4c4 so your version may be different.

So what I usually did was just use tisfc010 - it has setup, run time, total production time and spent time. I would generate a report off just those things. Then if I wanted I could into a report - lets say you sorted on order number - add a after order number layout to go query tihra100. But we developed sessions to do all this reporting - most of them are simple queries against tisfc010, then do other things like ask for date range to see what was charged between 2 dates, etc. I recommend sessions, but if you can't do that, then you can query just tisfc010 (assuming your version close to 4c4).

You can select like tisfc001.pnno, tisfc001.mitm link to tisfc010 and pull like sutm, rutm, prtm and sptm or what ever other fields you want.
