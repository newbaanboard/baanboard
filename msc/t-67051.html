Hi Joy:
As per the definition and property of Phantom items, there has to be no inventory of these items. The possible causes could be:
1. Somebody has made a sales / production / purchase / RPL order for the phantom item
2. Somebody has made inventory adjustment of the phantom item.
Due to the above reasons, the phantom item looses its "phantomness". To restore the condition, please remove all the allocations / inventory on order for this phantom item and also make the on hand inventory 0 (zero) for all these phantom items. There should not be any planned inventory movement of the phantom items.
Once you carry out the above, then run the planning engine and it should generate the correct estimated material for the production orders which are confirmed and transferred.
If you still have any issues, then please do let me know.
