There is a field Correct Std/Est for Rejects per Operation. By defualt this field is set to 'No'. The impatc of this is thta if your actual production is different from planned quantity, this session compares the standards for planned production and not actual production. If your actual production is likely to be different from palnned production this field should be set to 'Yes'.

Secondly the next field is 'report Standard or Estimated values' Thsi field determines which values should be compared to the actual values- Standadrd or Estimated. In finance the production result is always on standard. If in your session the default is set to estimated, this could also be the reason for difference between this session and Baan Finanace

Hope this helps
