Hi,

Yes that is possible in LN. What you need to do is, go to Purchase Order Parameters (tdpur0100m400) session. Under Parameter 2 tab, there is a field called Delete Order Data if Received Completely. Set it to "Order Line".

Now, run the session Delete Purchase Orders(tdpur4224m000) for your Order.

Please Note -

1. Run the delete purchase orders session only if it has one line
2. If it has multiple lines, run the session only if you want to delete all the lines which are fully received.
3. The Purchase Order Line should be fully received
4. Once done, revert the changes in Parameters to how it was.
5. There is no warning message, so make sure you give proper selection.
6. Simulate it on your Test Environment before running it in Production.

Hope that helps.

In BAAN 4 (at least BAAN IVc3) there is no session to delete purchase orders.

Here is what the business process is:
- Run the session Maintain Backorders (tdpur4130m000) and make the remaining order quantity (the Back Order Quantity) to zero.

This will 'functionally' close out the Purchase Order.

After that, if the balance quantity is still required, the option is to add another line to the same purchase order or to create a new purchase order.

The old order remains in the system as reference for the quantity that was received.
