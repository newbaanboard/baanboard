hi joemon

in routing 1 (standard routing) u can set effective date 150110
in routing 2 u can set effective date 050110
in BOM u create 3 line:
line 10-1=item A eff date 150110
line 10-2 = item C eff date 050110
line 20-2 = Item B eff date 050110
when prod ord created it will take routing 1 by default.
u can change the item routing to routing 2 , then change the ref date to 050110.

standard ln function is not supported it because it related with the standard cost.
the item must have 1 standard cost.
so this is the work around.
Except u wanna use Unit Effective or Engineering Revision.
