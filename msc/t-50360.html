this error usually happens when there are some changes in the "routing set-up " of mfd items (sub assy) after generating order planning, that means, you're "planned orders" for transfer is superseded..
do try two things
1. delete the affcted (FG) item from the planned orders and re-do generation of order plan. and then, try to transfer again... if does not work, try #2
2. check if all sub assemblies (mfd item) of the affected FG item have "item-planning" data. then if found one, maintain item-planning data and do-step# 1
