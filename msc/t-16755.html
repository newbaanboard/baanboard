I don't know how locked you are on part numbers, but I seem to recall doing something similar for a client. The base part number was 1234 then added a P for the Purchased Item, and Part number 1234 we added an M for the manufactured part.

Standard costs were established for both individually, then Order Parameters set, and we made them alternate and interchangable on the BOM's.

Accounting was satisfied with the cost issue, and planning although they had to do some mental manipulation knew exactly what was on order and the status of each purchased and or manufactured.

I hope this helps you

No need to have two seperate items.

The item should be a Purchased MPS item.
Create a Sales Contract for the item
Create a Purchase Contract for the item
Create MPS Plan Item data for the item
If you now create a sales schedule for the item and then run MPS, a Purchase Schedule will be created for the item.
