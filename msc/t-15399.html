As Ed says this flag will be set to "No" if the "Product Configurator" session tipcf5120s000 has not been "Exited" but "Aborted" instead. It is only set to "Yes" if all constraints validate without error.

Another situation you will, by default, get the tipcf500.vali flag set to no is if a product variant is received via EDI.

The session tipcf5200m000 "Validate Product Variants" can be used to check a whole range of product variants (i.e. check the tipcf500.vali flag) Beware, there have been some recent solutions for this session - check on http://www.support.baan.com
