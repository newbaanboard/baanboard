Couple of possibilities:
1. Try changing the Work Center first and then the machine. A work center can have several machines connected to it, but a machine can be connected to only one work center.

2. Still in Production Planning, Add an operation with the new data you want and delete the old operation. Assuming that your operation numbering scheme was set up to allow insertion - 10, 20, 30.... so that you can put in an OP 11, 23.....

Partly correct!
Adding new operations for running orders is possible when
1. Costing parameter "Production results per workcenter" is off
2. Above parameter is on, but workcenter is not changed.

This option 2 (parameter = on) will create so called operation sets wherein the cost per workcenter are collected. And here changing between workcenters is not possible anymore after freezing the estimated costs.

In BaaN 5 you have the option of when to freeze estimated costs for production orders. In tisfc0502m000 - Default Production Order Data you can select to freeze estimates at Order Creation; at Order Release; or Before the first WIP transaction. So, it is possible to make changes to the order that would not get reported as variance if this is set to and changes were made before the first WIP transaction.
