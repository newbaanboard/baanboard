I really can not answer 2 or 3. I have only used boi.call once and that was based on another posters recommendation. I can come up with several other scenarios that MAY cause problems. Notice I say MAY - I never really know what I may encounter, but I have so far always found work arounds. I also do not know Baan V, so what you encounter could be different. Here are some scenarios I have run into:
(1) Limiting the number of records processed using API. When I firsted start doing function servers I was able to process 1000's of records, but within the last year 1 or so, some of my most commonly used sessions require a limit to the number of records.
(2) Controlling subsessions can be problematic. Sometimes it requires a little work to get the control down and yes sometimes we have used Qkey to work around problems. Also sometimes I actually had to kill subsessions and then reactivate them to get them to work correctly using api.
(3) Sometimes all fields do not update. Used various work arounds like putting the data twice and doing two saves. Also I have done direct table updates.

Can not seem to remember any more big problems. It is really to hard to say without actually trying to do the coding. But I never shy away from using api type function servers.

Just a thought on #3 - using EDI instead of AFS:

The only thing that you can use EDI for is to load PO's (and maybe modify them) for you. Standard EDI functionality does not include 'receiving POs' or other things that we have used AFS to solve.
