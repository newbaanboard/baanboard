Products like RMCgen, QKEY and the freeware on these boards (search for 'rush') fit this description. These tools do not require source code. They provide a wrapper around the source code where you can add to or remove the standard sections. You cannot change standard logic. This can be used similar to a screen scraper. In fact we have used this product to replace the screen scrapers and include logic in these automations. Our tool also provides the ability to make a session wrapper or create a library out of the session. This does not use AFS functionallity and was developed specifically for versions not supported by AFS.

I am not sure of the availability of RMCgen all version of Triton, are you running 31b2? If you think it this is something that may match your requirements, let me know and I can research the version availability. You can also research the other products.

With RMCgen, you can create a 3GL script/library or a 4GL program script. If you choose the latter route, things will execute using the Baan's standard program. If you go the 3GL route, then yes you would have to ensure that the appropriate code "sections" are called in the correct/desired order. Much of this, however, is defaulted for you when you do the generation.
