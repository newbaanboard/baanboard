Usually the the macro GET.DATES.OF.PERIOD (see help text for description) is used. That macro will return the start date and the end date for the period currently processed.

Then you have to include this start date and end date in your select statement. (E.g. you can select all records from gld106 where the finalization date is between start date and end date. )

If you don't include start date and end date of the period in the select statement, then everything is selected for each period.

The system runs your script for each period, that it finds in the calender code, which you're using.

E.g. If you've got 2 years with 12 periods in your calender, then it runs your script 24 times and writes the result for each year/period to the "PI Data by Period" table. If you want to limit the number of periods, you have to set up a calender code with only those periods, that you want to be processed.

When you calculate the PI data for the second time, the system starts processing at the period of the last calculation date. If you want it to start at the first period again, you have to delete the last calculation date from "Maintain Calender Codes"
