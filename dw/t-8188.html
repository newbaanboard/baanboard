Hello kps_menon,

I can answer your question, but only partly. Workflow would be a pefect solution to do all kinds of handling after the business event has been recognized (i.e. the balance falling below 80%). Workflow could send emails, create work items for the receivers (if they are users of the system) and do all kinds of routing/escalation activities and track any required user action or external answers.

However, still the Baan session needs to report this event to workflow. I am not aware of a way of configuring the session in this way. Maybe there is a chance to achieve this by tools programming? It would be sufficient if an API call could be issued (ideally) or a command line could be called (fallback). Database triggers would be a way to do that, but this is very complex and not optimal from performance perspective, if the value is stored in distributed tables.

If somebody can answer this second part for you, then you are there.

This can be done, but you need to build a Baan 3 GL session, which is sort of polling to check the balance and then when it detects the level on which you want to take action, it then can send a call to the WF database to startup a Workflow process instance, which then can take care of the handling of this exception.

In fact WF with KPN is used in such a way, also there from an automatic batch handling of salesorders on exceptions WF processes are triggered to handle te exceptions by people through the Cosa WF client.
