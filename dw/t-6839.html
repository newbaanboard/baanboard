1- the DEM modeler need a normal Baan User login to be able to start Baan and create a Model
2- an DEM End-user need to be defined as Employee within the DEM Module
3- the latter DEM End-user need to be linked to a Baan user in order to have access to the Baan ERP application
4- in Baan ERP 5.0c, you need to define some 'runtime user' settings, to ensure proper access/operation for the end-user.

These steps are for a DEM generated role in Baan IV, running on Unix. There are similar steps for NT, etc.

SETTING UP A NEW USER FOR DEM


UNIX

1. Log onto Unix (as root).
2. Enter SAM, after the $ prompt, enter
3. Select Accounts for Users and Groups, enter.
4. Select Users, enter.
5. Tab and select Action, select Add, enter.
6. Complete the user screen entries:
· X for Create home directory
· Select bsp as Group, click OK.
· Enter password, re-enter password, enter
7. Tab File, space, exit.

Baan

1. Log onto Baan as bsp.
2. Go to Tools, User Mgmt, Maintain User.
3. Insert new user (use same name as for Unix user).
4. Under Startup Options tab, select DEM browser.
5. Under authorization tab, select role to be used for the user.
6. Under the Tool Bar, Special:
· Convert Baan user to Database user.
· Convert user to Runtime Data Dictionary.

DEM

1. Go into DEM process model
2. Select the user in session tgbrg4250s000 (accessed from the Project model, generate user dialogue button.
3. Generate the desktop for that employee and role.
