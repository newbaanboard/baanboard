EZ-Workflow
Automated Flow Control & User Notifications for Business transactions



To complement the EZ-Process family and to support & optimize the daily execution of business transactions, the EZ-Workflow module provides automated transaction routing and network-based tasks notification to users for all steps in route. Very efficient in complex and time-consuming approval processes (Requisition, PO, Expenses, etc...) or any other human-intensive tasks, it enable significant increase in productivity by triggering each tasks as the one before just ended, eliminating the non-productive delays between them. The web-based feedback and electronic signature enable real-time transaction tracking for each tasks in progress, and enable detailed audit trail of "Who did What When".
FUNCTIONS & FEATURES:

EZ-Workflow FlowControl module:
ability to define unlimited customized workflow routes for different business purposes
usage of existing email infrastructure to transmit task notifications to users
automated merge of Baan transaction data into the notification email
once the initiation of the route is performed, all dispatch and completion of steps are automated by the EZ-Workflow engine (server-based)
notification emails can be configured to launch applications or invoke content based on user selection
user feedback performed using web-based form, operated within the user browser (no application to be installed)
supports the user "substitute" concept enabling validated delegate to act upon task notification
each step completion can be accompanied by a user note (free format) kept in history of transaction
real-time monitoring of on-going tasks to identify bottlenecks and inactive transactions/steps
full audit trail for each Business/Baan transaction, including detailed steps information (Who did What When) and the transaction data (Baan or others)



EZ-Workflow integration with Baan:
ability to capture Baan generated Requisition/PO/etc and initiate appropriate workflow route
ability to notify specific user that the workflow route has been completed to trigger a Baan transaction to be executed
