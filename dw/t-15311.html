Below solution text 107069.

VERSION(S):
B40_c

SITUATION IDENTIFIED IN:
"Dynamic Menu Browser" (tgbrg4260m000)

SITUATION DESCRIPTION:
Currently there is no sort possibility available for the "activities" based on the "External Code" in the "Dynamic Menu Browser"

SOLUTION DESCRIPTION:
Now it is able to sort "Activities" in the "Dynamic Menu Browser" ascending, descending or not sorted depending on the "External Code". Therefor a specific resource setting has to be set:
Start session "Maintain Runtime Resources(ttdsk3160m000)". There an entry for that user(s) has to be made or changed (package "tg" and resource set "datamodel"). The value of the option "Snap to grid" has to be set to respectively "UP" for ascending, "DOWN" for descending. No value or something else will result in a not sorted "Dynamic Menu Browser".

As the "External Code" is a string and not a numeric field, you have to consider that the 1 (one) becomes for of the 2 (two) when sorted. This means that 100 (one hundred) becomes for of 20 (twenty). So if you have many activities in one Business Process you have to make sure that al "external codes" are in the same "numeric" range (e.g. starting with 1000, 1020 etc. etc.), to make sure your sorting query will be as expected.
