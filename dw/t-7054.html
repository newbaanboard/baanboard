Dear H3nry,

From the problem text, I derive that you're working with version Baan IV. In that version, DEM always import into the same version as the export came from. So I'm pretty sure that in (target) company 820 a version "Grafitec" exist, containing your exported process.

Simplest way to resolve is to copy the process in company 820 with session "Copy Version Data to Other Version" (tgbrg1200m000) to version Printec.
Note that components such as new created utilities are to be checked separately. They're not copied automatically with the process.

For company 830 you'll need to do the same trick.

I hope this answers your question.

You need to be careful with sharing tables in Baan IV. All texts (except the helptexts) are stored in the current company, and with session 'Convert Text to Help Text' converted to company 000. Sharing texttables can cause problems.
I've a document describing Baan IV DEM data sharing in more detail. If you send an email to the mailbox below, I can reply with the description attached.
