10th November 2002, 23:02
It is not very difficult to generate DEM users.
1) First define roles
2) Attach the main processes to each role
3) Create employees
4) link roles to employees
5) run session tgbrg4250m000 (create desktop)
If the user then will log on to Baan, one will go directly in the DEM, not in the menu structure
It is advisable to create a project model first
Tools and DEM authorisations work totally independent of each other and can therefore for sure co exists

The generation of user authorizations is changed between BaanIV and BaanERP.
In BaanIV real entries in the Tools tables are generated using session tgbrg4250s000. (this one is started from the Project Model)
In Baan 5.0b this session is called tgbrg4250m000 and in fact is in Tools tables only changing the startup menu in the User Data. No Tools entries are stored anymore.
In Baan 5.0c the session is called Configure Baan Employees and its code is tgbrg8240m000.
