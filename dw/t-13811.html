The only way you can limit the permissions on sub-sessions is by entering "session authorisations" in Baan tools. By setting the to and from times to the same time DEM will not overwrite these authorisations when re-generating desktops.

In BaanERP (5.0c) it is possible to give the application an option, how it should start (e.g. with full authorization or only display). This parameter is on the second form of the application, when youÂ´re modelling the process and start an activity.

Further it is possible to use the subapplication modelling. This functionality is used, to give authorizations to the subapplications - in which way the Subsession should start (e.g. no authorization or only display mode). But this function comes with a solution (please search the Knowledge Base on Baan Support).
