Hi Rahul,

You probably have used Baan IV before. DEM in Baan IV used the current version both for end-users as well as for modelers. For modelers to work on models in that version, end-users to let them work with models in that version.

As off Baan ERP (5), these 2 user-types (modeler and end-user) no longer share usage of the same 'current version' mechanism. The current version setting (in tgbrg1110m000) is only required and relevant for the modeler, to enable EDITING of models. The end-users however are now controlled in a separate table (Baan User DEM Settings, tgbrg8136m000). In that table, the Baan user is linked to a 'default version' (the version that will be initially used when launched a DEM desktop), a 'default project model' and a 'default optimization phase'. Next to that, some specific user settings are defined in that session.
Advantage of this concept is that the end-user can be active in multiple project models in the same company. In his user-desktop, the end-user can switch from one project model to another (if properly authorized) within the same company. For customers centralizing their logistics into one (logistical) company, this concept has it's advantages.
