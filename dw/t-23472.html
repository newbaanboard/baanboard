Marius

As you have probably noticed, the run program session itself won't open from a DEM activity. As a workround, set up an activity within one of your user's processes and link it to session ttdskprogman. This will open up a blank tools desktop session, from where the run program session is available from the File menu bar option.

Hi Marius,

Jeff is right, session run program does not show on menu. You have to do the following to give him/her the permission for run program.

BaaN Tool -> User Management -> General User Data -> Maintain User Data

Click on Maintain User Data and go to form Authorisation and click the option call session by code and then go to special and do convert to runtime data dictionery.
