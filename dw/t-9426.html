It all depends on how you have your authorizations setup. Here, we use DEM menus but we don't generate session authorizations through DEM. We let tools take care of the authorizations. It gets complicated but if you take away from all the file, run program option, then the user can not access a session that is not on his menu anyway.


Through tools, once a user is logged into Baan, you can setup key maps, where certain sessions are mapped to certain "F" keys on the keyboard. Then, once a user is logged in, he can use the "F" keys as shortcuts. They won't automatically start.

through tools, you can also maintain startup sessions which automatically starts sessions upon log on into Baan, However, the problem with this method is that the session start in the order of fastest to slowest, not in any predefined order.
