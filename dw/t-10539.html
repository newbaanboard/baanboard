Hi Slawek,

Answering your question, requires more information regarding the Baan version you're working with. Between Baan IVc, Baan ERP 5.0a and Baan ERP 5.0c there are differences in how to set up DEM. What is similar in all versions:

- create process in repository
- copy process to project model
- define role and link it to process (or actvity)
- define employee and link it to role
- generate desktops

From the fact that you've the session 'Install Client applications', I derive that you've at least Baan ERP 5.0a. Main difference between 5.0a and 5.0c is the authorization structure (in Baan 5.0a you have to define authorizations in the project model, in Baan 5.0c you can do that in the repository) and the component/application structure.

Changing in 'user data' the startup program to DEM Browser (what you did) is not necessary. That is automatically done while generating the desktop.

Hi Slawko,

Your description of creating processes in tgbrg5500m000 and project model in tgbrg4500m000 is looking ok. What you need to do next is:

- verify that you've DEM employee(s) defined (and they're related to an existing Baan user)
- verify that in DEM parameters (tgbrg0135s000) the checkbox 'Use employee - project model link' is UNchecked. (If this parameter is marked, the employee need to be linked to the project model separately.)
- Create an entry in session 'Runtime Version, Project Model, and Optimization Phase per Company'. Make sure you select the version and project model you've created before.
- You can 'preview' the DEM browser from the 'employees' session (specific -> 'start screen employee')
- By running session 'Configure Baan users by DEM employees' (tgbrg8240m000), the DEM settings are written into the user tables of the Baan user. After that, you can logon with the employee you've selected in this session tgbrg8240m000. The menu you should get is the DEM desktop.

Success!

Jeff Ball
18th November 2003, 12:27
Hi sado

no problem at all!

If you look at the DEM parameters, one of these controls the display sequence of objects in the desktop. Select the option for ascending order of external codes.

Then go to your process in the editor, and give the activity that you want to move an external code between the values of the two menu lines you want to insert it between. Save the process, and that's it. Next time users log on, the menu line will have moved.

There is an option within the editor to renumber external codes, which will automatically reset them all from the start state through to the end state, but if you have branches in your process the renumbering goes all the way down one, then the other, which might not result in the desktop you want. Renumbering some or all of the external codes allows you to display the menu any way you want.

Have fun
