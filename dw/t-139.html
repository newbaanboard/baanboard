You don't say what version of the application you are using, but it sounds like a known 5.0c problem. When you import the applications by component using tgbrg5155m000, then even though your component and release are correctly specified (the usual convention is BaanERP and B50_c) what gets filled into the cmpt and rele fields of tgbrg520 and tgbrg560 is BAAN and CORELLI due to some misplaced hard coding.

You can tell if this is the case by going into any process with missing descriptions, open the activity properties and re-selecting the component and release fields using the zoom buttons. If the activity descriptions appear when you save, this is your problem. The easiest way to solve it is to run an sql routine that will replace the values in the tables with the correct ones.

Good Luck!

For those interested or that helped with adding information on this situation that was occuring here. This had been logged with Baan and I received a solution today (11/9), the solution was 119637. Here is a brief description of the situation and solution:

SITUATION DESCRIPTION:
Session descriptions are not always shown.

Specialy when the session exist in the highest VRC but only in 1 language.
(Not the current language) and for the other languages the session exist in
a lower vrc

SOLUTION DESCRIPTION:
The search for language dependent description for a session is improved so
that it will search via the vrc structure

Hope this helps anyone else that is noticing this happening.
