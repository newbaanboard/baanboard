Jeff Ball
6th February 2008, 00:22
The processes need not be in the same version as the project model, but they must be in the derivation path for the version in which the project model sits. The logic is the same as VRCs for the software objects - the user exists in a current version (for normal desktops this will be the one used in the default project model). If the process quoted by the project model is not present in that version, the version it is derived from is checked, then the one that is derived from, and so on until it is found. If the process is not present in the derivation path, it cannot be seen.
