Labels are used in DEM to store the descriptions.
So it is possible to use the same DEM version in different languages, because the description is per language separate saved in the label for that language.
One of the key fields of the label-code is a company-number and that can give problems if your current label-codes are not the same as the current company number.

In 5.0c you should not share DEM tables anymore, for that the Central Model company concept is introduced and the whole working of DEM is based on that. For example also helptexts.

The company number is part of the label-code.
And sometimes this is re-formed at runtime for searching the description instead of the label-code from the table, because this is quicker.
However that causes problems when you are not in the company in which the label-codes are made and so the re-forming uses the wrong company code and doesn't find the right label.

The only really reliable option is to do the Import/Export, because then the labels are created also for the new companies and so everything can be found.
