Yes, in Baan 4 it's possible to "archive back" with several sessions. It's not always good to do this though. You have to check what the archive session does:

When you archived, the session tdinv7210m000 moved data from table tdinv700 to the archive company and updated the fields "Historical Inventory Balance" and "Date of Historical Inventory Balance" in table tdinv001 in the original company. The historical inventory balance is the starting point for inventory valuations.

I don't see a problem in moving the data in tdinv700 back, since the inventory records are dated. Therefore there is no danger that new data that has been entered in the meantime will be overwritten. You can do that either with the archive session or just copy the table back. Afterwards you have to run the program "Rebuild 'Historical Inventory Balance' (tdinv0253m000)" to correct the historical inventory balances though. This session recalculates and corrects the 'Historical Inventory Balance' in table tdinv001 (visible in session 'Maintain Items by Warehouse (tdinv0101m000)'). As always: if possible try this in a test company first and have proper backup procedures in place.
