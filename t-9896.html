28th March 2006, 16:08
Yes - it was a success. Maintain receipts does have some issues with lot controlled items. It seems like with each service pack release I need to make changes to it. The inbound and outbound sessions are pretty stable. I have not done anything with cycle counting.

We creat tags at certain points in the process. At receipt time we generate receipt tags for the stores group. OH - we contacted our suppliers had them start sending us Bar codes to use at receipt time. At pick time we generate pick tags - used to maintain outbound. We printed bar codes long before we automated the sessions. See the attached for some screen shots of what we did at each point. If you search around in here you will see some of the subroutines I use.

The best thing I can recommend is that you document your process as best as possible. See where it makes sense to create tags to make life easier for the users. Then you can see how you can streamline the process use API code. For all I know you might be able to generate and release outbound all at once. In our case they had to be separate.
