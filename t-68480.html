How do you use IWM... with Openworld or Webservice for communicating with baan?
if it is with OpenWorld, the scanners might get hung due to fatal error as there is no refresh mechanism unless ASM is used. In such case, look for the error logs and solve one by one.
Webservice has a auto restart mechanism... However, if error occurs it will still be in hung status until restart happens.

Can you please explain to me what is the difference here?
To my understanding, if you are on LN10.4+ the default communication method is webservice, but it can be configured to use tmboaserver. less than LN10.3 default is tmboaserver, but it can be configured to use Webservice. So, it depends on how the installation was configured.


Where can i find this?
If you are using webservice these settings would be available on the C4WS (Connector for Webservice) server or WSTK (Webserver Tool Kit) server.

If you dont have details about any of the above webservice servers, then the system should be running based on OpenWorld (tmboaserver). You can directly start looking at $BSE/log/log.* for fatal errors and start solving one by one. Specifically look for fatal errors which are caused by the userID which was used to start the tmboaservers.
