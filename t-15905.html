Hi -
You do not need to start the DDC Server to use Test Function Server.

The error 'Incorrect Enum', you need to type in the numeric equivilant to what you want to say. For example:

Enum Domain: tcyesno
Type in 1 for yes
TYpe in 2 for no

You must type in a value for each enumerated input field, you can't leave them blank.

To get further functionality there, IE, accept 'y' or 'Y' for Yes and 'N' no 'n' for no, then you need to update the script and bring in a string for that field instead of the enumerated domain, and validate / translate yourself.


Good luck.

You do not need to put strings in quotes in Test Function Server. I don't get an error when I use 1 or 2 for tcyesno. Our scripts have been updated some, but that should be the standard. You should be able to debug, and trace the value: ddc.error (or ddc.errors..) and notice when it changes. The domain error would be in the check.variables() section, make sure it's coming up when you expect it, and that those statements are correct in there, checking the right domains for the right variables. That section gets called automatically. Try commenting out the check.domain call for that field and see what happens...
