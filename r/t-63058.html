-programming language used to program business objects?

Mainly a proprietary 4GL language, called "Baan C", which allows for inline SQL. Good engine for standard sessions that does all the CRUD stuff without programmers having to do it manually. On top (very) new, GUI oriented developments are done in .net environments

-languages supported in the GUI?

See above

-your satisfaction with support/consultancy (in which region? US/Europe/Asia)
-is Enterprise Modeler still the way to go in Infor ERP? (got the info from Baan V6.1)

Still used in projects as a documentation tool, but it's not that you can't do without it

-is customizing functions/business objects an easy task?

Yes, if you know how :-)

-Webclient: does it provide full functionality? if not how does it differ from the Worktop (as it was called under 6.1)

Offers more(!) functionality compared to the Worktop. Definitely recommended to be used as end user interface, even if it requires an additional technology stack (two different stacks, if you add Workspace (Sharepoint based interface) on top).
