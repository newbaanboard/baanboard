I've had similar strange effects when trying to print Russian texts in a report (the rest of the report was in german). It also happened in a programm where I write a "normal" textfile.

The problems I had where because I used a very basic technic of cutting strings after a given length (like dummy = output(1;20) ). With output defined is non multibyte string (= ascii string)
This worked well when using normal ascii based strings. But I did assign a multibyte string to output beforehand, which worked quite fine - until some multibyte coded character was cutted right in middle of its 3 bytes.

This may not be the cause in your example, but it is worth checking. It may also be caused by some codebase problem.
