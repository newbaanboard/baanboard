OK, the final solution worked:

If you don't have central deployment enabled and/or don't have any idea of how to turn it on [like I do] then use this method as a workaround:

(1) Create the .wtd locally on each terminal server. I just saved mine with same name, to the default location
(2) call it as follows "c:\...\worktop.exe" "c:\...\doc1.wtd"
(3) if you choose to publish the app, it's easiest for the .wtd and the worktop.exe to be in the same location on each server, by the same name.
