Third part solutions might work too. We use Softshare here with Baan EDI.

So we can use Baan's EDI functionality to generate or read flat files, and use Softshare to convert them to/from XML data. It will also communicate with the customer.

I'm not saying this is better or worse than the other methods you ask about: it is just an alternative to consider. If you already have some EDI software in place, you might want to see it if can handle this or if it can be upgraded. Many traditional EDI vendors have added this functionality to their software.
