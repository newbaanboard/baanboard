Yes, Norwin is right.

I faced the same problem on Saturday.

With rexecd make sure the IP of the PC can be properly resolved (either within DNS or in /etc/hosts).

Alternatively switch the GUI config to use baan login daemon instead of rexecd, which will not require this.
