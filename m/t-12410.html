I can guess though...

You cannot see the reference counter within Baan. But since your database is Oracle, you can query the reference fields as if they are normal table fields within Oracle (with sqlplus).

Actually, within the RDBMS they act as "normal" fields, to be maintained by the database driver, not by the application.

There is however no direct way to check the validity of the reference counter... you can try with session ttaad4229m000: Tools- Database Management - Miscellaneous - Print References To Parent. But if you have stored your base data in a different company, you may need to run it for all operational companies and add the counts per company.

The other way is: check reference counter in Oracle, run refint6.2 with correct options (start with refint6.2 -U :)) and check reference counter again. And trust refint6.2 :P.
