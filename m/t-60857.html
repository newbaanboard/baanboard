Hello,

if you're a "superuser" you are able to give the authorization for all Package VRC's.
Or define a user who's in the proper PVRC.

From outside it may be difficult to extract the information you want, because of the outhorization system:

e.g.

autorizations can be set at:

Package-, Module-, Session-, Table-, Tablefieldlevel.

For example:

A user can have modify rights at session level, but only "read rights" at a table field used by the session.

or

a user has rights to the distribution package (td) but not to the purchase modul in distribution package (td pur) a.s.o.

Therefor i think, it's easier to use the application built ins.
