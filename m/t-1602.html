This behaviour is definately related to the version of the standard object.

The behaviour of this object was changed to remove the defaults sometime early 2000 and then following complaints to support it was reinstated in Feb 2001. See sol 108130.

So if you update both environments with the latest solution for the standard program (10923) then the behaviour should be identical.
