Refer to the Baan Support site for more info:

SolutionID 216739
OT: Enhancement Request

Solution Description
English Session Change Password provided.
TOOLS VERSION(S): B40_c, 7.1_a

SITUATION IDENTIFIED IN: "Change Password" (ttstpchangepw)

SITUATION DESCRIPTION:
End-users can only change their passwords at the time of password expiration/aging.

SOLUTION DESCRIPTION:
Session Change Password (ttstpchangepw) is provided. With this session end-users can always change their passwords, also outside the period of password expiration/aging. Session is present in menu User Management/Miscellaneous.

But, it's still maintaining multiple user accounts. I'd like for my end users to have only one user account to maintain ... single sign-on.
SolutionID 232281
QR: Functional Change

Solution Description
English SSO (Single Sign On) configuration BaanIV
TOOLS VERSION(S): B40_c

SITUATION IDENTIFIED IN: "Maintain User Data" (ttaad2100m000)

SITUATION DESCRIPTION:
In BaanIV Tools it is not possible to set-up and configure Infor SSO (Single Sign On). This is required for Infor MyDay (Role Based Homepages).

SOLUTION DESCRIPTION:
A new session Maintain SSO Parameters (ttams0100m000) is introduced. With this session it is possible to activate and configure SSO (Single Sign On).
Session Maintain User Data (ttaad2100m000) has been modified. It now shows a few extra fields which can be used to configure some user specific attributes related to SSO (Single Sign On).
Session Convert User Data to Runtime Data Dictionairy (ttaad4200m000) has been modified. In case SSO (Single Sign On) is active it will dump an extra runtime file per user which is used by the portingset during the login process of the user.
See the Session and Field Help of these sessions for more information about the SSO (Single Sign On) configuration.

To be honest, I haven't installed or configured the system. But I might give you some hints (I hope). First of all, run swlist as root and check if you see "LDAP-UX" and "PAM-Kerberos and Kerberos Support". If you don't see them, you'll probably have them installed first.

If you see them, you might have eg. a directory /etc/opt/openldap that contains some config files. Maybe these haven't been be configured yet. After that you should have a look at /etc/nsswitch.conf which might then contain a line "passwd: files ldap".

Last but not least: google "LDAP HPUX" will lead you to hp's download and documentation pages.
