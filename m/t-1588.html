Hi,
We wrote a custom interface which does credit card processing as well as refund. Basically, we use API's from verisign. The verisign API is called from a java program. It works like this.
1. Persom enters credit card information in Baan session
2. Baan creates an ascii file and then it calls the java program. It then waits for the response
3. Java program does the calls verifyingsite and reads the response and writes the response to ASCII file
4. Baan program will read the ASCII file and marks the transaction as approved.

We are able to get rid off all he card machines that were used before. It happens in real time. Also, we don't store the card number in our database. If we need to do refund we use the original transaction number.
