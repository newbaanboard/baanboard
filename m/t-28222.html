Dear newtobaan,

you could use

(1) Exchange module
Import data, specify field separator, modify data while importing (3gl functionality)

(2) Import sequential dump (dump file created with script or e.g. excel)

Important:

The data from the old system has to match the domain restriction of the baan system.

If you import old order numbers you have to specify a number range (old data) new data created with baan.

You have to set up the tcmcs047 (first free number) table.

Maybe better to import data in own company number or seperate tables.

When we migrated we just put what history data we had to have in Access so the users could view it. For WIP Production orders what we did was create the production order. Adjust on the material already issued and re-issue it to the production order. I think, but can't recall, that for purchase orders we only moved the ones that were open. When I think back I probably would have moved the history data into some new baan tables and created some reporting around that. I do not think I would attempt to load all the baan tables that would be involved.
