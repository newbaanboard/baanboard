Solved.

The funny thing is, that the Baan IV GUI uses GIF files and display them as if the were transparent - but they arent. You can see this if you change the settings of you windows desktop (e.g. windows classic, windows XP style); the grey background of the GUI will change AND the background of the buttons too.

If you check one of the button files e.g. $BSE/gif/gfd0002/find1.gif, you will see that it is a GIF87a file with a dark grey background. GIF87a doesn't support transparent background!

By accident, I finally found the trick: It's the color of the pixel in the lower left corner that tells baan which color has to be treated as transparent!

So I could use GIMP to repaint some of the gif files that drove me crazy, and now all works fine.
