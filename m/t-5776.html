Dear N. Prashanth Rao,

I understand that sometimes the installation and especially the configuration of Webtop is quite cumbersome. The reason for this is that it uses a number of (Baan) software components like OW adapter, ASM, and a web server. We recognized this, and try to help as much as possible by delivering a Diagnostics Tools together with Webtop, and moreover, by improving the configuration proces and administration console significantly in the upcoming Webtop 2.0 release.

In Webtop 1.1, things have improved. I recommend to install this version and use the documentation for Webtop 1.1. With Webtop 2.0, which will be released around September, we support web deployment, making life a lot easier.

Webtop (1.0/1.1/2.0) can be run on any web server supporting Java servlets. On IIS, you need to install a separate servlet engine (like e.g. Servlet Exec of NewAtlanta); if you would like to use e.g. WebSphere or iPlanet, the servlet engine is included in this software.

Support on Webtop 1.0 and 1.1 is partly shared between the Baan Support organization and Baan Development, this might explain your experiences with support issues.

If you have any questions, please contact Support in the USA first, but also feel free to contact me whenever you want on any Webtop issue.
