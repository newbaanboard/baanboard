Hi,
Java Connector Archietcture(JCA) is nothing do with BaaN. JCA is a framework for integrating heterogeneous ERP systems from any J2EE complaint Application Server that supports JCA.

JCA provides CCI ( Common Client Interface) that need to be implemented by EIS systems. In case of BaaN, the JavaClient that calls BaaN Business Objects (BOI) should be implemented as CCI, which talks to the J2EE Enterprise Java beans residing on the Application Server.

Also u can skip JCA framework and directly call the Javaclient that calls BOI from an Enterprise Java bean deployed at J2EE Application Server.

In order to integrate BaaN with Java, You need to write BaaN Business Objects in 4GL and u need some middleware to sync BOI with Java.

BaaN Connect Kit has the middleware BCBE API's(BaaN Connection Basic Edition ) but no more supported by BaaN. Also its very complex process.

I prefer Cordys . OpenWorld is also equally good. You can try other integration methodologies like DDC, Internet Applets but its pretty complex when compare to BOI.
