Originally posted by vishbaan
One problem I remember of is the hash key issue in baan. If you want to insert a record in baan table from outside baan, the hash key goes corrupt. But with the use of qptool the db operations, will it be handled by baan. Does a connection from VB to Baan using qptool cost a license in baan.
To answer the last question first: qptool does not need a licence.

The bad news: you cannot do updates with qptool. You'll have to use either gcommand, or write an interface. This last is again bad due to the licence it takes...
