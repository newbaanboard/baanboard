jim s
12th September 2006, 19:17
Quite simply, you're using two different units and Baan needs to know how to get from one to the other. You can have global conversions like 1 FT = 0.30479 Meters, and you can have specific conversion factors set up per item. This is where Baan looks first. You wouldn't want to globally convert SET to METERS because SET might convert differently for a different item. Therefore the conversion factor needs to be set up for that individual item, allowing Baan to get from here to there.

In case where different units are involved, baan looks for conversion facturs between these units. In your case baan will check if there is a direct conversion factor defined between set and m. Since this conversion factor is not defined, baan looks for two step conversion factor, whereby it is trying to first convert set to pcs (thi s is probably defined) and then pcs to m (this is missing) and since it does not find pcs to m conversion factor, you are getting this message.
